## Що і чому?

Робота з JSON - це процес кодування та декодування данних у форматі JSON у Bash-скриптах. Це робиться для зручності обміну даними між різними програмними продуктами та сервісами. Програмісти використовують JSON, оскільки він є одним з найпоширеніших форматів обміну даними у сучасному програмуванні.

## Як це зробити:

Для роботи з JSON у Bash необхідно встановити пакет jq. Це інструмент командного рядка, який дозволяє обробляти та змінювати данні у форматі JSON. Нижче наведено приклад коду та його виведення, яке демонструє, як можна використовувати jq для роботи з JSON:

```Bash
# Створення JSON об'єкту
data='{"name": "John", "age": 30}'

# Виведення значення ключа "name"
jq '.name' <<< "$data"

# Зміна значення ключа "age" на 35
jq '.age = 35' <<< "$data"

# Виведення всього об'єкту зі зміненим значенням "age"
jq '.' <<< "$data"
```

Виведення:

```
"John"
{
  "name": "John",
  "age": 35
}
```

## Глибше

JSON (або JavaScript Object Notation) був створений у 2001 році як легковажний та зручний формат обміну даними між різними сервісами. Його легко зрозуміти людям та програмам, тому JSON став популярним у веб-розробці та інших сферах програмування. 

В Bash, крім jq, існує кілька альтернативних варіантів для роботи з JSON, наприклад, jshon та jsawk. Також існують спеціалізовані пакети для роботи з JSON у великих програмах, які дозволяють більш ефективно та швидко обмінюватися даними у цьому форматі.

Щоб детальніше дізнатися про роботу з JSON в Bash та інші корисні команди, рекомендуємо прочитати документацію jq та відвідати форуми та спільноти програмістів.
