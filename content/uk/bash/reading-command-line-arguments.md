## Чому

Програмування у Bash мові є надзвичайно корисним для роботи у командному рядку. І часто, коли ми запускаємо скрипт у терміналі, ми хочемо передати йому якісь параметри або аргументи. Тут на допомогу приходять командні аргументи, які дають можливість вказати деталі у скрипті під час його запуску. Далі ми розглянемо, як можна зчитувати ці аргументи.

## Як це зробити

Для того, щоб зчитати аргументи командного рядка у Bash, використовуємо змінну $1, $2, $3, тощо. Де $1 - це перший аргумент, $2 - другий, і так далі. Давайте перевіримо це на прикладі:

```Bash
# Створюємо скрипт з ім'ям script.sh
# Який приймає два аргументи - ім'я та прізвище
# І виводить повідомлення "Привіт, [ім'я] [прізвище]!"

#!/bin/bash
echo "Привіт, $1 $2!"
```

Тепер запустимо наш скрипт із аргументами "Іван" та "Петров":

```
$ bash script.sh Іван Петров
Привіт, Іван Петров!
```

Як бачимо, скрипт успішно зчитав передані аргументи та вивів необхідне повідомлення. Також можна зчитувати аргументи за допомогою циклу, щоб обробляти більшу кількість аргументів. Наприклад, якщо ми бажаємо вивести всі аргументи, передані у скрипт, можемо використати цей код:

```Bash
#!/bin/bash
for arg in "$@"
do
    echo "$arg"
done
```

Тоді при запуску скрипту з аргументами "one", "two", "three" отримаємо наступний результат:

```
$ bash script.sh one two three
one
two
three
```

## Profound Dive

Не тільки змінні $1, $2, $3 дозволяють зчитувати командні аргументи, також існує можливість використовувати спеціальні функції, які дозволяють оброблювати аргументи більш тонко. Наприклад, функція "$#" дозволяє визначити загальну кількість аргументів, "$@" повертає всі аргументи у вигляді масиву, а "$?" показує код виходу останньої виконаної команди.

Зчитування аргументів командного рядка є важливим та корисним елементом програмування у Bash мові. Це дозволяє налаштувати скрипт для різних ситуацій та спростити його використання.
