## З чого це почати
Якщо ви вже працюєте з командним рядком або вивчаєте програмування, то ви, напевно, чули про Bash - швидкий і потужний інтерпретатор команд на основі тексту. Однак, чи знаєте ви, що Bash також може працювати з YAML? 

ЯМЛ, або  YAML Ain't Markup Language, це мова розмітки, яка використовується для представлення даних у вигляді ключ-значення. Вона стала дуже популярною серед розробників через свою простоту та читабельність. Але чому використовувати Bash для роботи з YAML? Далі ми розглянемо це детальніше.

## Як це зробити
Загальна структура YAML файлу складається з упорядкованих пар "ключ: значення", де значення може бути як простим текстом, так і складнішими структурами даних, такими як масиви або об'єкти. Для роботи з YAML файлами в Bash, нам потрібно використовувати утиліту `yq`, яка є частиною пакету `yq` у Unix-подібних операційних системах.

Ось приклад використання команди `yq` для створення нового YAML файлу:

```Bash
yq eval '.key = "value"' > new_file.yaml
```
У цій команді ми використовуємо опцію `eval`, щоб дозволити обробку JavaScript-коду у YAML файлі. Надалі, ми встановлюємо значення для ключа `key` рівним "value" і зберігаємо новий файл під назвою `new_file.yaml`.

Щоб отримати значення з YAML файлу, ми можемо використовувати такий синтаксис:

```Bash
yq eval '.key' file.yaml
```
Ця команда поверне значення ключа `key` з файлу `file.yaml`.

## Розкопки
Що ж, ви можете запитати, чому використовувати Bash для роботи з YAML, коли є багато інших мов програмування, що підтримують цей формат даних. А ось чому:

- Швидкість: Bash є дуже швидкою мовою в порівнянні з іншими високорівневими мовами програмування, такими як Python або Ruby. Це означає, що робота з великими YAML файлами може бути значно швидшою за допомогою Bash.
- Доступність: Bash є стандартною оболонкою у більшості Unix-подібних операційних системах, що означає, що ви можете використовувати його безкоштовно та без додаткових інсталяцій.
