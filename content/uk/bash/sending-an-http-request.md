## Чому

В програмуванні HTTP-запитів може бути корисно для зв'язку з серверами та обміну даними в інтернеті. 

## Як це зробити

Найпростіший спосіб надіслати HTTP-запит у Bash - це за допомогою команди `curl`. Вона дозволяє відправляти запити на будь-яку URL-адресу та отримувати відповідь в форматі тексту або JSON.

```
curl https://example.com
```

У даному прикладі ми надсилаємо GET-запит на веб-сайт `example.com`. Результатом буде отримання HTML-коду головної сторінки сайту у відповідь.

Також можна задати додаткові параметри команді `curl`, наприклад, вказати метод, заголовки або передати дані у тілі запиту.

```
curl -X POST -H "Content-Type: application/json" -d '{"name": "John", "age": 25}' https://example.com/api
```

У цьому прикладі ми надсилаємо POST-запит на веб-сайт `example.com/api` з JSON-даними у тілі запиту. Результатом буде отримання відповіді з сервера з кодом 200, що означає успішне виконання запиту.

## Глибока занурення

HTTP-запити базуються на протоколі передачі даних HTTP, що використовується для комунікації між клієнтом та сервером. Цей протокол визначає правила та формати обміну даними, такі як методи, заголовки та тіло запиту.

У Bash HTTP-запити можна надіслати також за допомогою команд `wget` або `nc` (netcat), але команда `curl` є найбільш популярною та потужною для роботи з HTTP.

Надсилання HTTP-запитів є важливою частиною для автоматизації та зв'язку програм, що взаємодіють з веб-серверами або API. У цій області можна докладніше ознайомитися із структурою та особливостями HTTP-запитів.
