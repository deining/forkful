## Чому

З більшим розвитком програмування і важливістю тестування у сфері ІТ, написання тестів стає необхідним елементом в процесі розробки програмного забезпечення. Це допомагає забезпечити високу якість коду, зменшити кількість помилок та збільшити ефективність розробки. Тому, написання тестів є важливою навичкою для будь-якого програміста.

## Як

Написання тестів у Bash може бути досить простим застосуванням команди `test`. Нижче наведено приклад коду, який перевіряє, чи є певна змінна додатнім числом та виводить відповідне повідомлення:

```Bash
num = 10
if test $num -gt 0; then
    echo "Змінна є додатнім числом"
else
    echo "Змінна не є додатнім числом"
fi
```

Вихідний код нашого тесту буде мати наступний вигляд:

```Bash
Змінна є додатнім числом
```

Цей приклад можна використати для перевірки різних умов, додавши нові блоки `if/else` за необхідністю.

## Глибока навалія

Окрім базових команд для тестування, в Bash є багато інших інструментів, які можна використовувати для написання більш складних тестів. Наприклад, можна використовувати оператор `&&`, щоб перевірити кілька умов одночасно, або використовувати цикли `while` та `for` для перебору елементів.

Також варто згадати про фреймворк для написання тестів, такий як Bats, який спрощує процес тестування та дозволяє писати більш організований та читабельний код.
