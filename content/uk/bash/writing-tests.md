# Чому

Навіть для початківців у програмуванні, написання тестів є важливою частиною розробки. Тести допомагають перевірити правильність коду, зменшити кількість помилок та полегшують роботу над проектом. Отже, написання тестів надає багато переваг для будь-якого програміста.

# Як

Написання тестів у Bash може здатися складним, проте це досить просто, якщо знаходити підхід до цього правильно. Нижче наведені декілька прикладів коду та очікуваного виводу.

```Bash
# Приклад перевірки виведення повідомлення

# Створення функції для перевірки тексту
message(){
  echo "Hello World!"
}

# Виклик функції та перевірка виведення "Hello World!"
if [[ $(message) == "Hello World!" ]]; then
  echo "Test passed"
else
  echo "Test failed"
fi

# Вивід: "Test passed"
```

```Bash
# Приклад перевірки додавання чисел

# Створення функції для додавання чисел
add(){
  result=$(( $1 + $2 ))
  echo $result
}

# Виклик функції та перевірка результату
if [[ $(add 5 10) -eq 15 ]]; then
  echo "Test passed"
else
  echo "Test failed"
fi

# Вивід: "Test passed"
```

На додаток до прикладів, важливо використовувати ключове слово `test` для перевірки умов у тестах.

# Deep Dive

Перевірка результатів написаного коду є ключовою частиною написання тестів. Вдало попереднє розуміння умов та уміння коректно використовувати команду `test` може забезпечити правильну перевірку в коді та допомогти виявити помилки. Також варто згадати про ключове слово `assert`, яке дозволяє перевірити, чи результат очікуваний. Для більш докладної інформації про написання тестів у Bash, рекомендуємо прочитати документацію за посиланням нижче.
