## Чому

Запитання без помилок є невідємною частиною будь-якого програмування. Воно допомагає уникнути багатьох проблем та забезпечити якість коду. У цій статті ми розглянемо, як написати тести в Bash, щоб зробити свої програми більш надійними та ефективними.

## Як це зробити

Найпоширенішим способом написання тестів в Bash є використання команди `test` або `[[ ]]`. Давайте розглянемо декілька прикладів з цими командами:

```Bash
# Перевірка чи файл існує

if [ -f file.txt ] # -f означає "файл існує"
then
  echo "Файл знайдено!"
else
  echo "Файл не існує."
fi

# Перевірка чи число є додатним

read -p "Введіть число: " num

if [ $num -gt 0 ] # -gt значить "більше за"
then
  echo "Ви ввели додатнє число."
else
  echo "Ви ввели від'ємне число."
fi
```

Вивід:

```
Файл знайдено!
Введіть число: 10
Ви ввели додатнє число.
```

Існує також можливість використовувати логічні оператори, такі як `&&` (логічне І) та `||` (логічне АБО) для написання більш складних умов. Наприклад:

```Bash
# Перевірка чи число є парним та більшим за 10

read -p "Введіть число: " num

if [[ $num % 2 -eq 0 && $num -gt 10 ]] # -eq означає "дорівнює"
then
  echo "Число є парним та більшим за 10."
else
  echo "Не виконано."
fi
```

Вивід:

```
Введіть число: 12
Число є парним та більшим за 10.
```

## Глибокий погляд

У разі коли в програмі використовуються багато вхідних даних і багато умов, писати всі ці перевірки вручну може бути досить втомлюючим та часозатратним процесом. Тому було створено фреймворк для написання тестів в Bash - `BATS` (Bash Automated Testing System). Він дозволяє автоматизувати процес тестування та допомагає зберегти час та уникнути помилок.

Наприклад, за допомогою `BATS` ми можемо легко перевірити, чи повертає функція правильне значення:

```Bash
@test "Додавання" {
  run ./calc.sh add 3 5
  [ "$status" -eq 0 ]
  [ "$output" -eq 8 ]
}
```
