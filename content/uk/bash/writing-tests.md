## Що і чому?
Написання тестів - це процес перевірки коду для ефективності та точності. Програмісти роблять це, щоб мати впевненість, що їх код працює як задумано та не містить помилок.

## Як:
Нижче наведені приклади коду та виводу з використанням ```Bash ...``` блоків.

Приклади коду для тестування назви файлу:

```Bash
#!/bin/bash
# Тестування назви файлу

filename="test_file.txt"

if [ -f "$filename" ] 
then
    echo "$filename існує."
else
    echo "$filename не існує."
fi
```

Вивід:

```Bash
test_file.txt існує.
```

## Глибоке занурення:
Написання тестів є важливою частиною процесу розробки програмного забезпечення та допомагає підтримувати якість коду. Раніше, використання тестів було складнішим, але з появою Bash та інших інструментів для автоматичного тестування, цей процес став більш доступним для програмістів. Є також альтернативи, як наприклад платформа Travis CI, яка дозволяє автоматично запускати тести при кожному оновленні коду в репозиторії. Для написання тестів використовують різні методи, включаючи ручне написання тестових сценаріїв та використання спеціальних програм для автоматизації процесу тестування.
