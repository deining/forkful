## Чому 

Писання до стандартного помилкового потоку може бути корисним для відлагодження та виведення повідомлень про помилки під час виконання скрипту в Bash. Це також може бути корисним під час роботи зі складними або великими програмами, де потрібно виводити різні типи повідомлень у різні потоки.

## Як

Приклад коду:
```Bash
#!/bin/bash
# Це приклад скрипта, який виводить повідомлення про помилку до стандартного помилкового потоку
echo "Стартуємо скрипт..."
# Це приклад функції, яка відображає повідомлення про помилку
function error_message {
  echo "Помилка: $1" >&2 # >&2 означає виводити до стандартного помилкового потоку
}
# Виклик функції з можливістю передачі текстового повідомлення про помилку
error_message "Неправильний параметр!"
```

Приклад виведення до стандартного помилкового потоку:
```
Помилка: Неправильний параметр!
```

## Глибоке занурення

Запис до стандартного помилкового потоку в Bash може бути виконаним за допомогою символу `>&2`, який перенаправляє виведене повідомлення до стандартного помилкового потоку замість стандартного потоку виведення. Це дозволяє виділити повідомлення про помилку в окремий потік, що дозволяє легше переглядати та відлагоджувати.
