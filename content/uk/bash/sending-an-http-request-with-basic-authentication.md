## Чому 

Якщо ви програміст або студент, який вивчає Bash, ви, можливо, зацікавлені в написанні скриптів, які здійснюють HTTP запити з основною аутентифікацією. Це дозволяє вам взаємодіяти з веб-сервером, який потребує унікального логіна та пароля для доступу до своїх ресурсів.

## Як

Для того, щоб надіслати HTTP запит з основною аутентифікацією використовуючи Bash, ми будемо використовувати утиліту `curl`. Ця утиліта дозволяє нам виконувати HTTP запити з командного рядка. Щоб надати управління аутентифікацією, ми будемо використовувати опцію `--user` і передавати туди логін та пароль в форматі `логін:пароль`. Ось приклад коду з виведенням результату запиту:

```Bash
curl --user username:password http://example.com/api/resource
```

Вивід:

```
{"id": 1, "name": "John Doe"}
```

Це описує простий запит GET до API, який повертає дані про користувача після авторизації з допомогою основної аутентифікації. Ви можете також використовувати утиліту `curl` для надсилання інших типів запитів, таких як POST або PUT, та підтримувати різні формати виведення, використовуючи різні опції.

## Глибша підготовка

Базова аутентифікація - це простий і недостатньо безпечний спосіб авторизації. Вона передає логін та пароль у відкритому вигляді, тому не рекомендується використовувати її для надійного захисту ресурсів. Якщо ви використовуєте базову аутентифікацію для своїх проектів, рекомендується додати шифрування або використовувати більш безпечні методи авторизації. Також варто звернути увагу на збереження логіна та пароля в безпечному місці і не передавати їх у відкритому вигляді.
