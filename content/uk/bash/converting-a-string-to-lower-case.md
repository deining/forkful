## Чому

Конвертація рядка до нижнього регістру може бути корисною для багатьох завдань, таких як знаходження співпадінь у тексті, робота з базами даних та багато іншого. Все це можна зробити швидше та ефективніше, якщо рядок буде в нижньому регістрі, тому ця функція є корисною для будь-якого, хто працює з рядками в Bash.

## Як зробити

Існує кілька шляхів для конвертації рядка до нижнього регістру в Bash. Один з них - використовувати утиліту ```tr```, яка змінює регістр всіх символів у вказаному рядку. Давайте подивимось на приклад:

```bash
# Початковий рядок
input="Привіт, СВІТ!"

# Використовуємо утиліту `tr`, для зміни регістру
output=$(echo "$input" | tr [:upper:] [:lower:])

# Виводимо результат
echo "$output"
# Результат: привіт, світ!
```

Інший спосіб - використовувати змінну ```${variable,,}```, яка змінює регістр всіх символів у вказаній змінній. Подивимось на цей приклад:

```bash
# Початковий рядок
input="Привіт, СВІТ!"

# Змінюємо регістр у змінній `input`
output="${input,,}"

# Виводимо результат
echo "$output"
# Результат: привіт, світ!
```

Як бачите, обидва способи дають одинаковий результат. Ви можете вибрати той, що вам більш зручний.

## Глибше занурення

Незважаючи на те, що конвертація рядка до нижнього регістру може здаватися простою задачею, внутрішній механізм виконання цієї операції не є таким простим. Під час конвертації, Bash додає до рядка додатковий символ ```\e``` перед кожним символом, для вказання зміни регістру. Це робиться для того, щоб виконати запит до зміни регістру не самим рядком, а шляхом роботи з командним рядком. Це може здаватися складною технікою, тому краще просто використовувати вбудовані функції, як показано вище.
