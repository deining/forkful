## Чому

Люди часто зіштовхуються з необхідністю порівняння дат в своїх програмах. Це може бути корисно для визначення періодів часу, для розрахунку різниці між датами або для подібних завдань. У цій статі ми детальніше поговоримо про порівняння двох дат у Bash.

## Як

Для порівняння дат у Bash ми можемо використовувати умовні вирази та команду `test`. Розглянемо наступні приклади:

```Bash
# Порівняння за допомогою умовного виразу
if [[ дата1 -eq дата2 ]]; then
    echo "Дата1 дорівнює даті2"
fi

# Порівняння за допомогою команди test
if test дата1 -gt дата2; then
    echo "Дата1 більша за дату2"
fi
```

У цих прикладах ми використовуємо опції `-eq` (дорівнює) та `-gt` (більше за) для визначення порівняння дат. Також, ми можемо використовувати інші опції, такі як `-lt` (менше за), `-ge` (більше або дорівнює), `-le` (менше або дорівнює) для інших типів порівняння.

Розглянемо приклад використання дат у форматі timestamp:

```Bash
# Переведення дат у timestamp
date1=$(date -d "01/01/2020" +"%s") # дата1 рівна 1577836800
date2=$(date -d "02/01/2020" +"%s") # дата2 рівна 1580515200

# Порівняння двох дат за timestamp
if test $date1 -eq $date2; then
    echo "Дата1 дорівнює даті2"
fi
```

У цьому прикладі ми використовуємо команду `date`, щоб перевести дати у формат timestamp (кількість секунд, що пройшла з 1 січня 1970 року). На цей раз ми використовуємо умовний вираз з опцією `-eq`, яка перевіряє рівність двох значень.

## Глибокий занурення

Більш детально розглянемо типи дат, які можна порівнювати у Bash.

### Дата у звичайному форматі
Цей тип дати має формат `MM/DD/YYYY`, де MM - місяць, DD - день, YYYY - рік.

### Дата у форматі timestamp
Як ми вже вказали раніше, це кількість секунд, що пройшла з 1 січня 1970 року.

### Дата у форматі Unix
Цей тип дати має формат `ГГГГММДДЧЧММ.СС` і використовується у системі Unix.

Цей тип дати має формат `YYYY-MM-DDTH
