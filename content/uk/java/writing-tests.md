## Чому

Написання тестів є важливою частиною процесу програмування в Java. Це дозволяє перевірити правильність роботи коду та знизити кількість помилок у програмі. Без тестів може бути важко виявити та виправити проблеми у коді, тому написання тестів є необхідною умовою для створення якісної програми.

## Як це зробити

Написання тестів в Java не є складною задачею. Ось приклад коду для перевірки правильності функції, яка додає два числа:

```Java
public int add(int a, int b){
  return a + b;
}
```
```Java
@Test
public void testAdd(){
  int result = add(3, 5);
  assertEquals(8, result);
}
```

У цьому прикладі використовується вбудована бібліотека JUnit, яка надає ряд функцій для перевірки роботи коду. Після запуску цього тесту, ми отримаємо виведення "Вірно!", що означає, що функція працює правильно.

Можна також створити більш складні тести, які перевірятимуть не тільки правильність роботи функцій, але і виконання певних умов. Для цього можна використовувати фреймворки, наприклад, Mockito або PowerMock, які дозволяють маніпулювати об'єктами у коді тестування.

## Глибше

При написанні тестів слід звертати увагу на кожну частину коду та враховувати всі можливі сценарії виконання. Це допоможе покрити якомога більше коду тестами та зменшити ймовірність виявлення помилок у програмі після випуску.

Також важливо дотримуватися деяких правил при написанні тестів, наприклад, давати зрозумілі назви для тестів та дотримуватися їх структури та логіки. Це дозволить легше зорієнтуватися у великих проектах та виявити проблеми у випадку їх виникнення.
