## Чому

У програмуванні існує важлива техніка виведення відладочної інформації, яку можна використовувати для відстеження та розбору проблем у ваших кодах. Незважаючи на те, що це може здатися рутинним або непотрібним, виведення відладочної інформації може бути цінним інструментом для вирішення проблем у вашому коді.

## Як

Використання методу ```System.out.println()``` є одним зі способів виведення відладочної інформації у вашому Java коді. Наведені нижче приклади показують, як використовувати цей метод для виведення значень змінних і повідомлень про стан вашого коду.

```Java
int number = 5;
System.out.println("Значення number: " + number);

String message = "Привіт усім!";
System.out.println(message);
```

Вищезазначені приклади виведуть наступне:

```
Значення number: 5
Привіт усім!
```

## Глибокий занурення

Наведені приклади є лише одними з безлічі способів робити виведення відладочної інформації у вашому коді. Є багато інших методів, таких як ```System.err.println()```, ```Logger``` і ```Debug()```, які можна використовувати для цієї мети. Ви також можете використовувати умовні оператори, такі як ```if```, для визначення, коли потрібно виводити відладочну інформацію, і коли ні. 

Останній совет - не забувайте видаляти виведену відладочну інформацію, коли ваш код готовий до переносу в продакшен. Жоден користувач не хоче бачити зайвих повідомлень у консолі.
