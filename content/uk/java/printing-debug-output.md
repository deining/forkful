## Чому

Ви, напевно, замислюєтесь, чому потрібно виводити відладочну інформацію (debug output)? Ну, все просто! Виводження відладочної інформації допомагає програмістам зрозуміти, що трапляється в коді під час виконання програми. Це надає можливість виявити помилки та виправити їх, що дозволяє програмі вишкроювати свій повний потенціал.

## Як виводити відладочну інформацію

Для виводу відладочної інформації використовуються методи ```System.out.println()``` або ```System.out.print()```, які дозволяють виводити повідомлення на екран. Наприклад:

```Java
int x = 5;
System.out.println("Змінна x має значення: " + x);
```
Виходом буде:
```
Змінна x має значення: 5
```
Також можна виводити відладочну інформацію в консоль, використовуючи методи класу ```java.util.log``` або шляхом використання логгера (logger). Це дозволяє зберігати історію виводу та відстежувати помилки.

## Глибоке вивчення

Окрім звичайного виводу повідомлень, ви також можете додавати до них додаткову інформацію, використовуючи різні методи форматування рядків. Наприклад, можна використати символи перенесення рядка ```\n``` для оформлення виводу в рядки або використовувати метод ```String.format()``` для використання змінних у виводі.
