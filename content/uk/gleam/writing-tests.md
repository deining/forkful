## Чому

Писати тести є важливою частиною процесу програмування. Вони допомагають переконатися, що ваш код працює правильно і захищений від змін.

## Як

Написання тестів у Gleam є простим та зрозумілим процесом. Для початку, необхідно імпортувати бібліотеку `testing` та оголосити тести у своєму модулі:

```
import testing

test "Додавання чисел" {
  assert_equal(2, 1 + 1)
}

```

Цей приклад містить тест, який перевіряє, чи дорівнює результат додавання чисел `1` та `1` значенню `2`. Якщо тест виконується правильно, ви побачите вивід `test "Додавання чисел"` success.

## Глибока погруження

Написання хороших тестів може зайняти час, але це варто зусиль. Коли ви пишете тести, ви можете розкрити помилки та забезпечити ще більшу стабільність вашої програми.

Наприклад, ви можете перевірити поведінку функції при неправильних вхідних даних:

```
test "Ділення на 0" {
  assert_error(%, 1, 0)
}

```

Такий тест повинен повернути помилку `DivByZero`, що допоможе вам виявити неправильне використання функції у вашій програмі.
