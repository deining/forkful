## Зачем

Написание тестов - это важная часть любого проекта, которая может сэкономить много времени и ресурсов на этапе разработки. Они позволяют обнаружить ошибки и проблемы до того, как они станут критическими. Благодаря тестам вы можете уверенно вносить изменения в свой код, зная, что все работает должным образом.

## Как

Начнем с создания нового проекта в Gleam. Для этого запустите команду `gleam new my_project` в терминале. Затем откройте созданный проект в любимом редакторе и перейдите в папку `tests`. В этой папке вы найдете файл `my_project_test.gleam`, который и будет содержать все наши тесты.

```Gleam
// Мы импортируем тестовый модуль
import gleam/test

// Определим нашу функцию, которую мы будем тестировать
pub fn add(a, b) {
  a + b
}

// Объявляем тестовую функцию
pub fn test_add() {
  // Мы ожидаем, что результат сложения 2 и 3 будет равен 5
  // Если это не так, то тест не пройдет
  test.assert_equal(add(2, 3), 5)
}
```

На этапе разработки удобно запускать тесты вручную, однако в дальнейшем вам понадобится автоматизация этого процесса. Для этого в файле `my_project.gleam` нужно добавить команду `gleam test`.

## Deep Dive

Как видно из примера выше, тесты пишутся в том же языке программирования, что и основной код. Это значительно упрощает процесс тестирования и позволяет сохранить единый стиль написания кода. В Gleam также есть возможность использовать различные модули для тестирования клиент-серверных приложений или тестирования интерфейсов. Подробнее об этом можно узнать в документации.
