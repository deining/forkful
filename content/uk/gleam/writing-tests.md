## Чому

В чому перевага написання тестів? Для чого це важливо?

Написання тестів допомагає впевнитися в тому, що ваш код працює правильно і не порушує функціональність інших частин програми. Використання тестів дозволяє вам виявити і виправити помилки на ранніх стадіях розробки, що зберігає час і зусилля у майбутньому.

## Як

Для того, щоб почати писати тести в Gleam, вам необхідно створити новий файл з розширенням `.test.gleam`. Для початку можна створити простий тест, який перевіряє, чи функція `add` правильно додає два числа:

```gleam
import test

test.test("2 плюс 2 дорівнює 4", {
  result = assert.equal(add(2, 2), 4)
})
```

У цьому прикладі ми імпортуємо модуль `test` для використання його методів. У блоках `test.test()` ми можемо передати опис тесту у першому параметрі, а у другому - функцію, яка містить сам тест. У цьому випадку, ми використовуємо метод `assert.equal()` для перевірки, чи дорівнює результат виклику функції `add` до 4.

Тепер, якщо ви запустите цей тест, ви повинні побачити успішний результат. Змініть один з параметрів `add`, наприклад, поміняйте 2 на 3, і запустіть тест знову. Ви повинні побачити повідомлення про помилку, так як результат вже не буде дорівнювати 4.

## Deep Dive

Gleam надає кілька можливостей для написання тестів. Наприклад, ви можете використовувати метод `assert.is_some()` для перевірки, чи є значення Some у Result або замість `test.test()` використовувати `test.test_with()` для передачі додаткових параметрів до функції тестування.

Також, ви можете створювати багато тестів для одної функції, використовуючи блоки `test.suite()` для групування тестів. Такий підхід полегшує організацію тестів у великих проектах.
