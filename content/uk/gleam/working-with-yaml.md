## Чому

Українські читачі, ви можете питати себе: "Чому мені варто спробувати працювати з YAML у Gleam?". Ну, тут є кілька причин! 

Перш за все, його легко зрозуміти та використовувати. YAML (YAML Ain't Markup Language) є текстовою мовою, спеціально призначеною для зберігання та представлення даних. Це означає, що ви можете легко читати та змінювати дані у YAML форматі, що робить його ідеальним для використання у веб-розробці, розробці додатків та багатьох інших проектах.

Крім того, Gleam надає простий та елегантний спосіб роботи з YAML. Ви можете швидко та легко зчитувати та записувати дані у YAML форматі за допомогою вбудованої функції "yaml".

## Як

Зараз ми розглянемо, як працювати з YAML у Gleam. Нижче наведені декілька прикладів коду та їхні вихідні дані.

```Gleam 
import gleam/yaml

// Зчитування даних у YAML форматі
let data = yaml.from_string("name: Gleam
age: 2
languages:
  - Erlang
  - Elixir
  - Rust")

// Виведення значення
println(data["name"]) // Виведе "Gleam"

// Запис даних у YAML форматі
let new_data = yaml.to_string(data)
println(new_data) // Виведе "name: Gleam
age: 2
languages:
  - Erlang
  - Elixir
  - Rust"
```

## Глибокий занурення

Тепер, коли ми зрозуміли, як працювати з YAML у Gleam, давайте глибше розберемося з цією темою. YAML підтримує різноманітні типи даних, такі як рядки, числа, булеві значення та масиви. Ви також можете вкладати дані одне в одного, що дозволяє створювати більш складні структури даних.

Зверніть увагу, що Gleam надає можливість задавати деякі перевизначення під час читання YAML, наприклад, можна задати, який тип даних повинен бути прочитаним з числа за замовчуванням.

Для отримання більш детальної інформації про роботу з YAML у Gleam, дивіться документацію до пакету "yaml" на офіційному сайті Gleam.
