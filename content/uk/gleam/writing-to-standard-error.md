## Той, у кого є сенс писати в стандартну помилку
Цілком можливо, що коли ви пишете свою програму, ви натрапили на помилку або вчинили помилку. У цьому випадку, вам скоріш за все знадобиться записати цю помилку, щоб дізнатися, що ж саме пішло не так. Використання команди за замовчуванням `io:format/2` може бути корисним для цієї задачі.

## Як використати запис до стандартної помилки в Gleam
Бібліотека `eio` надає нам функцію `print/1`, яка дозволяє нам використовувати команду `io:fwrite/2` для запису до стандартної помилки. Вона приймає як параметр строку, яку ви хочете написати до стандартної помилки.

```Gleam 
import eio

fn my_error_message() {
  let message = "Помилка: це не працює."
  eio.print(message)
}
```

Вивід цього коду буде виглядати так:

```
Помилка: це не працює.
```

## Глибинний аналіз запису до стандартної помилки
Використання команди `io:fwrite/2` дозволяє нам здійснювати форматований вивід до стандартної помилки. Це означає, що ми можемо додати змінну до нашого повідомлення і передати значення цієї змінної в другому параметрі функції `print/1`.

```Gleam 
import eio

fn my_error_message(error) {
  let message = "Помилка {}: це не працює."
  eio.print(message, [error])
}
```

Зверніть увагу на `[]` в другому параметрі функції `print/1`, що вказує на те, що це масив з одним елементом. Як результат, вивід буде виглядати так:

```
Помилка 500: це не працює.
```
