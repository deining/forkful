## Чому

Перевірка наявності каталогу може бути необхідною для програмістів, які розробляють додатки, які працюють з файловою системою. Це може допомогти уникнути помилок і зберегти час, перед тим як намагатися звернутися до неіснуючого каталогу. 

## Як

```Gleam
fn check_dir(directory) {
    match fs::dir_exists(directory) {
        Ok(exists) => {
            if exists {
                println!("Каталог існує!");
            } else {
                println!("Каталог не існує.");
            }
        },
        Err(error) => {
            println!("Помилка: {}", error);
        }
    }
}

check_dir("директорія/перевірка");
```

В даному коді ми використовуємо функцію `dir_exists()` з модуля `fs`, щоб перевірити, чи існує заданий каталог. Якщо каталог існує, то ми отримаємо повідомлення "Каталог існує!" на екрані. Якщо ж каталог не існує, то виведеться повідомлення "Каталог не існує." У разі якщо виникне помилка, то на екрані буде виведено повідомлення про неї.

## Детальний аналіз

При роботі з файловою системою важливо перевіряти наявність каталогів, перед тим як приступати до роботи з ними. Це може допомогти уникнути непередбачуваних ситуацій, таких як виконання операцій з неіснуючими каталогами або падіння програми через помилки. Модуль `fs` в Gleam дозволяє перевіряти наявність каталогів за допомогою функції `dir_exists()`. Ця функція повертає значення типу `Result`, яке містить `Ok(exists)` у разі, якщо каталог існує і `Err(error)` у разі, якщо виникла помилка. Тому ми використовуємо пошукові вирази `match` для обробки цих випадків і виконання необхідних дій.
