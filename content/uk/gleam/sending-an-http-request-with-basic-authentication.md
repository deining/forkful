##Чому

Основна причина виконання HTTP-запиту з базовою аутентифікацією полягає у зв'язку з веб-сервером, який вимагає ідентифікаційних даних для отримання доступу до своїх ресурсів. Ця методика є широко використовуваним способом захисту веб-додатків від несанкціонованого доступу.

##Як

Перед тим як надсилати HTTP-запит з базовою аутентифікацією, необхідно встановити авторизаційні дані у вигляді логіну та паролю. Для цього використовуються два ключові слова `BasicAuth.Username` та `BasicAuth.Password`, які приймають відповідне значення. Приклад реалізації коду у мові Gleam:

```Gleam
let request =
  Request.builder()
  |> Request.set_url("example.com")
  |> Request.set_header("Authorization", BasicAuth.build("username", "password"))
  |> Request.send()
```

Результатом виконання коду стане успішний HTTP-запит з базовою аутентифікацією, що дозволяє отримати доступ до ресурсів веб-сервера.

##Глибше копання

Метод базової аутентифікації описаний в стандартному RFC 7617, який встановлює спосіб передачі ідентифікаційних даних у вигляді логіну та паролю та вказує на необхідність кодування цих даних у форматі Base64. У випадку якщо авторизаційні дані введені некоректно або відсутні, сервер повернеться HTTP-статус 401 Not Authorized, який свідчить про невдалу спробу доступу.

##Дивись також
