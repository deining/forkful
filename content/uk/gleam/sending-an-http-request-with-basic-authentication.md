Що і чому?

Відправлення HTTP-запиту з базовою аутентифікацією - це методіка, яка дозволяє програмістам отримувати доступ до захищених областей веб-сайтів або додатків. За допомогою особистих ідентифікаторів, таких як ім'я користувача та пароль, HTTP-запит з базовою аутентифікацією дозволяє перевірити автентичність користувача та дозволити доступ до захищених даних.

Як це зробити?

Використовуючи мову програмування Gleam, можна виконати HTTP-запит з базовою аутентифікацією з допомогою наступного коду:

```
жог
імортеяр "gleam / http"
імортеяр "gleam / base64"

уиконати(http.запрос()
  .метод("GET")
  .url("https://example.com/protected")
  .аутентифікація(base64.кодування("username:password"))
)
```
Результатом буде веб-сторінка або дані з захищеного джерела. У разі невірного ім'я користувача або пароля, буде повернуто помилку аутентифікації.

Глибокий погляд

HTTP-запит з базовою аутентифікацією був першим способом використовувати ідентифікатори для доступу до захищених даних у веб-серверах. Як альтернативу, можна використовувати більш сучасні методи, такі як OAuth або ключі API. У реалізації HTTP-запиту з базовою аутентифікацією, ідентифікатори можуть бути передані в заголовку аутентифікації або в URL-adress.

Для ознайомлення з іншими способами автентифікації в HTTP, рекомендуємо переглянути матеріали про OAuth та ключі API.
