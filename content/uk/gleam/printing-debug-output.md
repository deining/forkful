## Чому:

Друк відладочної інформації може бути важливим етапом у процесі написання програми. Це може допомогти відстежувати і виправляти помилки в коді, а також зрозуміти, як працює програма.

## Як це зробити:

Перший крок - додати модуль отладки у проєкт:

```
Gleam debug
```

Потім можна використати функцію `debug.trace` для виведення різної інформації у консоль:

```
Gleam debug.trace("Ця функція виконується")
```

Цей код виведе рядок "Ця функція виконується" у консоль під час виконання програми. Також можна вивести значення змінної у консоль, щоб перевірити, чи вона має правильне значення:

```
let some_variable = "Some value"
Gleam debug.trace(some_variable)
```

Цей код виведе "Some value" у консоль. Не забувайте видалити всі виклики функції `debug.trace` перед тим, як завершити написання програми.

## Глибоке занурення:

Іноді буває потрібно вивести більш докладну інформацію, наприклад, стек викликів функцій або значення змінних на певних етапах виконання коду. Це можна зробити за допомогою функції `debug.inspect`:

```
let some_list = [1, 2, 3]
let result = some_list
|> List.map(fn(n) { n * 2 })
|> List.filter(fn(n) { n > 3 })

Gleam debug.inspect(result)
```

Цей код виведе у консоль список `[4, 6]`, що показує результати виконання функцій `List.map` і `List.filter`.

Ви також можете використовувати логічні умови для виведення інформації лише у визначених сценаріях, що є дуже зручним для відлагочення:

```
let some_number = 10

Gleam debug.trace("Це число менше 15") when some_number < 15
```

Цей код виведе рядок "Це число менше 15" у консоль лише у випадку, якщо значення змінної `some_number` менше 15.
