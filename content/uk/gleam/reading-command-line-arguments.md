## Чому

Командний рядок - неодмінна частина програмування, але використання аргументів командного рядка може бути особливо корисним, коли ви хочете змінити поведінку своєї програми, не вносячи зміни у сам код. Це дозволяє користувачу вказувати параметри при запуску програми, що робить його більш гнучким та необхідним для різних сценаріїв використання.

## Як

Щоб прочитати аргументи командного рядка у Gleam, використовуйте функцію `gleam/io/cmdline:read_args` з модуля `gleam/io/cmdline`. Ця функція повертає список аргументів у вигляді рядка, які можна прочитати за допомогою функції `String.to_bytes` для подальшого аналізу. Для прикладу, якщо ми запустимо програму з наступними аргументами `hello world`, то вивід буде виглядати наступним чином:

```Gleam
let args = gleam/io/cmdline:read_args()
gleam/io:format("Вхідні аргументи: {}", Tuple.to_list(args))
```

А вивід виглядатиме так:

```
Вхідні аргументи: ["hello", "world"]
```

## Глибока занурення

Крім функції `read_args`, у Gleam також є можливість читати інші аргументи, такі як прапори (флаги) та значення ключів. Для цього використовуйте функцію `gleam/io/cmdline:read_options`, яка повертає кортеж з двома списками - перший містить прапори, а другий - значення ключів. Наприклад, якщо ми запустимо програму з такими аргументами `--hello "Привіт"`

```Gleam
let {flags, options} = gleam/io/cmdline:read_options()
gleam/io:format("Прапори: {}", Tuple.to_list(flags))
gleam/io:format("Ключі: {}", Tuple.to_list(options))
```

Вивід буде наступним:

```
Прапори: ["--hello"]
Ключі: ["Привіт"]
```

Також, важливо пам'ятати про порядок аргументів при використанні функції `read_options`. Якщо ми запустимо нашу програму з аргументами `--age 25 --name "Іван"`, то порядок виведених значень буде відповідним - `["25", "Іван]`.
