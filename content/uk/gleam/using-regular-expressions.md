# Чому

Використання регулярних виразів дозволяє ефективніше та точніше обробляти текстові дані в програмуванні. Вони можуть бути корисними в багатьох сценаріях, таких як пошук та заміна певних шаблонів у тексті, перевірка правильності введеного користувачем даних, або фільтрація великих обсягів даних.

## Як це зробити

Для того, щоб розпочати використовувати регулярні вирази в Gleam, спочатку необхідно імпортувати модуль ```Regex```. Потім можна створити об'єкт регулярного виразу за допомогою функції ```Regex.from_string```, вказавши потрібний шаблон у вигляді рядка. Наприклад, якщо нам потрібно знайти всі слова, що починаються з букви "a" у тексті, можна використовувати шаблон ```"a\w+"```. 

Після створення об'єкту можна застосовувати різні методи для пошуку, заміни або перевірки збігів з даною регуляркою. Наприклад, функція ```Regex.find_all``` поверне список з усіма збігами, а метод ```Regex.replace``` може використовуватись для заміни збігів на певний текст. 

```Gleam
import Regex

regex = Regex.from_string("a\w+")
string = "I have an apple and a banana."
matches = Regex.find_all(regex, string)
// виведе ["apple", "and", "a"]
new_string = Regex.replace(regex, string, "a fruit")
// виведе "I have a fruit and a fruit."
```

## Глибоке погруження

Основою регулярних виразів є спеціальні символи, що описують певні шаблони та умови для пошуку. Наприклад, символ ```\w``` відповідає будь-якому символу слова, а ```+``` вказує на наявність одного або більше повторень попереднього символу. Багато символів може мати різні спеціальні значення залежно від контексту.

Також важливо знати про такі аспекти, як регістрозалежність, яка впливає на збіги регулярки з великими та малими літерами, нульова довжина, яка описує обрану частину тексту без наявності конкретного символу, та інші. Більше про це можна дізнатися у документації з регулярних виразів у Gleam.
