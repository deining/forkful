## Чому

В програмуванні часто зустрічається ситуація, коли потрібно перевірити виконання певної частини коду або виявити помилку. Для цього часто використовуються спеціальні команди, які виводять необхідну інформацію в консолі чи лог-файлі. У цьому блозі ми розглянемо детальніше процес друку відладочних повідомлень у мові програмування C# та навчимося правильно використовувати цей інструмент.

## Як це зробити

Для того, щоб друкувати відладочні повідомлення, ми використовуємо метод `Console.WriteLine()` у C#. Приклад коду з виведенням повідомлення "Hello World!" у консолі:

```C#
Console.WriteLine("Hello World!");
```

В результаті ви побачите у консолі повідомлення "Hello World!".

Також, для того, щоб зробити відладку більш зручною, можна додати до повідомлення значення змінних, які використовуються у даній частині коду. Приклад використання змінних:

```C#
string name = "Oksana";
int age = 25;
Console.WriteLine($"My name is {name} and I am {age} years old.");
```

У результаті ви побачите повідомлення "My name is Oksana and I am 25 years old.".

## Глибоке занурення

При роботі з великими проектами декілька рівнів відладки можуть бути дуже корисними для виявлення проблем. У C# існує можливість використання різних рівнів відладки, які дозволяють виводити різну інформацію залежно від поточного стану програми.

Наприклад, для виведення повідомлення з рівнем важливості "Debug" використовується метод `Debug.WriteLine()`. Приклад:

```C#
string text = "This is a debug message.";
Debug.WriteLine(text);
```

Також, для більш точної відладки можна використовувати різні флаги, які встановлюються для окремих повідомлень та дозволяють розмежовувати їх в залежності від важливості.
