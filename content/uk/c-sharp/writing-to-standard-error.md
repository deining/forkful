## Чому
Ви можете запитати: "Навіщо я маю писати в стандартну помилку?". На жаль, навіть у найкращих програмах можуть виникати помилки. Писати в стандартну помилку допомагає з'ясувати причину цих помилок та дозволяє легше відлажувати програму.

## Як
```C#
try
{
    // код, який може викликати помилку
}
catch (Exception e)
{
    Console.Error.WriteLine($"Помилка: {e.Message}");
}
```

Вище наведений код показує, як можна застосувати писання в стандартну помилку в C#. У розділі `try` виконується код, який може викликати помилку. Якщо помилка виникне, то вона буде перехоплена у розділі `catch` та повідомлена користувачеві через `Console.Error.WriteLine()`. Це допоможе вам виявити та виправити помилки у вашій програмі.

## Глибоке занурення
В C# існує кілька способів запису в стандартну помилку. Окрім використання `Console.Error`, ви можете також скористатися `Trace` та `Debug`. Вони дозволяють писати в стандартну помилку з різним рівнем деталей, що може бути корисним для відлагодження. Також, ви можете використовувати `TextWriter` для записування власних повідомлень в стандартну помилку.
