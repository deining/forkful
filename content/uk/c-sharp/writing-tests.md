## Чому

Коли ми програмуємо, ми часто просто пишемо код, перевіряємо його роботу і переходимо до наступного завдання. Але чи ви знали, що написання тестів може суттєво полегшити нашу роботу і зекономити час в майбутньому? Тести дозволяють перевірити, чи функціональність нашого коду працює як потрібно, і дозволяють швидко виявити і виправити можливі помилки.

## Як написати тести в C#

Процес написання тестів досить простий і добре підходить для мови програмування C#, оскільки вона має вбудовану підтримку для тестування. Для початку, нам потрібно встановити пакет NUnit, який дозволить нам створювати тести. Ось приклад тесту, який перевіряє, чи метод "Add" коректно додає два числа:

```C#
[Test]
public void TestAddMethod()
{
     Calculator calculator = new Calculator();
     int result = calculator.Add(5, 10);

     Assert.AreEqual(15, result);
}
```

Тут ми спочатку створюємо екземпляр класу, який буде тестувати, потім викликаємо метод "Add" і перевіряємо, чи отримуємо очікуваний результат. Якщо метод коректно додає числа, тест буде пройдений. В іншому випадку, він спрацює і повідомить про помилку.

Щоб перевірити кілька варіантів для одного методу, ми можемо використовувати атрибут "TestCase", який дозволяє передавати різні значення в метод:

```C#
[TestCase(2, 3, 5)]
[TestCase(10, -5, 5)]
public void TestAddMethod(int num1, int num2, int expectedResult)
{
     Calculator calculator = new Calculator();
     int result = calculator.Add(num1, num2);

     Assert.AreEqual(expectedResult, result);
}
```

Цей приклад покаже, як перевірити, чи метод "Add" коректно обробляє декілька різних пар чисел.
