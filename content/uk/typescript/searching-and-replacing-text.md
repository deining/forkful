##Чому

Написання програм - це широка сфера, яка може бути корисною для багатьох галузей. Заміна тексту - це одна з основних задач програмістів. Від маніпулювання даними до створення регулярних виразів, ця процедура є важливим елементом у написанні ефективних програм.

##Як це зробити

Найпростіший спосіб заміни тексту в TypeScript - використання методу `replace ()`. Наприклад, ми можемо замінити всі символи "a" на "b":

```
TypeScript
let str = "This is a sample string.";
let newStr = str.replace(/a/g, "b");
console.log(newStr); // Output: This is b sbmple string.
```

У цьому прикладі ми використовуємо регулярний вираз `/a/g`, щоб знайти всі входження "a" і замінити їх на "b". Ключове слово `g` позначає глобальну заміну, тобто всі входження будуть замінені, а не тільки перше.

Також існує метод `replaceAll ()`, який замінює всі входження без використання регулярних виразів:

```
TypeScript
let str = "This is a sample string.";
let newStr = str.replaceAll("a", "b");
console.log(newStr); // Output: This is b sbmple string.
```

Обидва цих методи повертають нову строку зі зміненим текстом, тому ми можемо зберегти її у змінну та використати далі у нашій програмі.

##Глибоке занурення

У TypeScript також є можливість шукати і заміняти текст за допомогою регулярних виразів. Синтаксис для цього дуже схожий на той, що використовується в JavaScript. Регулярні вирази дозволяють шукати більш складні шаблони тексту, що дозволяє замінювати різні комбінації символів.

Наприклад, за допомогою регулярних виразів ми можемо замінити всі слова, які починаються з букви "a", на слово "apple":

```
TypeScript
let str = "This is a sample string.";
let newStr = str.replace(/\ba\w*/g, "apple");
console.log(newStr); // Output: This is apple apple apple apple.
```

У цьому прикладі ми використовуємо `\b` для пошуку окремих слів, а `\w*` для знаходження будь-якої кількості символів після "а". Також ми встановлюємо `g` для глобальної заміни всіх входжень.

Щоб дізнатися більше про регулярні вирази в TypeScript, можна ознайомитися з [офіційною документацією](https://www.typescriptlang.org/docs/handbook/regular-expressions.html).
