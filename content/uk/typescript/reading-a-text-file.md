Перш за все, текстові файли є популярними форматами для збереження даних, так як вони легкі для читання та редагування. Часто програмісти повинні зчитувати текстові файли для обробки та аналізу даних. У цій статті ми розглянемо, як зчитати текстовий файл за допомогою TypeScript, мови програмування, яка надає сильну типізацію та підтримку ООП.

## Чому

Перш за все, зчитування текстових файлів може бути корисним для обробки великих обсягів даних. Також це може бути корисним для створення скриптів, які автоматизують певні завдання. Наприклад, можливо, вам потрібно аналізувати лог-файли вашого веб-сервера або заповнювати базу даних зі специфікаціями продуктів. Читання текстових файлів допоможе вам швидко та ефективно робити ці задачі.

## Як

За допомогою TypeScript ви можете зчитати текстовий файл за допомогою вбудованої функції `readFile` з пакету `fs`.

```TypeScript
import * as fs from 'fs';

// зчитування текстового файлу
const data = fs.readFileSync('file.txt', 'utf8');
```

У цьому прикладі ми імпортуємо пакет `fs` та використовуємо функцію `readFileSync`, яка зчитує файл і повертає його вміст у вигляді рядка, який ми можемо зберегти у змінній `data`.

Також ми можемо використовувати функцію `readFile`, яка приймає колбек-функцію як параметр.

```TypeScript
// зчитування текстового файлу за допомогою колбек-функції
fs.readFile('file.txt', 'utf8', (error, data) => {
  if (error) throw error;
  console.log(data);
});
```

У цьому прикладі ми передаємо колбек-функцію, яка буде виконана після того, як файл буде зчитаний. Якщо виникне помилка під час зчитування файлу, вона буде виведена у консоль.

## Глибше

Цікавим фактом є те, що TypeScript автоматично інферує типи для зчитаного з файлу значення. Якщо ваш файл містить числа, значення буде інтерпретовано як тип `number`, якщо рядки - як тип `string`. Це забезпечує більшу безпеку та надійність вашого коду.

Крім того, ви можете використовувати бібліотеку `@types/node` для отримання автодоповнення та допомоги під час написання коду, пов'язаного з файловою системою.
