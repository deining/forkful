## Чому
Тестування є важливою частиною розробки програмного забезпечення. Воно дозволяє перевірити, чи працює наш код коректно та визначити помилки, які можуть виникнути під час виконання програми. Зрозуміти, чому слід писати тести, допоможе покращити якість програм та скоротити час налагодження.

## Як
Для написання тестів у TypeScript необхідно використовувати бібліотеку для тестування, наприклад, Jest або Mocha. Далі ми продемонструємо, як написати простий тест для функції додавання за допомогою бібліотеки Jest.

```TypeScript
// Імпорт бібліотеки Jest
import { add } from './calc';

// Описуємо тестовий випадок
describe('Додавання', () => {
  it('Повинно додавати два числа', () => {
    // Очікуваний результат
    const expected = 9;
    // Результат виклику функції додавання
    const result = add(4, 5);
    // Перевірка, чи співпадає очікуваний результат з отриманим
    expect(result).toEqual(expected);
  });
});
```

В результаті виконання цього тесту, ви повинні отримати повідомлення, що тест пройшов успішно.

## Deep Dive
Написання тестів дозволяє забезпечити надійність та стабільність програмного продукту. Важливо створювати якомога більше тестів для перевірки різних варіантів використання програми та усунення потенційних помилок. Також тестування допомагає полегшити роботу команди розробників та збільшити прозорість процесу розробки.
