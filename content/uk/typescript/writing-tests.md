## Що & Чому?
Написання тестів - це процес, коли програміст створює спеціальні фрагменти коду для перевірки правильності роботи іншого коду. Програмісти роблять це, щоб переконатися, що код виконує своє завдання і не має помилок.

## Як:
```TypeScript
// Створення тесту
it('повинен правильно обробити коректні дані', () => {
  let result = деяка_функція (5, 3);
  expect(result).toEqual(8);
});

// Виклик тестів
npm run test
```

В даному прикладі ми створили тест, щоб перевірити, чи функція правильно додає два числа. Використовуючи команду `npm run test`, ми запускаємо всі тести у нашому проекті і дивимося результати.

## Глибока поглибка:
Написання тестів є важливою частиною розробки програмного забезпечення і дозволяє зекономити час і гроші у майбутньому. Існує багато інструментів для написання тестів, таких як Jest, Mocha і Jasmine. Крім того, тести можна використовувати для тестування не тільки коду, але і інтерфейсів та баз даних. Для написання тестів у TypeScript використовуються такі засоби, як Jest або Ava.
