## Чому

Написання тестів є важливою частиною написання програмного коду в будь-якій мові програмування, а в TypeScript воно особливо важливо. Тести дозволяють перевірити правильність роботи коду та запобігти появі багів у майбутньому.

## Як це зробити

Написання тестів у TypeScript є досить простим, але потребує знань певних функцій та концепцій. За допомогою засобів, таких як `describe` та `it`, можна описати поведінку тесту та очікуваний результат. Давайте розглянемо приклад тесту для функції додавання чисел:

```TypeScript
describe('додавання', () => {
  it('повинно повернути правильну суму', () => {
    const result = sum(5, 3);

    expect(result).toEqual(8);
  })
})
```

У цьому прикладі ми описуємо поведінку тесту з допомогою `describe`, а з допомогою `it` вказуємо очікуваний результат застосування функції `sum` до чисел 5 і 3. Далі, за допомогою `expect`, перевіряємо, чи повернув функція правильну суму. Якщо тест не буде пройдений, то отримаємо повідомлення про помилку та можемо внести зміни до коду.

## Основи написання тестів

Щоб бути впевненим у правильності тестів, потрібно мати розуміння основ TypeScript та їх використання при писанні коду. Наприклад, можна використовувати `import` для імпортування потрібних функцій чи класів до тестів. Також, корисно уявляти тести як користувальницькі сценарії, які повинні бути пройдені для того, щоб програма працювала правильно.
