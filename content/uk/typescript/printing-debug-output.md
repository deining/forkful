## Чому

Друк вихідних даних для налагодження часто є необхідною частиною процесу розробки програмного забезпечення. Він дозволяє нам крок за кроком перевіряти правильність нашого коду та виявляти помилки, що допомагає вдосконалювати якість програмного забезпечення.

## Як це зробити

Для друку вихідних даних у TypeScript потрібно використовувати функцію `console.log()`. Наприклад, якщо ми хочемо вивести повідомлення про призначення змінної `name`, ми можемо використовувати такий код:

```TypeScript
let name: string = "Софія";
console.log("Змінна name має значення " + name);
```

Якщо запустити цей код, ми отримаємо наступний вихід:

```
Змінна name має значення Софія
```

Це дуже простий приклад, але він демонструє, як ми можемо використовувати `console.log()` для друку вихідних даних та відстеження значень змінних під час виконання коду.

## Глибоке занурення

Друк вихідних даних також може бути корисним при відлагодженні складних функцій або виявленні причин помилок. Ми можемо використовувати різні види вихідних даних, наприклад, виводити масиви об'єктів, об'єкти зі складними структурами даних або навіть детальний запис послідовності виконання коду.

Крім того, у TypeScript є можливість використання вбудованого модуля `util` для друку більш структурованих вихідних даних за допомогою функції `inspect()`. Це дозволяє нам зручно виводити дані у форматі JSON та легко аналізувати їх.
