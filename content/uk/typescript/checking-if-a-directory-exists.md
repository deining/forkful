## Чому

Перевірка наявності каталогу є важливою частиною програмування в TypeScript. Це дозволяє нам перевірити, чи існує даний каталог перед виконанням дій з ним, такими як створення файлу або читання вмісту каталогу. Це допомагає обробляти помилки та забезпечує більш стійку роботу програми.

## Як

```TypeScript
if(fs.existsSync('./каталог')) {
    console.log('Каталог існує!');
} else {
    console.log('Каталог не існує!');
}

// Виведе "Каталог існує!" якщо каталог існує, в іншому випадку виведе "Каталог не існує!"
```

Перше, що ми робимо, це встановлюємо модуль `fs`, який дозволяє нам взаємодіяти з файловою системою. Далі ми використовуємо метод `existsSync()`, який повертає `true`, якщо файл або каталог існує, і `false`, якщо не існує. Ми передаємо шлях до каталогу у якості аргументу цьому методу. Якщо каталог існує, ми виводимо `Каталог існує!`, у протилежному випадку - `Каталог не існує!`.

## Глибоке дослідження

Що ж робить метод `existsSync()` для перевірки наявності каталогу в TypeScript? Він перевіряє, чи існує файл з таким самим ім'ям, як і каталог, і повертає `true` або `false` відповідно. Це дозволяє нам перевірити наявність каталогу на будь-якому рівні вкладеності. Якщо ваш каталог містить підкаталоги, метод `existsSync()` також перевіряє, чи існують вони, тобто він є рекурсивним. Якщо дослідити детальніше, то ви зможете побачити, як виконується перевірка наявності каталогу на різних операційних системах.
