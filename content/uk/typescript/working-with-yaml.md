## Чому

Найімовірніше, ви вже чули про YAML як формат для збереження структурованої конфігураційної інформації в зрозумілому і легкому для комп'ютерів форматі. Проте, чи знали ви, що YAML також є основним форматом для Front Matter у файлів Jekyll і є часто використовуваним у багатьох інших програмах, що потребують конфігураційних файлів? Навіть якщо ви не маєте великого досвіду з YAML, він може бути корисним у вашій розробці.

## Як

Щоб почати, нам знадобиться встановити бібліотеку, яка дозволить нам читати і записувати YAML файли. Для TypeScript цими можливостями наділені бібліотеки `js-yaml` і `yamljs`. Ось приклад розпарсення YAML файлу за допомогою `js-yaml`:

```TypeScript
import * as fs from 'fs';
import * as jsYaml from 'js-yaml';

const yamlFile = fs.readFileSync('config.yml', 'utf8');
const config = jsYaml.safeLoad(yamlFile);

console.log(config.key); // виведе значення ключа з конфігураційного файлу
```

Також ми можемо використати `yamljs` для того, щоб читати YAML файли безпосередньо у форматі об'єкта:

```TypeScript
import * as yamljs from 'yamljs';

const config = yamljs.load('config.yml');

console.log(config.key);
```

Обидві ці бібліотеки також підтримують запис YAML файлів за допомогою методів `safeDump` та `stringify`.

## Глибоке дослідження

Якщо ви плануєте використовувати YAML в своєму проекті, варто ознайомитися з його структурою. YAML складається з ключів і значень, що розділяються двокрапками та відокремлені пробілами. Крім того, ви можете використовувати масиви, об'єкти та багаторядкові дані. Ці можливості дозволяють вам створювати структуровані конфігураційні файли в зрозумілому форматі.

Також важливо зазначити, що YAML дозволяє використовувати коментарі, щоб пояснити структуру файлу або додаткову інформацію. Це може бути корисним для спільної роботи над конфігураційними файлами і для збереження важливих пояснень для майбутніх розробників.
