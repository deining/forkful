## Чому

Вміст тестів - це важлива частина розробки будь-якої програми. Використання тестів дозволяє пришвидшити процес розробки, покращити якість коду та знизити ймовірність помилок у програмі.

## Як

Написання тестів у C++ не є складною задачею, якщо ви знаєте основи мови програмування. Нижче показано приклад написання тесту за допомогою бібліотеки Google Test:

```C++
#include <iostream>
#include "gtest/gtest.h"

// Вказуємо, яка функція буде тестовою
TEST(squareTest, positiveNumbers) {
    // Очікуваний результат
    int expected = 25;

    // Фактичний результат
    int actual = square(5);

    // Перевірка на рівність
    ASSERT_EQ(expected, actual);
}

int main(int argc, char** argv) {
    // Запуск тестування
    testing::InitGoogleTest(&argc, argv);

    // Повертаємо результат
    return RUN_ALL_TESTS();
}
```

Вище показано створення тесту для функції `square`, яка повертає квадрат числа. При запуску програми, буде перевірено, чи функція `square` правильно повертає квадрат переданого числа. У разі, якщо тест не пройде успішно, програма видасть повідомлення про помилку та її місце.

## Deep Dive

Написання тестів дозволяє розбити програму на менші частини та перевірити їх роботу окремо. Це дозволяє зрозуміти, які саме функції або частини коду працюють неправильно, що допомагає знайти та виправити помилки більш ефективно.

Крім того, наявність тестів допомагає при внесенні змін до програми. При використанні практики розробки "тест-перш", коли спочатку пишуться тести, а вже потім код, зберігається варіант, що при змінах у коді не будуть зіпсовані раніше написані функціонуючі частини програми.
