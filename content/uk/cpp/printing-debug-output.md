## Чому

Під час написання програмного коду, виникає необхідність відстежувати та аналізувати різні етапи виконання програми. Виведення невідправних повідомлень у термінал або файл допоможе вирішити цю задачу.

## Як

Для виведення повідомлень використовуйте функцію `cout` з бібліотеки `iostream`. Також можна використовувати `cerr` для виведення повідомлень про помилки. Нижче наведено приклад використання цих функцій у коді:

```C++
#include <iostream>

using namespace std;

int main()
{
    int number = 5;

    // виведення повідомлення з використанням cout
    cout << "Число: " << number << endl;

    // виведення повідомлення про помилку з використанням cerr
    if (number == 10)
    {
        cout << "Число дорівнює 10" << endl;
    }
    else
    {
        cerr << "Число не дорівнює 10" << endl;
    }

    return 0;
}
```

Результат виконання програми буде виглядати наступним чином:

```
Число: 5
Число не дорівнює 10
```

## Глибші занурення

Виведення повідомлень для налагодження коду може бути корисним, але також може призводити до зайвого виведення даних та збільшення розміру програми. Якщо потрібні детальніші повідомлення для налагодження, можна використовувати умовну компіляцію з допомогою директиви `#define DEBUG`, яка дозволяє включати і виключати виведення повідомлень за допомогою змінної `DEBUG`. Нижче наведено приклад коду для цього методу:

```C++
#include <iostream>

using namespace std;

// визначення DEBUG для умовної компіляції
#define DEBUG

int main()
{
    int number = 5;

    // перевірка, чи включене DEBUG
    #ifdef DEBUG
        cout << "Номер: " << number << endl;
    #endif

    return 0;
}
```

Результат виконання цього коду буде пустим, оскільки DEBUG не включено. Але якщо розкоментувати директиву `#define DEBUG`, повідомлення буде виведено. Цей метод можна використовувати для вирішення проблем з надмірним виведенням повідомлень.
