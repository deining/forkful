## How to: (Як це зробити:)
Як із нуля створити PHP проєкт? Нижче основні кроки:

```php
<?php
// Крок 1: Встановіть Composer (Dependency Manager для PHP)
// Зробіть це з офіційного сайту: getcomposer.org

// Крок 2: Створіть папку для вашого проєкту і перейдіть в неї
mkdir my_new_project
cd my_new_project

// Крок 3: Ініціалізуйте новий проєкт з Composer
// В кінсоль введіть:
composer init

// Відповідайте на запитання й ініціалізуйте проєкт

// Крок 4: Ваш проєкт тепер має файл composer.json, де можна вказати залежності

// Крок 5: Створіть основний PHP файл
touch index.php

// Крок 6: Напишіть "Hello, World!" на PHP
<?php
echo "Привіт, світ!";

// Запустіть файл
// В консолі введіть:
php index.php
```

Очікуваний вивід у консоль:
```
Привіт, світ!
```

## Deep Dive (Поглиблено):
Перші PHP скрипти були простими, часто без структури. З часом з'явились фреймворки (Laravel, Symfony) та менеджери пакетів, як Composer, що дали стандарти та інструменти для керування складністю. Сучасні проєкти часто використовують автозавантажувачі (autoloaders), простори імен (namespaces) і залежності (dependencies), які можна оголосити в `composer.json`. Файли конфігурації та завантаження (зазвичай `.env` і `bootstrap`) також стали нормою. Важливо дотримуватись стандартів PSR для забезпечення сумісності між PHP компонентами.
