## Чому

Написання тестів є важливою частиною процесу розробки програмного забезпечення, оскільки вони допомагають виявляти та виправляти помилки ще до того, як код буде запущений в продакшн середовище. Тести також забезпечують впевненість у тому, що зміни, внесені до коду, не порушать функціональність програми.

## Як

Існує декілька способів написання тестів в PHP, тому ми розглянемо два найпоширеніші підходи: PHPUnit та Laravel Dusk.

### PHPUnit

PHPUnit є найбільш популярною бібліотекою для тестування PHP-коду. Вона надає багатий набір функцій для написання тестів та перевірки результатів. Давайте розглянемо простий приклад тесту, який перевіряє, чи правильно додаються два числа:

```PHP
public function testAddition()
{
    $calculator = new Calculator();
    $result = $calculator->add(2, 3);

    $this->assertEquals(5, $result);
}
```

У цьому тесті ми створюємо екземпляр класу калькулятора, викликаємо метод `add()` з аргументами 2 та 3 і перевіряємо, чи результат дорівнює 5. Якщо тест пройшов успішно, ми отримаємо повідомлення `Passed`, інакше буде виведено повідомлення про помилку.

### Laravel Dusk

Laravel Dusk - це інструмент для автоматизованого тестування інтерфейсу користувача веб-додатків. Він дуже корисний для тестування функціональності, яка пов'язана з веб-сторінками, наприклад формами або кнопками. Давайте подивимося на приклад тесту, який перевіряє, чи користувач може успішно увійти до свого акаунту:

```PHP
public function testLogin()
{
    $this->browse(function (Browser $browser) {
        $browser->visit('/login')
                ->type('email', 'test@example.com')
                ->type('password', 'password')
                ->press('Log in')
                ->assertPathIs('/home');
    });
}
```

У цьому тесті ми використовуємо Laravel Dusk для запуску браузера та виконання різних дій на сторінці входу. В результаті, ми перевіряємо, чи потрапили на сторінку `/home`, що є підтвердженням успішного входу до акаунту.
