З чого почати писання тестів:
Чому:
Записання тестів є необхідною складовою процесу програмування, оскільки вони дозволяють перевірити та впевнитися в тому, що ваш код працює правильно. Із ростом об'єму коду тестування стає ще більш важливою задачею, оскільки допомагає запобігти неочікуваним помилкам та зберегти час та зусилля у подальшій розробці.

Як:
Найпростіший спосіб написання тестів - це використання функції `assert()`, яка перевіряє, чи правильний результат повертає ваш код. Давайте розглянемо приклад:

```PHP 
function addition($a, $b) {
  return $a + $b;
}

//Перша тестова умова
$first_result = addition(5, 2); //буде 7
assert($first_result == 7); //перевірка на співпадіння з очікуваним результатом

//Друга тестова умова
$second_result = addition(10, -5); //буде 5
assert($second_result == 5); //перевірка на співпадіння з очікуваним результатом
```

Таким чином, ми перевіряємо, чи повертає функція правильні значення у випадку різних вхідних умов.

Глибше:
Написання тестів - це процес, що допомагає гарантувати якість вашого коду. Тестування можна провести на різних рівнях: модульне тестування, інтеграційне тестування та тестування з прийняттям вирішення. Кожен рівень тестування має свої особливості та дозволяє перевірити різні аспекти функціональності вашого коду.
