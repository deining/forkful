## Чому 

Як програмісти, ми часто працюємо зі стрічками - незалежно від того, чи це дані з форми, запити до бази даних або просто повідомлення для користувача. Іноді нам потрібно нормалізувати ці стрічки, щоб зменшити ризик помилок або для правильного відображення на екрані. Одним з таких прийомів є перетворення стрічки до нижнього регістру.

## Як це зробити?

```Elm
toLower "HELLO WORLD" == "hello world"
 
```

Цей простий приклад показує, як реалізувати функцію `toLower` у Elm. Ми передаємо стрічку "HELLO WORLD", і отримуємо назад стрічку "hello world" у нижньому регістрі. Elm має вбудовану функцію `toLower`, яка дозволяє нам легко здійснити цю операцію.

## Глибокий занурення

Але що якщо ми хочемо реалізувати це власноруч? Це може бути корисно для тих, хто вивчає мову Elm і хоче краще зрозуміти, як ці функції працюють. Для початку, давайте подивимося на дефолтну функцію `toLower` у Elm:

```Elm
toLower : String -> String
toLower string =
  toLowerMut (String.toCodePoints string) ""
```

Ми бачимо, що вона приймає стрічку і повертає іншу стрічку. Однак, її реалізації ми не бачимо, тому давайте глибше розберемося. Elm використовує функцію `toLowerMut`, яка приймає два параметри - список кодових точок та пусту стрічку. Вона виконує заміну на кожній кодовій точці, де це необхідно, і повертає оновлену стрічку з нижнім регістром.

Тепер ми бачимо, як Elm використовує цей метод для перетворення стрічки до нижнього регістру. Це може бути корисно для розуміння алгоритмів та реалізації власних функцій у мові Elm.
