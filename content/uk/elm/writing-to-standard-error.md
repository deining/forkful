## Чому

Найперше, давайте з’ясуємо, що таке стандартний потік помилок (standard error).
Коли ви запускаєте програму, він автоматично створює два потоки: стандартний потік вводу (stdin) та стандартний потік виводу (stdout). Стандартний потік помилок же (stderr) використовується для виведення повідомлень про помилки та інформації про виконання програми.

Для чого ж потрібно писати до стандартного потоку помилок? Це дозволяє зручно відстежувати та аналізувати повідомлення про помилки під час розробки програми.

## Як

У мові Elm для запису до стандартного потоку помилок можна використати функцію `Debug.crash`, яка приймає рядок з потрібним повідомленням.

```Elm
import Debug exposing (crash)

main = 
    crash "Повідомлення про помилку"
```

Виконання цього коду виведе повідомлення до стандартного потоку помилок:

```
Повідомлення про помилку
```

Зазвичай ця функція використовується при випадках непередбачених помилок або при відлагодженні коду.

## Глибоке занурення

Варто зазначити, що використання цієї функції не замінює практику обробки помилок в програмі. Вона слугує для зручного виведення повідомлень в розробці та налагодженні коду. 

Крім `Debug.crash`, у мові Elm є ще декілька функцій для запису до стандартного потоку помилок, таких як `Debug.log` та `Debug.todo`. Кожна з них має свої особливості та призначення.
