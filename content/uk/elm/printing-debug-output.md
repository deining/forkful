## Why (Чому)

В цій блозі я розберуся з тим, чому ви можете знадобитися віддрукування відладочного виводу під час програмування на Elm. 

## How To (Як)

```Elm
import Debug exposing (toString)

cat = {
	name = "Fluffy",
	age = 5,
	color = "gray"
}

Debug.log "my cat" (toString cat)
```

Результат:

```
my cat = { name = "Fluffy", age = 5, color = "gray" }
```
 
 Для початку, імпортуйте модуль Debug, який має функціонал для виводу відладочного повідомлення. Далі, створіть об'єкт, який ви хочете перевірити і використовуйте функцію `Debug.log` для виводу повідомлення з ім'ям та значенням змінної. Дуже корисною функцією є `toString`, яка перетворює значення в рядок. Це допоможе вам вивести на екран складніші дані, такі як списки або кортежі. 

## Deep Dive (Глибоке погрузження)

При програмуванні на Elm виникає багато ситуацій, коли потрібно перевірити значення змінних або стану програми під час її виконання. При цьому, відладочний вивід стає незамінним інструментом для швидкого з'ясування проблем.

У випадках, коли ви використовуєте функції `Debug.log`, важливо пам'ятати про те, що вони будуть виконуватися під час роботи програми, навіть у продакшені. Тому слід забезпечити, щоб видалити всі виклики `Debug.log` перед збіркою проекту для випуску. 

Також варто згадати про те, що Elm має функцію `Debug.toString`, яка дозволяє вам перетворювати значення в рядок x безпосередньо в коді, без використання модуля Debug. Це може бути корисно при необхідності виводу даних в консоль, наприклад, під час використання функцій `Debug.log` в інтерактивній консолі.
