# Чому?

У програмуванні іноді може виникнути потреба у виведенні додаткової інформації для відлагодження коду. Це допоможе зрозуміти, що саме відбувається в процесі виконання програми та знайти помилки. Використовуючи вбудований інструмент Elm для друківки відладкового виводу, це роботу можна значно полегшити.

# Як це зробити?

Щоб вивести додаткову інформацію під час відлагодження коду, необхідно додати функцію `Debug.log` до свого коду. Ця функція отримує два параметри: назву виводу та значення, яке потрібно вивести. Наприклад:

```elm
-- Код Elm
import Debug exposing (log)

users = ["John", "Jane", "Bob", "Alice"]

-- Вивід довжини масиву з Іменами користувачів
log "Довжина масиву з іменами: " (List.length users)
```

При запуску програми ви побачите наступний вивід в консолі:

```
Довжина масиву з іменами: 4
```

# Глибокий погляд

Функція `Debug.log` дозволяє друкувати будь-які дані: числа, строки, лісті та навіть кортежі. Важливо пам'ятати, що в output повинен бути еквівалентний Elm типу, тому використовувати його для друку функцій не рекомендується. 

Також, варто знати, що після завершення роботи з програмою необхідно видалити всі виклики `Debug.log`. Це потрібно зробити, оскільки друкований вивід займає пам'ять й може вплинути на продуктивність програми.
