## Чому

Написання тестів є важливою частиною процесу розробки програмного забезпечення. Вона дозволяє перевірити правильність функціонування коду і запобігти можливим помилкам у майбутньому. Це зберігає час і зусилля, тому що помилки можуть бути виявлені та виправлені рано в процесі розробки.

## Як

Щоб почати, створіть новий файл `test.c` у вашій обраній текстовому редакторі. Далі потрібно підключити бібліотеку `assert.h`, яка дозволить нам перевірити умови для наших тестів. Далі необхідно створити функцію `main` і додати декілька тестів у наш код.

```C
#include <stdio.h> 
#include <assert.h>

// Функція, яка перевіряє чи є аргумент рівним нулю 
int is_zero(int num) { 
    return (num == 0) ? 1 : 0; 
} 

// Основна функція з тестами 
int main() { 
    // Тест 1: перевірка чи функція is_zero повертає 1 для нульового аргументу 
    assert(is_zero(0) == 1); 
    printf("Test 1 passed\n"); 

    // Тест 2: перевірка чи функція is_zero повертає 0 для не нульового аргументу 
    assert(is_zero(5) == 0); 
    printf("Test 2 passed\n"); 

    // Тест 3: перевірка чи функція is_zero повертає 0 для від'ємного аргументу 
    assert(is_zero(-2) == 0); 
    printf("Test 3 passed\n"); 

    return 0; 
}
```

Виконайте цей код у вашому терміналі за допомогою наступної команди: `gcc -o test test.c` та запустіть, ви отримаєте наступний вихід:

```
Test 1 passed
Test 2 passed
Test 3 passed
```

Тепер ми перевірили правильність функціонування нашої функції `is_zero` за допомогою трьох тестів.

## Глибша поглиблення

Написання тестів повинно бути стандартною процедурою у розробці програмного забезпечення. Це дозволяє впевнитися у правильності працюючого коду та полегшує процес виправлення будь-яких помилок. Крім того, наявність тестів підвищує якість програми та здатність до змін, оскільки вони забезпечують простоту і надійність коду.
