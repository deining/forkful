## Що & Чому?
Написання тестів - це процес написання спеціального коду, який перевіряє правильність роботи програми. Програмісти роблять це, щоб переконатися, що їх програми працюють так, як очікується і не мають помилок.

## Як:
Нижче наведено приклад коду на мові ```C```, який перевіряє правильність роботи функції додавання:

```C
#include <stdio.h>

int add(int a, int b) {
   return a + b;
}

int main() {
   // перевірка функції додавання
   if (add(2, 3) == 5) {
      printf("Функція додавання працює коректно!");
   } else {
      printf("Упс! Щось пішло не так.");
   }
   return 0;
}
```

Висновок:
```
Функція додавання працює коректно!
```

## Детальний огляд:
Написання тестів - це важлива частина розробки програмного забезпечення, яка допомагає програмістам виявляти та виправляти помилки в своїх програмах. Цей підхід став популярним у 2000-х роках з поширенням практики тестування, так званого "розробки базового проекту". Є інші способи тестування програм, наприклад, ручне тестування або тестування на основі контрактів. Хоча написання тестів може вимагати більше часу на початкових етапах розробки, воно може заощадити час і зусилля на подальших етапах та запобігти виникненню серйозних помилок у програмі.
