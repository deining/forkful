##Чому

Написання тестів є ключовою частиною процесу розробки програмного забезпечення. Вони допомагають переконатись, що ваш код працює згідно очікувань та допомагають уникнути нових помилок при внесенні змін.

##Як

Нижче подані приклади коду та виводу для найпоширеніших способів написання тестів у мові C.

```C
// Приклад використання асерта 
#include <assert.h>

// Тестова функція, яка перевіряє, чи діє функція "sum" правильно
void test_sum(){
  assert(sum(2, 2) == 4);
  assert(sum(-4, 5) == 1);
  // Додайте сюди більше тестів, якщо потрібно
}

// Код з функцією "sum", яку ми тестуємо
int sum(int a, int b){
  return a + b;
}

int main(){
  // Викликаємо нашу тестову функцію
  test_sum();
  
  // Виведе повідомлення про успішне проходження тестів
  printf("Всі тести пройдено успішно!");
  
  return 0;
}

```

Вивід:

```
Всі тести пройдено успішно!
```

##Глибокий занурення

Написання тестів є важливим елементом тестування з урахуванням багатьох аспектів, таких як функціональність, регресія та відлагодження помилок. Крім того, інструменти для написання тестів у мові C, такі як CUnit та Unity, надають більше можливостей для зручного та ефективного тестування вашого коду.
