## Чому
Написання тестів є важливою частиною процесу розробки програмного забезпечення, оскільки вона дозволяє перевірити наявність помилок та забезпечити якість коду. Крім того, тести дозволяють ефективно виявляти та виправляти помилки на ранніх етапах розробки, що допомагає зекономити час та зусилля.

## Як
Написання тестів у мові С є простим та зручним завданням. Для цього необхідно скористатися фреймворком для тестування, наприклад, Unity, CppUTest або CUnit. Далі наведено приклад написання тесту за допомогою фреймворку Unity:

```C
#include "unity.h"

void test_addition(void) {
    int a = 2, b = 3;
    int result = addition(a, b);

    TEST_ASSERT_EQUAL_INT(5, result);
}
```

Після запуску даного тесту ми отримаємо результат:

```C
--------------------
RUN: test_addition
--------------------
-------------------
TEST SUMMARY
-------------------
test_addition: PASS
-------------------
```

Таким чином, ми можемо переконатися, що функція "addition" працює правильно. 

## Глибоке погруження
Написання тестів є важливою частиною тестування програмного забезпечення і може бути використано для багатьох цілей, таких як перевірка правильності реалізації функцій, виявлення помилок та забезпечення оглянутості коду. Слід дотримуватися певних правил при написанні тестів для досягнення найкращих результатів, таких як незалежність тестів та використання належних імен для функцій.
