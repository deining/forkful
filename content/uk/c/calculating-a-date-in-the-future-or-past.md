# Що і Чому?

Підрахунок дати в майбутньому або минулому є важливою частиною багатьох програм. Він дозволяє програмі оперувати часом і датою для виконання різних завдань, таких як планування подій або визначення тайм-стемпів. Виконання правильних розрахунків дати є важливою задачею для програмістів.

# Як це зробити:

Використовуючи мову програмування C, ми можемо легко реалізувати підрахунок дат у майбутньому або минулому. Для цього нам потрібно використовувати бібліотеку `time.h`. Наприклад, можна використовувати функцію `tm` для створення структури, яка буде містити поточну дату та час. Потім ми можемо змінити значення цієї структури, використовуючи функцію `mktime` для обчислення дати в майбутньому або минулому. Далі - приклад коду:

```C
#include <stdio.h>
#include <time.h>

int main() {
	
	struct tm current_time = {0}; // створюємо структуру для поточної дати та часу
	
	time_t time_now; 
	time(&time_now); // отримуємо поточний час та дату
	
	current_time = *localtime(&time_now); //  копіюємо час та дату до структури сurrent_time
	
	current_time.tm_year += 1; // додаємо рік до поточної дати
	
	mktime(&current_time); // обчислюємо нову дату та час
	
	printf("Дата через 1 рік: %d/%d/%d", 
		current_time.tm_mday, 
		current_time.tm_mon + 1,
		current_time.tm_year + 1900
	); // виводимо обчислену дату
	
	return 0;
}
```

Вивід коду буде наступним:

```
Дата через 1 рік: 10/6/2022 
```

# Глибоке погруження:

В минулому програмування дати було більш складною задачею, оскільки різні системи використовували різні бази для числення років. Наприклад, JDN (Юліанська денна нумерація) спочатку використовувала 365 днів на рік і її новий рік починався 1 січня, а потім Ден Гаак з'явився і додав до цього новий день у високосні роки. Це частково призвело до виникнення стандартної системи лічби дат, відомої як Григоріанський календар. Цей календар був прийнятий у більшості країн, а деякі інші використовують власні календарі.

Існують також інші альтернативи для підрахунку дати, зокрема алгоритм Зелінського та алгоритм Монтгомері. Вони також використовуються для обчислення дат у майбутньому або минулому.
