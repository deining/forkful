## Тому


Програмістам часто доводиться зіткнутися зі складними помилками при розробці програм. І в таких випадках особливо важливо мати засоби для відлагодження коду. З однієї сторони, це допомагає знайти та виправити помилки. З іншого боку, воно також допомагає встановити, чи програма виконується правильно. У цьому блозі ми розглянемо найпоширеніший спосіб відлагодження коду - вивід дебаг-інформації.


## Як


Один із найпростіших способів виведення дебаг-інформації у С є використання функції `printf`. Ця функція дозволяє вивести будь-який текст, об'єкт, або змінну на екран. Вона має таку ж саму структуру, як і функція `printf`, але виводить дебаг-інформацію у вікно терміналу. Ось приклад використання:

```C
#include <stdio.h>

int main() {

    int num1 = 5;
    int num2 = 10;

    printf("Значення першої змінної: %d\n", num1);
    printf("Значення другої змінної: %d\n", num2);

    return 0;
}

```

Ви можете запустити цей код у своєму IDE або скомпілювати його з командного рядка. Результат виводу буде наступним:

```
Значення першої змінної: 5
Значення другої змінної: 10
```

Це лише початок того, як ви можете використовувати функцію `printf` для виводу дебаг-інформації. Ви також можете форматувати виведення за допомогою різних специфікаторів формату, які дозволяють вам вставляти значення змінних на потрібних вам місцях.

## Глибоке погруження


Хоча функція `printf` є корисною для виводу дебаг-інформації, є і інші способи докладніше аналізувати ваш код. Наприклад, ви можете використати функцію `fprintf`, яка дозволяє вивести дебаг-інформацію не тільки на екран, але і у файл.
