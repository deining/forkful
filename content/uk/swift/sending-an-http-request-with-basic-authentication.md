Що & Чому?

Відправлення HTTP-запиту з базовою аутентифікацією - це процес, коли програміст надсилає запит до сервера з вказанням логіна та пароля для перевірки доступу до об'єкта. Це дуже важливий інструмент для забезпечення безпеки та захисту інформації.

Як це зробити:

```Swift
let username = "user1"
let password = "password1"

guard let url = URL(string: "https://example.com"),
    let loginData = "\(username):\(password)".data(using: .utf8)?.base64EncodedString() else {
    fatalError("Invalid URL or credentials")
}

var request = URLRequest(url: url)
request.httpMethod = "GET"
request.setValue("Basic \(loginData)", forHTTPHeaderField: "Authorization")

let task = URLSession.shared.dataTask(with: request) { data, response, error in
    if error != nil {
        print(error!)
    } else if let data = data {
        print(String(data: data, encoding: .utf8)!)
    }
}
task.resume()
```

Вдруковано: Вітаємо! Ви успішно відправили HTTP-запит з базовою аутентифікацією та отримали відповідь від сервера. Тепер ваша інформація захищена!

Детальніше:

Надсилання HTTP-запитів з базовою аутентифікацією стало стандартним способом для забезпечення безпеки та захисту інформації в Інтернеті. Цей метод широко використовувався від початку розвитку Інтернету і до сьогоднішнього дня залишається досить популярним. Є альтернативні методи, такі як Digest аутентифікація, яка також пропонує використовувати логін та пароль для перевірки доступу до об'єкта, але базова аутентифікація є простішою та більш ефективною для багатьох типів додатків.

Щоб надіслати HTTP-запит з базовою аутентифікацією, необхідно установити правильний заголовок «Authorization». У цьому прикладі ми використовуємо базову кодування Base64 для створення логіну та пароля, які передаються разом до сервера.

Дивись також:

Для докладнішої інформації про HTTP-запити та базову аутентифікацію, рекомендуємо ознайомитися з офіційною специфікацією HTTP/1.1, а також переглянути статті на тему настройки захисту веб-серверів.
