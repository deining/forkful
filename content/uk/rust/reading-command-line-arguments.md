## Чому

Використання аргументів командного рядка є важливим елементом програмування на мові Rust, оскільки це дозволяє користувачам взаємодіяти з програмами через термінал і передавати їм необхідну інформацію. Ця здатність значно полегшує роботу з програмами і робить їх більш універсальними.

## Як

Використання аргументів командного рядка можна реалізувати за допомогою стандартної бібліотеки Rust - "std::env". Давайте розглянемо невеликий приклад коду, який отримує аргументи командного рядка і виводить їх на екран:

```Rust 
use std::env;

fn main() {
    // Передаємо аргументи в змінну "args"
    let args: Vec<String> = env::args().collect();

    // Виводимо аргументи на екран
    println!("Аргументи командного рядка: {:?}", args);
}
```

При запуску програми з деякими аргументами командного рядка, ви отримаєте наступний результат:

```Shell
$ ./program argument1 argument2
Аргументи командного рядка: ["./program", "argument1", "argument2"]
```

Також можна перевірити кількість переданих аргументів і отримати конкретний аргумент за його індексом, як показано в прикладі нижче:

```Rust
use std::env;

fn main() {
    let args: Vec<String> = env::args().collect();

    // Виводимо кількість переданих аргументів
    println!("Кількість аргументів: {}", args.len());

    // Виводимо другий аргумент - "argument2"
    println!("Другий аргумент: {}", args[2]);
}
```

В результаті ви отримаєте:

```Shell
$ ./program argument1 argument2
Кількість аргументів: 3
Другий аргумент: argument2
```

Більш детальну інформацію про роботу з аргументами командного рядка на мові Rust можна знайти в офіційній документації: https://doc.rust-lang.org/std/env/index.html

## Глибші води

За замовчуванням, усі аргументи командного рядка є рядками (String) в мові Rust. Щоб обробляти їх як числа або інші типи даних, необхідно здійснити відповідну конвертацію. Наприклад, для отримання цілих чисел з аргументів можна використати метод "parse()" і перетворити рядок на тип "i32".

Також слід звернути увагу на обробку помилок при роботі з аргументами командного рядка. Якщо використати метод "unwrap()" для недійсного аргументу, програма потенційно може викинути помилку. Тому рекомендовано використовувати валідацію аргументі
