Що & Чому?
Відправлення HTTP-запиту з основною аутентифікацією - це спосіб забезпечення безпечного з'єднання між клієнтом та сервером за допомогою ідентифікаційних даних. Програмісти виконують цю дію, щоб забезпечити захист від несанкціонованого доступу до серверних ресурсів.

Як зробити:
Приклад коду й вихідних даних з використанням мови програмування Rust:

```Rust
use ureq::*;

// Встановлення основного з'єднання з сервером і передача основного призначення та пароля
let response = ureq::get("http://example.com/")
                .auth("username", "password")
                .call();

// Виведення статусного коду та відповіді
println!("{:?}", response.status());
println!("{:?}", response.into_string());
```

Глибша підготовка:
1. Історичний контекст: основна аутентифікація була запроваджена в HTTP в 1999 році для забезпечення збереження аутентифікаційних даних.
2. Альтернативи: існують інші методи аутентифікації, такі як OAuth та JWT, що забезпечують більш безпечний спосіб передачі ідентифікаційних даних.
3. Деталі виконання: при використанні основної аутентифікації, ідентифікаційні дані передаються в абстрактний метаданий рядок у HTTP-заголовках.
