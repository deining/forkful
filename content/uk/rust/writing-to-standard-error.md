## Чому

Запис до стандартного виводу помилок є важливою складовою процесу програмування і може допомогти полегшити налагодження коду та знайти помилки, які можуть призвести до небажаних результатів виконання програми.

## Як це зробити

```Rust
use std::io::{stderr, Write};
use colored::Colorize; // для кольорового виводу помилок

fn main() {
    let error_message = "Помилка: не вдалося знайти файл";
    let stderr = stderr();
    // виводимо повідомлення про помилку червоним кольором до стандартного виводу помилок
    writeln!(&mut stderr.lock(), "{}", error_message.red()).expect("Не вдалося записати до стандартного виводу помилок");
}
```

Вивід: `Помилка: не вдалося знайти файл`

## Глибоке вивчення

За замовчуванням, всі помилки записуються до стандартного виводу помилок (stderr), а не до стандартного виводу (stdout). Це дозволяє відрізнити помилки від звичайного виводу програми. Запис до стандартного виводу помилок також може бути використаний для логування різних подій під час виконання програми.
