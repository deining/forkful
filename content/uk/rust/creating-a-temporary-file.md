## Чому

Створення тимчасових файлів є корисним та популярним підходом у багатьох програмах. Тимчасові файли можуть бути використані для збереження даних, які потрібні тимчасово, та після закінчення роботи з ними, вони автоматично видаляються, що допомагає економити місце на жорсткому диску.

## Як

```Rust
use std::io::prelude::*;
use std::fs::File;
use std::env;
use std::io::Error;
use std::path::PathBuf;
use tempfile;

fn main() -> Result<(), Error> {
    // створення тимчасового файла
    let temp_file = tempfile::NamedTempFile::new()?;
    // отримання шляху до створеного файла
    let path = temp_file.path().to_owned();
    // вивід шляху
    println!("Тимчасовий файл створено за шляхом: {:?}", path);
    // отримання тимчасового файла за допомогою File
    let mut temp_file = File::create(path)?;
    // написання даних у тимчасовий файл
    temp_file.write_all(b"Це приклад даних, що зберігаються у тимчасовому файлі")?;
    // отримання поточної директорії
    let current_dir = env::current_dir()?;
    // створення нового тимчасового файлу у заданій директорії
    let temp_file = tempfile::Builder::new()
        .prefix("temp_file")
        .suffix(".txt")
        .tempfile_in(current_dir)?;
    // отримання шляху до нового тимчасового файла
    let path = temp_file.path().to_owned();
    println!("Новий тимчасовий файл створено за шляхом: {:?}", path);
    Ok(())
}
```

При запуску цього прикладу, буде створено два тимчасових файлу: перший у поточній директорії зі стандартною назвою та другий у поточній директорії зі спеціально заданою назвою та розширенням. Виведення шляху до кожного з файлів демонструє їхнє існування та можливість доступу до них.

## Поглиблення

Створення тимчасових файлів може бути корисно в багатьох випадках, наприклад, якщо потрібно тимчасово зберігати проміжні дані під час виконання програми, або для тестування функціоналу збереження файлів. У Rust існує багато бібліотек для роботи з тимчасовими файлами, які допоможуть зробити цей процес більш простим та зручним.
