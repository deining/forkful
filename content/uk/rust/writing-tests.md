## Чому

Написати тести є важливим етапом у програмуванні, оскільки вони допомагають перевірити коректність роботи коду і запобігти потенційним помилкам.

## Як

Нижче наведено приклад коду на мові Rust, який демонструє використання тестів:

```Rust
fn add(x: i32, y: i32) -> i32 {
    x + y
}

#[test]
fn test_add() {
    assert_eq!(add(2, 3), 5);
}
```

Ви можете бачити, що ми використовуємо макрос ```#[test]``` для означення тесту, а також макрос ```assert_eq!``` для перевірки коректності результату. Якщо тест буде успішним, ви побачите повідомлення про успішне виконання тесту, а якщо є помилки - ви отримаєте повідомлення про невдалий тест разом з виводом, що не пройшло перевірку. Таким чином, ви можете переконатися, що ваш код працює належним чином.

## Глибокий занурення

Написання тестів - це складний процес, і вимагає багато практики та розуміння окремих аспектів мови програмування. Для подальшого вивчення запропонований детальний документація про тестування в мові Rust.
