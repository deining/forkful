## Чому

Написання тестів є важливою частиною процесу розробки програмного забезпечення, оскільки вони допомагають забезпечити, що код працює правильно та не викликає помилок.

## Як

Для написання тестів в Rust необхідно знати декілька основних концепцій. Найпростішим способом написання тесту є використання макросу `assert_eq!`, який перевіряє рівність двох значень. Наприклад:

```Rust
fn add(a: i32, b: i32) -> i32 {
  a + b
}

#[test]
fn test_add() {
  assert_eq!(add(2, 3), 5);
}
```

Цей тест перевірятиме, чи функція `add` додає два числа і повертає правильний результат.

## Огляд

Крім перевірки рівності значень, Rust також надає можливість тестувати панічні ситуації, використовуючи макроси `assert` та `assert!`. Крім того, можна створювати різні групи тестів та використовувати різні асерти для більш гнучкого тестування. Детальнішу інформацію про написання тестів в Rust можна знайти у [документації](https://doc.rust-lang.org/book/ch11-01-writing-tests.html).
