## Чому
Коли ви програмуете на Rust, ви хочете бути впевненими, що ваш код працює правильно і не видаватиме непередбачувані помилки. Писати тести допоможе вам перевірити правильність роботи програми.

## Як це зробити
Існує багато різних бібліотек для написання тестів на Rust, але ми рекомендуємо використовувати стандартну бібліотеку `std::test`. Ось простий приклад тесту, який перевіряє, чи функція `add` додає числа правильно:

```Rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}
```
```Rust
#[test]
fn test_add() {
    assert_eq!(add(2, 2), 4);
    assert_eq!(add(10, -5), 5);
}
```
Результат виконання цього тесту буде наступним:
```
running 1 test
test test_add ... ok
```

## Глибинний дослідження
Написання тестів може бути трохи складнішим, ніж просто викликати функцію та перевіряти результат. Можна використовувати анотації, такі як `#[ignore]` і `#[should_panic]`, для контролю поведінки тестувального фреймворку. Також можна використовувати макроси для збільшення зрозумілості коду тестів. Більше інформації про написання тестів можна знайти в [документації стандартної бібліотеки](https://doc.rust-lang.org/std/test/index.html).
