## Що і Чому?

Відправлення HTTP-запиту - це процес, коли програма надсилає запит до сервера, щоб отримати певну інформацію. Програмісти виконують цю дію для забезпечення взаємодії між своєю програмою та веб-сервером.

## Як:

```Rust
use reqwest::blocking::Client;
use reqwest::StatusCode;

fn main() {

    // Створюємо об'єкт клієнта
    let client = Client::new();

    // Створюємо запит GET до вказаної URL-адреси
    let response = client.get("https://example.com")
        .send()
        .expect("Не вдалося відправити запит");

    // Переконуємося, що відповідь успішна (код 200)
    assert_eq!(response.status(), StatusCode::OK);

    // Отримуємо вміст відповіді у вигляді рядка
    let body = response.text()
            .expect("Не вдалося отримати тіло відповіді");

    // Виводимо отриманий вміст у консоль
    println!("{}", body);
}
```

## Глибше розуміння:

Історичний контекст:
Відправлення HTTP-запитів було вперше використано в 1990-х роках, коли веб-сервери стали популярними. Початково це було зроблено за допомогою спеціальних програм, але з часом ця функціональність була включена безпосередньо в різні мови програмування.

Альтернативи:
Крім Rust, існують і інші мови програмування, які дозволяють відправляти HTTP-запити, такі як Python, Java та JavaScript. Однак, через свої особливості, Rust може бути більш атракційним варіантом, зокрема, через свою безпечність та швидкодію.

Деталі реалізації:
У прикладі використовується бібліотека ```reqwest```, яка дозволяє легко та зручно відправляти HTTP-запити в Rust. Однак, щоб скористатися цією функціональністю, необхідно встановити її за допомогою менеджера пакетів Cargo. Детальніше про це можна дізнатися у документації бібліотеки.
