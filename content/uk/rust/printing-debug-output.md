## Чому

Найпростіше кажучи, під час розробки програмних продуктів часто виникає потреба у виведенні дебаг-інформації для з'ясування роботи програми та виявлення помилок. І в цьому випадку виручають команди для виведення дебаг-інформації, які дозволяють бачити поточний стан програми та значення змінних на різних етапах виконання коду. В цій статті ми розглянемо основи виведення дебаг-інформації у мові програмування Rust.

## Як це зробити

Приклад друкованої дебаг-інформації за допомогою функції `println!`:

```Rust
let x = 5;
let y = 10;

println!("Змінна x має значення {}", x);
println!("Змінна y має значення {}", y);
```

В результаті виконання цього коду буде виведено таке повідомлення:

```
Змінна x має значення 5
Змінна y має значення 10
```

У Rust також існують багато інших функцій для друкованої дебаг-інформації, наприклад `dbg!`, `eprintln!`, `format!` та `trace!`. Щоб ознайомитися з ними та дізнатися більше про їхнє використання, рекомендуємо прочитати офіційну документацію мови Rust.

## Глибока розкопка

Окрім простого виведення дебаг-інформації, у мові Rust також є можливість використовувати макроси для динамічного створення дебаг-інформації в залежності від поточного контексту. Це дозволяє знаходити та виправляти помилки швидше та ефективніше.
