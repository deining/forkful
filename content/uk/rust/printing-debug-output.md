Логування в програмуванні є одним з найбільш важливих і корисних інструментів, який допомагає вирішувати проблеми та відлагоджувати ваші програми. Зокрема, логування відбувається шляхом друку відладочних повідомлень або помилок у вигляді тексту, що дозволяє програмістам отримати більш детальну інформацію про те, що відбувається під час виконання програми.

## Чому

Нерідко під час написання програми з'являються різні помилки або невідповідності, які можуть бути важко відслідковувати. В цьому випадку, логування відбувається дуже до речі, адже воно дозволяє отримати більш детальну інформацію про те, що трапилося в програмі, що значно полегшує її відлагодження та виправлення.

## Як використовувати

Для того, щоб використовувати логування у своїй програмі, потрібно використовувати бібліотеку "log" та "env-logger". Перша з них надає нам можливість використовувати різні рівні відладки, такі як "trace", "debug", "info", "warn" та "error". Друга бібліотека за допомогою змінної середовища дозволяє вибрати, який рівень логування буде використовуватися під час виконання програми.

Приведемо приклад коду, який виводить повідомлення про помилку, та в кінці програми друкує кількість разів, коли виникав елемент масиву з індексом, який є меншим за нуль:

```Rust
extern crate log;

use log::{error, info, warn};

fn main() {
    let arr = vec![1, 3, 5, -2, 7];
    for i in 0..arr.len() {
        if arr[i] < 0 {
            error!("The value at index {} is negative!", i);
        }
    }

    let mut negative_count = 0;

    for value in arr {
        if value < 0 {
            negative_count += 1;
        }
    }

    info!("Number of negative values: {}", negative_count);
}
```

В результаті виконання цього коду в консолі ми побачимо наступне:

```
ERROR - The value at index 3 is negative!
INFO - Number of negative values: 1 
```

Це дозволить нам зрозуміти, який саме елемент масиву викликає помилку, та дозволить зосередитися на певних частинах програми при виправленні помилки.
