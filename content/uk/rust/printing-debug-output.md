## Чому

Розробка програмного забезпечення може бути непростим іноді. Часом, коли ми застрягли на певному кроці, ми захочемо вивести деяку важливу інформацію для розуміння, що відбувається в нашій програмі. Для цього і існує інструмент друкованого виводу в Rust.

## Як

Для початку, нам потрібно додати бібліотеку `debug` до нашого проекту:

```Rust
use std::fmt::Debug;
```

Далі, ми можемо використовувати `println!` для друкованого виводу:

```Rust
println!("Значення змінної: {:?}", x);
```

Однак, якщо ми хочемо друковати більше значень або структуру даних, нам потрібно додатково імпортувати бібліотеку `Debug` і використовувати `{:?}` для форматування:

```Rust
use std::fmt::Debug;

#[derive(Debug)]
struct Point {
    x: i32,
    y: i32,
}

let p = Point { x: 5, y: 10 };

println!("Координати точки: {:?}", p);
```

В результаті, ми отримаємо друкований вивід зі значеннями наших змінних або структури даних.

## Глибоке погруження

Важливо пам'ятати, що друкований вивід повинен використовуватись для відладки та налагодження програми, а не як постійний метод виведення інформації. Також, ми можемо використовувати команди `eprintln!` і `dbg!` для виведення у випадку помилки та виведення з додатковою інформацією, відповідно.
