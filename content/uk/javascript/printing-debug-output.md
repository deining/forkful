## Чому

В програмуванні важливо контролювати процес виконання коду та виявляти можливі помилки. Виведення дебаг-виводу є одним зі способів, яким можна вивчити, що відбувається в програмі під час її виконання. Це дозволяє виявити проблемні місця та виправити їх для кращої продуктивності програми.

## Як це зробити

Для виведення дебаг-виводу використовують функцію `console.log()`. Наприклад, якщо потрібно перевірити значення змінної `x`, можна ввести `console.log(x);` у потрібному місці у коді. При виконанні програми, це значення буде виведено в консоль і дозволить дізнатися, яке саме значення має ця змінна.

```
// Приклад виведення дебаг-виводу
let x = 10;
console.log(x); // виводить 10 в консоль
```

Іншим способом для виведення дебаг-виводу є `debugger`. Використовуючи цю команду, програма зупиняє своє виконання у потрібному місці та дозволяє крок за кроком вивчити значення змінних та поточний стан програми. Для цього потрібно встановити точку зупинки у потрібному місці у коді та виконати програму у режимі дебагу.

## Глибше розбираємося

Використання `console.log()` та `debugger` є лише одними зі способів виведення дебаг-виводу. У Javascript є багато інших інструментів для налагодження програм, таких як `console.error()`, `console.warn()`, `console.info()`, які дозволяють виводити різноманітну інформацію про стан програми. Також можна використовувати спеціальні інструменти для налагодження, такі як `Chrome DevTools` або `Node.js debugger`, які надають більш зручний та докладний аналіз програми.
