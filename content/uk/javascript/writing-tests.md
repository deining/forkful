Що & Чому?
Писання тестів - це процес написання спеціальних функцій, які перевіряють правильність роботи програми. Програмісти пишуть тести, щоб переконатися, що їх програми працюють точно так, як очікувалося, і не мають помилок.

Як це зробити:
```Javascript
// Приклад тесту для функції додавання
function add(a, b) {
   return a + b;
}

// Запускаємо тест
console.log(add(2, 3)); // Виводить 5 - тест пройшов успішно!
console.log(add(5, 7)); // Виводить 12 - тест пройшов успішно!
console.log(add(10, 4)); // Виводить 14 - тест пройшов успішно!
```

Глибоке поглиблення:
Написання тестів - це не новий концепт, він використовується вже десятиліттями у розробці програмного забезпечення. Наприклад, в одній команді може бути кілька програмістів, які працюють над однією програмою. Перевірка роботи програми за допомогою тестів дозволяє забезпечити єдність та правильність роботи коду, а також зменшити кількість помилок, які можуть виникнути у процесі розробки.
