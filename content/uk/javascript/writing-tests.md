## Чому

Написання тестів є невід'ємною частиною процесу програмування, оскільки це дозволяє перевірити правильність коду та забезпечити стабільність програми. Це також допомагає зекономити час та зусилля при виявленні та виправленні помилок.

## Як

```Javascript
function add(num1, num2) { // Функція додавання двох чисел
  return num1 + num2;
}

const result = add(2, 3); // Змінна для збереження результату

console.log(result); // Виведе 5 у консоль
```

Один з способів написання тестів - використання фреймворку для тестування, такого як Jest чи Mocha. Для цього потрібно створити окремий файл з розширенням `.test.js` та використовувати команду `jest` чи `mocha` для запуску тестів. У коді вище ми створили просту функцію додавання та перевірили її результат за допомогою тесту.

## Глибоке занурення

Написання тестів вимагає аналізу та розуміння кожного кроку у коді програми. Це дозволяє виявити та виправити помилки в ранній стадії розробки, що зберігає час та гроші в майбутньому. Тестування також допомагає забезпечити якість коду та зменшити кількість багів у програмі.
