Що & Чому?
Перевірка наявності каталогу є процесом, за допомогою якого програмісти перевіряють, чи існує певний каталог на комп'ютері. Це дає змогу запобігти помилкам та забезпечити більш гладкий виконання програм.

Як це зробити?
Використовуючи Javascript, ми можемо використовувати вбудовану функцію ```fs.existsSync()``` для перевірки наявності каталогу. Якщо каталог існує, то ця функція поверне значення ```true```, якщо ні - ```false```. Давайте подивимось на приклад коду:

```Javascript
const fs = require('fs');

if (fs.existsSync('folder')) {
  console.log('Каталог існує');
} else {
  console.log('Каталог не існує');
}
```

Виходячи з цього прикладу, ми можемо побачити, що ми використовуємо умовний оператор ```if``` для перевірки значення, яке повертає функція ```fs.existsSync```. Якщо каталог існує, умова буде виконана і в консоль буде виведено повідомлення "Каталог існує", в іншому випадку - "Каталог не існує".

Глибше погляньмо
Історичний контекст:
Перевірка наявності каталогу є частиною більшої функціональності, що пов'язана з керуванням файловою системою. Історично, існувало багато способів перевірки наявності каталогу, але з появою Javascript цей процес став набагато простішим.

Альтернативи:
Однією з альтернатив до використання вбудованої функції ```fs.existsSync()``` є використання зовнішніх бібліотек, таких як ```fs-extra``` або ```path-exists```. Ці бібліотеки надають більш розширені можливості та дозволяють зручніший доступ до файлової системи.

Деталі реалізації:
Вбудована функція ```fs.existsSync()``` використовує під капотом системні виклики для перевірки наявності каталогу. Це означає, що вона працює швидше та надійніше, порівняно з іншими способами перевірки.
