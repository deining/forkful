## Чому

Перевірка існування каталогу є важливою частиною програмування, особливо коли працюєте з файловою системою та потрібно перевірити, чи існує необхідний каталог для подальшої роботи. Це також допомагає уникнути ошибок та збереже ваш час при виконанні програми.

## Як

```Javascript
const fs = require('fs');
const path = '/Приклад/шлях/до/каталогу';
fs.access(path, (err) => {
  if (err) {
    console.error('Каталог не існує');
  } else {
    console.log('Каталог доступний');
  }
});
```

У цьому прикладі ми використовуємо модуль `fs` для роботи з файловою системою та метод `access()`, який перевіряє доступність файлу чи каталогу за заданим шляхом. Якщо файлу не існує, видається помилка та програма припиняє своє виконання. Якщо файл або каталог існує, виводиться повідомлення про його доступність.

## Deep Dive

При використанні методу `access()` існують різні режими доступу, які дозволяють перевірити як саму наявність файлу чи каталогу, так і права доступу до нього. Наприклад, за допомогою режиму `fs.constants.F_OK`, який перевіряє наявність файлу, можна також використовувати режим `fs.constants.R_OK`, щоб перевірити право на читання файлу.

Також, використовуючи `fs.access()` можна перевірити існування не тільки каталогів, але і файлів, що дозволяє більш детально контролювати роботу з файловою системою.
