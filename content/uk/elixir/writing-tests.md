## Чому

Написання тестів - це важлива частина розробки програмного забезпечення і допомагає забезпечити високу якість коду та запобігти появі багів у вашому продукті. Дізнайтеся, чому написання тестів є необхідною практикою для освідчених програмістів.

## Як це зробити

Написання тестів в Elixir легко та просто. Ми можемо використати вбудований інструмент ExUnit, щоб створити тести для нашого коду за допомогою блоку ```defmodule ... do```. Давайте подивимось на приклад коду та результат його виконання:

```Elixir
defmodule Calculator do
  def add(a, b) do
    a + b
  end
end

defmodule CalculatorTest do
  use ExUnit.Case
  test "додавання" do
    assert Calculator.add(2, 2) == 4
  end
end
```

Ви можете побачити, що ми створили модуль ```Calculator``` з функцією ```add``` та її тест у модулі ```CalculatorTest```. Запустивши тест, ми побачимо такий результат:

```
1 test, 1 assertion, 0 failures.
```

Тепер ми можемо бути впевнені, що функція ```add``` працює правильно та наш код не має помилок.

## Глибше

Крім базового синтаксису, Elixir має багато корисних функцій для написання тестів. Наприклад, ми можемо використовувати такі функції як ```assert_receive``` та ```assert_raise``` для перевірки поведінки наших функцій в різних умовах. Також ми можемо використовувати анотації ```@tag``` та запускати тести з певними тегами, щоб тестувати певні частини нашого коду.  Наприклад:

```Elixir
defmodule CalculatorTest do
  use ExUnit.Case
  @tag :slow
  test "додавання" do
    assert Calculator.add(2, 2) == 4
  end
  @tag :fast
  test "швидке додавання" do
    assert Calculator.add(1, 1) == 2
  end
end
```

Ми можемо запустити тести з тегом ```slow``` або ```fast``` окремо, щоб перевірити швидкість нашого коду.
