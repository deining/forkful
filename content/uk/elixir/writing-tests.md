# Про що і чому?

Написання тестів у програмуванні - це процес створення автоматичних перевірок для перевірки правильності роботи вашого коду. Програмісти цього роблять, щоб забезпечити якість свого коду та упевнитися, що додаток працює так, як очікується.

# Як це зробити:
```Elixir
defmodule Calculator do
  # Тест для додавання
  test "додавання" do
    assert Calculator.add(2, 3) == 5
  end

  # Тест для множення
  test "множення" do
    assert Calculator.multiply(2, 3) == 6
  end
end
```

У цьому прикладі ми створили модуль ```Calculator``` та додали два тести для перевірки функцій ```add``` та ```multiply```. У обох тестах ми використовуємо функцію ```assert```, яка перевіряє правильність обрахування та повертає ```true``` у випадку успішного результату.

# Поглиблене вивчення:
Написання тестів почало набувати популярності в програмуванні наприкінці 1990-х років, коли багато компаній почали використовувати методологію розвитку програмного забезпечення TDD (Test Driven Development). Існують інші альтернативи такі як BDD (Behaviour Driven Development) та ATDD (Acceptance Test Driven Development), які також займаються написанням автоматичних тестів. У Еліксирі для написання тестів використовується фреймворк ExUnit, який надає зручний синтаксис для тестування.
