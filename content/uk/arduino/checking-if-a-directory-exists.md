## Що та чому? 

Перевірка наявності директорії - це процес, який дозволяє програмістам перевірити, чи існує директорія з певною назвою на пристрої, на якому виконується програма. Це дозволяє визначити, чи можна працювати з цією директорією та іншими файлами, що знаходяться в ній. Результатом перевірки буде повідомлення про наявність чи відсутність директорії.

## Як робити:

Для перевірки наявності директорії використовуються функції `exists()` і `mkdir()`. Перша перевіряє чи існує директорія з вказаною назвою, повертаючи `true` або `false`. Друга створює нову директорію з вказаною назвою, якщо її ще не існує. Ось приклад коду для перевірки поточна директорії та створення нової, якщо потрібно:

```
if (exists("тестова директорія")) {
  Serial.println("Директорія вже існує!");
} else {
  mkdir("тестова директорія");
  Serial.println("Директорія створена!");
}
```

## Глибокий профіль:

Перевірка наявності директорії є важливою частиною програмування, оскільки дозволяє забезпечити правильну роботу з даними та уникнути можливих помилок. Ця функція також є важливою при написанні програм для зберігання та обробки важливих даних, коли потрібна відповідна структура для роботи з файлами.

Існує також альтернативний підхід до перевірки наявності директорії, який використовує функцію `open()`. Цей метод простіший, але менш надійний, оскільки може викликати помилку, якщо директорія не існує.

Важливою деталлю у перевірці наявності директорії є використання відповідного шляху до неї. Для Arduino, шляхи задаються відносно кореневої директорії. Наприклад, якщо ви перевіряєте наявність директорії `тестова директорія` у кореневій директорії, шлях до неї буде `тестова директорія/`. Якщо ви перевіряєте директорію, що знаходиться в іншій директорії, наприклад, `Папка1/Папка2`, шлях до неї буде `Папка1/Папка2/тестова директорія/`.
