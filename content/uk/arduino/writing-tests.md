## Чому

Написання тестів є важливим етапом при розробці програмного забезпечення. Воно дає можливість перевірити правильність роботи програми та виявити можливі помилки, що дозволяє забезпечити якість коду і полегшує подальше вдосконалення проекту.

## Як це зробити

Нижче наведені приклади коду та приклад виводу програми для написання тестів на платформі Arduino.

### Перевірка роботи LED-світлодіода

```Arduino
const int LED_PIN = 13;

void setup() {
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_PIN, HIGH); // включення світлодіода
  delay(1000); // затримка 1 секунда
  digitalWrite(LED_PIN, LOW); // виключення світлодіода
  delay(1000); // затримка 1 секунда
}
```

Вивід програми має виглядати так:

```
Включення світлодіода
Затримка 1 секунда
Виключення світлодіода
Затримка 1 секунда
```

### Перевірка роботи аналогового датчика

```Arduino
const int ANALOG_PIN = A0;

void setup() {
  Serial.begin(9600); // ініціалізація порту для виводу даних
}

void loop() {
  int sensorValue = analogRead(ANALOG_PIN); // зчитування значення аналогового датчика
  Serial.println(sensorValue); // вивід значення на монітор порта
  delay(500); // затримка 0.5 секунди
}
```

Вивід на моніторі порта має мати наступний вигляд:

```
Значення датчика: 200
Затримка 0.5 секунди
Значення датчика: 400
Затримка 0.5 секунди
```

## Глибинна розробка

Написання тестів є важливою складовою частиною процесу розробки програмного забезпечення. За допомогою тестів можна переконатися, що програма працює правильно у всіх сценаріях, враховуючи різні вхідні дані та можливі помилки. Це допомагає виявити та виправити проблеми ще на ранніх етапах розробки, що зберігає час та зусилля у подальшій роботі над проєктом.

Крім того, написання тестів дозволяє забезпечити стабільність та надійність програми, а також полегшує впровадження змін та додавання нового функціоналу у майбутньому.
