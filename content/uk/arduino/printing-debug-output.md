## Чому

Використання друкованої відладкової інформації дозволяє зрозуміти роботу програми та знайти помилки в коді. Це допоможе вам покращити якість проекту та зменшити кількість помилок.

## Як

```Arduino
void setup() {
  Serial.begin(9600); // ініціалізація порту для друкованого виводу
}

void loop() {
  int sensorValue = analogRead(A0); // отримання значення з аналогового піна
  Serial.println(sensorValue); // друк значення на моніторі порту
  delay(1000); // затримка на 1 секунду
}
```

### Використання функції Serial.print()

Функція `Serial.print()` дозволяє вам вивести значення будь-якого типу даних (рядок, числове значення, змінну). Наприклад, для виведення повідомлення "Hello World!" на моніторі порту ви можете використати наступний код:

```Arduino
Serial.print("Hello World!");
```

### Використання функції Serial.println()

Функція `Serial.println()` друкує строку та автоматично додає символ нового рядка (\n) після виводу. Таким чином, кожен новий вивід починається з нового рядка, що полегшує читання друкованих повідомлень. Наприклад, для виведення числового значення на монітор порту ви можете використати наступний код:

```Arduino
int number = 10;
Serial.println(number);
```

### Використання спеціальних символів

У функціях `Serial.print()` та `Serial.println()` можна використовувати спеціальні символи для форматування друкованого повідомлення.

- `\n` - символ нового рядка
- `\t` - символ табуляції
- `\r` - символ повернення каретки (повертає курсор на початок рядка)
- `print()` та `println()` можна комбінувати, наприклад: `Serial.print("Hello");` `Serial.println("World!");` // виведе "Hello World!" на двох різних рядках

## Deep Dive

У середовищі Arduino IDE є спеціальне вікно для перегляду друкованої відладкової інформації - "Serial Monitor". Щоб його відкрити, натисніть "Tools" -> "Serial Monitor" або скористайтеся комбінацією клавіш `Ctrl+Shift+M`. Програма повинна бути завантажена на плату Arduino, щоб ви могли бачити друковану інформацію.

Однією з переваг використання відладкового друку є можливість відслідковувати процес виконання програми. Наприклад, якщо ми виводимо значення з сенсора, можна спостерігати, як воно змінюється
