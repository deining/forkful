# Чому
Написання програм для Ардуіно може бути веселим та захоплюючим, але часом можуть виникнути проблеми з кодом. Використання виведення налагодження може стати корисним інструментом для виявлення та вирішення цих проблем.

## Як
Використовуйте функцію ```Serial.print()``` для виведення відлагоджувальної інформації на моніторі послідовного порту. Наприклад, при можливих проблемах з часом виконання коду, можете вивести час, який був витрачений на виконання певного фрагмента коду:

```Arduino
unsigned long start_time = millis(); // запам'ятайте час в початку коду

// ваш код

unsigned long time_elapsed = millis() - start_time; // обчисліть, скільки часу пройшло

Serial.print("Час виконання коду: ");
Serial.print(time_elapsed); // виведіть час на монітор послідовного порту
Serial.println(" мс"); // додайте перенос рядка, щоб забезпечити читабельність
```

Це дозволить вам відслідковувати час виконання ваших функцій та з'ясувати, який фрагмент коду може бути причиною повільності в програмі.

## Глибокий пір
Крім простого виведення часу виконання, ви можете також вивести значення змінних або рядки, щоб побачити, що відбувається в програмі на різних її етапах.

Наприклад, якщо ви створюєте програму для зчитування даних з датчика температури та вологості, ви можете вивести відповідні дані на виході послідовного порту для перевірки їх правильності.

```Arduino
// прочитати значення температури і зберегти його у змінну
float temperature = analogRead(A0) * 0.48828125; // конвертувати значення АЦП на вольтаж, а потім на температуру

Serial.print("Температура: ");
Serial.print(temperature); // виводимо значення температури на монітор послідовного порту
Serial.print(" °C");
```

Виведення цих значень дозволить вам перевірити правильність зчитування датчика та відповідність даних в реальному світі.
