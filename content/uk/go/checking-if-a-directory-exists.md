## Що це і чому: 
Перевірка наявності директорії є процесом виявлення та перевірки існування вказаної директорії в файловій системі. Це важлива функція для програмістів, оскільки дозволяє виконувати операції з файлами та директоріями, що залежать від наявності необхідних ресурсів у файловій системі.

## Як зробити: 
```Go
func checkDir(path string) (bool, error) {
    _, err := os.Stat(path)
    if err == nil {
        return true, nil
    }
    if os.IsNotExist(err) {
        return false, nil
    }
    return false, err
}
```
Перш за все, потрібно імпортувати пакет `os`, який надає доступ до функцій для роботи з файловою системою. Далі, використовуючи функцію `os.Stat()` та перевіряючи отриманий об'єкт помилки, можна визначити наявність директорії і повернути відповідний результат. Також, можна використати функцію `os.IsNotExist()` для перевірки, чи існує помилка `os.ErrNotExist` та повернути відповідний результат. 

## Глибоке занурення: 
Історичний контекст:
Перевірка наявності директорії стала доступною з пакетом `os` в Go версії 1.0. Раніше, для виконання цієї дії потрібно було використовувати функцію `open()`, що було менш зручно та ефективно.

Альтернативи:
Можна також використати пакет `filepath` та функцію `Exists()` для перевірки наявності файлів та директорій.

Деталі імплементації:
Функція `os.Stat()` повертає об'єкт типу `FileInfo`, який містить інформацію про файл або директорію, включаючи назву, розмір, час створення та інше. Якщо об'єкт помилки не є `os.ErrNotExist`, це означає, що директорія існує.
