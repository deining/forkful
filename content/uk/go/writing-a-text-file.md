## З чого почати програмувати на Go

Написання текстового файлу є необхідною здібністю для програмістів будь-якої мови програмування, в тому числі і для Go. Текстові файли - це головний інструмент для зберігання та обміну даними між програмами та користувачами. Наприклад, вони можуть використовуватися для збереження налаштувань програм або для зберігання важливої інформації. Тому, вивчення як правильно створювати та зчитувати текстові файли є важливим для будь-якого початківця програміста.

## Як писати текстові файли в Go

У Go, є кілька основних шляхів, якими можна створювати текстові файли. Давайте подивимося на кожен з них.

Найпростіший і швидкий спосіб - використати функцію ```WriteFile``` з пакету ```io/ioutil```. Для цього необхідно використовувати кратку форму запису, де спочатку вказується назва файлу, потім його вміст у вигляді масиву байтів.

```Go
package main

import (
	"io/ioutil"
	"fmt"
)

func main() {
	content := []byte("Це текст, який буде записаний в файл.")

	err := ioutil.WriteFile("file.txt", content, 0644)
	if err != nil {
		fmt.Println(err)
	}
}
```

Також можна використати пакет ```os``` для створення файлу та його дальшого запису за допомогою методу ```WriteString```. Після того, як файл був створений, він повинен бути закритий за допомогою методу ```Close()```.

```Go
package main

import (
	"os"
	"fmt"
)

func main() {
	file, err := os.Create("file.txt")
	if err != nil {
		fmt.Println(err)
		return
	}
	defer file.Close()

	content := "Це текст, який буде записаний в файл."
	_, err = file.WriteString(content)
	if err != nil {
		fmt.Println(err)
	}
}
```

## Глибоке погруження в написання текстових файлів

При роботі з текстовими файлами важливо врахувати деякі нюанси. Наприклад, необхідно дбати про кодування файлу, оскільки різні мови та системи можуть використовувати різні кодування символів. Також важливо, щоб файл був закритим після завершення запису, щоб уникнути проблем з доступом до нього у майбутньому.

Крім того, важливо пам'ятати про перевірку результуючого об'єкту помилки після запису файлу. Це дозволить уникнути можливих проблем з виконанням програми.
