## Чому

Написання тестів є важливою частиною процесу розробки програмного забезпечення. Вони допомагають перевірити правильність реалізації функціональності та запобігти появі помилок в майбутньому.

## Як

```Go
func AddNumbers(x, y int) int {
  return x + y
}
```
Використання функції для додавання двох чисел за допомогою оператора `+` є популярним прикладом для написання тесту. Ми можемо перевірити чи функція повертає очікуваний результат за допомогою наступного тесту:

```Go
func TestAddNumbers(t *testing.T) {
  result := AddNumbers(2, 3)
  
  if result != 5 {
    t.Errorf("Expected result to be 5, but got %d", result)
  }
}
```

Якщо тест знаходиться у тому ж самому пакеті, що й код, то ми можемо використовувати метод `AddNumbers` без префіксу пакету.

## Deep Dive

Написання детальних тестів допомагає покрити різні сценарії використання та перевірити правильність програми. Ми також можемо використовувати різні методи для перевірки різних типів даних. Наприклад, ми можемо використовувати `testing.T.Log()` для виведення додаткової інформації про тест, або `testing.T.Fatalf()` для припинення тесту при зустрічі помилки.
