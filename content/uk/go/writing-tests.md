## За що

Виконання тестів є важливою частиною процесу розробки програмного забезпечення. Вони допомагають перевірити, чи працює програма так, як слід і виявляють помилки та проблеми, що можуть виникнути в майбутньому.

## Як

Найпростішим способом написання тестів у Go є використання пакету "testing", який надає стандартну бібліотеку. Почнемо з створення тесту для функції, що додає два числа:

```Go
// Приклад сигнатури функції, яку ми будемо тестувати
func add(x, y int) int {
    return x + y
}

// Приклад тесту
func TestAdd(t *testing.T) {
    // Будемо перевіряти, чи правильно функція додає числа
    result := add(2, 3)
    expected := 5
    if result != expected {
        t.Errorf("Додавання вдалося з помилкою: отримано %d, очікувалось %d", result, expected)
    }
}
```

Після того, як тест був написаний, його можна запустити з командного рядка за допомогою команди `go test`. Якщо тест пройшов успішно, ви побачите повідомлення "PASS", якщо ні - "FAIL". Також можна використовувати ключ `-v` для отримання детальнішої інформації про проходження тестів.

## Глибше

У Go є декілька функцій, призначених для важелювання тестових скриптів. Одна з них - `t.Errorf()`, яка використовується для виведення повідомлення про помилку. Інша - `t.Helper()`, яка допомагає вказати рядок, де виникла помилка в тесті.

Також, варто звернути увагу на розділення тестів на пакети та файли. Для цього необхідно створити папку з назвою `tests` у тій же директорії, де знаходиться пакет, який потрібно перевірити. У цій папці можна створювати тестові файли з назвою, яка починається на `*_test.go`.
