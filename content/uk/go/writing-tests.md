## Переваги написання тестів у Go

Можливо, багато програмістів питають себе: "Чи потрібно писати тести для своєї програми у Go?" Наша відповідь: так, дуже потрібно. Написання тестів допоможе вам певніти, що ваш код працює правильно та продуктивно. Крім того, це зекономить ваш час та зусилля у майбутньому, оскільки виявлення та виправлення помилок в тестах вимагає менше часу, ніж вирішення проблем у робочій програмі.

## Як написати тести у Go

Написання тестів у Go досить просте та зручне. Ви можете використовувати вбудовану функцію `testing` та додавати свої власні тести у функцію `TestName(t *testing.T)`. Нижче наведено приклад структури тесту у Go:

```Go
package main

import (
	"testing"
)

func TestSum(t *testing.T) {
	numbers := []int{1, 2, 3, 4, 5}
	result := sum(numbers)

	if result != 15 {
		t.Errorf("Expected sum to be 15, got %d instead", result)
	}
}
```

У цьому прикладі ми перевіряємо функцію `sum`, яка повинна обраховувати суму чисел у списку. Якщо результат відрізняється від очікуваного, ми використовуємо функцію `t.Errorf` для виведення повідомлення про помилку.
