## Чому

Writing tests є важливою частиною у програмуванні з будь-якою мовою, але у Go вони можуть бути особливо корисними. Тести допомагають підтвердити правильність коду та запобігти появі багів і помилок у майбутньому. Вони також допомагають зменшити час на відлагодження програми та забезпечити більш стабільну та надійну роботу програми. Отже, писати тести варто для поліпшення якості та швидкості розробки програм на Go.

## Як

Існує кілька основних методів написання тестів у Go. Усі тести розташовуються в тому ж самому пакеті, що і код, який вони перевіряють, та мають суфікс `_test`. Для запуску тестів використовується команда `go test`. Ось приклад тесту для функції `Sum`, яка повинна повертати суму двох чисел:

```Go
func Sum(a, b int) int {
  return a + b
}

func TestSum(t *testing.T) {
  sum := Sum(5, 7)
  if sum != 12 {
    t.Errorf("Expected sum to equal 12, got %d", sum)
  }
}
```

В результаті виконання команди `go test` повинен бути виданий наступний результат:

```
ok      example.com/pkg    0.001s
```

Це означає, що тест пройшов успішно. У випадку, якщо тест не пройшов, буде виданий детальніший результат та повідомлення про помилку.

## Глибоке вивчення

Написання тестів у Go також може бути більш продуманим та ефективним, якщо використовувати додаткові бібліотеки, наприклад, `goconvey` чи `ginkgo`. Ці бібліотеки дозволяють писати більш детальні тести та зручно організовувати код. Також варто розглянути використання тестового покриття за допомогою `go test -cover` для вказівки відсотка покриття коду тестами.
