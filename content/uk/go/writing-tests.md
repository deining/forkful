Що та чому?

Тестування коду - це процес написання додаткового коду, який перевіряє, чи працює відлагоджуваний код коректно. Програмісти роблять це для того, щоб перевірити, що їх код виконує потрібну функцію і не містить помилок.

Як?

```Go
func add(x, y int) int {
  return x + y
}

func TestAdd(t *testing.T) {
  total := add(2, 3)
  if total != 5 {
    t.Errorf("Expected 5, but got %d", total)
  }
}
```

Вглиб?

Тестування коду виникло з потреби впевнитися, що програмне забезпечення працює коректно і не містить помилок. Існують інші методи випробування коду, такі як ручне тестування, але автоматизоване тестування дозволяє швидше і ефективніше перевіряти більші обсяги коду. В Go використовується пакет ```testing``` для написання тестів. Тебе можуть зацікавити інші бібліотеки для автоматизованого тестування, такі як ```GoConvey``` або ```Testify```. У процесі написання тестів важливо розуміти, як правильно писати тести і як підтримувати їх у майбутньому.
