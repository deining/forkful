## Чому

Писати в стандартний вихідний потік помилок (*standard error*) часто буває корисним для збереження та відлагодження програм. Це дозволяє швидко визначити проблеми у випадку, якщо програма не працює як очікувалося.

## Як

```Go
package main
import "fmt"
func main(){
    fmt.Fprintln(os.Stderr, "Повідомлення про помилку")
}
```

Вищенаведений приклад демонструє як можна використовувати функцію `Fprintln` з пакету `fmt`, щоб написати повідомлення про помилку у стандартний вихідний потік помилок. Вихідний потік можна перенаправити у файл за допомогою команди `./program 2> error.log`.

## Глибинний аналіз

Написання в стандартний вихідний потік помилок також дозволяє виконувати розширений аналіз помилок за допомогою пакету `log`. Цей пакет надає функціональність для додавання додаткової інформації про помилки, такої як дата та час виникнення, до повідомлення про помилку.
