## Чому

Користуватися HTTP запитом з основною автентифікацією можна для безпечного доступу до захищених ресурсів або API, так як цей спосіб передачі даних дозволяє перевірити ідентифікацію користувача перед отриманням доступу до запитуваних даних. 

## Як це зробити

```Go
// Імпорт необхідних пакетів
import (
  "net/http"
  "fmt"
  "io/ioutil"
  "encoding/base64"
)

url := "http://example.com/" // URL, куди буде відправлений HTTP запит
username := "username" // Ім'я користувача
password := "password" // Пароль

// Створення об'єкту для виконання запиту
client := &http.Client{}

// Створення запиту та додавання заголовків для основної автентифікації
req, err := http.NewRequest("GET", url, nil)
auth := username + ":" + password
req.Header.Add("Authorization", "Basic " + base64.StdEncoding.EncodeToString([]byte(auth)))

// Відправлення запиту та отримання відповіді
resp, err := client.Do(req)

// Обробка помилок
if err != nil {
  fmt.Println("Помилка виконання запиту:", err)
}

// Читання відповіді та виведення її у консоль
body, err := ioutil.ReadAll(resp.Body)
fmt.Println("Відповідь:", string(body))
```

Вище наведений код демонструє, як створити HTTP запит з основною автентифікацією в мові Go. Перш за все, ми імпортуємо необхідні пакети, включаючи пакет `net/http`, який містить функції для роботи з HTTP протоколом. Далі, ми створюємо об'єкт `client` для виконання запитів, і створюємо сам запит за допомогою функції `http.NewRequest()`. Основною частиною цього коду є створення заголовка `Authorization`, який містить закодоване значення імені користувача і пароля. Після відправки запиту та отримання відповіді, ми читаємо її та виводимо у консоль.

## Поглиблене вивчення

Основна автентифікація використовує механізм, який називається "Basic access authentication" і передбачає кодування імені користувача і пароля в форматі base64 та додавання цього значення до заголовка `Authorization` в запиті. Одна з особливостей цього методу - його відкритий характер, оскільки ім'я користувача і пароль не шифруються, а тільки кодуються. Це означає, що при використанні основної автентифікації потрібно забезпечити безпеку передачі даних на бік сервера
