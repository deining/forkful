## Чому

В програмуванні, надсилання HTTP запиту є дуже важливим кроком для отримання інформації з інтернету. Незалежно від того, чи ви розробляєте веб-сайт, додаток або скрипт, ви часто будете використовувати HTTP запити для отримання даних з веб-сервера.

## Як

Для початку, нам потрібно встановити бібліотеку Requests для здійснення HTTP запитів у нашому коді. Можна встановити цю бібліотеку за допомогою pip у наступний спосіб:

```Python
pip install requests
```

Після встановлення бібліотеки, вам потрібно імпортувати її до свого коду, що дозволить вам використовувати її функції. Ось приклад коду, який буде надсилати GET запит на сайт "https://example.com" та роздрукує відповідь сервера:

```Python
import requests

r = requests.get("https://example.com")

print(r.text)
```

Ви можете бачити, що ми використовували метод `get()` з бібліотеки Requests та передали в нього URL для запиту. Метод `get()` повертає об'єкт Response, який містить всю інформацію про відповідь сервера, таку як статус коду, заголовки та тіло запросу. Метод `text` дозволяє нам отримати тіло відповіді у вигляді тексту.

Також, ми можемо встановити додаткові параметри для запиту, як наприклад, передавати дані у вигляді POST запиту чи встановити заголовки. Ось приклад використання POST запиту з параметрами:

```Python
import requests

payload = {"key1": "value1", "key2": "value2"}
r = requests.post("https://example.com", data=payload)

print(r.text)
```

Зверніть увагу, що ми використали параметр `data`, який дозволяє нам передати дані у POST запиті. Також, ви можете встановити заголовки за допомогою параметру `headers`, як показано нижче:

```Python
import requests

headers = {"Content-Type": "application/json"}
r = requests.get("https://example.com", headers=headers)

print(r.text)
```

## Глибока назва

Крім базового використання Requests для надсилання простих HTTP запитів, ви можете також дізнатися більше про специфікації HTTP протоколу та використовувати додаткові функції бібліотеки, такі як встановлення cookies, аутентифікація та робота з HTTPS. Це дозволить вам створювати більш складні та функціональні HTTP запити у вашому коді.
