## Чому

Виведення помилок для відлагодження є важливим етапом при програмуванні, оскільки це дозволяє легше з’ясувати, що відбувається у вашому коді та виявити помилки. Надруковані вихідні дані можуть бути корисними для відстеження значення змінних та контролю поведінки програми. 

## Як це зробити

Існує кілька способів друку вихідних даних в Пайтон. Один з них - це використання функції `print()`, яка дозволяє друкувати будь-який об'єкт в терміналі. Наприклад:

```Python
age = 25
print("Моя вік -", age, "років")
```

Виходом цього коду буде `Моя вік - 25 років`. 

Крім того, можна використовувати команду `logging` для запису вихідних даних до файлу замість терміналу. Наприклад:

```Python
import logging

logging.basicConfig(filename = 'debug.log',level = logging.DEBUG, format = '%(asctime)s - %(levelname)s - %(message)s')

number = 10
logging.debug("Значення змінної number: " + str(number))
```

Після виконання цього коду, у файлі `debug.log` буде записане повідомлення: `2021-07-09 15:00:00,123 - DEBUG - Значення змінної number: 10`.

## Поглиблене вивчення

Друкування вихідних даних також може допомогти під час відлагодження складніших програм. Якщо у вас є багато змінних та функцій, то може бути важко відслідковувати, що відбувається в коді без друку вихідних даних. 

Також варто пам'ятати про те, що надмірне друкування вихідних даних може сповільнити виконання програми. Тому важливо обговорити з командою, які дані потрібно виводити для відлагодження та як часто робити розпечатки для забезпечення оптимальної продуктивності.
