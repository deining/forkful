# Чому

Виведення відладних повідомлень є важливою частиною процесу розробки програмного забезпечення, оскільки це дозволяє програмістам відслідковувати та виправляти помилки у своєму коді. Використання виведення відладки може значно спростити та прискорити процес виправлення помилок.

# Як це зробити

```
Синтаксис для виведення повідомлень відладки використовує ключове слово `Debug.Trace.trace`.

```Haskell
import Debug.Trace

-- Приклад виведення повідомлення "Hello World!"
main = do
    trace "Hello World!"
    putStrLn "Hello World!"
```

В терміналі ви отримаєте наступний вивід:

```
Hello World!
Hello World!
```

Зверніть увагу, що повідомлення "Hello World!" було виведене двічі - один раз від `trace` та ще один раз від `putStrLn`. Це показує, що відладне повідомлення було виконано перед виконанням інших дій у вашій програмі.

# Глибокий занурення

Виведення відладних повідомлень дозволяє розробникам детально аналізувати процес виконання їх програми. Крім виведення простих повідомлень, можна використовувати функціональні властивості мови Haskell для більш складного виведення.

```
Наприклад, ви можете використати ключове слово `show` для перетворення значень у рядки та виведення їх:

```Haskell
-- Приклад програми, яка обчислює площу кола
import Debug.Trace

main = do
    let pi = 3.1415926
        radius = 5
        area = pi * radius * radius
    trace ("Радіус: " ++ show radius ++ ", Площа: " ++ show area)
    print area
```

Ви отримаєте такий вивід:

```
Радіус: 5, Площа: 78.539815
78.539815
```

Ви також можете використовувати `traceShow` для виведення значень з власним попереднім текстом:

```Haskell
-- Приклад перевірки, чи є число парним
import Debug.Trace

isEven :: Int -> Bool
isEven x = traceShow x (x `mod` 2 == 0)

main = do
    print $ isEven 2
    print $ isEven 3
```

Ви отримаєте такий вивід:

```
2
True
3
False
```

Також варто звернути увагу на те, що `trace` та `traceShow` є чистими функціями, тобто вони не мають побічних ефектів та не модифікують дані. Це важливо уникати непередбачуваних помилок під час використання відладних повідомлень.
