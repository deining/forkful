# З чого почати

У сучасному світі програмування мало хто може уявити свою роботу без використання відладки. Це не тільки дозволяє знайти та виправити помилки у коді, але й спрощує розробку та підтримку програм. Але якщо мова йде про функціональну мову програмування, таку як Haskell, то може виникнути питання, чи існує можливість проводити відладку з виводом даних у консоль. У цій статті ми розглянемо, як саме можна виконати таку дію та чому це важливо для програмістів Haskell.

## Як виводити дані у консоль у Haskell?

Для того, щоб виводити відлагодженими дані у консоль у Haskell, нам знадобиться використовувати функції з модуля `Debug.Trace`. Давайте проведемо декілька прикладів для кращого розуміння.

```Haskell
import Debug.Trace

-- Приклад з додаванням двох чисел
addTwo :: Int -> Int -> Int
addTwo x y = trace ("Додаємо " ++ show x ++ " та " ++ show y) (x + y)

-- Виведе: "Додаємо 5 та 7"
-- Повертає: 12
addTwo 5 7
```

Користуючись функцією `trace`, ми можемо вивести будь-яку інформацію у консоль під час виконання програми. Це дозволяє нам відстежувати значення змінних та контролювати правильність роботи функцій.

```Haskell
-- Приклад зі змінною
greet :: String -> String
greet name =
  let
    message = "Привіт, " ++ name
  in
    trace ("Створюємо повідомлення: " ++ message) message
  
-- Виведе: "Створюємо повідомлення: Привіт, Іван"
-- Повертає: "Привіт, Іван"
greet "Іван"
```

Також ми можемо виводити значення змінних у будь-який момент часу, наприклад, коли ми бажаємо дізнатися, що ж саме передається у функцію.

## Глибше про відлагодження в Haskell

При використанні функції `trace` необхідно враховувати деякі особливості. По-перше, краще уникати використання її в коді, який потім буде запущений у продакшні, так як це може повільнити роботу програми. Код з функцією `trace` також може бути складнішим для читання та розуміння, тому краще використовувати її в прикладних цілях або для тестування.
