## Що & Навіщо?

Використання регулярних виразів - це інструмент для пошуку та заміни тексту в рядках. Програмісти використовують їх, щоб більш ефективно маніпулювати рядками даних та виконувати ряд роботів, таких як перевірка валідності формату, вилучення певної інформації та заміна частин тексту.

## Як:

```Haskell 
import Text.Regex
```

Надаючи команду ```import Text.Regex```, ми робимо доступними функції для роботи з регулярними виразами у Haskell. Наприклад, за допомогою функції ```matchesRegex```, ми можемо перевірити, чи входить певна частина тексту в регулярний вираз.

```Haskell
matchesRegex "Hello" "Hello World" -- повертає True
matchesRegex "Hello" "Goodbye World" -- повертає False
```

Існує також можливість використовувати регулярні вирази для заміни тексту. Наприклад, функція ```replaceRegex``` дозволяє замінити всі входження певного виразу у рядку на інший текст.

```Haskell
replaceRegex "world" "universe" "Hello world" -- повертає "Hello universe"
```

## Глибші деталі:

Ідея регулярних виразів була створена в 1950-х роках математиком Стівеном Клінеом. У той час, ця технологія використовувалася для роботи в різних текстових редакторах та інших програмах. В даний час, існують альтернативи для роботи з регулярними виразами, такі як пошук та заміна у текстових процесорах або використання парсерів у програмуванні.

Під капотом, реалізація регулярних виразів у Haskell використовує скомпільовані функції для більш ефективної обробки великих обсягів даних.
