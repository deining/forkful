## Чому

Якщо ви працюєте зі складними задачами, що вимагають передавання змінних при запуску програми, ви, напевно, вже знаєте, що необхідно користуватися аргументами командного рядка. Але як це зробити в Haskell? Цей пост допоможе вам розібратися з цим.

## Як

Завдяки функціям `getArgs` та `getProgName`, ви зможете без проблем зчитувати аргументи командного рядка у своєму коді Haskell. Для початку, імпортуємо потрібні модулі:

```Haskell
import System.Environment
```

Після цього можна використати функцію `getArgs`, яка повертає список аргументів командного рядка у вигляді рядка. Наприклад, якщо у вас є програма, яка приймає два аргументи:

```Haskell
main = do
  args <- getArgs
  putStrLn ("Перший аргумент: " ++ args !! 0)
  putStrLn ("Другий аргумент: " ++ args !! 1)
```

Запускаючи цю програму з наступними аргументами: `prog аргумент1 аргумент2`, ви отримаєте наступний результат:

```
Перший аргумент: аргумент1
Другий аргумент: аргумент2
```

Крім того, `getProgName` поверне назву самої програми, що дуже зручно, якщо ви хочете реалізувати динамічну поведінку.

## Глибша розгляд

Функція `getArgs` не повертає список типу `String`, але список типу `IO String`, оскільки вона використовує монади. Це дозволяє отримати доступ до аргументів у будь-якому місці програми, а не тільки в `main`. 

Якщо вам потрібна більш складна обробка аргументів командної строки (наприклад, перевірка на наявність певних опцій), можна скористатися додатковими бібліотеками, такими як `cmdargs` чи `optparse-applicative`.
