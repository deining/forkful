## Тому

Написання тестів є важливою частиною процесу розробки програмного забезпечення, оскільки допомагає перевірити, чи функції та методи виконують те, що очікується. Це зменшує кількість помилок та збільшує якість програмного продукту.

## Як

Найпростіший спосіб написання тестів у Haskell - використання функцій `assert` та `eq` зі стандартної бібліотеки `Test.HUnit`. Почнемо з встановлення пакету `hunit` за допомогою команди `cabal install hunit` або `stack install hunit`.

Давайте створимо простий тест для функції `add`:

```Haskell
import Test.HUnit

add :: Int -> Int -> Int
add x y = x + y

testAdd :: Test
testAdd = TestCase (assertEqual "1 + 2 = 3" 3 (add 1 2))

main :: IO Counts
main = do
  runTestTT testAdd
```

Ви можете запустити цей код та побачити, чи проходить тест чи ні. Якщо все проходить успішно, ви побачите поточну кількість тестів, які були виконані.

## Глибоке погруження

Написання власних тестів дозволяє більш детально перевіряти функціональність вашого коду. Наприклад, ви можете використовувати функції `assertTrue` та `assertFalse` для перевірки булевих умов, `assertRaises` для перевірки винятків та багато інших корисних функцій.

Також, ви можете створювати групи тестів за допомогою функцій `TestList` та `TestLabel`, щоб зберігати ваші тести більш структурованою та зрозумілою формою.

У якійсь мірі, написання тестів є мистецтвом - чим більше ви практикуєте, тим більш ефективними та корисними стають ваші тести.
