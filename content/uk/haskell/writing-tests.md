## Чому

Написання тестів є важливою частиною процесу розробки програмного забезпечення. Вони допомагають перевірити правильність роботи коду, зменшити кількість помилок і забезпечити стабільність програми.

## Як це зробити

Існує багато різних способів написання тестів у Haskell. Одним з них є використання пакету `HUnit`, який надає потужні інструменти для створення тестів. Давайте подивимось на невеликий приклад тестування функції додавання у Haskell:

```Haskell
-- Для початку імпортуємо пакет HUnit
import Test.HUnit

-- Оголошуємо функцію додавання
add :: Int -> Int -> Int
add x y = x + y

-- Створюємо тестовий випадок для перевірки функції add
testAdd :: Test
testAdd = TestCase $ assertEqual "Повинно бути 10" 10 (add 4 6)

-- Запускаємо тест та дивимось на результат
main :: IO ()
main = do
    runTestTT testAdd
```

В консоль ми побачимо `Cases: 1  Tried: 1  Errors: 0  Failures: 0`, що означає, що наш тест пройшов успішно. Якщо б функція додавання повертала неправильний результат, ми побачили б помилку в консолі.

## Глибоке погруження

Написання тестів в Haskell є більш складною темою, ніж може здатися на перший погляд. Є багато пакетів для написання тестів, які надають різні функціональності для тестування різних аспектів програмного забезпечення. Щоб дізнатися більше про написання тестів у Haskell, рекомендуємо вам ознайомитися з цією [статтею](https://www.haskell.org/hunit/introduction.html) та ознайомитися з іншими пакетами для тестування, такими як `QuickCheck` та `tasty`.
