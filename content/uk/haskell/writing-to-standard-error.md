## Чому

Програмістам часто потрібно виводити повідомлення про помилки напряму в консоль, використовуючи стандартний потік помилок (standard error). Це дозволяє навіть при виникненні помилки продовжувати виконання програми та обробляти несправності.

## Як використовувати

Написання в стандартний потік помилок можна зробити за допомогою функції `hPutStrLn` з модуля `System.IO`.

```Haskell
import System.IO

main :: IO ()
main = do
    hPutStrLn stderr "Повідомлення про помилку"
```

Після запуску програми, повідомлення про помилку з'явиться в консолі.

```
$ stack run
Повідомлення про помилку
```

Цей спосіб виведення дозволяє зручніше та швидше виявляти та вирішувати помилки в програмі.

## Глибший аналіз

Існує ще один підхід до виведення повідомлень про помилки за допомогою функції `hPutStr`. Ця функція дозволяє виводити повідомлення до будь-якого потоку. Якщо потрібно вивести повідомлення в консоль, його можна передати в якості першого аргументу функції `hPutStr`, а потім вже передати необхідний потік, наприклад, `stderr`.

```Haskell
import System.IO

main :: IO ()
main = do
    hPutStr "Повідомлення в консоль"
    hPutStr "Повідомлення в файл"

-- Виведе:
-- Повідомлення в консоль
-- Повідомлення в файл
```

Більш глибоке розуміння потоків та їх викорістання можна отримати, вивчаючи модуль `System.IO`, який містить багато корисних функцій для роботи з потоками у Haskell.
