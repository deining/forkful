## Чому

Створення тимчасового файлу може бути корисним для тих, хто працює зі скриптами або автоматизованими процесами. Тимчасові файли можуть зберігати проміжні дані або функціонувати як місце для тимчасового збереження результатів обробки. 

## Як

```Fish Shell``` надає нам можливість легко створити тимчасовий файл з використанням вбудованих властивостей шелу. Для цього ми можемо використати наступний синтаксис:

```
### Створення тимчасового файлу

set -l tmp (mktemp)
echo "Це тимчасовий файл" > $tmp
cat $tmp
```

В результаті отримаємо наступне:

```
Це тимчасовий файл
```

Ми можемо також змінити шлях, де буде створений тимчасовий файл, використовуючи опцію ```-p```:

```
### Зміна шляху тимчасового файлу

set -l tmp (mktemp -p /tmp/output)
echo "Це тимчасовий файл в /tmp/output" > $tmp
cat $tmp
```

В результаті отримаємо наступне:

```
Це тимчасовий файл в /tmp/output
```

## Глибокий захід

Щоб краще розібратися з процесом створення тимчасових файлів, давайте розглянемо деякі зауваження. Спершу, варто знати, що ```mktemp``` гарантує унікальність створюваного тимчасового файлу, тобто ім'я файлу міститиме в собі випадкові числа та букви, що запобігає можливим конфліктам при одночасному використанні шелом.

Також слід врахувати, що тимчасові файли бувають двох типів: файли з іменем та без імені. Файли з іменем створюються в конкретному шляху та містять його в назві, тоді як файли без імені створюються в поточній директорії.
