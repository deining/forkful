## Чому

Передрук вихідних даних необхідний для відлагодження програми та знаходження помилок. Він надає можливість бачити стан змінних та іншу важливу інформацію, яка допоможе виправити неполадки в коді.

## Як

```Fish Shell``` була розроблена спеціально для взаємодії з іншими програмами, в той час як ви кодуєте. Щоб надрукувати вихідні дані для відлагодження, скористайтеся командою ```echo```:

```Fish Shell
echo "Змінна Х дорівнює $X"
```

Це надрукує повідомлення зі значенням змінної Х у консоль. Також, можна використовувати спеціальні комбінації знаків, щоб надрукувати іншу корисну інформацію:

```Fish Shell
echo "Я зараз перебуваю у директорії $PWD виконуючи команду $history"
```

Із цією командою ви можете надрукувати будь-яку важливу інформацію, яка допоможе вам знайти помилки у своєму коді.

## Глибокий занурення

Надрукування вихідних даних - це один із елементів відлагодження. Подробиці, що були надруковані, можуть вказати на те, що сталося під час виконання програми. Також, ви можете використовувати практичну команду ```set -x```, яка надрукує всі команди, які виконуються в процесі вашої програми. Це допоможе зрозуміти логіку вашого коду та знайти місця, де виникають помилки.
