## Чому

Написання до стандартного виводу помилок є важливим аспектом програмування в оболонці Fish. Це допомагає уникати конфліктів зі стандартним виводом та дає змогу ефективно обробляти помилки у вашому коді.

## Як

Існує кілька способів запису до стандартного виводу помилок в оболонці Fish. Один з них - використання команди `echo` з флагом `-e`, щоб додати спеціальний символ `\e[1;31m`, який змінює колір виводу на червоний.

```Fish Shell
echo -e "\e[1;31mЦе повідомлення буде виведено червоним кольором до стандартного виводу помилок!\e[0m"
```

Ви також можете використовувати конструкцію `2>&1` для перенаправлення стандартного виводу на стандартний вивід помилок.

```Fish Shell
ls -l file_that_doesnt_exist 2>&1 | grep "Немає доступу"
```

## Deep Dive

Щоб краще розібратися у процесі писання до стандартного виводу помилок, корисно знати кілька тонкощів:

- Файл стандартного виводу помилок зазвичай називається `stderr`.
- `>&` і `|&` безпосередньо перенаправляють помилки на стандартний вивід помилок.
- Флаг `-f` для команди `echo` прибирає потворні символи (`\e[1;31m` в нашому прикладі).
