## Чому

Послання HTTP-запиту з базовою аутентифікацією є важливою частиною веб-розробки, оскільки дозволяє з'єднувати сервер із захищеними ресурсами за допомогою імен користувача та пароля. Це важливо для забезпечення безпеки при передачі конфіденційної інформації.

## Як Виконати

Базова аутентифікація відправляє дані про ім'я користувача та пароль в заголовках запиту. Ці дані зазвичай передаються у перекодованому форматі, використовуючи base64 кодування. Для виконання такого запиту у Fish Shell можна скористатися командою `curl`, але спершу потрібно встановити необхідний пакет `curl-basic-auth`, використовуючи команду `fisher`.

```Fish Shell
fisher install jorgebucaran/curl-basic-auth
```

Тепер можна використовувати `curl` команду з додатковими параметрами для виконання запиту з базовою аутентифікацією.

```Fish Shell
curl -u "username:password" http://example.com
```

Це вкаже серверу, що дані авторизації мають бути передані у вигляді ім'я користувача та паролю в заголовках запиту. Після успішної авторизації, сервер надішле відповідь із запрошеним ресурсом.

## Глибоке Занурення

Для більш розуміння базової аутентифікації, варто розглянути формат, у якому передаються дані. Як вже зазначалося, дані про ім'я користувача і пароль перекодовуються у base64 формат. Це робиться для збереження безпеки, оскільки такий формат легко прочитати, але складно розшифрувати. Крім того, базова аутентифікація не надійна і може бути легко обійдена, тому рекомендується використовувати більш захищені методи аутентифікації.
