## Чому

Передплатити у розрізненні командного рядка є корисним навичкою для кожного програміста, особливо в сучасному світі орієнтованому на командна рядка. Незалежно від того, чи ви працюєте з власним комп'ютером або в командному шлюзі, знання використання аргументів командного рядка дозволяє вам ефективно взаємодіяти з вашою системою і виконувати завдання швидше і ефективніше.

## Як

Команда ```Fish Shell``` має вбудовані інструменти для зчитування аргументів командного рядка. Вони надають кращу продуктивність та лаконічний синтаксис по порівнянню з іншими шелами. Щоб зчитати аргументи командного рядка за допомогою ```Fish Shell```, вам потрібно використовувати спеціальну змінну ```$argv``` та функцію ```set```. Наприклад:

```
fish $ set arg1 arg2 arg3
fish $ echo $arg1
arg1
fish $ echo $arg2
arg2
fish $ echo $arg3
arg3 
```

Зверніть увагу, що ім'я файла виконувань (наприклад, ```fish $``` в нашому випадку) також рахується як один з аргументів командного рядка, тому перший аргумент (```arg1``` в нашому випадку) починається з другого елемента в змінній ```$argv```. Ви можете також використовувати параметри командного рядка за допомогою змінної ```$argc``` для отримання загальної кількості аргументів. Приклад:

```
fish $ set arg1 arg2 arg3
fish $ echo $argc
3
```

Використовуйте ці інструменти, щоб легко отримувати і обробляти аргументи командного рядка для вашої програми.

## Глибоке погруження

Скористайтеся функцією управління помилками ```set -e``` для перевірки наявності аргументів командного рядка. Якщо аргумент не було надано, ```Fish Shell``` поверне помилку і зупинить виконання програми. Приклад:

```
fish $ set -e arg1 arg2
```

Також, використовуйте функцію управління помилками ```set -q``` для перевірки, чи існує змінна. Наприклад:

```
fish $ set arg1 arg2
fish $ if set -q arg3
  echo "arg3 is set"
else
  echo "arg3 is not set"
end
```

Зверніть увагу, що ```arg3``` не було об'явлено, тому функція управління помилками ```set -q``` повертає значення ```false```.
