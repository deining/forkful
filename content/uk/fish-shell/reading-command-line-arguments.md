## Той, Хто: Зрозуміти, Чому Необхідно Читати Аргументи Командного Рядка

Як програмісти, ми можемо пройти довгий шлях від того, щоб ставати експертами в своєму мистецтві. Однак, одна навичка, яка залишається важливою для будь-якого програміста, це здатність до роботи з командним рядком. Навіть більше, робота з командним рядком повністю витісняє інші інтерфейси при роботі з файловою системою та іншими завданнями. У цій статті ми розглянемо, чому необхідно здатися на читання аргументів командного рядка.

## Як Це Зробити: Приклади Коду та Виведення для *Fish Shell*

Попередження: Цю статтю було перевірено і протестовано на роботоспособність з використанням *Fish Shell* версії 3.0.0.

Щоб отримати доступ до аргументів командного рядка, ми будемо використовувати команду `argparse` в *Fish Shell*. Перш за все, давайте створимо новий файл під назвою `args.fish`. У цьому файлі ми будемо робити всю нашу роботу.

Введіть наступні командні рядки в файл `args.fish`:

```Fish Shell
#!/usr/bin/env fish

argparse -n ARG1 "Наш перший аргумент"
argparse -n -s ARG2 "Наш другий аргумент"
argparse -n -f ARG3 "Наш третій аргумент"
```

Цей код означає, що ми створюємо три аргументи, які можна передати при запуску команди, а саме `ARG1`, `ARG2` та `ARG3`. Слово після флагу `-n` це назва аргументу, яке ми використовуємо для звернення до нього, а після `""` ми маємо опис аргументу.

Тепер давайте спробуємо запустити наш файл із деякими аргументами.

У терміналі введіть:

```cmd
fish args.fish one -s test -f file.txt
```

При цьому ми передаємо значення для кожного з аргументів. Щоб побачити, як ці значення отримано в коді, використовуйте наступний код:

```Fish Shell
IFS=$'\n'
echo "ARG1: $ARG1"
echo "ARG2: $ARG2"
echo "ARG3: $ARG3"
```

Вивід буде наступним:

```
ARG1: one
ARG2: test
ARG3: file.txt
```

Тепер ми можемо використовувати ці значення в нашому коді і зробити з ними щось корисне.
