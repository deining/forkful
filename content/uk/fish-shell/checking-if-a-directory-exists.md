## Чому

Перевірка існування директорії є важливою частиною програмування, оскільки може допомогти у виконанні конкретних дій з файлами та директоріями, а також уникнути помилок у випадку, коли вони не існують.

## Як

Проверка того, чи існує директорія виконується за допомогою спеціальної команди **test** в Fish Shell. Приклад коду для перевірки існування директорії «documents» у домашньому каталозі користувача виглядає так:

```Fish Shell
if test -d ~/documents
    echo "Директорія існує"
else
    echo "Директорія не існує"
end
```

Результат виконання цього коду буде залежати від того, чи існує така директорія. Якщо директорія не існує, буде виведено повідомлення "Директорія не існує".

## Deep Dive

Перевірка існування директорії може бути використана не тільки для перевірки наявності директорії, але і для перевірки її типу. Наприклад, ```test -d``` перевіряє, чи є об’єкт директорією, тоді як ```test -f``` перевіряє, чи є об’єкт файлом. 

Крім того, використання команди **test** можна поєднувати з іншими командами для більш складної логіки перевірок. Наприклад, команда ```test -d ~/documents && ls ~/documents``` виконає перевірку наявності директорії та виведе список файлів у ній, тільки у випадку, якщо директорія існує.
