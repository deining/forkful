Що і чому?

Робота з JSON це особливий спосіб взаємодії з даними. Це формат збереження та обміну інформацією, який дозволяє зберегти структуру та значення даних, що полегшує їх обробку та передачу. Це дуже зручно для програмістів, тому що дозволяє отримати доступ до конкретних даних та обробляти їх швидко та ефективно.

Як?

Зразки коду та вихідні дані:

```Fish Shell 
# Створення json об'єкту з даними 
set -l person '{"name": "John", "age": 30, "pets": ["dog", "cat"]}' 

# Виведення значень "name" та "age" з об'єкту за допомогою команди `jq` 
echo $person | jq '.name, .age'

# Виведення значення "cat" зі списку "pets" 
echo $person | jq '.pets[1]'

# Встановлення значення "city" у об'єкті та збереження його у файлі "profile.json" 
echo $person | jq '. + {"city": "Kyiv"}' > profile.json 

```

Очистий політ

Стисла інформація про роботу з JSON:

- Часто використовується для збереження конфігурацій та налаштувань в програмах.
- Дозволяє легко та швидко отримувати доступ до конкретних даних.
- Існує альтернатива - формат XML, але JSON є більш зручним для програмістів.
- Реалізація роботи з JSON може відрізнятися у різних мовах програмування.

Детальніше

JSON - це формат обміну даними, який був розроблений у 1999 році та широко використовується у програмуванні. Його синтаксис схожий на мови програмування, що полегшує розуміння та роботу з ним для програмістів.

Використання більш старого формату XML може бути менш зручним та більш громіздким, тому JSON став популярним у світі програмування. Однією з переваг JSON є можливість збереження як списків, так і об'єктів зі значеннями. Це дозволяє структурувати дані та легко отримувати до них доступ.

Іншими форматами обміну даними є YAML, CSV та INI, проте JSON є більш універсальним та дозволяє зберегти складну структуру даних.

Для роботи з JSON у Fish Shell також доступні інші корисні утиліти, які допоможуть зчитати та обробити дані у форматі JSON. Наприклад, використання команди `curl` для отримання даних з Інтернету та команди `grep` для роботи зі зчитаними даними.
