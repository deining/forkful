## Чому

Тестування є важливою частиною процесу розробки програмного забезпечення. Написання тестів допомагає виявляти помилки та забезпечує більш стабільну та надійну роботу програми. Під час розвитку проєкту, тестування зберігає відновлювальну здатність коду і дозволяє швидше реагувати на зміни та оновлення.

## Як виконати тестування в Kotlin

Написання тестів в Kotlin можна виконати за допомогою фреймворка JUnit. Розглянемо приклад тестування простої функції, яка повертає додану суму двох чисел:

```Kotlin
fun add(a: Int, b: Int): Int {
    return a + b
}
```

Для створення тесту викликаємо функцію `assert()` з вказанням очікуваного та фактичного результатів:

```Kotlin
class AddTest {
    @Test
    fun addTest() {
        assert(add(2, 3) == 5)
    }
}
```

Після запуску тесту ми отримуємо вихід `0 tests passed, 0 tests failed`, що свідчить про те, що тест успішно пройшов.

## Поглиблене вивчення тестування

Написання тестів допомагає більш явно визначити очікування написаного коду та дозволяє швидше впевнено внести зміни. Крім того, наявність тестів спрощує відлагодження програми, дозволяючи швидко локалізувати та виправляти помилки.

Наприклад, при написанні тестів для функції `add()`, ми можемо встановити додаткове очікування та перевірити, що функція також повертає коректний результат для від'ємних чисел:

```Kotlin
@Test
fun addNegativeTest() {
    assert(add(-2, -3) == -5)
}
```

Також можна використовувати анотації `@Before` та `@After` для запуску певних дій перед та після виконання кожного тесту.
