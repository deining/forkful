## Чому

Компарування дат є важливим у багатьох програмах. Воно дозволяє порівнювати часові маркери, щоб знайти найновіші або найстаріші події, визначити тривалість події чи визначити час до наступної події. Для українського читача це може бути корисним для створення програм, які використовують український календар або специфічні локації з часовим поясом.

## Як

Для порівняння двох дат використовується метод `compareTo()` з класу `LocalDate` з бібліотеки Java Time. Наприклад, ми маємо дві дати: `2021-01-01` і `2021-01-05`. Давайте перевіримо, яка з них є раніше за допомогою умови `if`.

```Kotlin 
val date1 = LocalDate.parse("2021-01-01")
val date2 = LocalDate.parse("2021-01-05")

if (date1.compareTo(date2) < 0) {
    println("$date1 є раніше, ніж $date2")
}
```
Виведе: `2021-01-01 є раніше, ніж 2021-01-05`

Також можна використовувати `compareTo()` для порівняння на рівність двох дат. Якщо дати рівні, метод поверне `0`.

```Kotlin
val date1 = LocalDate.parse("2021-01-01")
val date2 = LocalDate.parse("2021-01-01")

if (date1.compareTo(date2) == 0) {
    println("$date1 і $date2 є однакові")
}
```
Виведе: `2021-01-01 і 2021-01-01 є однакові`

## Глибинний аналіз

Метод `compareTo()` використовує порівняння за замовчуванням у форматі `YYYY-MM-DD`, але можна використовувати й інші формати для додаткового порівняння. Наприклад, можна порівнювати дати за поточним часом або часом з різницею в 24 години.

```Kotlin
val now = LocalDate.now()
val tomorrow = now.plusDays(1)

if (now.compareTo(tomorrow) == 0) {
    println("Зараз є те саме, що завтра")
}
```
В даному прикладі ми порівнюємо дату за поточним часом і часом через 24 години. Оскільки ці дати не рівні, умова не спрацює і рядок не виведеться.

Для додаткової гнучкості можна використовувати клас `LocalDateTime`, який включає в себе інформацію про дату та час, адже декотрі події можуть відбуватись у той же час, але у різні дні.
