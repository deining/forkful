## Що & чому?
Похиляти до стандартного виводу помилок (standard error) означає виводити інформацію про помилки або непередбачувані ситуації прямо на екран. Програмісти роблять це для того, щоб дізнатись про можливі проблеми з програмою та вдосконалити її якість.

## Як це зробити:
Приклади коду та вихідні дані можна подивитись у блоках ```Kotlin ...```.

Щоб вивести повідомлення про помилку в стандартний вивід помилок, можна використовувати функцію ```System.err.println()```. Наприклад, якщо ви хочете повідомити користувачу про помилку з коментарем "Некоректні дані", то можете написати такий код:

```Kotlin
System.err.println("Некоректні дані")
```

Вихідні дані будуть виглядати наступним чином:

```
Некоректні дані
```

## Глибокий занурення:
Історичний контекст: раніше програмісти використовували стандартний вивід помилок для відлагодження програм, адже це було єдиним доступним способом виводу інформації прямо на екран. Однак, зараз є багато інших рішень для відлагодження програм, таких як логування або використання спеціальних програм для налагодження.

Альтернативи: як було сказано вище, для відлагодження програм можна використовувати логування або спеціальні програми для налагодження. Також деякі мови програмування мають свої власні методи для виводу повідомлень про помилки, наприклад, в Java це метод ```System.out.println()```.

Деталі реалізації: стандартний вивід помилок в Kotlin реалізований за допомогою менеджера виводу (output manager). Він відповідає за вивід інформації на екран та може бути перемиканий на різні канали (наприклад, на файл або в іншу програму для налагодження).

## Дивіться також:
Докладнішу інформацію про стандартний вивод помилок у Kotlin можна знайти на офіційному сайті мови програмування Kotlin (https://kotlinlang.org). Також, додаткове вивчення цієї теми можливе за допомогою документації Java (https://docs.oracle.com/javase/tutorial/essential/io/index.html) або за допомогою статтей на сайтах для програмістів, таких як https://tproger.ru.
