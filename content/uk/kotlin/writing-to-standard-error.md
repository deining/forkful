## Чому
Існує багато ситуацій, коли програмістам потрібно виводити повідомлення про помилки або статус виконання програми. Використання стандартного потоку для цього - швидкий та простий спосіб зробити це.

## Як
Один з способів виводу повідомлення в стандартний потік помилок (standard error) - за допомогою функції ```error()```. Наприклад:
```
Kotlin
fun main() {
    error("Це повідомлення про помилку")
}
```
В результаті виконання програми, на консоль буде виведено повідомлення про помилку:
```
Це повідомлення про помилку
```
Також, програміст може вручну отримати стандартний потік помилок за допомогою об'єкту ```System.err```. Для цього необхідно використати метод ```println``` та подати йому повідомлення у вигляді рядка:
```
Kotlin
fun main() {
    System.err.println("Це повідомлення про помилку")
}
```

## Детальніше
Використання стандартного потоку помилок має свої особливості. Наприклад, він не може бути перенаправлений за допомогою оператора ```>``` та піп (|). Це означає, що важливо правильно обробляти виводи програми до стандартного потоку помилок і не користуватись ним для звичайного виводу.
