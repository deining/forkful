# Чому

Часто під час програмування ми стикаємося з помилками, які можуть ускладнити процес налагодження і виправлення коду. Таке виникає, коли програма використовує стандартний потік виводу для повідомлень про помилки. Проте, щоб зробити цей процес більш ефективним і швидким, ми можемо писати повідомлення про помилки у стандартний потік помилок - stderr. Це дозволить нам швидко виявляти та вирішувати проблеми у програмі.

# Як

Для цього нам потрібно використати вбудований тип даних "System.err" у Котліні. Далі, ми можемо використати функцію "print()" або "println()" і передати їй повідомлення про помилку. Наприклад:

```Kotlin
fun main() {
    val age = -5
    if(age < 0) {
        System.err.println("Вік не може бути від'ємним")
    }
}
```

Після запуску програми, ми побачимо таке повідомлення про помилку у консолі:

```
Вік не може бути від'ємним
```

# Глибокий занурений

Стандартний потік помилок, або stderr, є одним з трьох основних потоків у програмі - разом зі стандартним потоком вводу (stdin) та стандартним потоком виводу (stdout). Він використовується для виведення повідомлень про помилки та винятки. Крім того, його можна перенаправити у файл для збереження цих повідомлень для майбутнього аналізу. 

Також варто зауважити, що при виконанні кількох команд одночасно, потоки можуть мішати один одному, що може призвести до некоректних відображень у консолі. Тому важливо правильно керувати потоками та їх перенаправленням у програмах.
