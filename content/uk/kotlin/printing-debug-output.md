## Чому
Вивід відлагоджувальних повідомлень є корисним для вирішення проблем з кодом та пошуку помилок.

## Як
Для виводу повідомлень в Kotlin можна використовувати функцію `print()` або `println()`, які приймають різні типи даних, наприклад:

```
Kotlin val num1 = 10
Kotlin val num2 = 20
print("Сума чисел: $num1 + $num2 =")
println(num1 + num2)

Kotlin val str = "Привіт"
println(str)
```

Результат виконання цього коду буде виглядати наступним чином:

```
Сума чисел: 10 + 20 = 30
Привіт
```

Також, можна використовувати функцію `debug()` з бібліотеки `kotlin-logging`, яка дозволяє виводити відлагоджувальні повідомлення з покращеним форматуванням та додатковими параметрами, наприклад:

```
Kotlin val logger = KotlinLogging.logger {}
val name = "Іван"
val age = 25
logger.debug { "Ім'я: $name, Вік: $age" }
```

Результат буде виглядати наступним чином:

```
Д [main] ivan_blog - Ім'я: Іван, Вік: 25
```

## Deep Dive
Вивід відлагоджувальних повідомлень дозволяє отримати більш детальну інформацію про виконання програми. Це може бути корисно при пошуку та виправленні помилок, а також для вивчення структури програми. Крім того, використання функцій `debug()` з бібліотеками, які підтримують логування рівня важливості повідомлень (наприклад, `info()`, `error()`, тощо), дозволяє зручно фільтрувати та відстежувати виконання програми.
