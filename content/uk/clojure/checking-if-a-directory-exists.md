## Чому
Можливо, ви бачили, як програми або скрипти перевіряють, чи існує певна директорія перед тим, як виконувати певні дії. Це робиться з метою упевненості, що папка, до якої нам потрібно звернутися, дійсно існує на операційній системі. У нашому блозі ми розглянемо основні причини, чому може знадобитися перевірка існування директорії у програмуванні на Clojure.

## Як
```Clojure
(def dir "/шлях/до/директорії")

(if (.exists (java.io.File. dir))
  (println "Директорія існує")
  (println "Директорія не існує"))
```

У цьому прикладі ми використовуємо функцію .exists з Java API для створення об'єкту File, який представляє певну директорію. Ця функція повертає булеве значення, яке показує, чи існує вказана директорія. Ми використовуємо умовне вираз if для виводу відповідного повідомлення у залежності від результату перевірки.

## Глибокий розгляд
Перевірка існування директорії є хорошим практичним рішенням для більшої надійності та безпеки програм. Якщо ми не перевіремо існування директорії перед доступом до неї, це може призвести до помилок або випадкового створення нової папки у несподіваному місці. Крім того, перевірка існування директорії також може допомогти уникнути втрати даних чи недосяжності файлів.
