# Чому

Написання тестів є важливою частиною будь-якої програми. Воно допомагає виявляти потенційні помилки та забезпечує якість коду. Крім того, тести дають впевненість у тому, що програма працює як очікувалося, що дозволяє швидше виявляти та виправляти будь-які проблеми.

# Як

Для написання та запуску тестів у Clojure використовуються різні бібліотеки, такі як clojure.test, clojure.spec і test.check. Давайте розглянемо приклади використання цих бібліотек.

```Clojure
(ns my-test-project.core-test
  (:require [clojure.test :refer :all]))

;; Ось приклад тесту для функції, яка додає два числа
(deftest add-two-numbers-test
  (testing "додає два числа"
    (is (= 5 (add-two-numbers 2 3)))) ;; виконати функцію add-two-numbers з аргументами 2 та 3 повинно повернути 5
```

Ви можете запустити цей тест, використовуючи команду `lein test`, і, якщо функція повертає очікуване значення, тест буде виконаний успішно. Для того, щоб перевірити, чи виконується певна умова в програмі, використовуються різні функції, такі як `is`, `are`, `isnt`, `are-not` та інші.

# Глибока занурення

Написання тестів важливо не тільки для перевірки функціональності програми, але й для створення документації та полегшення розробки. У Clojure є можливість описувати специфікації для функцій за допомогою бібліотеки clojure.spec. Це дозволяє визначити типи даних та обмеження для аргументів та повернутих значень функцій. Крім того, використання тестів допомагає виявляти та локалізувати проблеми в коді та полегшує рефакторинг.
