## Чому

Надсилання HTTP-запиту з базовою аутентифікацією - це потужний інструмент для обміну даними між клієнтом та сервером. Використання цього методу дозволяє забезпечити безпеку передачі даних та зменшити ризик несанкціонованого доступу до важливих інформаційних ресурсів. 

## Як користуватися

Нижче наведений приклад коду Ruby, який показує, як надіслати HTTP-запит з базовою аутентифікацією. Потрібно вказати свої дані для з'єднання (URL, логін та пароль), а також тип запиту (GET, POST, PUT, DELETE). Запустіть код у своєму терміналі та перевірте вихідний код, щоб побачити результат запиту.

```Ruby
require 'net/http'

# Вказати дані для з'єднання
url = URI.parse("https://example.com/api")
req = Net::HTTP::Get.new(url.path)
req.basic_auth("username", "password")

# Виконати запит та отримати відповідь
res = Net::HTTP.start(url.host, url.port) {|http|
  http.request(req)
}

# Вивести результата запиту
puts res.code
puts res.body
```

Вихідний код має повернути статус код 200 та тіло запиту, якщо ви правильно вказали ваші дані для з'єднання.

## Deep Dive

Надсилання HTTP-запиту з базовою аутентифікацією вимагає від клієнта передачі в заголовку запиту поля "Authorization", яке містить ваш логін та пароль, закодовані у базовому кодуванні. Сервер, на який надсилається запит, перевіряє це поле для підтвердження автентифікації користувача перед наданням доступу до ресурсів.

На відміну від інших методів аутентифікації, базова аутентифікація не надійна, оскільки дані передаються у відкритому вигляді. Щоб збільшити рівень безпеки, рекомендується використовувати HTTPS замість HTTP для передачі даних.
