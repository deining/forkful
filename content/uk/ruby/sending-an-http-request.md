## Зачем

В програмуванні, і особливо в програмуванні на Ruby, надсилання HTTP-запитів є однією із найбільш важливих операцій. Це дає можливість взаємодіяти з різними сервісами та отримувати необхідні дані. Також це є необхідною умовою для розробки веб-додатків із використанням API. 

## Як

В Ruby, для надсилання HTTP-запиту існують декілька способів. Розглянемо найпростіший з них за допомогою прикладу.

```
require 'net/http'

url = URI('https://example.com') # замінити на посилання, яке потрібно запитати
res = Net::HTTP.get_response(url)
puts res.body
```

В цьому прикладі ми використали стандартну бібліотеку Ruby - `net/http`. За допомогою методу `get_response` ми отримуємо відповідь на запит та виводимо тіло відповіді у термінал.

Існують також бібліотеки, які спрощують надсилання запитів, такі як `httparty` та `rest-client`. Розглянемо такий самий приклад, але вже за допомогою `httparty`:

```
require 'httparty'

url = 'https://example.com' # замінити на посилання, яке потрібно запитати
res = HTTParty.get(url)
puts res.body
```

Як бачимо, код став коротшим та зрозумілішим. Детальніше про ці бібліотеки можна прочитати у наступному розділі.

## Deep Dive

Надсилання HTTP-запитів в Ruby є досить простим, але це лише вершина айсберга. Щоб самостійно вирішувати різноманітні задачі пов'язані з HTTP-запитами, необхідно розуміти принципи роботи цього протоколу та мати глибокі знання про нього.

Для початку, рекомендується ознайомитися зі специфікацією HTTP та розібратися з основними методами: GET, POST, PUT, DELETE. Також важливо знати про заголовки (headers) та параметри запиту (query parameters).

Після цього, варто розглянути різні бібліотеки, які покращують роботу з HTTP-запитами. Наприклад, `faraday` або `httpclient` мають більше можливостей та можуть бути корисними для більш складних завдань.
