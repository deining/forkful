## Чому

Перевірка та порівняння двох дат є важливим аспектом програмування, особливо в контексті роботи зі строками та роботи з даними, пов'язаними з часом. Це дозволяє зробити програму більш гнучкою та ефективною.

## Як

Найпростішим способом порівняти дві дати є використання методу `.==` для об'єктів типу `Date` або `DateTime`. Для цього необхідно використовувати один з таких варіантів коду:

```Ruby
require 'date'

Date.new(2021, 9, 12) == Date.new(2021, 9, 12)  # → true
DateTime.new(2021, 9, 12, 12, 0, 0) == DateTime.new(2021, 9, 12, 12, 0, 0)  # → true
```

Однак, якщо нам потрібно перевірити, чи одна дата більша або менша за іншу, ми можемо використовувати методи `.>` та `.<=` або ж використовувати оператори `>` та `<=`.
Наприклад:

```Ruby
require 'date'

Date.new(2021, 9, 10) > Date.new(2021, 9, 8)  # → true
DateTime.new(2021, 9, 12, 12, 0, 0) <= DateTime.new(2021, 9, 13, 12, 0, 0)  # → true
```

## Глибоке занурення

Для того, щоб краще зрозуміти, як працює порівняння дат в Ruby, давайте розглянемо base class `Date`.

У даному класі визначені всі методи для порівняння дат, включаючи `.==`, `.<`, `.>`, `.<=` та `.>=`.  Однак, інші класи, які успадковують від `Date` (наприклад, `DateTime` або `Time`), визначають власні методи порівняння, базуючись на своїй індивідуальній внутрішній структурі. Це може призвести до неочікуваної поведінки порівняння між об'єктами з різних класів.

Необхідно бути уважним та чітко розуміти, з якими об'єктами ми маємо справу при порівнянні дат.
