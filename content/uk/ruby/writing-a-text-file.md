## Початок роботи із написанням текстового файлу

Написання текстових файлів є важливим аспектом програмування, який дозволяє зберігати дані, такі як рядки тексту, у структурованому форматі. Це необхідно для зберігання і передачі інформації між програмами та системами. У цій статті ми розглянемо деталі написання текстових файлів у мові Ruby для українських читачів.

## Як це зробити

Написання текстових файлів у мові Ruby досить просте. Нижче наведені базові кодові приклади для створення і записування даних у текстовий файл. 

```Ruby
# Відкриття файлу для запису
file = File.open("data.txt", "w")

# Запис рядка тексту у файл
file.puts "Привіт, у світі Ruby!"

# Закриття файлу
file.close
```

При виконанні цього коду, у файлі "data.txt" буде збережений рядок "Привіт, у світі Ruby!". Можна також додавати більше даних у файл, використовуючи метод `puts`, що дозволяє додавати рядки у кінець файлу.

```Ruby
# Відкриття файлу для дописування
file = File.open("data.txt", "a")

# Дописування рядка тексту у файл
file.puts "Це ще один рядок тексту"

# Закриття файлу
file.close
```

Також можна зчитувати вміст текстового файлу та оброблювати дані. Наприклад, ми можемо зчитати кожен рядок файлу та вивести його на екран:

```Ruby
# Відкриття файлу для читання
file = File.open("data.txt", "r")

# Зчитування та виведення кожного рядка файлу
file.each do |line|
  puts line
end

# Закриття файлу
file.close
```

## Глибший опис

У мові Ruby існує багато методів та класів для роботи з текстовими файлами, що дозволяє зберігати та оброблювати дані у них. Методи `open`, `puts`, `gets` та `close` використовуються для відкриття, запису, зчитування та закриття файлів відповідно. Також є методи для перевірки наявності файлу, видалення файлу та багато інших.

Також важливо зазначити, що при роботі з текстовими файлами, потрібно пам'ятати про обробку винятків (exceptions), оскільки можуть виникати помилки при зчитуванні або записуванні файлу. Для цього можна використовувати блок `begin/rescue/end` для обробки помилок та виведення відповідного повідомлення.
