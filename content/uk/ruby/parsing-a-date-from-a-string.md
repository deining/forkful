## Що & Чому?
Toast шукаємо 2002 року поминальних, ми парсимо дату з рядка. Це означає, що ми перетворюємо текстову інформацію, таку як "серпня 15, 2021" у вигляді об'єкту дати, щоб комп'ютер міг легше з ними працювати. Програмісти часто використовують цей процес для обробки дати з різних форматів, таких як уранційні строки або строки, згенеровані користувачами.

## Як зробити:
```Ruby
require 'date'

Date.strptime("August 15, 2021", "%B %d, %Y")
#=> #<Date: 2021-08-15 ((2459431j,0s,0n),+0s,2299161j)>
```

```Ruby
require 'date'

Date.parse("15/08/2021")
#=> #<Date: 2021-08-15 ((2459431j,0s,0n),+0s,2299161j)>
```

У цих прикладах ми використовуємо модуль дати та його методи - `strptime` та `parse` - щоб перетворити рядок у форматі дати Ruby міжнародного стандарту, який комп'ютер може правильно інтерпретувати. З використанням відповідних форматувань дати можливе парсінг дат з різних форматів.

## Глибокий погляд:
Історичний контекст: Парсінг дат зробив гігантський стрибок наперед з появою Електронної машини. Раніше жодних бібліотек для цього не було, тож люди використовували власні методи парсінгу дати. Сьогодні існує кілька альтернативних бібліотек для парсінгу дат, таких як Chronic та Time.parse, які можуть бути корисними в деяких випадках.

Деталі реалізації: У Ruby є кілька вбудованих методів для роботи з датами, таких як `strptime`, `parse` та `Time`. Ці методи дозволяють використовувати різні форматування та параметри для парсінгу та роботи з датами. Крім того, деякі сторонні бібліотеки надають додаткові функції та зручний синтаксис для парсінгу дат.
