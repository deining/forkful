## За що?

Перевірка існування директорії є важливою для багатьох програм на Ruby, щоб переконатися, що потрібні файли можуть бути знайдені та оброблені правильно. Це також може стати корисним, коли потрібно виконати певні дії, якщо директорія вже існує.

## Як?

```Ruby
# Перш за все, потрібно підключити бібліотеку 'FileUtils'
require 'fileutils'

# Перевірка існування директорії за допомогою методу 'exist?'
if File.exist?('my_directory')
  puts "Директорія існує!"
else
  puts "Директорія не існує!"
end

# Створюємо нову директорію за допомогою методу 'mkdir'
# Якщо директорія вже існує, то буде викинуто помилку
FileUtils.mkdir('my_directory')
```

### Вихідні дані:
```
Директорія існує!
```

## Глибоке занурення

Для більш детального розуміння роботи і функцій методів `'exist?'` та `'mkdir'`, давайте розглянемо їх докладніше.

Метод `'exist?'` перевіряє, чи існує файл або директорія за вказаним шляхом. Він повертає `true`, якщо файл або директорія існує, і `false`, якщо не існує. Цей метод також приймає необов'язковий другий аргумент - режим перевірки, що дозволяє враховувати власника, права доступу і час зміни файлу.

Метод `'mkdir'` створює нову директорію за вказаним шляхом. Якщо директорія вже існує, викидається помилка `Errno::EEXIST`. Цей метод також приймає кілька необов'язкових аргументів, які дозволяють задати права доступу, створити декілька директорій одночасно та інші використовуються для додаткової налаштування.

## Використовуйте ці методи розумно

Важливо використовувати метод `'exist?'` перед викликом методу `'mkdir'`, щоб переконатися, що директорія не існує вже. Також не забувайте перевіряти результати виклику методу `'mkdir'`, оскільки він може викинути помилку у випадку, якщо директорія вже існує.
