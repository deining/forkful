# Чому

Багато програмістів так чи інакше зв'язані з обробкою даних. Велика кількість цих даних, особливо у великих проектах, потребує ефективного способу їх зберігання і обробки. І це саме те, що пропонує YAML!

## Як

YAML - це формат збереження та обробки даних в зручному для читання і написання вигляді. Це особливо корисно, якщо ви працюєте зі структурованими даними, такими як списки або словники. Давайте розглянемо кілька прикладів, щоб зрозуміти, як працювати з YAML у Ruby.

```Ruby
require 'yaml'

# Створення словника даних
data = {
  'name' => 'Іван',
  'age' => 25,
  'hobbies' => ['футбол', 'малювання', 'музика']
}

# Збереження даних у форматі YAML
yaml_data = YAML.dump(data)
# Виведення збережених даних
puts yaml_data

# Виведення даних у зручному для читання вигляді
puts 'Ім\'я: ' + data['name']
puts 'Вік: ' + data['age'].to_s
puts 'Хобі: '
data['hobbies'].each { |hobby| puts '- ' + hobby }
```

Результат виконання коду:

```yaml
--- # Розпочинаємо збереження даних у форматі YAML
# Інформація про користувача
name: Іван
age: 25
hobbies:
- футбол
- малювання
- музика
```

Ім'я: Іван
Вік: 25
Хобі:
- футбол
- малювання
- музика

## Глибше

YAML також може бути використаний для збереження складніших структур даних, таких як об'єкти або навіть класи. Крім того, ви можете використовувати спеціальні теги YAML для визначення типу даних або інших параметрів. Наприклад:

```Ruby
require 'yaml'

# Створення класу
class User
  def initialize(name, age)
    @name = name
    @age = age
  end
end

# Створення об'єкта і збереження у форматі YAML
user = User.new('Катя', 30)
yaml_user = YAML.dump({'user' => user})
# Додавання тегу для визначення типу даних
yaml_user.gsub!(/!ruby\/object\:User/, '!User') # Це може бути зроблено звичайним використанням глобальної підстановки

# Виведення збережених даних
puts yaml_user
```

Результат виконання коду:

```yaml
--- !User
user:
  age: 30
  name: Катя
```
