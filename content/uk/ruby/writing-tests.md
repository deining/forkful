## Чому

Написання тестів є важливою частиною розробки програмного забезпечення, оскільки воно дозволяє перевірити правильність роботи коду та виявити можливі помилки. Без тестів важко гарантувати якість програми та забезпечити її стабільність та відповідність бізнес-вимогам.

## Як

Написання тестів в Ruby може бути досить простим та ефективним завданням. Для початку, тести слід розподілити на три типи: одиниць, інтеграційні та прийняття. Для того, щоб створити тест, слід виконати наступні кроки:

1. Спочатку потрібно встановити фреймворк для тестування, наприклад, RSpec або MiniTest. Встановлення можливе за допомогою менеджера пакетів Ruby - Bundler.
2. Створюйте файли з розширенням .rb та надавайте їм назви, що описують тести, наприклад, "user_spec.rb".
3. Встановіть посилання на бібліотеки фреймворку у файлі тесту, щоб отримати доступ до методів, необхідних для написання тестів.
4. Використовуйте "describe" для опису тестування конкретної функції або класу.
5. Використовуйте "it" для опису конкретного тесту та його очікуваного результату.
6. Виконайте необхідне перевірка за допомогою вбудованих методів (приклади наведено нижче).
7. Запустіть тести при використанні команди "rspec <файл тесту>".

Перевірка результатів може виглядати наступним чином:

```Ruby
describe User do # описуємо клас User
  it "should return the correct full name" do # описуємо очікуваний результат
    user = User.new(first_name: "John", last_name: "Doe")
    # створюємо новий екземпляр класу User з переданими аргументами
    expect(user.full_name).to eq("John Doe") # перевіряємо, чи співпадає результат з очікуваним
  end
end
```

## Deep Dive

Написання тестів вимагає розуміння концепцій організації коду та його структури. Найчастіше використовувані методи для перевірки результатів це "assert" (твердження) та "expect" (очікування). Також важливо визначити граничні випадки для тестування, щоб переконатися, що програма поводиться правильно в усіх сценаріях.
