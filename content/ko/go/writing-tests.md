# 왜 테스트를 작성해야 하나요?

따뜻한 여름날, 우리를 놀라게 한것은 한 채의 집이다. 그것은 제대로 지어져 있다가도 파도에서 떠내려갔다. 이 역사는 문제를 찾아내는 신화같은 파도의 핸들로 여러분을 기습할 수 있다.

공학적인 프로젝트를 작업 할 때도 마찬가지입니다. 작성되지 않은 테스트가 있을 때, 어떤 코드가 작동하지 않고 프로그램이 예기치 않게 충돌 할 지도 모릅니다. 이러한 상황에서 테스트 작성의 중요성을 깨닫게됩니다.

테스트가 없다면, 코드 수정시 그 깊은 기본면을 이해하기 어렵고 불확실합니다. 모든 변경 사항이 예상치못한 결과를 초래하고 최악의 경우 팀에 자금을 흘려버릴 수도 있습니다.

하지만 테스트를 작성하여 어려운 상황을 효율적으로 관리 할 수 있습니다. 그래서 지금부터 우리는 Go 프로그래밍에서 테스트가 왜 중요한지 살펴보겠습니다.

## 테스트 작성 방법

먼저 "```Go ... ```" 코드 블록 내에서 코딩 예시와 출력을 제공합니다.

예를 들어, 우리가 간단한 함수를 작성할 때 어떻게 테스트를 작성할 수 있을까요? 아래의 예시를 봅시다.

```Go
func multiply(x, y int) int {
	return x * y
}

func TestMultiply(t *testing.T) {
	result := multiply(2, 3)

	if result != 6 {
		t.Errorf("Expected 6, but got %v", result)
	}
}
```

위의 코드는 두 수를 곱하는 간단한 함수를 작성한 후 테스트를 작성하는 방법을 보여줍니다. "```Go ... ```" 코드 블록을 사용하여 코드 조각을 포함시켜 코드가 가독성 있게 나타나도록합니다.

## 깊이 파고들기

테스트를 작성하는 것은 간단한 것처럼 보일 수 있지만, 실제로는 코드의 깊은 부분을 이해하는데 많은 도움이 됩니다. "테스트 주도 개발"이라는 개념은 개발자가 테스트를 먼저 작성하고 그에 맞게 코드를 작성하도록 유도하는 것을 말합니다.

또한 Go 언어에서는 테스트 코드를 작성하는데 있어서 자체적으로 편리한 기능과 도구를 제공합니다. 가장 많이 사용되는 "testing" 패키지를 이용하여 코드를 테스트하는데 필요한 다양한 메서드를 제공합니다.

따라서 테스트 작성은 단순히 코드를 테스트하는 것 이상의 의미가 있습니다. 실제로는 개발자의 관점을 확대하고 프로그램을 더욱 견고하게 만드는데 도움이 됩니다.
