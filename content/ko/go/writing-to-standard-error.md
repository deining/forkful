## 왜

Go 프로그래밍은 마이크로서비스 아키텍처 또는 분산 시스템과 같은 복잡한 응용 프로그램에서 높은 수준의 견고함을 제공합니다. 하지만 이러한 응용 프로그램에서 오류를 디버깅할 때 가장 어려운 부분 중 하나는 오류 메시지를 추적하는 것입니다. 내장된 로그 함수를 사용하여 이를 처리할 수 있지만, 때때로 디버그를 위해 표준 오류에 직접 로그를 쓰는 것이 더욱 효과적일 수 있습니다.

이 블로그 포스트에서는 Go 언어에서 표준 오류에 로그를 쓰는 방법과 그 이유를 알아보겠습니다.

## 하우 투

Go 언어에서는 os 패키지의 Stderr 변수를 사용하여 표준 오류에 로그를 쓸 수 있습니다. 아래 코드 예제를 참고하세요.

```Go
import "os"

func main() {
    f, err := os.Open("test.txt")
    if err != nil {
        // 표준 오류에 로그를 출력합니다.
        fmt.Fprintln(os.Stderr, "파일을 열지 못했습니다.", err)
    }
}
```

위 코드에서는 fmt 패키지의 Fprintln 함수를 사용하여 표준 오류에 로그를 기록합니다. 이를 통해 표준 오류에 로그를 출력하므로써 오류를 추적할 수 있게 됩니다.

## 딥 다이브

실제로 표준 오류에 로그를 출력하는 것은 매우 간단하지만, 이를 통해 어떤 추가적인 이점을 얻을 수 있을까요?

Go 언어에서는 표준 오류에 로그를 출력하는 것을 권장하고 있습니다. 이는 표준 출력보다 기능이 더 많기 때문입니다. 예를 들어, 표준 오류에는 색상을 적용하여 오류를 시각적으로 더 잘 구분하거나, 오류가 발생한 라인 수와 같은 추가 정보를 포함할 수 있습니다.

또한, 미들웨어나 로깅 라이브러리에서는 이러한 기능을 활용하여 더욱 가독성 높은 오류 로그를 생성할 수 있습니다.
