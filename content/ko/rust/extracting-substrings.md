# 왜

서브스트링을 추출하는 것에 참여하는 이유는 무엇일까요? 이것은 불변 데이터 형식과의 작업을 더욱 효율적이고 안전하게 만들기 위해 중요합니다.

서브스트링은 문자열 내에서 문자의 부분집합을 의미합니다. Rust는 문자열을 조작하는 유용한 기능들을 제공하지만, 때때로 문자열의 일부분만 필요한 상황이 있을 수 있습니다. 이를 위해 Rust는 내장 `slice` 타입을 제공합니다. `slice`는 문자열의 일부분을 가리키는 포인터로, 불변 데이터를 보장합니다.

하지만, `slice`를 사용하기 위해서는 문자열의 길이와 같은 문자열의 정보를 알아내는 추가적인 작업이 필요합니다. 이러한 작업은 복잡하고 오류가 발생하기 쉽습니다. 그렇기 때문에 서브스트링 추출이 필요하게 됩니다.

# 방법

다음은 Rust에서 서브스트링을 추출하는 방법의 예시 코드입니다.

```Rust
let s = "안녕하세요! 여러분";
let hello = &s[0..2]; // 0부터 1까지의 인덱스를 추출하여 Hello라는 변수에 저장
```

위의 코드에서 `&s[0..2]`는 s 문자열의 0부터 1까지의 인덱스를 추출하여 `slice` 타입 변수에 저장하는 것을 의미합니다. 여기서 첫 번째 숫자는 추출을 시작할 인덱스를 나타내고, 두 번째 숫자는 추출할 인덱스 바로 다음의 인덱스를 나타냅니다.

다음은 위의 코드의 실행 결과입니다.

```bash
Hello
```

위의 예시 코드에서는 문자열의 일부분만 필요한 상황이었지만, 만약에 `slice`를 사용하지 않고 문자열 전체를 복사하게 된다면 메모리 낭비가 발생합니다. 그렇기 때문에 Rust는 `slice` 타입을 이용하여 메모리를 절약하고, 문자열의 일부분만을 사용할 수 있도록 합니다.

또한 `slice`는 문자열의 일부분만을 가리키는 포인터이기 때문에, 원본 문자열이 수정되어도 서브스트링에 영향을 주지 않습니다. 서브스트링을 추출할 때 원본 문자열을 수정해도 안전하다는 것을 보장할 수 있습니다.

# 더 깊게

Rust에서는 `slice`를 사용하여 문자열의 일부분을 추출하는 것 뿐만 아니라, 벡터 등의 다른 데이터 형식에서도 사용할 수 있습니다. 또한 마찬가지로 `slice`를 이용하여 메모리 절약과 데이터의 안전성을 보장할 수 있습니다.

이처럼 Rust에서 `slice`는 매우 유용한 타입이며, 잘 활용한다면 효율적인 프로그래밍을 할 수 있습니다.
