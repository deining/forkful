## 왜: 날짜를 미래나 과거로 계산하는 프로그래밍을 수행하는 이유는 무엇인가요?

조선 배포일이나 과거에 발생한 이벤트의 날짜를 알고 싶거나, 지정된 일정을 확인하기 위해 날짜를 계산하는 등 다양한 상황에서 날짜를 계산하는 프로그래밍이 필요합니다.

## 계산하는 방법: "```Javascript ... ```" 코드 블록 안에 코딩 예제 및 샘플 출력을 넣어 설명합니다.

계산하는 방법에는 다양한 방법이 있지만, 가장 간단하고 흔한 방법은 Date 객체를 이용하는 것입니다. 정확한 날짜를 계산하기 위해서는 날짜와 시간을 나타내는 다양한 속성들을 이용해야 합니다. 예를 들어, 년도를 계산할 때는 getFullYear() 메소드를 사용하고, 월을 계산할 때는 getMonth() 메소드를 사용합니다.

### 코드 예제:

```Javascript
// 오늘 날짜와 시간을 담은 Date 객체 생성
var today = new Date();

// 년도 계산
var year = today.getFullYear();

// 월 계산 (0부터 시작하므로 1을 더해줌)
var month = today.getMonth() + 1;

// 예상 배포일 계산 (오늘 날짜에서 3일 후)
var deliveryDate = today.getDate() + 3;

// 출력
console.log("오늘 날짜:", year + "년", month + "월", today.getDate() + "일");
console.log("예상 배포일:", year + "년", month + "월", deliveryDate + "일");
```

### 샘플 출력:

```
오늘 날짜: 2021년 6월 14일
예상 배포일: 2021년 6월 17일 
```

## 깊이 파헤치기: 미래나 과거로 날짜를 계산하는 더 깊은 정보

날짜를 계산하는 프로그래밍에서 가장 중요한 부분은 기준이 되는 시간이라는 것입니다. 기본적으로 Date 객체는 현재 시간으로 초기화되기 때문에, 날짜를 계산하는 디폴트 값은 오늘 날짜와 시간을 기준으로 합니다. 하지만, 원하는 날짜와 시간을 기준으로 계산하고 싶을 경우에는 해당 날짜와 시간을 명시적으로 설정해주어야 합니다.

또한, 날짜를 계산할 때 주의해야 할 점은 각 나라마다 사용하는 시간대(Timezone)가 다르다는 것입니다. 날짜를 계산할 때 이를 고려하지 않으면 원하는 날짜와 다른 결과가 나올 수 있습니다. 따라서, 정확한 날짜를 얻고 싶다면 시간대를 명시적으로 설정해주어야 합니다.
