## 왜 만들어야 하나요?

만약 여러분이 프로그래밍 작업을 하다가 임시 파일을 만들어야 할 일이 있다면, 일반적으로 이유는 그 파일을 사용하는 프로그램들이 특정 형식의 파일을 필요로하기 때문입니다. 이럴 때 우리는 임시 파일을 만들고, 필요한 작업을 수행한 뒤에는 그 파일을 삭제합니다.

## 임시 파일 만드는 방법

임시 파일을 만드는 방법에는 여러 가지가 있지만, 여기서는 Fish Shell을 사용한 예제를 보여드리겠습니다. 우리가 사용할 명령어는 `mktemp`입니다. 마크다운 코드 블록 안에 있는 코드를 실행해 보세요.

```Fish Shell
# 임시 파일 만들기
mktemp

# 파일 이름 지정하기
mktemp example.txt
```

위 코드를 실행하면 임시 파일이 현재 디렉토리에 생성될 것입니다. 또한 파일 이름을 지정하여 생성할 수도 있습니다. 방금 만든 임시 파일은 다음과 같은 결과를 출력합니다.

```
/tmp/tmp.GLcAzy1fUC/example.txt
```

위와 같이 임시 파일의 경로가 출력되는 것을 볼 수 있습니다.

## 깊게 파고들기

임시 파일을 생성하는 명령어는 다양하게 있지만, 가장 많이 사용되는 것 중 하나는 `mktemp`입니다. 이 명령어는 운영체제의 기본 임시 디렉토리에 파일을 만드는 것이 기본 설정입니다. 다만, 여러분이 원하는 디렉토리에 파일을 만들고 싶다면 `--tmpdir` 옵션을 추가하여 경로를 지정할 수 있습니다.

또한, `mktemp`는 특정 파일 형식의 파일을 생성하고 싶을 때 유용합니다. `--suffix` 옵션을 사용하면 파일 이름 뒤에 원하는 접미사를 붙일 수 있습니다. 예를 들어, `.csv` 접미사를 붙이면 CSV 파일을 생성할 수 있습니다.
