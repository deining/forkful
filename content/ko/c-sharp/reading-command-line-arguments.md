## 왜

커맨드 라인 인자를 읽는것에 대해 관심이 있는 분들이 이 글을 읽는 이유는 무엇일까요? 커맨드 라인 인자는 프로그래밍에서 상당히 중요한 역할을 합니다. 사용자가 프로그램을 실행할 때, 특정한 값을 인자로 전달하여 프로그램의 결과를 바꿀 수 있습니다. 이를 통해 사용자는 동적으로 프로그램의 실행을 조정할 수 있고 더욱 효율적으로 사용할 수 있습니다. 따라서, 커맨드 라인 인자를 읽는 방법에 대해 알고 있는 것은 중요합니다.

## 하는 법

커맨드 라인 인자를 읽는 방법은 매우 간단합니다. 먼저, ```Main``` 메서드의 파라미터에 ```string[] args```를 추가해야 합니다. 이것은 ```args``` 배열에 사용자가 입력한 인자들이 저장됩니다. 그리고 이 배열을 반복문을 통해 접근하여 원하는 작업을 수행하면 됩니다. 예를 들어, 아래의 코드는 사용자가 입력한 인자를 반복문을 통해 출력하는 예제입니다.

```C#
static void Main(string[] args)
{
    foreach (string arg in args)
    {
        Console.WriteLine(arg);
    }
}
```

만약 사용자가 "dotnet run arg1 arg2"와 같은 인자를 입력하면, 아래와 같은 출력결과를 얻을 수 있습니다.

```
arg1
arg2
```

이렇게 간단하게 커맨드 라인 인자를 읽을 수 있습니다. 또한, 인자에 대한 유효성 체크나 형변환 등 다양한 작업도 가능합니다.

## 깊게 파헤치기

커맨드 라인 인자를 읽는 것에 대해 더 깊이 알아보겠습니다. 먼저, 우리는 배열의 인덱스를 사용하여 특정 위치의 인자를 가져오거나 수정할 수 있습니다. 또한, ```Environment.GetCommandLineArgs()``` 메서드를 사용하여 모든 인자를 배열로 반환받을 수도 있습니다. 이 메서드는 현재 프로세스의 커맨드 라인 인자를 모두 반환해주는데, 첫 번째 인자는 실행 경로가 들어가게 됩니다.

또 다른 중요한 포인트는 인자의 옵션을 설정하는 것입니다. 인자의 옵션은 ```-```나 ```--```로 시작하는데, 이를 통해 어떤 인자가 어떤 의미를 가지는지 명시할 수 있습니다. 예를 들어, ```--help``` 옵션은 해당 프로그램의 도움말을 출력하는데 사용될 수 있습니다. 이러한 다양한 옵션들이 있기 때문에, 인자를 읽을 때 유용하게 활용할 수 있습니다.
