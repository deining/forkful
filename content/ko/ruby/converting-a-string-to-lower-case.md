글 쓴 이유: 문자열을 소문자로 변환하는 일에 참여하는 이유를 1-2문장으로 설명

많은 프로그래머들은 문자열을 처리하고 다루는데 많은 경험이 있습니다. 문자열은 자주 사용되는 데이터 타입이기 때문에 우리는 이것을 다루는데 익숙해져야 합니다. 하지만 때로는 문자열 중에서도 대문자와 소문자를 구별해야 할 필요가 있습니다. 예를 들어, 사용자의 입력을 받는 기능을 만들 때 이름에 대문자가 들어가도 문자열을 저장할 때 소문자로 변환하여 저장하는 경우가 있습니다. 또는 학교나 회사에서 사용하는 이메일 주소는 대부분 소문자로 이루어져 있기 때문에 입력 받은 문자열을 소문자로 변환하여 검증하는 경우도 있습니다. 

이처럼 문자열을 소문자로 변환하는 일은 우리의 일상 생활에서 자주 발생하는 일입니다. 따라서 프로그래머로서 우리는 문자열을 소문자로 변환하는 기능을 자유롭게 다룰 수 있어야 합니다. 이를 위해서는 Ruby에서 제공하는 메소드를 활용하여 코드를 작성할 수 있어야 합니다. 다음으로는 문자열을 소문자로 변환하는 방법에 대해 알아보겠습니다.

방법: 예시 코드와 결과를 보여주는 "```Ruby ... ```" 코드 블록

우선 Ruby에서 문자열을 소문자로 변환하는 가장 간단한 방법은 String 클래스의 downcase 메소드를 활용하는 것입니다. 예시 코드는 다음과 같습니다.
```
name = "JOHN"
puts name.downcase
```

결과는 다음과 같습니다.
```
john
```
위의 코드에서 downcase 메소드를 사용하면 name 변수의 값인 "JOHN"이 소문자로 변환되어 출력됩니다.

하지만 downcase 메소드는 모든 문자열을 소문자로 변환하는 것이기 때문에 모든 문자가 소문자인 문자열은 변화가 없게 됩니다. 따라서 만약 모든 문자가 소문자인지 확인한 후에 변환하고 싶다면 downcase! 메소드를 사용해야 합니다. 예시 코드는 다음과 같습니다.
```
email = "example@example.com"
if email == email.downcase
  puts "이메일 주소가 유효합니다."
else
  puts email.downcase!
end
```

결과는 다음과 같습니다.
```
example@example.com
```

위의 코드에서 downcase! 메소드는 email 변수의 값인 "example@example.com"이 모두 소문자인지 확인한 후에 모두 소문자로 변환해주는 역할을 합니다. 따라서 이를 이용하면 모든 문자가 소문자인지 확인한 후에 변환할 수 있습니다.

이외에도 Ruby에는 문자열을 소문자로 변환하는 다양한 메소드들이 있습니다. 이 메소드들을 사용하여 다양한 상황에 맞게 문자열을 소문자로 변환할 수 있습니다. 

더 알아보기: 문자열을 소문자로 변환하는 방법에 대해 깊이 알아보겠습니다. Ruby에서 문자열을 소문자로 변환하는 방법에는 downcase 외에도 upcase, swapcase 등의 메소드가 있습니다. 이 메소드들은 각각 대문자를 소문자로, 소문자를 대문자로, 대소문자를 바꿔서 변환하는 역할을 합니다.
