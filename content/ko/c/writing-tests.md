## 왜 테스트(test)를 작성해야 할까요?

코딩을 배우고 실제로 프로그래밍을 하면서 테스트를 작성하는 걸 들어본 적이 있을 것입니다. 그러나 왜 테스트를 작성해야 하는지에 대해서는 잘 이해하지 못했을 수도 있습니다. 그래서 오늘은 여러분과 함께 테스트에 대해 이야기해보고 이를 작성하는 법을 알아보겠습니다.

테스트를 작성하는 이유는 무엇일까요? 테스트는 코드에 버그가 있는지를 확인하기 위해 사용됩니다. 즉, 테스트를 작성함으로써 코드를 더 안전하고 신뢰성 있게 만들 수 있습니다. 또한, 테스트를 작성함으로써 매번 코드를 수정하고 나서도 수동으로 실행하여 오류가 있는지를 확인할 필요가 없어집니다. 간단히 말해서, 테스트를 작성하면 시간과 노력을 절약할 수 있습니다.

## 테스트를 어떻게 작성할까요?

이제 테스트를 작성하는 방법을 알아보겠습니다. 테스트를 작성할 때는 C 언어의 `assert()` 함수를 사용합니다. 이 함수는 인자로 전달된 값이 `false`이면 프로그램을 멈추고 오류를 출력합니다. 이를 이용하여 간단한 예제를 작성해보겠습니다. 

```C
#include <stdio.h>
#include <assert.h>

int sum(int a, int b) {
    return a + b;
}

int main() {
    // sum 함수가 기대한 결과값을 반환하는지 확인합니다
    assert(sum(3, 6) == 9);
    // 또 다른 예제
    assert(sum(10, -2) == 8);
    // 다음 코드는 오류가 발생합니다
    assert(sum(5, 6) == 12);
    
    printf("테스트가 모두 통과되었습니다!");
    return 0;
}

```

위 코드를 실행하면 `테스트가 모두 통과되었습니다!`라는 출력문이 나오게 됩니다. 그러나 `sum(5, 6) == 12`가 `false`이므로 프로그램이 그 이후로 실행되지 않습니다. 이를 통해 오류를 쉽게 찾아낼 수 있음을 알 수 있습니다. 

또 다른 예시를 살펴보겠습니다. 

```C
#include <stdio.h>
#include <assert.h>

void print_message() {
    printf("우리 모두 C 언어를 잘 배웁시다!");
}

int main() {
    // print_message 함수가 제대로 실행되는지 확인합니다
    assert(print_message() == NULL);
    
    printf("테스트가 모두 통과되었습니다!");
    return 0;
}

```

위 코드를 실행하면 같은 오류가 발생하지 않습니다. `print_message()` 함수는 `void`형식이기 때문에 `NULL`을 반환하는 것이 정상적입니다. 이와 같이 테스트를 통해 오류를 찾는 것이 효율적입니다.
