## 왜 테스트 코드를 작성해야 할까요?

취업 시장은 매우 경쟁적입니다. 따라서 개발자로서 우리는 자신의 능력을 증명할 필요가 있습니다. 하지만 코드 작성만으로는 충분하지 않습니다. 다른 사람들과 마찬가지로 우리도 실수를 할 수 있고 버그를 낼 수 있습니다. 더구나 이런 일이 발생한다면 많은 시간과 비용이 소요될 수 있습니다.

테스트 코드를 작성함으로써 우리는 우리가 작성한 코드의 정확성을 쉽게 확인할 수 있습니다. 또한 코드 수정을 할 때도 테스트 코드가 우리에게 큰 도움이 되어 실수를 줄일 수 있습니다. 따라서 우리는 테스트 코드를 작성하는 것이 중요하다고 할 수 있습니다.

## 어떻게 테스트 코드를 작성할 수 있을까요?

C 프로그래밍에서는 `assert()` 함수를 사용하여 간단하고 효율적으로 테스트 코드를 작성할 수 있습니다. 아래 예시 코드를 확인해 보세요.

```C
#include <assert.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    // 두 수를 더한 결괏값이 맞는지 확인하는 테스트 코드
    assert(add(2, 3) == 5);

    // 더한 결과가 0이 아닌지 확인하는 테스트 코드
    assert(add(5, -5) != 0);

    // 실제 코드와 다른 결과를 내는 잘못된 테스트 코드
    assert(add(2, 2) == 5); // 이 코드는 실패합니다.
    
    return 0;
}
```

위 코드에서 `assert()` 함수는 괄호 안의 조건이 참이 아닐 경우 `Assertion failed` 메시지를 출력하고 프로그램을 종료합니다. 따라서 프로그램을 실행할 때 `assert()` 함수는 우리가 작성한 코드의 정확성을 검증하는 역할을 합니다.

만약 테스트가 실패한다면 우리는 코드를 수정하여 다시 실행하거나 `assert()` 함수의 조건을 수정하여 예상하는 결과를 얻을 수 있게 됩니다.

## 깊이 알아보기

테스트 코드를 작성하는 것은 단순히 정확성을 검증하는 것만이 아닙니다. 테스트 코드는 더 깊이 들어가면 더 많은 이점을 얻을 수 있습니다. 예를 들어, 우리가 작성한 함수의 성능을 측정하고자 할 때도 테스트 코드를 활용할 수 있습니다.

또한 협업을 할 때도 테스트 코드는 매우 유용합니다. 다른 개발자들과 협업하여 큰 프로젝트를 진행할 때 우리가 작성한 코드를 검증하기 위해서는 테스트 코드가 필수적입니다. 테스트 코드가 작성되어 있다면 다른 개발자들은 우리가 작성한 코드의 정확성을 쉽게 확인할 수 있고 의사소통의 오류를 줄일 수 있습니다.
