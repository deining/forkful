# 왜?

보통 우리는 프로그래밍을 할 때 출력을 많이 사용합니다. 하지만 때로는 디버깅을 위해 출력을 표준 에러로 보내는 것이 더 좋을 때도 있습니다. 이를 통해 더 나은 오류 추적 및 디버깅이 가능합니다.

# 방법

자 아두이노에서 표준 에러로 출력하는 방법을 알아봅시다. 아래 코드 블록은 간단한 예제입니다. 주석을 통해 설명을 추가하였습니다.

```
Arduino void setup () {
  Serial.begin (9600); // 시리얼 통신을 시작합니다.
  Serial.println ("시리얼 출력 예제"); // 시리얼에서 출력합니다.
  Serial.printError (F ("표준 에러 출력 예제")); // 표준 에러로 출력합니다.
}
```

위 코드를 실행하면 시리얼 모니터에서는 "시리얼 출력 예제"가 출력되지만 시리얼 통신이 아닌 다른 디바이스에서는 "표준 에러 출력 예제"가 출력됩니다.

# 심도있는 분석

표준 에러로 출력을 보낸다는 것은 컴퓨터 프로그램에서 오류 추적 및 디버깅을 위해 더 많은 정보를 제공하는 것을 의미합니다. 기본적으로 코드를 실행하는 동안 시스템은 표준 에러 출력 버퍼로 메시지를 보냅니다. 이 메시지는 실행 중인 프로세스의 표준 에러 스트림에 저장되어 디버깅에 사용될 수 있습니다. 따라서 이를 통해 더 빠르고 쉽게 오류를 식별하고 수정할 수 있습니다.
