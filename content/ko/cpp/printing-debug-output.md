## 왜

프로그래밍을 하다보면 디버그 출력을 할 필요가 생기는 경우가 있습니다. 디버그 출력은 코드에서 문제를 찾고 수정하는 데 도움이 되기 때문에 중요합니다.

## 어떻게

디버그 출력을 하는 데는 여러 가지 방법이 있지만, 가장 일반적인 방법은 ```cout```을 사용하는 것입니다. 예를 들어, 다음과 같은 코드를 작성해보세요.

```C++
#include <iostream>

int main() {
  int a = 5;
  std::cout << "변수 a의 값: " << a << std::endl;
  return 0;
}
```

위 코드를 실행하면 아래와 같은 출력이 나옵니다.

```
변수 a의 값: 5
```

또 다른 방법으로는 ```cerr```을 사용하는 것입니다. ```cerr```은 ```cout```과 비슷하지만, 에러 메시지를 출력할 때 사용됩니다. 예를 들어, 다음과 같은 코드를 작성해보세요.

```C++
#include <iostream>

int main() {
  int x, y;
  std::cerr << "두 수를 입력하세요: ";
  std::cin >> x >> y;
  if (y == 0) {
    std::cerr << "0으로 나눌 수 없습니다." << std::endl;
    return 1;
  } else {
    std::cout << "나눈 결과: " << x / y << std::endl;
    return 0;
  }
}
```

위 코드를 실행하면, 두번째 수를 0으로 입력하면 아래와 같은 에러 메시지가 출력됩니다.

```
두 수를 입력하세요: 10 0
0으로 나눌 수 없습니다.
```

## 딥 다이브

위에 나열한 방법 외에도 디버그 출력을 하는 데에는 다양한 방법이 있습니다. 예를 들어, 사전에 정의해둔 매크로(Macro)를 사용하거나 디버깅 툴을 사용하는 등의 방법이 있습니다. 하지만 디버그 출력에 대해 깊이 알아보기 위해서는 프로그래밍 언어의 특성과 어떤 상황에서 어떤 방법을 사용하는 것이 좋은지 등을 공부하는 것이 필요합니다.
