## למה

בשנים האחרונות, כתיבת בדיקות התפתחה לתחום משמעותי בתכנות בפייתון. בשלבים מוקדמים של פיתוח קוד, היא נראית כמעט כמו עבודה מיותרת ומאמץ נוסף על המתכנת. אך אף על פי כן, מכיוון שכתיבת בדיקות חיונית לשמירת איכות הקוד, היא הפכה לדרישה חיונית לכל מתכנת פייתון. כתיבת בדיקות מאפשרת שיפור בתהליך הפיתוח וגם מאפשרת לנו לזהות בעיות בקוד לפני שהן משפיעות על המשתמשים.

## איך לכתוב בדיקות

כתיבת בדיקות בפייתון נעשית בעזרת הספרייה של pytest. נוכל לכתוב בדיקות לפונקציות, מחלקות וכל חלק אחר של הקוד שנרצה לבדוק. ניתן להפעיל את הבדיקות על ידי הרצת הפקודה "pytest" מתוך התיקייה שבו ממוקם קוד התגידות. לדוגמה:

```Python
# קבלת הערך המינימלי מבין תווים
def min_char(str):
    min = str[0]
    for c in str:
        if c < min:
            min = c
    return min

# בדיקת תגובה תקינה של הפונקציה
def test_min_char():
    assert min_char('apple') == 'a'
```

פרטי הבדיקה יוצאים כתוצאה מפעילת הפקודה "pytest" כמו כן:

```
$ pytest
============================= test session starts ==============================
platform darwin -- Python 3.6.6, pytest-3.9.3, py-1.7.0, pluggy-0.8.0
rootdir: /Users/user/code/test, inifile:
collected 1 item

test_file.py .                                                 [100%]

=========================== 1 passed in 0.01 seconds ===========================
```

אנחנו רואים שהבדיקה עברה בהצלחה ושהפונקציה מחזירה את הערך הראוי.
