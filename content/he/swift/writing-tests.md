# למה:

כתיבת בדיקות היא חלק חשוב מתהליך הפיתוח ב-Swift שכדאי לכל מפתח ללמוד ולהשתמש בה. בעזרת הבדיקות, ניתן לוודא שקוד המפתח שלנו עובד כצפוי ולזהות בעיות כבר בשלב הפיתוח ולא בגלל המשתמשים.

# איך לעשות זאת:

כדי לכתוב בדיקות יעילות ב-Swift, ישנם כמה דברים שצריך לקחת בחשבון. נבצע דוגמאות של קוד כדי להדגים את התהליך.

```Swift
// ייבוא המודולים הנדרשים
import XCTest
@testable import MyApp

// הגדרת פונקציה לבדיקת התוצאה
func testAddNumbers() {
    // הגדרת משתנים לבדיקה
    let num1 = 5
    let num2 = 7
    
    // הגדרת התוצאה הצפויה
    let expected = 12
    
    // בדיקה שהתוצאה של הפונקציה מתאימה לתוצאה הצפויה
    XCTAssertEqual(addNumbers(num1, num2), expected)
}
```

בכדי לבדוק את הקוד כולו, נוכל להשתמש בפונקציות `XCTAssert` שמאפשרות לנו לבדוק טיעונים מסוימים. בנוסף, כדי לבדוק פונקציות שמחזירות ערך `Void` כמו פונקציות הדפסה, יש להשתמש בפונקציות `XCTExpectation` ו-`waitForExpectations` כדי לוודא שהקוד נכנס לתוך הפונקציה המתאימה.
