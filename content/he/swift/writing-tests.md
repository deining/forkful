## למה

כשמתחילים לתכנת בשפת Swift, ייתכן שלא נדמה שכתיבת בדיקות (tests) הוא חלק חשוב מהתהליך. אבל בפועל, כתיבת בדיקות יכולה להיות כלי עז מאוד בפיתוח תוכניות. במאמר זה נלמד למה כדאי לכתוב בדיקות וכיצד לכתוב אותן בשפת Swift.

## כיצד לכתוב בדיקות

כתיבת בדיקות בשפת Swift היא פשוטה ונוחה. לפניכם כמה דוגמאות של כתיבת בדיקות עם הסברים קצרים:

```Swift
// דוגמא בדיקה של פונקציה
func add(_ x: Int, _ y: Int) -> Int {
    return x + y
}

// בדיקה אם מצביע ריק מתאים לאופציונל של מחרוזת
var name: String? = nil
XCTAssertNil(name)

// בדיקה שמפעילה השוואה בין שני מספרים
let height = 183
let expectedHeight = 180
XCTAssertEqual(height, expectedHeight)
```

## Deep Dive

בנוסף לבדיקת הקוד שכתבנו, ישנם עוד כמה נקודות שכדאי לקחת בחשבון בכתיבת בדיקות:

- הרצת בדיקות באופן אוטומטי תוך הרצת התוכנית, על מנת למנוע כשלים עתידיים ולבטח מוצר איכותי.
- כתיבת בדיקות ניתן להפעיל כאשר מבצעים תיקון שגיאות או שינויים בקוד, כדי לוודא שהשינויים לא גורמים לכשלים בפונקציונליות קיימת.
- השתמשו בדיקות כדי לבדוק פעולות מיוחדות או תנאים מיוחדים של הקוד, על מנת להבין טוב יותר את פעולתו.

כתיבת בדיקות היא כלי חשוב וקריטי בתהליך התכנות ויכולה לחסוך המון זמן וכאבי ראש בעתיד. בואו ננסה ונתמקד בהוספת כתיבת בדיקות לתהליך התכנות שלנו.
