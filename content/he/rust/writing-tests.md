# מה ולמה?
כתיבת בדיקות היא תלת מחקר בו מתבצעת בדיקת קוד כדי לוודא שהוא עובד כצפוי. תהליך זה משמש כדי לוודא את תקינות ויציבות התוכנה ולמנוע באגים ותקלות אפשריות בהמשך. כמו כן, כתיבת בדיקות מסייעת למתכנתים להבין את הקוד ולהתמודד עם שגיאות ובעיות שעלולות לצוץ בעתיד.

# איך לעשות זאת:
לבדיקת קוד באמצעות השפת Rust ישנם כמה דרכים. הדרך הפשוטה ביותר לעשות זאת היא להשתמש במסגרת הבדיקות המובנית של Rust שמציעה הרבה כלים ותכונות נוחות כדי לבצע בדיקות קוד בצורה ברורה ויעילה.

```Rust
#[cfg(test)]
mod tests {
    #[test]
    fn test_function() {
        // אנו יכולים לכתוב כאן בדיקות של הפונקציות שלנו
        assert!(1 + 1 == 2);
    }
}
```

בכדי להריץ את הבדיקות, ניתן להשתמש בפקודה `cargo test` בכותרת הפרוייקט שלנו. בכדי להכין תצורת משתנים שונה לכל מבחן, ניתן להשתמש במילת המפתח `#[test]`. למידע נוסף על ההגדרות והאפשרויות של הבדיקות בשפת Rust ניתן לקרוא בעמוד הרשמי של השפה.

# התעמקות נוספת:
בנוסף לבדיקות המובנות של Rust, ישנם גם מספר כלים סטנדרטיים נוספים וספריות קשורות המשמשות לבדיקות קוד והפקת תוצאות. לדוגמה, ספריות כמו `assert_eq!` ו- `expect` משמשות לבדיקת התאמה והערכת פעולות. כמו כן, קיימות גם כלים חיצוניים וספריות נוספות שניתן להשתמש בהן לבצע בדיקות מתקדמות יותר ולנהל חלק מהסיפריות החיצוניות של הפרוייקט.
