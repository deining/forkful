## למה

כתיבת טסטים היא חלק חשוב בכתיבת קוד בשפת Rust. הטסטים מאפשרים לנו לבדוק בצורה אוטומטית את הפונקציונליות של הקוד ולוודא שהוא עובד כפי שצריך. הם מקלים על תהליך הפיתוח ומנגנים בכמה שלבים בלתי ניתנים לטעינה. אז למה לא ללמוד כיצד לכתוב טסטים ב-Rust ולקחת את הקוד שלנו לרמה הבאה?

## איך לעשות זאת

כתיבת טסטים ב-Rust היא פשוטה ונגישה. אנחנו נדגים את זה באמצעות דוגמאות בקוד הבא:

```Rust
fn add(x: i32, y: i32) -> i32 {
    x + y
}

#[test]
fn test_add() {
    assert_eq!(add(2, 3), 5);
}
```

הכינו את הפונקציה add שמקבלת שני מספרים שלמים ומחזירה את הסכום שלהם. לאחר מכן, המילה השמורה #[test] מציינת כי זאת פונקציה בעלת תכלול טסטים. בקוד הטסט, אנחנו משתמשים בפונקציה assert_eq כדי לוודא שהערך שהפונקציה מחזירה הוא באמת תואם לתוצאה שאנחנו מצפים. אם הערך אינו תואם, הטסט יכשל ויתן לנו התראה.

בכדי להריץ טסטים, נכתוב בטרמינל ```cargo test```. מהעזרה שלנו בכתיבת קוד, נקבל את התוצאה הבאה:

```
running 1 test
test test_add ... ok

test result: ok. 1 passed; 0 failed; 0 ignored; 0 measured; 0 filtered out
```

אנחנו יכולים לראות שהטסט שלנו עבר בהצלחה ולא היו שגיאות. הכנו את הקוד שלנו כדי לקבל תוצאה של 5 כאשר אנו מחברים את המספרים 2 ו-3, כך שהטסט שלנו התחשב בזה.
