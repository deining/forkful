## מדוע

משהו שכולנו מאוד, אבל מאוד מאוד מאוד לא אוהבים לעשות הוא לכתוב בדיקות לקוד שלנו. אבל יש לנו כאן חדשנות, הצטרפו ל־Rust! הִשְׁתַּמְשׁו בשפת תכנות הזאת במיוחד כדי לכתוב בדיקות איכותיות ויעילות לקוד שלכם. היתרון הגדול של לכתוב בדיקות הוא כי הן עוזרות לנו להיות בטוחים שהקוד שלנו עובד כמצופה ושהדברים לא נשברים כל פעם שאנו משנים את הקוד שלנו.

## איך לעשות

כדי לכתוב בדיקות ב־Rust, צרו זוג פונקציות עבור כל פונקציה שאתם רוצים לבדוק. יש מספר שיטות שונות לבדיקות, ואתם יכולים לבחור את השיטה שמתאימה הכי לקוד שלכם. בדרך כלל, כדאי להשתמש בתוכנת הבדיקה המובנית והעוצמתית של Rust הידועה בשם "assert!".

לדוגמה, אם יש לכם פונקציה שמחשבת את חברות של מספרים, ניתן לבדוק בעזרת הבדיקה הבאה:

```Rust
fn sum(x: i32, y: i32) -> i32 {
    x + y
}

#[test]
fn test_sum() {
    assert_eq!(sum(2, 2), 4);
}
```

הוסיפו את הפונקציה "sum" ואת הבדיקה שלה לקובץ הטסטים של Rust והריצו אותו באמצעות "cargo test". אם הפונקציה היא תקינה, תקבלו את התוצאה "test test_sum ... ok". תוכלו כמובן לכתוב עוד ועוד בדיקות באופן דומה.
