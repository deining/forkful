## מדוע

קריאת ארגומנטי שורת הפקודה היא כלי שימושי בפיתוח תוכניות ואפליקציות בשפת Rust. היא מאפשרת למשתמש להעביר מידע מהסביבה החיצונית ישירות לתוך התוכנית באמצעות פרמטרים שהוא מזין לשורת הפקודה. כך, ניתן לשנות לפי צורך מאפיינים של התוכנית בזמן ריצה.

## איך לעשות זאת

כדי לקרוא ארגומנטי שורת הפקודה בשפת Rust, נשתמש במודול "std::env" (סטנדרטית). נתחיל על ידי הגדרת משתנה שיכיל את הארגומנטים שהזנו על ידי המשתמש, ונאתחל אותו בפיתור הפונקציה "args()" של המודול. לאחר מכן, נוכל לגשת לכל ארגומנט ולהשתמש בו באופן נוח על ידי מתודות כמו "next()" ו-"unwrap()". נציג דוגמא למקרה שבו המשתמש מגדיר את הפרמטר "name" ואת הערך "John" כארגומנטים של התוכנית:

```Rust
use std::env;

fn main() {
    let args: Vec<String> = env::args().collect();
    let name = args[1].clone();
    println!("Hello, {}", name);
}
```

פלט:

```bash
$ ./program name John
Hello, John
```

## צלילת למעמקים

כמו שראינו בדוגמא, פתירת הפונקציה "args()" מציעה לנו גישה קלה ונוחה לארגומנטי השורת הפקודה. עם זאת, כדאי להיזהר מספר שיקולים: ראשית, אם המשתמש לא יזין את הארגומנט שעבורו אנו מחזירים ערך (כמו במקרה הדוגמא שלנו), התוכנית תתרוסק בשגיאה. לכן, כדאי להשתמש במתודה "expect()" במקום "unwrap()", כדי לטפל במצב זה בצורה הולמת יותר. ראשית, כדאי להתבונן במספר האגרגט
