## למה

מדוע מישהו ירצה לכתוב לפלט שגיאות סטנדרטי ברסט? נכון לעכשיו, ברסט נחשבת ללשונית גיימר בעולם התכנות ואינה מיועדת לפיתוח כללי. אך לפני כמה שנים, יצאה תוכנית לשם משטרה כדי להשתמש ברסט לכתיבת קוד ננטטיבי צנוע ובטוח. כתיבת שגיאות לפלט סטנדרטי היא דרך נוספת לפיתוח ברסט כדי לשפר את תקינות הקוד ולזהות ולתקן שגיאות בקוד בזמן הריצה.

## איך לעשות זאת

לכתוב לפלט שגיאות סטנדרטי ברסט פשוט, רק ניתן להשתמש בפונקצית "eprintln!" כך:

```Rust 
fn main() {
    eprintln!("זוהי שגיאה סטנדרטית.");
}
```

כאשר התכנית מורצת, הפלט יהיה:

```
זוהי שגיאה סטנדרטית.
```

עם קוד זה, לא יהיו ייבואים נוספים או השמות הפונקציונליים העליונים שגיאת פלט סטנדרטי כתיבות שגיאות לפלט באופן חוזר הליך באמצעות השימוש ב-exclam אלא יריצו יד השיטות כי אינן פותחות בצורה פרושטית.

## צלילה עמוקה

כאשר נכתב פלט סטנדרטי, הפלט הוא מיועד לצורך שימוש בינארי. פלט סטנדרטי הינו מסורתי, פלט אולי לא יבין שיגרת התנועה שנמצאת או שניתן להתאים את משתנים העלאיום לתוכן שהודפס על ידה כדי לשמור את הפלמט המתאים. לדוגמה, ניתן להדפיס קוד מועצה או משתני תא ישיר כדי לנתק את הפלמט המתאים.
