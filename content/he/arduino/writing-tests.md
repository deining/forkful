# למה:

כאשר מתכנתים עובדים עם Arduino, הם נתקלים בכמה סיבות לכתוב בדיקות. ראשית, כתיבת בדיקות עוזרת לנו לוודא שהקוד עובד כפי שצפינו ושהמודולים שלנו תואמים את הציפיות שלנו. נוסף לכך, כתיבת בדיקות עוזרת לנו למצוא ולתקן בעיות בקוד בצורה מהירה ויעילה. בקיצור, כתיבת בדיקות היא חלק חשוב מתהליך הפיתוח על מנת להבטיח קוד בריא ויעיל.

# איך לכתוב בדיקות בארדואינו:

```arduino
#include <ArduinoUnit.h>

test(functionName) {
    assertEqual(1, 1); // בודקים את התוצאה הצפויה מול התוצאה המתקבלת
}

test(anotherFunction) {
    assertTrue(true); // בודקים שהתנאי הוא נכון
}

test(failingFunction) {
    assertEqual(1, 2); // הבדיקה תכשל וזה יציג הודעת שגיאה
}

// יציג את התוצאות של כל הבדיקות
arduinoUnit.runAllTests();
```

בדוגמה זו, אנחנו משתמשים בספריית ArduinoUnit כדי לכתוב בדיקות. הייתכנן את הפונקציות שלנו ואז משתמשים בפקודות הassert בכדי לוודא שהתוצאות שלנו הן מה שציפינו לקבל.

# העומק של בדיקות:

כתיבת בדיקות אינה רק על להריץ קוד ולוודא שהוא עובד, אלא כן על לבצע בדיקה מקיפה של מערכת הקוד שלנו. ישנם מספר מתכונות של בדיקות שיכולים לעזור לנו לכתוב ולשמור על בדיקות בריאות:

- כתיבת בדיקות מקיפות: נסביר בצורה יעילה את התהליך לכתוב בדיקות המקיפות את כמעט כל התחומים של הקוד שלנו.
- גבולות: מביאים לידי דמיין את התנאים האפשריים בקוד שלנו
