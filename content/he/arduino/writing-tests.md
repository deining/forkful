## מדוע
נכון לתחילת הדרך, זה יכול להיות מפתיע כמה משתמשים חדשים ב-Arduino לא מתבקשים לכתוב בדיקות לקוד שלהם. אם זה המקרה עבורך, מוטב לך ללמוד את הדרך הנכונה לכתוב בדיקות עכשיו כדי לחסוך זמן וכאב ראש בעתיד.

## איך לעשות זאת
כדי לבדוק את הקוד שלך במהירות, אנחנו ממליצים להשתמש ב-FastLED כספרייה לבצע בדיקות יחידה. הנה קטע קוד פשוט על מנת להראות לך איך ליצור פונקציה נפרדת לבדיקות.

```Arduino
#include <FastLED.h> // הוספת ספרייה לקוד

#define NUM_LEDS 10 // מספר ה-LEDים ברשת שלך

CRGB leds[NUM_LEDS]; // יצירת מערך הכולל את כל ה-LEDים

void showPattern() { // פונקציה להצגת פטרן על ה-LEDים
  // כאן תכתוב את הקוד שלך להצגת הפטרן
}

void testPattern() { // פונקציה לבדיקת הפטרן
  // כאן תכתוב את הקוד שלך לבדיקת הפטרן
  for (int i = 0; i < NUM_LEDS; i++) {
    leds[i] = CRGB::White;
  }

  FastLED.show(); // הצגת הפטרן על ה-LEDים
}

void setup() {
  FastLED.addLeds<WS2812, DATA_PIN, GRB>(leds, NUM_LEDS); // רשת של כוללי LEDים
}
```

וכעת, בשביל לבדוק האם הקוד שלך עובד, תחבר את ה-LEDים ואת הקוד ללוח ה- Arduino וכתוב את הפקודה הבאה:

```Arduino
testPattern();
```

אם הכל עבד כראוי, אתה תראה את ה-LEDים מציגים צבע לבן. כעת, כשהפטרן תקין, המשך לכתוב את הקוד של הפטרן עצמו.
