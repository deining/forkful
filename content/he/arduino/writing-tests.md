## למה

ניתן להגיד שכתיבת בדיקות היא חשובה להצלחת כל פרויקט תכנות. בואו נכיר כמה הסיבות העיקריות לכתיבת בדיקות בארדוינו.

## איך לעשות זאת

כאשר מתחילים לכתוב קוד בארדוינו, יש להציג לעצמם את השאלה הבאה: אם לאחרונה אין לי כמות גדולה של זמן, למה כדאי לבצע את פענוח לא מבוטל במקרה זה? האם יש אולי דרך אחרת ליצור בסיס טוב לפרויקט שלי?

כתיבת בדיקות מוגדרת כהייה אחת מהדרכים הטובות ביותר לבנות טוב את המתחילה של הפרויקט שלכם. הקוד הבא מדגים כיצד ליצור בדיקת יחידות פשוטה בארדוינו:

```Arduino
int x = 10;
int y = 5;
int result = x + y;
if (result == 15) {
  Serial.println("הבדיקה עברה בהצלחה!");
} else {
  Serial.println("הבדיקה נכשלה");
}
```

כמו שאתם רואים, הקוד מגדיר שתי ערכים ומבצע בדיקה פשוטה על ההם. במידה זו, הוא צריך להדפיס את ההודעה "הבדיקה עברה בהצלחה!" אם הבדיקה צריכה להחיל נכשלה. בדוגמה זו, הוא אמור להדפיס "הבדיקה עברה בהצלחה!" לכן, הבדיקה זהה ל- Pass.

## עומק הבדיקה

כעות כתיבת מעורבות ב- Arduino, יש לכתוב חבילות בדיקה סטנדרטיות ובמקרה זה היא מכשירה את הפעולות הבאות:

- בסיס בדיקות: כשאנחנו רוצים לבצע תביעה פשוטה כדי לבדוק אם תכנית ה-Arduino שלנו ניצלה כראוי

- שאילתות: אתה יכול להריץ בדיקות שאילתות מסוימות כדי לברר האם שאלה שלך מצליחה כנכונה
