## Why

אז למה בכלל לקרוא על פרמטרי שורת הפקודה? אנחנו כבר יש לנו ממשק גרפי נוח עם כמעט כל יישום אפשרי, אז למה להשתמש בשורת הפקודה ולא אפשר בפשטות עם לחצן כמה פעמים?

טוב, הנה כמה סיבות: ראשית, רוב הטכנולוגיות והתוכניות ישמרו עדיין על קיומן עם הוצאת פרמטרי שורת פקודה. אם אתה עובד עם מערכות לינוקס או מבוססות יוניקס, הגדרות והתצורה של יישומים עשויים להיות בתוך פרמטרי הפקודה.

בנוסף, ייתכן ואתה רוצה להריץ תוכניות עם הגדרות שונות בכל פעם שאתה מפעיל אותן, ופרמטרי שורת הפקודה יכולים לעזור לך לעשות זאת בצורה נוחה ומהירה.

## How To

כתבתי עבורכם כמה דוגמאות קצרות כדי להמחיש איך לקרוא פרמטרים מצד שורת הפקודה בשפת Bash.

```Bash
#!/bin/bash

# פונקציה המדפיסה את ארגומנט השורה הראשונה
function print_first_argument {
    # להדפיס את הארגומנט הראשון
    echo "הארגומנט הראשון הוא: $1"
}

# מפעיל את הפונקציה עם ארגומנטים שונים
print_first_argument "שלום"
print_first_argument "אני"
print_first_argument "Bash"
```

פלט:

```
הארגומנט הראשון הוא: שלום
הארגומנט הראשון הוא: אני
הארגומנט הראשון הוא: Bash
```

כמו שאתה יכול לראות, הארגומנט הראשון הוא מיוחד מכיוון שהוא מכיל את הטקסט שאתה מעביר לפונקציה.
