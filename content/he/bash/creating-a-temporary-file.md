## למה 

טוב, בדיוק מדוע בתוך התכנית שלכם כדאי ליצור קובץ זמני? יתרה מזה שזה ניקוי קוד ואורך מספר שורות- ישנן מספר דרכים נוספות לנקות את הסביבה שלכם עם `bash`.

## איך לעשות

למטרת ניקוי קוד ובדיקת בדיקה כמותית את הבסיס של פקודות נתונים תמימות, צרו קובץ זמני באמצעות הפקודה `mktemp`:

```Bash
temp_file=$(mktemp)
```

הוסיפו את התוכן שלכם לקובץ הזמני על ידי כתיבה לשימוש בפקודה `echo` או `printf`:

```Bash
echo "Hello World!" > $temp_file
```

אתה יכול להשתמש בקובץ הזמני כחלק מהתוכנה שלך כפי שבא לך. כאשר אתה מסיים להשתמש בקובץ, כדאי למחוק אותו עדיין לסיום:

```Bash
# לפני פקודות אחרות
rm -f $temp_file
```

אתה יכול גם ליצור קובץ זמני עם שם ייחודי על ידי הוספת פרמטר לפקודה `mktemp`:

```Bash
temp_file2=$(mktemp example.XXXXXX)
```

שם הקובץ כולל את הנתונים הכלליים של שם המחשב, כך שאתה יכול לשנות אותו ולהשתמש בו עבור תוכניות שונות באותו זמן.

## נחקור עמוק

קובץ הזמני שנועד להוצאת פרטים זמנים של סדר בתוך דירקטוריה אנשי קשר עבור יכולת להשתמש הוא `--tmpdir`, למעשה עבור כל מאפיין לשם כך, אתם משתמשים `--mktemp`.

דרך מדויקת בכדי למקסם את גודל מאפיין זה זולת הלא מבוטלים שלכם בגירסה הזו הוא כאשר אינו שיש "מקור" חופשי בקוד שלכם משאב, .שם הפקודה שתואר כאשר
