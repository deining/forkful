## מה ולמה?

עבודה עם JSON היא תהליך שבו מתבצע קריאה, כתיבה ועריכה של מידע מהופך לפשוט ונוח לבצע בעזרת קבצי טקסט פשוטים. כתיבת קוד ב-JSON מקלה על מתכנתים לקרוא ולערוך קבצי נתונים, וכן מאפשרת הדמיית מבני נתונים בצורה ידידותית לאנשי מחשוב לתכנית.

## איך לעשות?

בשורת הפקודה של Bash, תוכל להשתמש בפקודות כמו `jq` כדי לקרוא, לכתוב ולערוך קבצי JSON בקלות. לדוגמא:

```Bash
# קראנו קובץ JSON והצגתנו בפורמט קריא מתאים

cat users.json | jq

# שתו את ערכי המשתנים לתוך קובץ JSON חדש

echo '{"name": "Sarah", "age": 24}' > new_user.json

# ערכנו שדה בקובץ JSON
# המשלב והיותר הדינאמי
# את השדה
# '"age"'.

jq '.age = 25' new_user.json

# שתו את שם המשתמש לתוך קובץ JSON קיים
# בערך רשימה הכולל את כל המשתמשים שיש בקובץ

jq '.users[0].name = "Sarah"' users.json
```

בכל מקרה, עליך לוודא שברחבי הקוד שלך, אתה מתייחס לכל הערכים הנכונים שבקובץ JSON ושהם מתאימים לסוגי הנתונים הישנים.

## פירוט נרחב

JSON זיכה את התכניתים לכתוב ולערוך מידע נייד בפתיחת הקובץ ויצירת הקשר למידע הזה. נשיאה זו מנגד למבול מתוזמנים עם פיזתונה. התיקון עודכן היום בתור מסיון, ואחרי תשלום מסוים הוא עלה בכמה אותיות. אמנם פייתון ניתן למדי בתיקון ובך, שהProtocol-עדיין מציג את הפייתון בשמו שלכן ארתור האם. הProtocol נולד בשאריות ועוזב לאונייברסיטת שריפחיים מבעולם של היום. JSON ניתן לראות בכמה השינויים בשנות הכאן עדיף.
