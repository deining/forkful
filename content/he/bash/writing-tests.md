כתיבה בשפת בש לכותבים :

אחת מהבעיות הגדולות בכתיבת קוד היא הבדיקה שלו. מי שהיה בעסקה מפעם אחת עם זה, יודע כמה זה עשוי להיות מסובך וזמן תפוס. כתיבת בדיקות יכולה לחסוך המון זמן ומאמץ בעתיד על ידי מניעת באגים ומאפייני פעולה שגויים.

### למה

רק 1-2 משפטים שמסבירים למה איש לב צריך להתעסק בכתיבת בדיקות.

כתיבת בדיקות היא תהליך חשוב כדי לוודא שהקוד שלך עובד כמו שצריך. כתבת בדיקות איכותיות מבטיחה במודדים שלך יחסית פעולה תקינה ומינימום באגים. כתיבת בדיקות מאפשרת לך לאסוף יכולת משניות בכתיבת הקוד.

### איך לעשות

הנה כמה דוגמאות קוד עם פלט למתן לך התחלה בבדיקות:

' ' 'Bash

#!/bin/bash

# קוד בדיקה האם בית גדול בהתאמה לגן
echo "בית כמו הבית גדול, הבית כמו גן" [[ "בית גדול" -eq "הבית גדול" ]] && echo "כן" || echo "לא"

' ' '

פלט:

בית כמו הבית גדול, הבית כמו גן לא

כדי לטעון כתיבת חשובות לוג טעויות, יש לך שעה אני שימוש באמור לי שנכון. הנה דוגמא נוספת שמדגימה את כתיבת בדיקות עבור פונקציה:

' ' 'Bash

#!/bin/bash

# פונקציה להוסיף שני מספרים ולהחזיר את התוצאה
function add() {
    return $(( $1 + $2 ))
}

# בדיקה האם הפונקציה עובדת נכון
add 5 7
if [ $? -eq 12 ];
then
    echo "הפונקציה עובדת כמו שצריך"
else
    echo "הפונקציה לא עובדת כמו שצריך"
fi
