## למה
ניתן לכתוב בש (גרסת היום) כדי לבדוק את הקוד שלך ולוודא שהוא עובד כמצופה. בכתיבת בדיקות ניתן להפחית מספר השגיאות ולשפר את יעילות התוכנית שלך, כך שתוכל לחסוך זמן ומאמץ בעתיד.

## כיצד לבצע
אם ברצונך לכתוב בדיקות בש, הנה כמה דוגמאות שבאמצעותן תוכל לממש את זה:

```Bash 
#!/bin/bash

# בדיקת האם מספר מסוים הוא מספר זוגי או אי-זוגי
num=2

if [ $((num%2)) -eq 0 ]
then
  echo "זהו מספר זוגי"
else
  echo "זהו מספר אי-זוגי"
fi

# צריך להדפיס "זהו מספר זוגי"
```

כאן אנו בודקים אם מספר שתוכנן מראש גם מספר זוגי או אי-זוגי, ובהתאם לתוצאה מדפיסים הודעה מתאימה.

```Bash
#!/bin/bash

# בדיקת כפולת ערך בתוך מערך
numbers=(1 2 3 4 5)
dup_num=2

if [[ "${numbers[@]}" =~ "${dup_num}" ]]
then
  echo "המספר $dup_num נמצא במערך"
else
  echo "המספר $dup_num אינו נמצא במערך"
fi

# צריך להדפיס "המספר 2 נמצא במערך"
```

כאן אנו בודקים האם מספר מסוים נמצא בתוך מערך כפול ערך, ומדפיסים הודעה מתאימה.

## צעדים נוספים
כדי לכתוב בדיקות בש, ישנם כמה נושאים שיכולים לעניינך:

- הצגת דוגמאות מלאות של בדיקות בש עם פלט מצופה ואיך הן יכולות לסייע לך בתהליך הפיתוח.
- יצירת קבצי בדיקה נפרדים כדי להפחית את כתיבת הקוד בקובץ הראשי.
- השתמש בפקודת "assert" כדי להוסיף בדיקות מדו
