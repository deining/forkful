# למה

בעולם התכנות, ל礁שום קובץ חדש לא תמיד הינה דבר פשוט. ייתכן שהקובץ כבר קיים או שהמסלול שנבחר תקין. בהכללה של דרכי הכתיבה של בם, לעיתים נדרשת לבדוק אם תיקייה קיימת לפני שממשיכים במעטפת הקוד שלנו. לכן, לדעת איך לבדוק אם תיקייה קיימת הינה כישור נחוץ לכל מתכנת.

## כיצד לבדוק אם תיקייה קיימת

לבדיקת קיום תיקייה בבם, ניתן להשתמש בתנאי if ובשליטה test. הנה דוגמה לבדיקת קיום של התיקייה "Documents" בתיקיית הבית שלך:

```Bash
if [ -d ~/Documents ]; then
    echo "התיקייה קיימת!"
else
    echo "התיקייה לא קיימת."
fi
```

אם התיקייה קיימת, הפלט של הקוד יהיה "התיקייה קיימת!" ואם התיקייה לא קיימת, הפלט יהיה "התיקייה לא קיימת." ניתן גם להשתמש בפקודת "ls" כדי לבדוק אם התיקייה קיימת ולהציג את תוכן התיקייה:

```Bash
if ls ~/Documents > /dev/null; then
    echo "התיקייה קיימת!"
else
    echo "התיקייה לא קיימת."
fi
```

המסלול "> /dev/null" מנתב את הפלט של פקודת "ls" אל התיקייה האיכותית מהכתיבה לדיסטריבוציות האחרות.
