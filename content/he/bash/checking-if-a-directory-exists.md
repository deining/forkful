## למה כדאי לבדוק אם תיקייה קיימת
אין ספק שבכל תכנות סקריפט בכשלון ריצה קיימת סיבה אחת או יותר לכך שקוראים יוכנסו לבדוק אם תיקייה קיימת. זה יכול להיות כדי לוודא שהמשתמש כתב את המקום הנכון לחיפוש קבצים, או כדי לוודא שהתוכנית תוכל לכתוב קבצים במקום תיקייה מסוימת.

## איך לבדוק אם תיקייה קיימת
ישנם מספר דרכים לבדוק אם תיקייה קיימת בבש, ונבחר להראות את שתי הפתרונות הכי נפוצים עבור כתיבת סקריפטים:

```Bash
# אם תיקייה קיימת, הודעה ריקה תוחזר
if [ -d "/path/to/directory" ]; then
  echo "" # אין שום פעולה ריקה מוצעת לביצוע
fi
```

```Bash
# אם תיקייה קיימת, התקן השער יחזיר 0
# התקן השער מייצג את הפעולה האחרונה שהייתה משולבת
# והמספר 0 מייצג פעולה הצלחת
[ -d "/path/to/directory" ] && echo "הודעת בדיקה"
```

## חקירה מעמיקה
כאשר שתי הדוגמאות שציינו עובדות, כדאי לשים לב לכמה מסקנות חשובות. כל דוגמה מכילה צהרת אם (if), אשר מוסיפה הגנה נוספת במקרה שתיקייה לא תהיה קיימת. בדוגמה הראשונה, תהגרה כמו ריקה תוכל לשמש כפעולה ריקה שתהיה למעשה כלום. בדוגמה השנייה, פקודת ההכנסה (&&) מיועדת להראות בצורה ישירה שהפעולה האחרונה שנתנו לתקן שער תבוצע רק אם הפקודה לפניה המדווחת תצליח גם היא. על ידי כתיבת && לפני הפע
