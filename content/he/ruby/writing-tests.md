## למה

כתיבת בדיקות היא חלק בלתי נפרד מתהליך הפיתוח של כל בונה תוכנה. כתיבת בדיקות מאפשרת לנו לוודא שקוד התוכנה שלנו עובד כפי שצריך ומונעת בעיות ותקלות בהמשך.

## איך לעשות זאת

למעלה אפשר לראות דוגמאות של קוד בשפת רובי עם היקפי קוד כמותות לשימוש ביצירת בדיקות. הנה דוגמא בסיסית:

```Ruby
# דוגמה של מחלקת בדיקה
class Calculator

  # דוגמה של פונקציית בדיקה
  def self.add(num1, num2)
    num1 + num2
  end

end

# בדיקת התוצאה באמצעות מספר אסרטיבי
puts Calculator.add(2, 5) == 7  # תוצאה צפויה: true
puts Calculator.add(-3, 10) == 7  # תוצאה צפויה: false
```

כחלק מהבדיקות, אנחנו משתמשים בפונקציות אסרטיביות שמשוות את התוצאה הצפויה לתוצאה האמתית. בדרך זו, אם יש תקלה בקוד התוכנה, נקבל התראה ונוכל לתקן את הבעיה.

## ירידה לעומק

היות וכתיבת בדיקות היא חלק חשוב בתהליך הפיתוח, כדאי ללמוד על הנושא עוד כדי להבין כיצד ניתן ליעץ ולכתוב בדיקות יעילות. כמה נקודות חשובות להקפיד עליהן:

- כתיבת בדיקות אנחנו משתמשים במספרים מיוחדים שקוראים "מספרי אסרטיביות" כדי לוודא שהתוצאה הצפויה היא מה שאמורה להיות.
- כתיבת בדיקות יעילות דורשת לבדוק מקצת קוד לפני כתיבת הבדיקות עצמן. אם השורות שלכם לא עובדות כמו שצריך, לא תהיו מסוגלים לדעת בדיוק מה לבדוק.
