## למה

כתיבת בדיקות בקוד היא חלק חשוב מתפקוד התיכנות. זה מאפשר לנו לוודא שהקוד עובד כפי שצופו ומונע תקלות פוטנציאליות בעתיד.

## כיצד

כדי לכתוב בדיקות בקוד Ruby, תצטרכו להשתמש בכמה מודולים שמאפשרים לנו לייצר בדיקות בקוד שלנו. הנה דוגמאות לבדיקות שאפשר לכתוב:

```ruby
# דוגמה לכתיבת מתודה של בדיקה
def test_addition
  assert_equal 4, 2 + 2
end
```

```ruby
# דוגמה לבדיקה של מחרוזת
def test_string
  assert_equal "hello", "hello"
end
```

```ruby
# דוגמה לבדיקה של מערך
def test_array
  array = [1, 2, 3]
  assert_equal 3, array.length
end
```

הנה פלט המתודה של בדיקות המחרוזת:

```ruby
2 tests, 2 assertions, 0 failures, 0 errors, 0 skips
```

## Deep Dive

כאשר אנו כותבים בדיקות, חשוב לזכור את כמה נקודות חשובות:

- תמיד ניתן להריץ בדיקות שהם פרטיות, כלומר רק בקובץ הקוד שלנו. זה מאפשר לנו לבדוק את הקוד בכפתור אחד בלי להיות תלויים בקוד אחר.
- בדיקות מאפשרות לנו לאתר תקלות בקוד ולתקן אותם לפני שהם נהפכים לתקלות גדולות יותר.
- כאשר מדברים על בדיקות בתיכנות, כלומר הם לא מתארים את הקוד כולו. כלומר, אנחנו מדברים על הבדיקות של הקוד בעצמו, אבל זה אינו מתאר את האפליקציה המלאה כולה.
