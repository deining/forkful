## למה

לכתוב בדיקות בתכנות Ruby הופך את תהליך הפיתוח ליותר מוכנת ויותר קל. הבדיקות מאפשרות לך לבדוק את תכני הקוד שלך לפני שהוא נכנס לאווירה החיה, מבטיחות שהוא עובד כראוי ומפחיתות את הסיכון לטעויות עתידיות.

## איך לכתוב בדיקות ב Ruby

הריצו את הפקודה הבאה על כו-מחשב שרונכבוחתי רונכה: 

```Ruby
def add(num1, num2)
  result = num1 + num2
  return result
end 

puts add(2, 3)
```

שורת הפקודה 1 מגדירה את הפונקציה add ומקבלת שני פרמטרים כקלט. שורות 2 ו-3 מחשבות את תוצאת ההוספה של שני המספרים ומחזירות את התוצאה. ובשורה 5 אנחנו מדפיסים את התוצאה באמצעות הפקודה puts. כעת נרוץ את הפעולה ונקבל את הפלט הבא: 

```
5
```

כעת נכתוב פונקציה תומכת על מנת לבדוק את הפעולה add. לשם כך, נשתמש בספריית "test/unit":

```Ruby
require 'test/unit' 

def add(num1, num2)
  result = num1 + num2
  return result
end 

class AddTest < Test::Unit::TestCase 
  def test_add 
    assert_equal(5, add(2,3))
  end 
end
```

בשורה 1 אנחנו מתחילים עם ייבוא הספריה. בשורה 5 אנחנו מגדירים את הפונקציה test_add שמבדקת את הפעולה add ומתייחסת לאחותו של assert_equal (שהיא משווה בין שני ערכים ומוודאת שהם שווים). לבסוף, בשורה 9 אנחנו מגדירים את המחלקה AddTest המרצה את הפונקציה test_add. נריץ את הקוד ונקבל שורת הפלט הבאה: 

```
Run options [ --seed 6049 ]

1 runs, 1 assertions, 0 failures, 0 errors, 0 skips 
```
