כתיבת פוסט בלוג על תכנות בפורמט אלם לקוראי עברית.

## למה

בכתיבת קוד ייתכן שנתקלנו בצורך לבדוק האם קיים קובץ או תיקייה במערכת הקבצים שלנו. בכתיבת פוסט זה, אני אסביר כיצד לבדוק בפשטות אם קיימת תיקייה ולמה זה חשוב.

## איך לבדוק אם קיימת תיקייה באמצעות אלם

כדי לבדוק אם קיימת תיקייה באמצעות אלם, אנו נשתמש בפונקציה ייחודית שנקראת `doesPathExist` מתוך חבילת `elm/file`. נוכל להשתמש בפונקציה זו על מנת לבדוק האם תיקייה מסוימת קיימת במערכת הקבצים שלנו.

```elm
import File exposing (exists)
import File.Path exposing (path)

checkIfExists : String -> Cmd Msg
checkIfExists path =
    path path |> exists
```

במקרה זה, אנו מזינים את הנתיב של התיקייה שאנו רוצים לבדוק בפונקציה `exists` ואנחנו מקבלים כתוצאה בוליאנית (`True` או `False`) שמציינת האם התיקייה קיימת או לא. באמצעות פונקציה זו, נוכל לבצע פעולות שונות על תיקיות בצורה קלה ויעילה.

## נחוצים

כדי לוודא שאנו בודקים אם התיקייה קיימת במקום הנכון במערכת הקבצים שלנו, ישנם כמה נתיבים שיכולים לעזור לנו להבין את התיקייה שאנו מנסים לבדוק. נדבר קצת על כמה מהם:

- נתיב יחסי: נתיב זה הוא נתיב של התיקייה מתוך התיקייה הנוכחית שאנו נמצאים בה. לדוגמה, אם נמצאים בתיקיית `root`, ואנו רוצים לבדוק אם תיקייה בשם `images` קיימת בתיקיית `root`, הנתיב יחסי יהיה פשוט `images`.
