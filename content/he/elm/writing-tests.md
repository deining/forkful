## למה

כשמתחילים לכתוב קוד ב-Elm, כנראה שהמטרה העיקרית היא ליצור אפליקציות טובות ויעילות. אבל לצד כל הקוד הזה, חשוב לכתוב גם בדיקות כדי לוודא שקוד ה-Elm שלכם עובד נכון ולא יפול בזמן ביצוע. בנוסף, בדיקות מסייעות גם בחיזוק הבטחון שלכם בקוד ובהבטחה שהאפליקציה שלכם עובדת כפי שצריך.

## איך לעשות זאת

כתיבת בדיקות ב-Elm היא פשוטה יותר ממה שאתם חושבים. מרבה הכבוד לרוב שלכם, בדיקות נכתבות באותו הדרך כמו כתיבת קוד ב-Elm. אבל כדי שתהיה לכם הבנה טובה יותר של הנושא, הנה כמה דוגמאות שיכולות לעזור לכם.

```Elm
-- גדירת פונקציה שמחשבת את הסכום של שני מספרים
add : Int -> Int -> Int
add x y =
    x + y
-- כתיבת תרגיל בדיקה
addTest : Test
addTest =
    describe "add function" <|
        test "returns correct sum" <|
            \_ ->
                Expect.equal (add 5 10) 15
```

בדיקות נכונות תשתמש בפונקציות כגון `describe` ו-`test` כדי להגדיר את המבנה של הבדיקות שלכם, והשתמשו בפונקציות כמו `Expect.equal` כדי לבדוק שהתוצאה המצופה של הקוד מתאימה לתוצאה שאתם מקבלים. בתור המתכנתים שלכם, אתם רק צריכים לפתור התרגילים בפנים, לכתוב את הקוד ולהגדיר את התוצאה המצופה. את הכלה של המשפט, אז הבדיקות אמורות להיות ממש פשוט.
