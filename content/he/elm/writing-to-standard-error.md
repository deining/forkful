## Why

כתיבה לתקליטור הסטנדרטי (standard error) היא כלי חשוב לכל בונה תוכנה, מייצרת תצוגה נוספת של הפלט של התוכנית שלכם. כתיבה לתקליטור הסטנדרטי יכולה לעזור למצוא ולתקן באגים, לאפשט את האיתות כאשר משתמשים בדבוגר ולפתח את התוכנית בצורה יותר תקינה.

## How To

פתחו את הקובץ המייצג את התוכנית שלכם והוסיפו את השורה הבאה להתחלתו:

```Elm
import Debug exposing (crash)

```

כעת, כל פעם שאתם רוצים לכתוב לתקליטור הסטנדרטי, פשוט הוסיפו את השורה הבאה:

```Elm
crash "הודעת שגיאה"

```

והעתיקו את הודעת השגיאה הרצויה במקום "הודעת שגיאה". למשל, אם אתם רוצים להדפיס את הערך של משתנה "x", אתם יכולים לעשות זאת באמצעות הפקודה הבאה:

```Elm
crash ("x = " ++ toString x)

```

קריאה של התוכנית שלכם כעת יוביל לכתיבה של ההודעה לתקליטור הסטנדרטי והפלט של התוכנית יכלול את ההודעה שלכם. לדוגמה:

```
x = 5
```

## Deep Dive

כעת שלמדנו איך לכתוב לתקליטור הסטנדרטי, אבקש להציג כמה רעיונות נוספים:

- בדיקת תנאים: אנו יכולים להשתמש בכתיבה לתקליטור הסטנדרטי כדי לבדוק תנאים ולהדפיס הודעות שגיאה אם הם לא עומדים. למשל, אם נרצה להסתבך כאשר מספר בחזקת המחבר שלנו גדול מ-10, נוכל להשתמש בכול אחד מן הפקודות הבאות:

```Elm
if n > 10 then
    crash "מספר גדול מ-10"
else
    n ^ n
```
