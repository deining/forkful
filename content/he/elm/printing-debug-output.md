## למה

אני בטוח שכולם מכירים את התחושה הקשובה שמתקבלת כאשר אנחנו משתמשים בשפת תכנות ומנסים לאתר ולתקן את הבעיות הקיימות בקוד שלנו. אם רק היינו יכולים להכניס כמה שהוא הדפסות בקוד כדי לבדוק את הערכים שנמצאים באותם מקומות או לראות את התהליך של הקוד שלנו, זה היה יכול לפתור את רוב הבעיות שנתקלים בהן. למזלנו, ב-Elm יש לנו את כלי הדיבאגינג שיכול להוסיף הדפסות בקלות לקוד שלנו ולעזור לנו למצוא את הבעיות הקיימות.

## איך לעשות זאת

להדפיס פלט של הדיבאגינג ב-Elm פשוט מאוד. לפניכם דוגמא קצרה עם קטע קוד ופלט:

```Elm
import Debug

main =
    let
        x = 5
        y = [1, 2, 3]
        z = Debug.log "x and y" (x, y)
    in
    z

-- פלט: (x and y, (5, [1, 2, 3]))
```

כפי שאתם רואים, יש לנו פונקציה שנקראת "Debug.log" שמקבלת שני פרמטרים - הודעה וערכים. הוא מדפיס במקרה זה את ההודעה ואת הערכים שהוא מכיל.

עוד דרך להדפיס פלט באמצעות הדיבאגינג היא לשבץ את הערך בתוך המחרוזת של ההודעה. לדוגמא:

```Elm
Debug.log "The value of x is " ++ (toString x)
```

הקוד הזה ידפיס את המחרוזת "The value of x is 5", כאשר x הוא הערך שלנו.
