##מדוע

נמחקת האותיות התואמות לתבנית במתיחה ספציפית עם הגרסה הנוכחית של PHP? הסיבה העיקרית היא להפסיק בעיות באמצעות מילות המפתח או באישור נתונים מסוגל שאינם נמצאים בתבנית נכונה. למשל, אם יש לנו משתנה שמכיל מספר טלפון, אבל הוא מכיל גם תווים לא רצויים, נוכל להשתמש בתכונת preg_replace של PHP כדי להסיר את התווים הלא רצויים ולהשאיר רק את המספר טלפון.

##כיצד לעשות זאת

כל הכיתוב מתוך "```PHP ... ```" לקונסולת המנסה את פונקציית ה-matching של Regex בשלמות תחת:

```PHP
$str = "אני יודע גם אם תנאי רצוי";
$str = preg_replace("/[^א-ת ]/", "", $str);
echo $str; // "אני יודע גם אם תנאי רצוי"
```

בדוגמא הנתונה, נבדוק אם ניתן לעשות MATCH עם מספר טלפון כאשר "```(000) 000-0000```" הוא התבנית. ניצוב בעזרת הפונקציה של MATCH ונתנה את המספר המבוקש מדוגמא זו:

```PHP
$str = "זה מספר טלפון: (123) 456-7890";
preg_match("/[(][0-9]{3}[)] [0-9]{3}-[0-9]{4}/", $str, $matches);
echo $matches[0]; // "(123) 456-7890"
```

כמו במשל "```str_replace()```", נוכל להשתמש בתכונה "```preg_replace()```" גם לממתוג את התווים שאנחנו פשוט רוצים למחוק. עם סימן או מילר פשוט, נוכל להשתמש בפונקציה למחוק עיצוב חוץ:

```PHP
$str = "<p>This is a <span>paragraph</span></p>";
echo preg_replace("/<[a-z]+>/", "", $str); // "This is a paragraph"
```

בדוגמא הנתונה, נוכל להסיר להבן כל רווחים לחוץ המילת אינספון שתכיל את המשתנים רק עם התכונה "```*```".
