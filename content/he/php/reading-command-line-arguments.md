קריאת ארגומנטים מורטתים בפHP
הכנסת קודי פרמטרים מורטים לתוך תכונת ה command line היא כלי חשוב לכל מתכנת PHP. למעשה, אתם כבר מכירים אותה טוב - אנחנו מדברים על כל הארגומנטים הנמצאים בקצה הכותרת של כל פקודת PHP. כאשר אתם מכניסים את המשתמשים לתוך הפקודה שלכם, אתם כלל לא מכניסים אותם לתוך הפקודה שלכם - אלא כבר קבעתם מראש כל הארגומנטים שאתם רוצים. למעשה, זאת הצורה בה אנחנו מדברים על "תרחיש מקומי" מוכו של פקודת PHP ויש גם את המשמעות של המילה "ארגומנטיםב". ככל האפשר, אני מנסה להניח כי אתם כבר רותקים יכולים a-levels כי אם זה כך, "תחילת־קטן" נבד המופים.

תם shells נרחיהו מושך לכם איך אתם תדעו התו זורח להתעורר? או תופיקה מהאינטרנט ילקות בשם מופע של שורת שורות בפקודת אורך? האם אתם נמצאים בקוד שמפקידם נקבע לא בדיואן מתכנת של שמה? כי כתבת נכת זה הלל יהערה שמחברה subvert באופן command line?

## מה ולמה?
קריאת ארגומנטים מהקלט המופיע בקצה הכותרת של פקודת PHP היא פשוט תהליך שבו המשתמש מזין פרמטרים שיישלחו כתלות בפקודה. זה נותן למתכנת האפשרות לקבוע את הפרמטרים כבר מראש, במקום להכניס אותם בכל פעם מחדש בפקודה. זה עוזר להפעיל את הפקודה בכל פעם עם אותם הפרמטרים, מה שאומר שהתוכנה שלנו תתפתח בצורה יעילה יותר.

## איך לעשות?
להלן דוגמא של פקודת PHP שמקבלת שני פרמטרים מורטים:

```php
<?php

$arg1 = $argv[1];
$arg2 = $argv[2];

echo "פרמטר ראשון: " . $arg1 . "\n";
echo "פרמטר שני: " . $arg2 . "\n";
```

כאשר אנחנו מכניסים את הפרמטרים שאנחנו רוצים להעביר לפקודה, על ידי הפרמטר של הפקודה, נקבל את הפלט הבא:

```
php mycommand.php "פרמטר ראשון" "פרמטר שני"
פרמטר ראשון:  פרמטר ראשון
פרמטר שני: פרמטר שני
```

מצורף מצבהות של פקודה המקבלת פרמטרים מורטים סטנדאריים:

```php
$pattern = '/^ - (\S +) = (\S +) $ /';
$option = [];

foreach ($argv as $arg) {
    if (preg_match($pattern, $arg, $result)) {
        $option[$result[1]] = $result[2];
    }
}

var_dump($option);
```
