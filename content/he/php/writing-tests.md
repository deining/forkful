## למה

בעולם התכנות, ישנו מושג שנקרא "כתיבת בדיקות". מה זה בדיוק אומר? זהו פשוט התהליך של יצירת קוד המתבצע באופן אוטומטי כדי לוודא שהתוכנית שנכתבת עבורה תחלוף כמו שצפוי גם אם הוא תקין ולא יכול להאריך עוד ביינקו אדומה,. זה מועיל במיוחד כאשר יש לך קוד מול קוד או יש אנשים מרובים העוסקים בו כדי להבין כיצד הקוד עובד ומה יכול לרעום.

## כיצד לעשות זאת

כדי לכתוב טסטים ב-PHP, יש להשתמש בפונקציות מובנות כמו "assert" ו-"assertEquals" לבדיקת קוד פרטי. ניתן גם להשתמש בכלי חיצוני כמו PHP Unit או Behat לכתיבת טסטים. הנה דוגמה לכתיבת בדיקות בשיטה של TDD (Test-Driven Development):

```PHP

//זהו קוד הפונקציה שנכתב בסטייל TDD
function addNumbers($x, $y){
 return $x + $y;
}

//בדיקת התוצאה הצפויה של הפונקציה
assert(addNumbers(2,3) == 5);

```

בכך אנו בודקים את הפונקציה הנתונה לפני שמשתמשים בה בשלבים מתקדמים של פיתוח.

## חצייה עמוקה

אם ברצונכם ללמוד עוד על כתיבת טסטים ב-PHP יש כמה מקורות מומלצים לכם להתחיל:

- המדריך המקורי של PHP Unit יכול לספק לכם ידע מעשי על כיצד לכתוב טסטים בכללי.
- בלוג של PhpStorm מככב עם מאמר על כתיבת טסטים ברמת חציית עמוקה שיכול לעזור לכם להתחיל מההתחלה.
- באתר Code Academy יש שיעורים מפורטים על כתיבת טסטים ב-PHP.
