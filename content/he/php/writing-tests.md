## למה

כתיבת בדיקות היא חלק חשוב מתהליך הפיתוח בכל שפת תכנות. בעקבות הסוגיות והטעויות המצופות בעבודה עם קוד, מתבקשת כתיבת בדיקות כדי לוודא את תקינות הקוד ולמנוע תקלות נוספות בעתיד.

## איך לעשות זאת

על מנת לכתוב בדיקות בצורה יעילה ותקינה, ישנם כמה כללים ועקרונות לצער. נצפה בדוגמאות של קוד נכון ושגוי ונתבונן בפלטים השונים.

```PHP
// קוד נכון
function add($num1, $num2){
    return $num1 + $num2;
}
echo add(3, 5);
// פלט: 8
```

```PHP
// קוד שגוי
function subtract($num1, $num2){
    return $num1 + $num2;
}
echo subtract(10, 4);
// פלט: 14
```

כמו כן, נלמד כיצד להשתמש בכלים שונים כדי לראות כיצד הבדיקות עובדות ולזהות את השגיאות בקוד שלנו.

## ירידה עמוקה

כתיבת בדיקות היא חלק מאוד חשוב בפיתוח תוכניות ובכתיבת קוד איכותי. באמצעות העריכה והשפרת הבדיקות שלנו, אנחנו יכולים להבטיח כי הקוד שלנו תמיד יכול לעבוד בהוראה ואינו מייצר תקלות למשתמשים ולקודם קודם. בנוסף, כתיבת בדיקות מאפשרת לנו לבצע שינויים ושדרוגים בקוד שלנו בצורה מבוטלת יותר, מה שמקל על תהליך הפיתוח.
