## למה

כתיבה לפלט המתאר למקום שגוי נעשה על מנת לאתר ולתקן בעיות בקוד. כתיבה לפלט המתאר אירעות כמו שגיאות, נתענות ומצבים לא צפויים עלולה לעזור למתכנתים להבין מה החלקים של הקוד גורמים לבעיות, ולאתר את המקומות שיש צורך לתלות דיבוג.

## איך לעשות זאת

אם אתם מקורה בשפת C++, אתם כבר מכירים את התנאי האחרון "cerr" השולח תוכן לפלט המתאר במקום הטקסטואלי עם התכווננות. למקרים כאלה, אתם יכולים ליצור פלט מתאר עם תמיכת עיצוב לקריאה קלה מצד המתכנת. לדוגמא:

```C++
#include <iostream>
using namespace std;

int main() {
  int x = 50;
  if (x < 100) {
    cerr << "הקבצים פחות מ100." << endl;
  }
  return 0;
}
```

פלט:

```sh
הקבצים פחות מ100.
```

שימו לב שהנתון המתאר מופיע כפי שהוא בתוך ההודעה האחרונה.

## חקירה עמוקה

הכתיבה לפלט המתאר הינה כלי חזק שמאפשר למתכנתים למצוא בעיות בקוד שאחרת היה קשה לזהות. הנה כמה טיפים לניצול המלא של פונקציונליות זו:

- תמיד השתמשו בסימני הפלט המתאר בהקשר מתאים. לדוגמא, במקרים של תנאים, נסו לפרט את המצב שהוביל להדפסה של ההודעה.
- נסו להגיש פלט מתאר בעזרת ספרות משמעותיות. זה יכול לסייע לכם לזהות במהירות את הבעיה ולאתר את המקום שגורם לה.
- כדי למנוע שגיאות במרץ-זה, ודברים לא צפויים על ידי השתמשו בתנאי "if/else" כדי לבחון ה
