## מה ולמה?
כתיבת מבחנים היא תהליך שבו מתכנתים יוצרים קוד יחידות כדי לבדוק את פונקציונליותם. כיוון שכתיבת קוד היא חלק חשוב מאוד בתהליך פיתוח תוכנה, זה חשוב לוודא שהקוד עובד כפי שציפינו לו.

## איך לעשות זאת:
כאשר אנו כותבים מבחנים, אנו מתבססים על ספריית הבדיקות המובנית של C++ - `assert.h`. בשימוש בפקודת `assert()` אנו משתמשים בטענת בדיקה בכדי לוודא שתנאי מסוים הוא נכון. הנה דוגמא פשוטה:

```C++
#include <iostream>
#include <cassert>

int main(){
  int num = 5;
  assert(num == 5); // אם התנאי אמת, לא יודפס כלום
  assert(num != 5); // אם התנאי שגוי, יודפסה הודעת שגיאה
  return 0;
}
```

פונקציה זו תמצא את הטעמים מתאימים לבד וכך תוודא שהקוד שלנו פועל כרצוי.

## העומק:
כמו תהליך הפיתוח, כתיבת מבחנים היא תהליך שפותח ומשתוקק לשיפור כל הזמן. כתיבת מבחנים איננה המתודה היחידה לבחון קוד, ישנן מספר אלטרנטיבות אחרות כגון יחידות הבדיקה. כיצד שאלור, מסגרת פרוגרמתית לכתיבה, מאפשר לנו לכתוב יחידות בדיקה שישמשו כפתרונות מצוינים לכתיבת מבחנים.

רעיון נוסף הוא כתיבת מבחנים תוך שימוש בכלי כתיבת מבחנים. אפשרויות נחוצות הן Google Test ו-Catch הן שניים מכלים המציעים מגוון יכולות ומערכות הפעלה באמצעות המתחשב במספר מגבלות כולל התאמה אישית למחשב שלך וכונן המערכת.

ניתן ליצור מנוע כתיבת מבחנים משלנו באמצעות המסגרת המסעירה שהיא הכתיבה בשפות מקבילות וגיליון מחושב בלתי מסולסל כדי להעניק לנו יכולות עשירות.
