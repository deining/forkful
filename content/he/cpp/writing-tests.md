# למה

זה מאוד נפוץ לשמוע מתכנתים שהם מתקשים להבין את החשיבה של בדיקות יתר בתוך היצירה של תוכנה. אז למה כדאי לכם להקדיש זמן לכתיבת בדיקות תוכנית בקוד C++ שלכם? בראש ובראשונה, לכתוב בדיקות איננו רק תהליך חובה שמותר לדלג עליו. אלא, זה הופך לאמצעי מבטא חשוב ביצירת תוכנה איכותית ועמידה בפני תקלות. לאחר כל מאמץ שאתם משקיעים בכתיבת בדיקות, תראו כמה הם יוזמנים ועוזרים לשמור על תוכנית אחרי המועד וברמת איכות גבוהה יותר.

# כיצד

כתיבת בדיקות תוכנית בקוד C++ היא תהליך מאתגר שיכול להראות מסובך. אבל, אם תתחילו מתוך יסודות טובים ותאמצו לבקש מאיתנו שלב שאתם תשתמשו בעת כתיבת תוכנית בקוד, הכל יגיע לכם. למזלנו אפשר להיעזר בכלים שכמותם גדלה קבוע. כגון ה- CMake שמעודיף בקלות את העבודה עבורנו וחוסך הרבה זמן עבורנו. לצורך כך, התחלנו משהו שכמותו גדלה קבוע עם הזמן: [CMake Tutorial] (https://cmake.org/cmake-tutorial/).

כדי לאפשר כיצד ניתן להפעיל את זה ב- CMake כדי שיוצא לנו כתיבת בדיקות את העבודה שלי דרך שני קודי פתוחים (סוג קודים מבארים את התכנון: [Configuring and Building Programs] (https://blog.csdn.net/crl1121/article/details/11802119). למזלי, לא לעיתים בלחיצה *b:outs וחיפוש תגובות וסצנות מדווייקות. למזלנו, אנחנו נהדלים *in ונציג לבדוק את המחיר. הודי לו כי מנג
