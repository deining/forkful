## למה

עבור מפתחי תוכנה, כתיבת בדיקות היא חלק חשוב מתהליך הפיתוח. בדיקות מאפשרות למנוע באגים ושגיאות בקוד, כך שהתוכנה תעבוד בצורה יעילה ותוכיח להיות איכותית יותר.

## איך לעשות זאת

כתיבת בדיקות אינה מסובכת כמו שרוב הנמענים חושבים. ניתן לכתוב בדיקות בעבור קוד C++ בצורה פשוטה ויעילה. לדוגמה, ניתן לאתר באגים באמצעות אתר שמכיל פונקציות כגון `assert` ו- `static_assert`. הנה דוגמא של צורת השימוש של `assert`:

```C++
void print_number(int num) {
  assert(num >= 0 && num <= 100);
  std::cout << "Number: " << num << std::endl;
}
```

Mם, אין נו גק הלקו ניכר לעין, אתה מודע לכך שהמספר שהועבר לפונקציה `print_number` נמצא בטווח שנקבע על ידי הפקודה `assert`. כאשר הפונקציה מופעלת, הנקודה בה האסרטציה לא מתקיימת, התוכנה תפסיק לעבוד ותדפיס שגיאה לעלון. זה יעזור לך למצוא את השגיאה ולתקן אותה לפני שהתוכנית מכשילה.

בנוסף, קיימים כמה כלים נוספים כמו `catch` ו- `try` שגם מאפשרים לך לנתח את הקוד ולמצוא באגים בדרך זו. כדי לממש מבנה זה, כל מה שאתה צריך לעשות הוא להתאים אותו להצהרת הכותרת התואמת לפניך.
