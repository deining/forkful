## למה
דוגמאות קצרות ופשוטות לבדיקת קלט ופלט הן דבר נפוץ בתחום התכנות ובעיקר בשפת סי++ . כפי שכולנו יודעים, ייבוא הקלט והמציאת בעיות בקוד הן חלק חשוב מתהליך הפיתוח. לכן די נפוץ לבצע בדיקות שונות על הקוד בעת הפעלתו בכדי לזהות בעיות ולאתר את מקורן. ואחד הכלים הנפוצים לצורך זה הוא הדפסת פלט נוסף המאפשר לנו לראות את הערכים שהושלמו על ידי התוכנית בכל נקודה בזמן הרצתה.

## כיצד לעשות זאת
דוגמאות הקדמה פשוטותים, אך יעילים. פשוט נוסיף פקודת הדפסה לקטע הקוד הרלוונטי ונוכל לראות את הערכים בפלט התצוגה שלנו. לדוגמה:

```C++
int x = 5;
float y = 3.14;
cout << "The value of x is: " << x << endl;
cout << "The value of y is: " << y << endl;
```

פלט:

The value of x is: 5
The value of y is: 3.14

כמו כן, אפשר להדפיס ערכים של משתנים באמצעות פונקציות פנימיות של סי++ כמו הפונקציות `printf()` ו- `sprintf()`. לדוגמה:

```C++
int hours = 12;
int minutes = 30;
int seconds = 45;
printf("The current time is: %d:%d:%d", hours, minutes, seconds);
```

פלט:

The current time is: 12:30:45

בנוסף לזה, ישנם כלים נוספים שניתן להשתמש בהם לצורך הדפסת פלט נוסף, כמו דברים מתקדמים יותר כמו `cout` וליברריס מתקדמים כמו `log4cpp` ו- `boost.log`. הבעיה היא שכמו כל כלי במתכנת, גם זה דורש היכולת להתאים אותו לשימוש המיוחד של היישום שלנו ולנהל את הפלט בתוך הקוד.
