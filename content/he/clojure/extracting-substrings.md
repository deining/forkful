קוד קלוז'ר (גירסה נוכחית) כתיבה מאמר לקוראי עברית בסגנון בלתי פורמלי עם צורת ביטוי ככל האפשר קצרה ולטובת הנקודת. להימנע משימוש מיותר במילים וכן במשפטים. הכתבה מכילה ארבעה קטעים המסומנים בכותרות שונות, כאשר כל קטע שייך תחת כותרת אחת מתוך ארבעת הכותרות הבאות מתורגמות לעבריית:

"## מה ולמה?": שתי עד שלוש משפטים המסבירים (1) מהי ריכוז התת מחרוזת, וגם (2) למה מתכנתים מבצעים זאת.

"## איך לעשות:": דוגמאות של קוד ותוצאות מוצגות בתוך מסגרת הקוד "קלוז'ר ...".

"## חקירה מקיפה": פרטים מעמיקים כגון (1) הקשר ההיסטורי, (2) אלטרנטיבות, וכן (3) פרטים בתחום היישום המדובר.

"## ראו גם": קישורים למקורות נוספים הקשורים.

**מה ולמה?**
ריכוז תת מחרוזת הוא תהליך שמאפשר למתכנתים לחלץ חלק מהמחרוזת המקורית, בהתאם לתבנית מסוימת. מתכנתים מבצעים את זה כדי להשתמש בתתי מחרוזת עבור פעולות שונות, כגון חיתוך המחרוזת לחלקים קטנים יותר, או לחילופין עיבוד הנתונים עבור פעולות נוספות.

**איך לעשות:**
בינתיים נהיה שימוש בלינק לתרגום הטוב ביותר עם קוד קלוז'ר המבוסס על הגירסה האחרונה של השפה. שתי דוגמאות:

### דוגמא
```
(clojure.string/substring "איך לשלוח הודעות מבלי לנמק את מספר ה-IP שלי" 13 20)
```

זה מה שאנחנו מקבלים:
```
 "הודעות"
```

### דוגמא לחילוצים נוספים
```
(clojure.string/substring "היכנסת לאתר וממלאת את הטופס מחדש עשרות פעמים? תתרגע - יש דרך קצרה וקלה!" 23 29)
```

זה מה שאנחנו מקבלים:
```
 "טופס"
```

**חקירה מקיפה:**
בתחילת שנות השמונים, ריכוז תת מחרוזת היה דבר מעניין וחדש בתחום המחשבים, ומתחשב הנהלי מערכת. בשנים האחרונות, התהליך הוא חלק מהן כמעט כל משך היום, והשימוש בתוכנת הכתיבה שלך. חילופי טיוטה קלים מאוד לבצע, וזה קל בשימוש בתוך מילים אחרות כדי להגיע לתוך קטעים חלקים.
