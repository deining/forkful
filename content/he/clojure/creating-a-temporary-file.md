## למה

תיקון ואחסן מידע זמני יכול להיות חשוב בעת מתן כתוקן מדורה אם יש צורך בך. יצירת קובץ זמני מאפשרת למתן בעיה זמנית לכתוב מידע, לקרוא מידע כתוקן מקסימום ולאחר מכן למחוקו.

## כיצד

תחילה, נטושו את קובץ עם הפרמטר `:deleteOnExit true` כדי שקריאת הפרמטר תחסך ממידע זמני. נכתוב כתבה בשפה טובה יותר שתוכל לקרוא את תכתונה של הקובץ חדש תוכל למצוא תבניות סמפל בתחתית הכתבה.

```Clojure
    (with-open [temp-file (java.io.File/createTempFile "prefix-" ".tmp")]
        (println "קובץ זמני נוצר במקום הבא:" (.getPath temp-file)))
```

`תיקיה/קובץ זמני` יוצר מחרוזת של התיקיה עם `prefix` קבוצת תווים מגדירה, עם האותיות ממקוריות לקובץ זמני. כן נדרוש לקרוא `.getPath` על התיקיה את `string` אחד הפרמטר. אנו יצרנו קובץ זמני עם השם `prefix-.tmp` בתיקיית העבודה הנוכחית. לסוף, אנו מדפיסים את הנתון הנוכחי במקום הבא.
