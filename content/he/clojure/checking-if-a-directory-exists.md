## למה

בכדי לבצע פעולות בתיקייה בקובץ Clojure, עלינו לוודא שהתיקייה קיימת במערכת שלנו. דרך זו נוכל להיות בטוחים שאנחנו נותן את הפעולות הנכונות לתיקייה הנכונה, ונמנע מהתרגלות תכניות או התקיה שלנו.

## כיצד לבדוק אם תיקייה קיימת

בכדי לבדוק אם תיקייה קיימת במערכת שלנו, נשתמש בפונקציה `clojure.java.io/file` שמקבלת את הנתיב של התיקייה שאנחנו רוצים לבדוק. למשל, אם נרצה לבדוק אם התיקייה 'לשוני העיתון' קיימת בנתיב "C:/Documents", הקוד יראה כך:

```Clojure
(def directory (clojure.java.io/file "C:/Documents/לשוני העיתון"))
```

לאחר מכן, נשתמש בפונקציה `clojure.java.io/file?`כדי לבדוק אם התיקייה קיימת או לא. אם התיקייה קיימת, הפונקציה תחזיר את הערך `true`, אחרת תחזיר את הערך `false`.

```Clojure
(if (clojure.java.io/file? directory)
  (println "התיקייה קיימת")
  (println "התיקייה לא קיימת"))
```

פלט:
```
התיקייה קיימת
```

## מעמקים עמוקים

כאשר אנחנו משתמשים בפונקציה `clojure.java.io/file`, אנחנו לא רק מקבלים את הנתיב של התיקייה, אלא גם מציינים את הנתיב לתיקייה האב שלה. למשל, אם הנתיב של התיקייה הוא "C:/Documents/לשוני העיתון", הנתיב לתיקייה האב הוא "C:/Documents".

ככה ניתן לבנות תיקיות בדינמיות וליצור סדר בין תיקיות שונות. כמו כן, ניתן להשתמש בפונקציות נוספות כמו `clojure.java.io/exists?` כדי לבדוק אם התיקייה קיימת אך לא נוצרה באופן פשוט.
