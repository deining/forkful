# למה

ברגע שמתחילים לפתח אפליקציות ואתרים, חשוב לוודא שהנתיבים היוצרים את המאגרים של הקבצים והתיקיות שלנו קיימים. בדיקה של אומרים, אנחנו רוצים לוודא שיכולה להתאים נתיב שכבר ניצב תחת קיימת.

# איך לעשות זאת 

כדי לבדוק אם תיקיית קייפֿ קיימת, נשתמש במתודת `exists()` ונשתמש בקוד שלנו כדי לוודא אם הנתיב קיים. הנה דוגמה ב- Kotlin:
```
Kotlin
fun main() {
    val filePath = File("C:/Documents/Example")
    if (filePath.exists()) {
        println("הנתיב קיים")
    } else {
        println("הנתיב לא קיים")
    }
}
```
משם, אם תיקיית הקייף לא תהיה קיימת, המסך יחזיר "הנתיב לא קיים". אחרת, תיקיית הקייפ נמצאת והמסך חזר "הנתיב קיים".

# מקרינה עמוקה

כאשר מיישמים לוח הבקרה על תיקיית הקייץ, נרצה לוודא בדיוק גם את הנתיבים וגם את הגישורים שלנו לכל קובץ תחתיו. למשל, אם תיקיית הקייף נמצאת תחת מדפסת משמשת ומכתיבה, נרצה לוודא שיש לנו את התיקיית קיית רב-גישה. אנחנו יכולים לעשות זאת באופן דומה על ידי שימוש במתודת `canRead()` עם בדיקת ההשתמשים שלנו. כדי להיות יותר מדויקים, נדרוש את הקוד שלנו לראות שכל שולמו במכתיבה אשר נמצא שכבר נתיייתפ כליכא נבדל. וכדי להיות יותר כנה, יש לנו לוודא שתיקיית הקיים היא שדרגת המתאים במטרה לפתח את התוכן.
