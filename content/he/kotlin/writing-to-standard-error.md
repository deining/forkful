## למה?

כתיבה לפלט השגיאה הסטנדרטי (standard error) היא חלק חשוב מניהול תהליכי תכנות בקוד קוטלין. בלי כתיבה לפלט השגיאה הסטנדרטי, יכול להיות קשה לעקוב אחרי שגיאות ולבדוק אם התכנית פועלת כראוי.

## איך לעשות זאת?

כדי לכתוב לפלט השגיאה הסטנדרטי בקוד קוטלין, ניתן להשתמש בפקודת "System.err.println()" ולהעביר לה את ההודעה של השגיאה בין הסוגריים. לדוגמה:

```Kotlin
System.err.println("זהו הודעת שגיאה")
```

פקודת כתיבה לפלט השגיאה הסטנדרטי כאשר יש תנאי בתוך פקודה if נראה כך:

```Kotlin
if (condition) {
    System.err.println("הודעת שגיאה")
}
```

ניתן גם להשתמש במתודת "printStackTrace()" כדי להדפיס את מיקום השגיאה בקוד:

```Kotlin
try {
    //קוד עם אפשרות לשגיאה
} catch (e: Exception) {
    e.printStackTrace()
}
```

בקוד זה, אם כללי קוד fail, נוכל לראות בפלט השגיאה את המיקום שבו הקוד נכשל.

## העמיקו יותר

השתמשנו כעת בדרכים נפוצות לכתוב לפלט השגיאה הסטנדרטי בקוד קוטלין. אך ישנם דרכים נוספות לנהל את הפלט הזה. ניתן להשתמש במחלקת "PrintStream" או ליצור מחלקת אבסטרקט משלנו שתייצג את ההודעות של השגיאות. כמו כן, ניתן להתאים את הפלט לכמה ניסיונות אחרי אשר פקודות הכתיבה יפסיקו לנתב את ההודעות לפלט המשתמש.
