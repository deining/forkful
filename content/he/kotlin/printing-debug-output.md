##מהראשוןים שהיהThe Debug Output

החזרת המידע המתקבל מתהליך הקוד פנים כבר היה חלק חשוב מתהליך התכנות. רוב מטרת המידע הזה היא לסייע למתכנת בבניית הודעות שיעזרו לו לפתור בעיות ולהבין את הדרך שבה הקוד רץ. כתוצאה מכך, פסדים קודמים היו משתמשים בהדפסה במסך כאמצעי להמצאת בעיות בקוד. כמובן שכיום ישנם כלים יותר מתקדמים, אבל תהליך זה עדיין חשוב כמו תמיד, ובפרט במידה וניתקלים בבעיות קשות לסילום.

##איך להדפיס פלט לדיבוג

כאשר ניתקלים בבעיות בקוד, לפעמים הדפסת הפלט הרגיל לא מספקת את המידע המתאים על המקרה. במקרה כזה, יתכן שנרצה להדפיס דברים נוספים כמו ערכי משתנים או הודעות מותאמות. כדי לעשות זאת בקלות בקוד קוטלין, ישנם כמה דרכים:

```Kotlin
// דוגמא להדפסת ערכי משתנים
val num1 = 5
val num2 = 10
val sum = num1 + num2
println("המכפלה של $num1 ו-$num2 היא $sum")

// דוגמא להדפסת הודעה מותאמת
val name = "יונה"
println("שלום $שם, ברוך הבא למשחק הניחושים!")
```

בקוד מעלה, אנחנו משתמשים בסימן $ לפני שם המשתנה כדי להכניס את הערך שלו לתוך ההודעה. אם נרצה להדפיס מספרים או סטרינגים מוחלפים יחדיו, נשתמש בסימן + כדי לחבר אותם.
