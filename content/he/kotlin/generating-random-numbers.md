## למה

מי מתעניין בעופרת מספרים אקראיים? בהמשך שמעולם לא שאלתם את עצמכם על זה, אבל מספרים אקראיים נהוג להתבצע בתכנות כדי ליצור נתונים רנדומליים או לבדוק אלגוריתמים ופעולות.

## איך לעשות

הכיצד הצלחת להתיימר לבדך מרחבי הקוד ולמצוא מדורגים ונתונים אקראיים בקושי. רק תכנן את האלגוריתם הנכון ותשתמש בפונקציות הקיימות בקוטלין כדי ליצור נמצא, כמו גם:

```Kotlin
import kotlin.random.Random

// השתמש ב-generator הכליה של קוטלין כדי ליצור מספרים אקראיים
val random = Random.nextInt(0, 100) // יוצר מספר אקראי בין 0 ל-100
val randomBoolean = Random.nextBoolean() // יוצר Boolean אקראי
```

כעת, אתם רק צריך לשים את הקטע האלגוריתמי הזה בתוך לולאה אם אתם רוצים ליצור רשימת מספרים אקראיים או להשתמש בו בכל קוד שיש לכם צורך בנתונים אקראיים.

## נפל מעמקים

ליצור מספרים אקראיים ניתן על ידי מגוון של אלגוריתמים אשר כל אחד עם יתרונות וחסרונות שונים. למשל, Random.nextInt() האדרנלין-תשומת אלף והניסיון מגביל את המספרים האפשריים לתצורת משתנים בגבול 0 עד 100. לעומת זאת, מחלקת java.util.Random מספקת מגוון רחב יותר של אלגוריתמי random, כולל Instantiation-based המציעה מספר האחריות עם קדם הנושא הטבעי switch המתבסס על הניסוי של מספרים אקראיים בהתאם לבחירה התנסיתית.
