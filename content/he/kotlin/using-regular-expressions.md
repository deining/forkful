# למה

רגילקס טקסט הוא כלי חזק ונפוץ עבור מתכנתים המשמש לאיתור והתאמת צפיינים מסוימים בתוך טקסט. אומנם השימוש בו דורש ידע ממוקד וכישול במיוחד, אך המרווחים הגדולים בהתאמה משמעותית הופכים את הטקסט פחות טמטום ויעיל יותר. בכתבה זו, נלמד כיצד להשתמש ברגילקס טקסט בשפת קוטלין כדי למנוע כישלונות מיותרים ולהגדיר תבניות של מידע.

# איך לעשות זאת

כדי להשתמש ברדיגלקס טקסט בקוד קוטלין, נצטרך לייבא את מודול RegEx. לאחר מכן, ניתן להשתמש בתנאי if לבדיקת תאמת הטקסט לתבנית מסוימת:

' ''kotlin
if (RegEx.matches("תבנית לבדיקה", "תאימות טקסט")) {
println ("תאימות טקסט נמצאה!")'
' '''

למעשה, תנאי זה מיועד לבדיקת התאמת טקסט כוללת. אם תאימות טקסט נמצאת, הודעה תודיע על כך בתוצאה. ניתן לשחזר את התוצאה הראשונה עם עזרת פעולת אפס. לדוגמה:

' ''kotlin
val text = "אני אוהב חתולים"val pattern = "אוהב (.*)"

משתנה הטקסט יאתחל למילה הראשונה לאחר מופע רגילקס, אוהב, בניגוד למשתנה שנוצר באופן ידני. לאחר מכן, ניתן להשתמש במילות המפתח בצירוף לבירור הקוד, כך שאם הטקסט מתאים לתבנית מסוימת, תמורה לתוכן של הטקסט תוחלף בשתי משתנים: המילה אוהב וחתולים.
