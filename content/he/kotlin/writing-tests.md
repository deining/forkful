## למה

למשוך דגש על ההגנה פנימית של קוד בכדי לוודא שהוא עובד כמו שצריך ומנגיש לשימושים מגוונים. בנוסף, כתיבת בדיקות עוזרת לודא ששינויים חדשים לא מפריעים לפונקציונליות קודמת.

## איך לכתוב בדיקות ב-Kotlin 

כתיבת בדיקות ב-Kotlin היא פשוטה ונוחה, למרבה המזל. כדי להתחיל יש ליצור פונקציה נפרדת עבור כל פונקציה שצריכה להיות נבדקת. בכדי לבדוק תוצאות מצופה, ניתן להשתמש בפונקציות כמו `assert()` ו- `assertTrue()`.

```Kotlin
// פונקציה לחיבור שתי מספרים ולבדוק אם התוצאה נכונה 
fun addNumbers(a: Int, b: Int): Int {
    return a + b
}

// בדיקה שהפונקציה מחזירה את התוצאה הנכונה 
fun testAddNumbers() { 
    assertEquals(addNumbers(2, 2), 4) 
}
```

בנוסף, ב-Kotlin ניתן להשתמש בספריית ייחודית שנקראת "JUnit 5" כדי למקסם את חזרת התוצאות ולשפר את השלבים השונים בבדיקה.

## טפסים עמוקים

כתיבת בדיקות מעודדת מתכננים לחבר עם פעולה בצורה שלמה יותר. כאשר צוותים מתחילים לבנות בדיקות בפרויקט, הם יכולים להתפתח על ידי כתיבת בדיקות לפני כתיבת הקוד הממשי. בנוסף, כתיבת בדיקות יכולה לעזור לטפל בפאר קוד כגון תקלות חומרה וזיהוי כתיבה כאשר הקוד משתמש בפונקציות חיצוניות.
