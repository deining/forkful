## למה

למה כדאי לכתוב בדיקות? לא משנה אם אתה ממתין לעזרה מהקבוצה שלך, או אם אתה חושש מפיגורים ותקתות בקוד שלך, בדיקות הן כלי חיוני לכתיבת קוד איכותי ויעיל. הן עוזרות לך לבדוק שהקוד שלך תואם את הדרישות שלך ושהוא עובד כצפוי ללא שגיאות ובאופן עקבי. כתיבת בדיקות מאפשרת לך להתקדם במהירות ובבטחון בפיתוח התוכנה שלך.

## איך לכתוב בדיקות בקוד Kotlin

כדי לכתוב בדיקות ב-Kotlin, יש להשתמש בספריית ה-```kotlin.test``` המובנית. כדי לתת לדוגמה, נכתוב בדיקה פשוטה לכן במחלקה הבאה שמציגה פעולות חשבון:

```Kotlin
import kotlin.test.assertEquals

class AccountTest {

    @Test
    fun `test deposit`() {
        val account = Account()
        account.deposit(100)
        assertEquals(100, account.balance)
    }
}
```

בקוד זה, אנחנו משתמשים בפונקציית הבדיקה המובנית ```@Test``` כדי לבדוק את פעולת הפקודה ```deposit()``` שמקבלת סכום כקלט ומעדכנת את המאזן של החשבון. כדי לוודא שהתוצאה שמקבלים היא המצופה, אנחנו משתמשים בפונקציית הבדיקה המובנית ```assertEquals()``` ומשווים את המאזן הנוכחי של החשבון לסכום שמצפים לקבל כתוצאה. עם יכולות הבדיקה המובנות של Kotlin, ניתן לכתוב ולנהל מגוון רחב של בדיקות ולוודא שקוד ה-Kotlin שלך עובד כאשר עושים כוונתך.
