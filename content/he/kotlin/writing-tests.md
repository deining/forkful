## למה

כתיבת בדיקות בתוכנות לא תמיד נראית כמשימה כיפית, אבל היא מצוינת להבטיח שהתוכנה שלכם עובדת בצורה נכונה ומתפקדת כפי שציפיתם. בדיקות גם יכולות לעזור לך למצוא בעיות באופן כללי ולתקן אותם לפני אישור או ייצוא של התוכנה שלכם.

## כיצד לעשות זאת

כדי לכתוב בדיקות עם Kotlin, תצטרכו להשתמש בספריית הבדיקות הפנימית של Kotlin. הנה דוגמא של סימולציה קצרה שמדגימה איך לבצע בדיקת מקרה קצה:

```Kotlin
@Test fun testEdgeCase() {
    val list = listOf(1, 2, 3)
    // אשכול קצר של בדיקות
    assertEquals(list.size, 3)
    assertNotEquals(list[1], 4)
}
```
הפלט של הקוד הנ"ל יהיה:
```
PASSED: testEdgeCase
```

## חפירה עמוקה

כעת שראינו דוגמא קצרה, הנה כמה עצות נוספות על כתיבת בדיקות ב-Kotlin:

- שימו לב שהתוויות @Test מופיעות לפני כל בדיקה, כדי שספריית הבדיקות תזהה אותן.
- ניתן להשתמש בשיטת assert והשיטות הנגזרות שלה (כמו assertEquals) כדי לוודא שהתוכנה שלכם מצית אנטיה או אמינית. 
- ניתן גם לבדוק מהאם כפולתם של מוחלט הגם טופיה אינה דומה לפעמיים. זה יכול להיות לאיתור בעיות כמו עריכות שגויות או נתונים שגויים.
