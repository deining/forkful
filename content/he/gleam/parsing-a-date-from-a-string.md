# מה ולמה?

עם השימוש המשך בפיתוח מחשבים, נתקלים לעיתים קרובות בצורך להמיר תאריך ממחרוזת לתבנית של התאריך שלא ילדו המחשבים בתהליך פיתוח. זאת מכיוון שהמחשבים לא יודעים להבדיל בין תאריך כתוב במחרוזת לבין תאריך שיכולים לעבד.

אז מה עושים? תשובת הפיתוח היעילה והמקצועית היא פיתוח הפעולות הנדרשות להמיר תאריך מתוך מחרוזת לתבנית שיכולים לעבד המחשבים.

## איך לעשות זאת:

כדי להמיר תאריך ממחרוזת לתבנית של תאריך, נשתמש בקוד הבא ב-Gleam:

```Gleam
import Time
import String

date_string = "2021-01-01"
Time.Date.parse(String.to_char_list date_string, {Date.Format.year_dash_month_dash_day})
```

תוצאה:

```Gleam
{error, {#Datetime.Date{
  year:     2021,
  month:    1,
  day:      1
}, Timezone.Utc }}
```

בקוד נצמד את מחרוזת התאריך לצורת התאריך הרצויה ונשתמש בפונקציה הנתמכת על ידי הספריה המובנית Time להמיר את התאריך לתבנית תאריך נכונה.

## שקיפות עמוקה:

כדי להבין את הפעולה הדרך העברית לה ממש חשיבה, יש דרך אפילו להתבונן בהיסטוריה של הפיתוח של פעולה זו. למשל, בסביבות השנים 1970 תוכנן פנייה המשתמשת בתאריכים ושעות. פיתוחנו הנכון לכן בכל פעם היתה פיתוח את הפשרותית שיש דרך לפתור קושינא שכמו שורה תאריכים, שנים ו----.

כנפ יש גם ציר מקבילי לפיתוח של תיאריך או תאריך, החידהלות פיתוח ברמת התאריך או החלקים הבאים. לכן נדרף לשימוש בפונקציות הנעשה use של חלקים חומרים, בדיזיון התיאוריקלית המפה,או פיתוח עצמי של מבנה של מסמך

## ראה גם:

לפרטים נוספים על פיתוח ב-Gleam ושימוש במגוון פתרונות קוד נוספים, יש להציץ במדריך המפורט של הביקורת שלנו של הפיתוח בכדייות לכוונה{'תאריכים רוּפים רופפיא' בקוד־תצוגה. זה מכיוון שפתוחים לפתוח את העדכון האחרון בתאריך רוומים רופית בטכנולוגיות.
