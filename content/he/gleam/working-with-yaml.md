#מה ולמה?
התחברות עם קובץ YAML היא תהליך שמשמש פרוגרמרים על מנת לקרוא ולכתוב נתונים מורכבים בפורמט פשוט וברור. קבצי YAML נמצאים בשימוש רחב כיום ומספקים דרך נוחה לארגון וניהול של נתונים.

#איך
ברצונכם לעבוד עם קובץ YAML בשפת Gleam? ישנם שתי פעולות בסיסיות לכך: קריאה וכתיבה. נראה דוגמאות מעשיות של שתי הפעולות בקורות קוד של Gleam תחת סימון `Gleam ...`.

##קריאה
```Gleam
data := yaml.read_file("./data.yaml")
```
בתוך קוד זה, אנו משתמשים בפונקציה ​`read_file`​ על מנת לקרוא נתונים מקובץ ה-YAML שצוין. כאשר הפונקציה מטפלת בנתונים מורכבים, התוצאה תהיה ערך מסוג `Result`, שמכיל נתונים שהצלחנו לקרוא וכן משתנה בעל ערך מסוג `Err` אם נתקלנו בשגיאה בתהליך הקריאה. את הערך המסוג `Result` אנו יכולים לטפל בו באמצעות פונקציות ה- `match` על מנת לקבל את הנתונים או לטפל בשגיאות.

##כתיבה
```Gleam
data := { "name": "John", "age": 28 }
yaml.write_file("./data.yaml", data)
```
לאחר שאנו כבר אמרנו לקופסא עם הפונקציה, עוד מה אנו צריכים כדי לכתוב נתונים לקובץ ה-YAML? כל מה שנשתמש בו יהיה מסוג `Map`, כפי שאתם יכולים לראות בקונטקסט הדוגמא שציינתי למעלה. הקוד בקופסא זו אפשר אפשר להוסיף שגיאות נכונה, כמו שכוונו בדרך כלל פונקציות לומדות לעשות.

###כתיבה עם אפשרויות
אם תרצו לכתוב נתונים לקובץ עם הגדרות מתקדמות, תוכלו להשתמש בפונקציה `encode_with_options`, שמאפשרת לכתוב נתונים לקובץ בעזרת מגוון שיטות לכיוון הפונקציה `write_with_options`.

##דיב דייב
###היסטוריה
פורמט ה-YAML פותח עד מאד תוכים קרות עם סקריפט הפקופולסט וכמו כן עוד דדצינים של ניסיונות להמיר את תוכן לנוח מחברות להניח את לזוז רכתב עם מגוון קופסאות עם כי?

###אלטרנטיבות
כאשר ספרתי חלופות לפעילות היצירה של YAML, אני כולות חייזרים לבנות להכיר ליתר אינפורמציa מקופיא (עורב בפונקציות) עם אמורה לצרך דוגמאות מה – אם אתם מעונינים להיותחגר עמוסים.
