כתיבת בדיקות היא חלק חשוב וקריטי בתהליך של כתיבת קוד, ובעיקר בתחום של תכנות בשפת Gleam. כתיבת בדיקות מאפשרת לנו להבין ולוודא שהקוד שכתבנו עובד כפי שציפינו, וזה יכול למנוע בעיות ותקלות בעתיד.

## Why (למה)

למה כדאי לכתוב בדיקות? הסיבה העיקרית היא שזה מפנה לנו זמן יקר וכסף על חיפוש ותיקון בעיות בשלב מאוחר יותר בתהליך של הפיתוח. בזמן שאנחנו כותבים את הקוד, יכולים להיות מספר טעויות ואמצעי נכשלים שאנחנו לא מתרשמים מהם בשלב זה של התהליך. כתיבת בדיקות עוזרת לנו לזהות ולתקן את הבעיות הללו במהירות, ולהבטיח שהקוד שלנו יעבוד באופן תקין מבלי לצורך בהתעסקות מיותרת בעתיד.

## How To (כיצד לכתוב בדיקות)

קוד של Gleam נראה כמו זה:

```Gleam
fn create_user(name) {
    fn validate_name(name) {
        match name {
            "" -> error("Name cannot be empty")
            _ -> ok()
        }
    }

    validate_name(name)
}
```

הקוד נמצא בתוך זוג ‪‪triple backticks‬‬, וזוהי הדרך הנכונה לכתוב קוד בדרך זו. כעת, נאמת את הקוד על ידי כתיבת טסטים עבורו:

```Gleam
test "create_user" {
    assert ok() == create_user("John")
}

test "empty_name" {
    assert error("Name cannot be empty") == create_user("")
}
```

כאן אנו בודקים שהפונקציה create_user מצפה לקבל תווים ותחזיר את הערכים הנכונים. אנו משתמשים ב־assert מכיוון שאנחנו מצפים שהערך שיחזור הוא ערך תקין.
