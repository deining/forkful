## למה

בעולם התכנות, כתיבת בדיקות היא חלק חשוב מאוד של תהליך הפיתוח. היא מאפשרת לנו לוודא כי הקוד שאנחנו כותבים עובד כמו שצריך ושהשינויים שאנחנו מבצעים לא מפרים את פונקציות הקוד הקיימות. כתיבת בדיקות מאפשרת לנו לצמצם שגיאות ולאמנם זה עלול להוסיף מספר נוסף של צעדים לתהליך הפיתוח, אך בעקבות שימוש בבדיקות בצורה נכונה, אנחנו יכולים לשמור על הקוד שלנו ברמה גבוהה יותר.

## איך לעשות זאת

כתיבת בדיקות ב-Gleam היא פשוטה וקלה ללמידה. נהוג לחלק את הקוד שלנו לפונקציות קטנות יותר וליצור קבצי בדיקה נפרדים עבור כל פונקציה. כאשר אנחנו כותבים בדיקות ב-Gleam, נשתמש בפונקציות מיוחדות כדי לוודא שהקוד שלנו מחזיר את הערך הנכון ושהתוכנית שלנו מתנהגת כפי שצריך. הנה דוגמא:

```Gleam

fn add(x, y) {
  x + y
}

test "Addition" {
  assert add(2, 3) == 5
}

test "Negative numbers" {
  assert add(-2, -5) == -7
}

```

בדוגמא זו, אנחנו מקפידים לבדוק שהפונקציה שלנו add מחזירה את הערך הנכון עבור כמה ערכים שונים. אנחנו יכולים להריץ את הבדיקות באמצעות פקודת ה-assert ולוודא שהערך שמתקבל הוא זה שאנחנו מצפים לו.
