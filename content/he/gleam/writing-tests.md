## למה
באחת מהכלים החשובים ביותר בתהליך התכנות, נדרש הסבר על נושא הטסטים. במאמר זה נפרט למה כדאי לכתוב טסטים וכיצד לעשות זאת ב- Gleam.

## איך לעשות זאת
בעזרת דוגמאות קוד ופלט לדוגמאות שונות, נסביר כיצד לכתוב טסטים ב- Gleam. הנה כמה דוגמאות לביצוע ב- Gleam:

```Gleam
// המודול Sut מכיל את הפעולה האינטרנלית addtwo
pub fn addtwo(x: Int, y: Int) -> Int {
    x + y
}

pub fn test_addtwo() {
    assert addtwo(2, 3) == 5
}
```

כאן אנו מגדירים את המודול Sut שמכיל את הפעולה addtwo שמקבלת שני פרמטרים מטיפוס Int ומחשבת את סכוםם. בסוף הקובץ, אנו מגדירים פונקציה בשם test_addtwo המכילה את הטסט של הפונקצה addtwo. באמצעות הפונקציה assert אנו מוודאים שהפעולה addtwo מחזירה את התוצאה הנכונה.

```Gleam
test "טסט לפונקציה פעוטה" {
    assert my_function(5) == 10
}
```

כאן אנו בודקים את תוצאת הפונקציה my_function על הקלט הנתון (5) ומוודאים שהתוצאה היא הנכונה (10).

כדאי לשים לב שבכדי להריץ טסטים צריך להוריד את הספרייה ```gleam.test``` על ידי הרצת הפקודה הבאה בשורת הכפתורים:

```sh
gleam install gleam.test
```

## מכניסה עמוקה
בתחום התכנות, ידוע כי כתיבת טסטים היא חשובה כמו כתיבת קוד עצמו. למרבה המזל, עם Gleam ניתן לכתוב טסטים פשוטים במיוחד בעזרת הסינטקס הנועד לכתיבת פונקציות מיוחדות לטסטים. כל מה שנדרש כדי לבחון פונקצ
