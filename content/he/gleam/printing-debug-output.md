# למה

דיוקוד יורה בעתים קרובות דפיסה לאוסף הערכים הוא דבר מאוד יעיל כשמדובר על תיקון באגים או בעיות בקוד. באמצעות הדפסת פלט של הצעדים הכלשהם שביצע הקוד, ניתן למצוא את הטעות ולתקן אותה בקלות. הדפסת פלט בעתיד לא תנצל יחסית לאופן השמה בדבריים אפשרי רק לסוף, אז למה שחלק מאתגר פתרונה? מאחוריהם (למשל, בשביל לסמוך על איזה משתנה)

# איך לעשות

כדי להדפיס צעדי קוד ב־Gleam, ניתן להשתמש בפקודת הדפסה `inspect()` ושמה לתוך אוסף הערכים הרצוי להדפסה. בנוסף, ניתן להשתמש ב־`println!()` עבור דפיסה של ערכים שאינם אוסף, כמו משתנים או תוצאות של פעולות. נהפוך לפעולה כדי להדפיס את הווריאנט של פונקציות או מה שלא, אנו נדפיס את עצמו. ברגע שעיית הדפסת פלט ניתן לראות את צעדי הפעולה בצורה ממוקדת ובכך לזהות בקלות את הבעיה.

```Gleam
import gleam/list

fn example_function(x) {
  y = x + 1
  z = x * y
  println!("x is {}", x)
  println!("y is {}", y)
  println!("z is {}", z)
}

fn main() {
  example_function(2)
}
```

פלט:

```
x is 2
y is 3
z is 6
```
