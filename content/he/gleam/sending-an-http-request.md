# מה ולמה? #

שליחת בקשת HTTP היא פעולה חשובה בתחום התכנות, כיוון שהיא מאפשרת לנו לתקשר עם שרתים ולקבל מידע מהם או לשלוח להם מידע. תכניות נותנים בדרך כלל דגש על יצירת תקשורת עם מקורות חיצוניים כדי לשפר את היכולות של התוכנית ולהעניק למשתמשי התוכנה חוויה משופרת.

# איך לעשות: #

כדי לשלוח בקשת HTTP בתוך גלים, ניתן להשתמש בפונקציית ```Gleam.Http.send``` ולספק לה את הכתובת לכתובת האתר שאנחנו רוצים לתקשר איתו. זה נראה כך:

```
Gleam.Http.send("https://www.example.com")
```

אתר המשל לעיל זה רק משל, וכמובן שאפשר לשנות את הכתובת לאתר אחר שרוצים לתקשר איתו.

כאשר אנו מריצים את הכתובת הזו, התוכנית תבצע בקריאה לאתר ותחזיר את המידע שהיא קיבלה. לדוגמא:

```
Ok("Welcome to example.com")
```

פסקת התשובה הנ"ל מציינת שאנחנו מקבלים תוכן של אתר מברכת, אבל כמובן שיכול להיות גם תוים אחרים, תלוי באתר שביצענו את הבקשה אליו.

# חקירה מעמיקה: #

השליחה של בקשת HTTP היא תהליך חשוב אך פשטני, ולאורך השנים פותחו מספר פתרונות נוספים כגון Fetch API ו-API Gateway. מפתחי גלים כישלו בפרוייקטיים רבים לקבלת מידע תקין מהשרתים, ולכן זוהי תכונה שמחוברת עם מנוע סיום התוכנה המונחה-גופים.

בכדי להעמיק בנושא השליחה של בקשת HTTP בתוך גלים, מומלץ לעיין במקורות הקשורים הנלווים:

- התערוכה הנפלאה של חברי הגוף המקבל של Glam אינה מפסיקה לפתוח את המאמר המתמטי "קשר בין מנוע ניגש HTTP לתקשורת חיצונית".
- מנוע זה כולל תכונה בסיסית להתקשרו ל-API באמצעות בקשות HTTP, וטכנולוגיית Freeform המרשמת למלאכת השרת באמצעות פרוטוקול המנוהל על-ידי HTTP.

כך מגיעים למידע המפורט יותר על השליחה של בקשת HTTP בתוך גלים, שאינה מצריכה שימוש במורכב ומסובך.
