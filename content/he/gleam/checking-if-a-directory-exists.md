#למה

בדיקת קיומו של תיקייה היא תהליך חשוב בתכנות ובמערכות המבוססות על קבצים. זאת מטען כי בדקוי את קיומו של התיקייה קודם כל מאפשר ייצור מסלולים וטיפולים אחרים בקבצים בתוך התיקייה החל אם היא קיימת. למעשה, לבדוק את קיומה של תיקייה היא צעד בסיסי בבניית פונקציות שיעזרו לך לנהל ולעבד קבצים תוך תיוג בדיוק את התיקיות הנכונות לעיבוד.

# איך לעשות זאת

בשפת גלמה, ישנן מספר דרכים לבדוק את קיומו של תיקייה. ניתן להשתמש בפונקציה "file.exists" אשר מקבלת את הנתיב של התיקייה כפרמטר ומחזירה ערך בוליאני, המציין אם התיקייה קיימת או לא. ניתן גם להשתמש בפונקציות נוספות כמו "file.is_dir" אשר בודקת אם הנתיב המצויין מתאים לתיקייה, ו-"path.canonicalize" אשר קובעת את הנתיב האבסולוטי של התיקייה מכל תוך התיקייה שנמצאת בתיקיות בנות נוספות וכו'.

כדי לבדוק אם תיקייה קיימת בגלמה ניתן להשתמש בקוד הבא:

```Gleam
import gleam/path

let directory = "/home/user/files"
let exists = path.exists(directory)

if exists {
  "The directory exists!"
} else {
  "The directory does not exist!"
}
```

מסלול התיקייה לבדוק נקבע כדי לצמצם הסתבכויות כמו החלפת כמו טכנולוגיה אחרית (למשל מ Windows ל Unix ולהיפך), ניתן להשתמש בפונקציות נוספות כדי לטפל ביתר דייקנות ברכיבי הנתיבים האבסולוטיים שנמצאים בתנאיים מיוחדים.
