# למה

כתיבת בדיקות בתכנות היא פעילות חיונית לכל מפתח תוכנה. הן מספקות ביטחון ומובטחות כי הקוד שכתבנו עובד כמו שציפינו שיעשה. בנוסף, הן עוזרות לנו לזהות באופן מהיר ויעיל בעיות כנגד כמה אינטגרציה קשורות וחלקי תוכנה שונים.

# איך לעשות

```Go
func Add(x, y int) int {
	return x + y
}

func TestAdd(t *testing.T) {
	sum := Add(5, 7)
	if sum != 12 {
		t.Errorf("Expected sum to be 12, but got %d", sum)
	}
}
```

תחילה, ייצרו פונקציית בדיקה בעזרת התגית `Test` ושם ממשו את הקוד שנרצה לבדוק (במקרה זה, פונקציית Add). לאחר מכן, השתמשו בפעולת ההערכת שגיאה `Errorf` כדי להשוות את התוצאה של הפונקציה לתוצאה הצפויה. כשנכיר את הפרמטרים המתקבלים והתוצאות הצפויות, ניתן להמשיך לכתוב בדיקות נוספות עבור פונקציות וחלקי קוד אחרים שרצינו לבדוק.

# עומק התיכון

כתיבת בדיקות היא מנגנון נהדר לזיהוי באופן מוקדם של בעיות בקוד ולמניעתן. בעזרת בדיקות ניתן לבנות קוד יותר יציב ומתחבר יותר, וגם לחסל באופן מהיר באגים כשהם מתגלים. חשוב לשים לב לנקודות התפר הטובות כדי לוודא שכל הפרטים המיוחדים של הפונקציה או המחלקה מתפקדים כמצופה.
