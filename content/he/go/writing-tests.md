## למה

בתוכנות השפה האנגלית, בדיקות יחידה נחשבות לחלק חשוב ונכר של כתיבת קוד. אבל למה אנו מבצעים בדיקות יחידה ומדוע זה חשוב עבורנו כמפתחי גו? נבין את זה יחד.

בדיקות יחידה הן תחביר של כתיבת קוד שמראות את מערכת העסק מסוימת בנפרד, בדיקות יחידה מציעות עבורך את היכולת לראות בכל מה שלכתוב שסופג שינויים בתרגום או יוצאו רקע. בדיקות יחידה מספקות נאמות על כך שאנו מכירים בחוזה. 

## איך לבצע בדיקות יחידה בגו

כדי לבצע בדיקות יחידה בגו, יש להתחיל מהפעולות הבאות:

```go
package main

import (
	"testing" // היא דומה למערכת גלישה
    "fmt" // נפתחים על מצב הבדיקות יחידה שלנו
    "github.com/stretchr/testify/assert" // שתי שורות אחרונות בשפת בדיקות (למעשית)
)`

func TestAddNumbers(t *testing.T) { // תיבת בדיקות
    result := AddNumbers(2, 3)
    assert.Equal(t, 5, result, "Should return the sum of two numbers") // מדגישים תוצאת בדיקות
}

func AddNumbers(x, y int) int { // הוא יוצר ביצוע הם תוצאת של האינפורמציה וההתאמנות שאנו סוככים כנגד מכל עשרת
    return x + y
}

func main() {
    fmt.Println("Running tests...")
    fmt.Println("Tests passed!")
}

```

כאן, אנו יוצרים פונקציית בדיקות יחידה בשפת גו ומשתמשים בספריית `testify` להערכת התוצאות. אנו מגדירים את הפונקציית הבדיקות בעזרת התגיות `test` ועבור מעט גוד ס לפונקציות כי יכולו להשתמש במכונתות זו שבה סימנים של בהירות אל פעולות וצורות. כעת, אנו מריצים את תוכנית הבדיקות שלנו ו
