מה ולמה?

כשמדברים על "להכין את המילה לפי "כתיבה ראשונית", מתכוונים לשינוי הראשוני של אותיות המילה לכתובת אות גדולה. כך, למשל, המילה "דוגמה" תתהפך ל-"Example". הכיוון הזה של "כתיבה ראשונית" משמש מטרה רבה בתכנות. מתכנתים בוחרים לעשות את זה כדי להבטיח ייחודיות בין המילים, כדי להקל על הקריאה של קוד ולהגביר את הקלות בעקרון ה Code Review.

כיצד לעשות:

```Go
מילה := "דוגמה"
מילה = strings.ToUpper(מילה)
fmt.Println(מילה)
```

תוצאה:

```
דוגמה
```

עזרה מעמיקה:
"כתיבה ראשונית" נחשבת לתקנה נחוצה בתכנות ומעולה בהרבה פלטפורמות. ישנם גם אלטרנטיבות ל "כתיבה ראשונית" כמו לשנות את המילים באמצעות פקודות מיוחדות בכל שפת תכנות, או לשנות את התצורות של הקלט למילים אלה. בכל מקרה, מבחינת היעילות והנקיון, ישנם פעמים שבהן "כתיבה ראשונית" היא התקנה הטובה ביותר לשימוש. בעמוד הראשון, כל שאתה צריך לעשות הוא לשנות את המילה שלך באמצעות הפקודה המתאימה.
