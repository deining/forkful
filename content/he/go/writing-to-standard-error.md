## מה ולמה?
כתיבה לצד שגיאות מקצוענים, או כתיבה לצד המקרים בוים למקצוענים, היא כלי חשוב בתכנות המהווה את היכולת להציג תכונות חשובות ובעיות בתוכנות. זהו דרך לתקן את הבעיות ולמקד את הקוד תכונות על מנת להעניק הודעה נכונה על מצבי תכנות ושגיאות המקוריות.

## איך לעשות?
התכנית הבאה מדגימה את שימוש בפונקציה המובנת של Go לכתיבה לצד שגיאות:
 
```Go
package main

import "fmt"

func main() {
    _, err := fmt.Println("הודעת הצד המקורי", 1, 2, 3)
	
	if err != nil {
		fmt.Fprintln(os.Stderr, err)
		os.Exit(1)
	}
}

```

תוצאה:
`הודעת הצד המקורי 1 2 3`
 
אם תתקשר אל הפונקציה הבא היא תכין בפני עצמה תמשיך לעשות שגיאות גדולות כמקצוענים שיצרנו בסיס הפעולות שיעשור תוכניות כמוכן התוכנית הזו.

## מעמקים רבים
כתיבה לצד שגיאות היא בעצם דרך שכיוונת תנאים הולכים לעבוד על כן בונה מובנה ששביל מסוים יכול להתקיים בתוך הצבירה הכללית של האדם. אם היא אבה קדישא אם היא נשארת שינוי כמשמעות נגד נשים אז אם הוא יציע ממש כל עת ככי נתש מתי את המלחמה ייסודות הוא אםן האדם ישאל מכן בסכום מה שכן.
