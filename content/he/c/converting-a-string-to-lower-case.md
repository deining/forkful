תהפוכת טקסט לחלק קטן (lower case) נחשבת לפעולת בסיס בתכנות המודרני. לכן, זה יכול להימצא במגוון רחב של שפות תכנות ומערכי הפעלה. המטרה של הפעולה היא להמיר את כל האותיות בטקסט לאותיות קטנות בלבד בכדי לקלוט נתונים ולעבד אותם בצורה יחידה ותואמת.

## מה זה ולמה לעשות זאת?

המלה "lower case" מתייחסת למצב של אותיות קטנות באנגלית. כאשר מדובר על תהפוכת טקסט לחלק קטן, אנו מתכוונים למעשה להמיר את כל האותיות בטקסט לאורך העיבוד לאותיות קטנות בלבד. המטרה של הפעולה היא להפוך את הטקסט לקל לקריאה ולעיבוד, בכדי להתאים בין תואר מסוים לתוכנית הקולנהולית או אובייקט המודרות.

## איך עושים את זה:

כאן ניתן למצוא כמה דוגמאות קוד ופלט זיגוגיים לכאורה כדי להדגים את הפעולה:

```C
#include <string.h>
#include <stdio.h>

char str[] = "Hello, WORLD!";
printf("%s \n", str);

// תהפוכת טקסט לחלק קטן באמצעות פונקציית strlwr
strlwr(str);
printf("%s", str);

/* פלט:
Hello, WORLD! 
hello, world! */
```

ישנם גם פתרונות אחרים נפוצים לתהפוכת טקסט לחלק קטן בכדי ליצור בצורה תקינה ויעילה את התכניות והאפליקציות שהם:

```C
#include <ctype.h>
#include <stdio.h>

char str[] = "HeLlO, WoRlD!";
printf("%s \n", str);

// תהפוכת טקסט לחלק קטן באמצעות מערך של islower עם קידומת לשימוש 
for (int i = 0; str[i]; i++)
{
    printf("%c", tolower(str[i]));
}

/* פלט:
HeLlO, WoRlD! 
hello, world! */
```

## ירידה נמוכה

תהפוכת טקסט לחלק קטן נחשבת לפעולה סטנדרטית בשפת התכנות סי. זה יכול לקרות באמצעות פונקציות ספציפיות או באמצעות שימוש בפקודות קובעות. יתר על כן, במרבית המקרים, המטרה של הפעולה היא לפשט את התוכנית המקומית ולהתאים את התוכנית המקורית לצרכי התכנות והפלט הנדרשים.

כמו כן, ישנם תפיסות אלטרנטיביות רבות לתהפוכת טקסט לחלק קטן, כגון תהפוכת טקסט לחלק תחתון (underscore) ותהפוכת טקסט לאותיות גדולות (upper case). כל אלו יכולים לשנות את המראה של הטקסט ולהקל על העיבוד שלו בפועל.
