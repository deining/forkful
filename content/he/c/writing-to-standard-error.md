## למה

כתיבה לפלט שגיאה תקני, או כתיבה לפלט לנתונים דיבוגינג נחוץ בקוד, יכול לסייע למתכנת לתקן באופן יעיל שגיאות ובאופן כללי, להוסיף חידושים ולשפר את הקוד שלו.

## כיצד

```C
#include <stdio.h>
int main() {
   printf("זהו פלט רגיל לפלט!\n");
   
   fprintf(stderr, "יש לי שגיאה כאן!\n");
   return 0;
}

```

פלט:

```
זהו פלט רגיל לפלט!
יש לי שגיאה כאן!
```

בדוגמה זו, הפלט הראשון הוא מתוך `printf` הקלאסי, שמדפיס לפלט הרגיל. הפלט השני הוא מתוך `fprintf`, שמדפיס לפלט שגיאה. ניתן לראות שהפלט השני נמצא בצבע אדום, מציין שזהו פלט לשגיאה.


## חקירה מעמיקה

כאשר משתמשים בפונקציות כמו `printf` לכתיבה לפלט, ההודעות מופעלות לתוך פלט מסוג stdout. אם יש צורך לכתוב הודעה לפלט שגיאה, ניתן להשתמש בפונקציות כמו `fprintf` ולכתוב לפלט מסוג stderr. כך, ניתן להגביר את אמינות הקוד ולזהות שגיאות בצורה יעילה יותר.

חשוב לזכור שכאשר כתיבה לפלט stderr נעשית, יש להשתמש בפונקציה `perror` על מנת לחזור עם תיאור השגיאה המדויק וכדי שהמשתמש יוכל להבין במה התרחשה השגיאה.
