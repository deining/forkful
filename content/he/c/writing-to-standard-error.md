## למה

כתיבה לתקליטור התקינה (standard error) היא חלק מרגיל וחשוב משימות תכנות. היא מאפשרת למפתחים לאתר באמצעות תקליטור התקינות את השגיאות והבעיות בקוד שלהם. כתיבה לתקליטור התקינות גם עוזרת לצרכנים למצוא פתרונות לבעיות שהם עלולים להתאתר בהם.

## כיצד לעשות זאת

```C
#include <stdio.h>

int main(void)
{
  fprintf(stderr, "הודעת טקסט לתקליטור התקינות\n");
  return 0;
}
```

כדי לכתוב לתקליטור התקינות בשפת C, יש להשתמש בפקודת fprintf ולשלוח את ההודעה שברצונך להדפיס כתחרות לפונקציית fprintf הסטנדרטית. בקוד המופיע למעלה, אנחנו משתמשים בלולאה כדי לחזור על ההדפסה כמה פעמים, אך בפועל ניתן להשתמש למשל בתנאי if לבחירת פעולת ההדפסה.

## מעמיקים יותר

כמו בכתיבה לתקליטור התקינות, כתיבה לתקליטור התקינות מאפשרת למפתחים לקבוצה לערוך תחזוקת תכניות. זה מאפשר להם לאתר בעיות ולתקן אותן מהר יותר, הכוללים את הקנה ארץ במעקב אחר השגיאות המתאות. בנוסף, כתיבה לתקליטור התקינות אינו משתמש בשפה הסטנדרטית של C, ובכך כתיבה לתקליטור התקינות משתמש במעקב אחר השגיאות של התקליטור התקינה וכדי לאמנות דיאלוג של השגיאות.
