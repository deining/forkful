## מה זה ולמה?

כשאתם מפתחים אפליקציות מרובות, נהורו להדפיס פלט תקין כדי לוודא שהתוכנה שלכם עובדת כראוי. כאשר אתם מדפיסים מידע נוסף במקום, הכל צריך להיות תקין ומדויק. זה מאפשר למפתחים לזהות במהירות את הבעיות בקוד ולתקן אותן.

## איך לעשות זאת:

מדפיסים רק כאשר רוצים לצפות פלט עם קבצי דבוג שלחה ל

```C
#include <stdio.h>
int main()
{
    int a = 5;
    float b = 2.5;
    char c = 'x';
    printf("a = %d, b = %f, c = %c\n", a, b, c);
    return 0;
}
```

כאן אנחנו מדפיסים את הערכים של המשתנים a, b ו-s. ניתן להוסיף גם הודעות תיבוליות כמו "התוכנית התחילה" או "ערך שגוי". כאשר התוכנית מריצה, הפלט יראה כך:

```C
a = 5, b = 2.500000, c = x
```

המשתנים יוצגו עם הערכים שלהם ומאחר שהוספנו שורת התיבול, ניתן להבחין בבקש לתיקון הערך של המשתנה.

## ים עמוק:

כאשר נעלמו ימים, המפתחים התמודדו עם תקשורת עיוורת ודרך לנתח קוד בלי לכתוב תואר כולל ויכולים כעת לדפוס את הערכים של המשתנים. ישנם כמה אפשרויות לעשות זאת, כמו כך, כאשר אתם כותבים דבוגים, תהיו מסוגלים לבחון את הפלט שלכם בקלות ולהבין את מי שמשנה את הקוד הפעולה של התוכנית.
