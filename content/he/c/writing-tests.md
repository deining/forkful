## למה 

בעולם של התפתחות תוכנה, כתיבת בדיקות היא חלק חשוב מהתהליך. היא מאפשרת לנו לוודא שהקוד שכתבנו עובד כפי שציינו ולזהות באופן מוקדם טעויות אפשריות. היא גם מאפשרת לנו להיות יותר בטוחים בקוד שאנחנו כותבים ולשפר את תהליך הפיתוח שלנו.

## איך לעשות

בדיקות בתכנות בשפת C נעשות באמצעות הפונקציות assert ו-printf. כאשר כותבים בדיקות של assert, אנחנו מוודאים שכל משתנה ותוצאה מתאימים למה שציפינו. לדוגמה:

```C
#include <assert.h>

int sum(int a, int b) {
  return a + b;
}

int main() {
  int result = sum(5, 5);
  assert(result == 10);
  printf("The sum of 5 and 5 is %d.", result);
  return 0;
}
```

כאן, אנחנו מוודאים שתוצאת הפונקציה sum תהיה 10 בכל מקרה, ואם לא תהיה כזו, התכנית תכשל. ה-printf יודפס רק אם הבדיקה עברה בהצלחה.

## לגלות עומק 

כעת שהבנתם את הבסיס של כתיבת בדיקות בשפת C, כדאי לחקור את ההגדרות העמוקות יותר כדי לפתח מיומנויות נוספות בהתפתחות התוכנה. ישנם כמה טיפים שמומלץ להתרגל אליהם כדי לבנות בדיקות איכותיות יותר, כמו להשוות ערכים במקום להשוות משתנים שלמים עם assert, להשתמש בלולאות לבדיקת צורות לולאה שונות, וכו'.
