## למה

תוכניות בשפת C מכילות תמיד טעויות ובאופן טבעי רוצים לתת לקוד שלנו לרוץ כמו שצריך. כשדיסמורונים מתגלים במהלך הפיתוח, כתיבת בדיקות עוזרת לזהות אותם ולתקן אותם בזמן כדי להבטיח תכונות ותפקידים נכונים ומדויקים לתוכנה שלנו.

## כיצד לכתוב בדיקות

את הקוד שלנו ניתן לנסח בעזרת הקבוצה המיוחדת החמודה של "```C ... ```". כאן אפשר להדגים דוגמאות של קוד בכדי להבין כיצד לכתוב בדיקות בשפת C ולהראות את הפלט שאמור להתקבל כאשר הבדיקה עובדת כראוי.

```C
#include <stdio.h>

int main(){
    // הדפסת הודעה לבדיקה
    printf("זוהי בדיקה ראשונה");
    
    // זוהי הבדיקה עבור קלט ופלט לא מסודרים
    int x = 5;
    int y = 10;
    printf("x הוא %d ו-y הוא %d", x, y);

    // בדיקה נוספת עבור קלט ופלט מסודרים
    int a = 10, b = 20, c = 30;
    printf("a הוא %d, b הוא %d ו-c הוא %d", a, b, c);
    
    return 0;
}
```

כאן אנחנו רואים שיש לנו שתי בדיקות בכדי לעבור: אחת עבור קלט ופלט לא מסודרים ואחת עבור קלט ופלט מסודרים. בכל בדיקה אנחנו מדפיסים כמה משתנים ומשווים אותם למתודות הרלוונטיות. אם הפלט תואם לקלט, הבדיקה עברה בהצלחה.
