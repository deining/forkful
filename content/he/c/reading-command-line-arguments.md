## למה

שלוש הסיבות העיקריות לקרוא ארגומנטים מצד שורת הפקודה בשפת `C` הן:

1. זהו דרך יעילה לקבל קלט מהמשתמש ולהשתמש בו בתוך התוכנית.
2. ארגומנטים מצד שורת הפקודה מאפשרים כניסה דינמית לתוכנית, במקום לקבוע ערכים קבועים מראש עם קלט מוקדם.

## איך לעשות

```C
#include<stdio.h>

int main(int argc, char *argv[]){
    printf("מספר הארגומנטים: %d\n", argc);

    int i;
    for(i=0; i<argc; i++){
        printf("ארגומנט %d: %s\n", i, argv[i]);
    }

    return 0;
}
```

הנה דוגמה של כיצד ניתן לקרוא ארגומנטים ממשתמש ולהדפיס אותם במספר שורות. לפתוח את הקוד בשם `read_args.c` ולהריץ אותו בכניסה מצד שורת הפקודה:

```shell
gcc read_args.c -o read_args
./read_args hello world!
```

הפלט של התכנית יהיה:

```
מספר הארגומנטים: 3
ארגומנט 0: ./read_args
ארגומנט 1: hello
ארגומנט 2: world!
```

גם נוכל להשתמש בסימון `argv[0]` כדי לקרוא את שם התוכנית המשתופצת. אם נשנה את הקריאה לפורמט הבא:

```C
printf("שם התוכנית החדשה: %s\n", argv[0]);
```

הפלט יהיה:

```
מספר הארגומנטים: 3
ארגומנט 0: ./read_args
ארגומנט 1: hello
ארגומנט 2: world!
שם התוכנית החדשה: ./read_args
```

כעת ניתן להשתמש בארגומנטים הנמצאים באינדקסים אחרים כדי לבצע פעולות מתאימות עליהם.
