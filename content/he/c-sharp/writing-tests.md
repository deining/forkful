# למה

כתיבת בדיקות בתוכנות שלנו היא צעד חשוב בבניית תוכניות איכותיות. בדיקות מאפשרות לנו לוודא כי הקוד שכתבנו עובד כפי שציפינו ומקיים את הפונקציונליות שלו. הן עוזרות לנו לזהות באופן מוקדם יותר תקלות ובאופן כללי, להפחית את הסיכון של באגים ותכשיטים.

# כיצד לכתוב בדיקות בבתים

נדמה שכתיבת בדיקות היא עבודה פשוטה, אבל זה ממש לא המקרה. כדי לכתוב בדיקות איכותיות, יש לנו כמה כללים חשובים לעקוב:

```C#
// יצירת מופע של מחלקה לבדיקה
Calculator calc = new Calculator();

// הגדרת את הקלט לפונקציה
int num1 = 5;
int num2 = 10;
                
// קריאה לפונקציה של המחשבון לבדיקה
int result = calc.Add(num1, num2);

// השוואת התוצאה לתוצאה צפויה
if (result == 15)
{
    // הדפסת הודעת הצלחה אם התוצאה תואמת לציפיותינו
    Console.WriteLine("בדיקה עברה בהצלחה!");
}
else
{
    // הדפסת הודעת כישלון אם התוצאה לא תואמת לציפיותינו
    Console.WriteLine("בדיקת היתה כישלון.");
}
```

# גיבור עמוק

כתיבת בדיקות מספקות לנו נקודת מבט נוספת על קוד שלנו. זה יכול לעזור לנו לזהות מקומות בקוד שאולי צריך תיקון או שיפור. בנוסף, בדיקות מאפשרות לנו לכתוב קוד נקי יותר ולשמור עליו תואם לסטנדרטים שלכנו.
