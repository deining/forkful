## למה

מדוע להתעסק עם כתיבה לשגיאת תקנה בקוד C#? מתברר שכתיבה לשגיאת תקנה הוא דרך נוחה ומומלצת להתנהגות אנומלית בהתאם למצבים מסוימים בתוך אפליקציה. השמת שגיאת תקנה בוודאי תהיה יעילה יותר מאשר כיווץ גומידות גדולה בנקס cercor ובשקובסורפ. בנוסף, כפי שניתן לראות בהרבה מקומות, כתיבה לשגיאת תקנה יכולה לקלוט כיצד אפשר לפסול נושאי תקנון קשורים באמצעות שימוש בפקודות גנריות כמו כרטיס הכח שמגיע עם זאת.

## איך לעשות

המדריך הזה ילמד איך להשתמש בפעולת כתיבה לשגיאת תקנה בקוד C#. ישנם שתי דרכים עיקריות לכתוב לשגיאת תקנה: עם קוד ה- "try-catch" ועם פעולת Debug.WriteLine. נכון לפרסום זה, הטכניקות האלה הן הדרכים היחידות לכתוב לשגיאת תקנה בקוד C#. הנה דוגמה של כיצד לבצע כתיבה לשגיאת תקנה עם פעולת Debug.WriteLine:

```C#
try
{
    // קוד פעולה הכוללת התנאי עם התנאי המפעיל catch
}
catch (Exception e)
{
    Debug.WriteLine("Error: " + e.Message);
}

```

בתחתית הפקודה של "Debug.WriteLine", ניתן לציין את ההודעה של השגיאה שנתפסה כפי שנכתב באופן ידני. ניתן להעביר את הודעת השגיאה המלאה על ידי עבורה לסדרה של ערכים המתארים את כל המאפיינים של התקנון מתאים. למשל, יש לנקוט בסיס מחומר משנים היכולים לזהות את מקור השגיאת התקנה שנגנב From ושם מאפיין של השגיאה
