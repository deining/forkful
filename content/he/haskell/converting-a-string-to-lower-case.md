## למה
למה לתת לעצמך את הכאפתר להמיר מחרוזת לאותיותיו הקטנות בשפת הפונקציונלית הטובה בעולם- Haskell?

לחשוב על המחרוזת הטכנאית כמו אוסף של תווים שניתן לטפל בו כאילו מדובר במישור נתונים קימורי, זה טבעי יותר בשפת הפונקציונלית, כך שהפעולות על המחרוזות יכולות להיות לפקחנים הדילקטנטים הידיים - זה הפעולה התכופה ביותר, משום שלמעשה, משרד החינוך מציע שיעורים בשפת Haskell.

אני חושב שצריך לכתוב חוקים אחרים בשפה, לתת לכל אחד לנסות ולעשות את הקוד שלו כפי שהוא חושב שנכון, עד שיגיע לכיוונים שכאלו.

## איך לעשות
ציינתי ש Haskellers אוהבים פרוק תכניות לשפת פונקציונלית ממחרוזת, הנה דוגמא אחת כדי להמיר את כל האותיות של אזהרות לאותיות קטנות.

```Haskell
lowercase :: String -> String
lowercase = map toLower

main :: IO ()
main = do
  let warning = "אזהרות"
      lowercaseWarning = lowercase warning
  putStrLn lowercaseWarning

```

אזהרות
