## למה

כתיבת בדיקות בתוכנית היא חלק חשוב ונפוץ של תהליך פיתוח תוכנה. במאמר זה אני אחלוק עםכם מדוע כדאי לכתוב ולשכפל בדיקות תוכנה בשפת הפסקל.

## איך לכתוב בדיקות תוכנה בהסקל

כתיבת בדיקות תוכנה בשפת הפסקל יכולה להיות מסובכת בתחילת הדרך, אבל עם קצת בקצב ותרגול היא משתלפת ולמדת. הנה תכנים בסיסיים שנוכל להתחיל כתיבת קטעי קוד בהסקל.

```Haskell
-- ייבוא מודולים נדרשים
import Test.HUnit
import Text.Printf

--פונקציית בדיקה כללית
testFunction :: Test
testFunction = TestCase (do
    -- הכנס את הטקסט או הערך שאתה רוצה לבדוק
    assertEqual "אתה צריך לכתוב את התוכן שלך לכאן" "אתה צריך ליצור ערך שלם כמו נמרוד"
)

--בדיקות יחידתיות
test1 :: Test
test1 = TestCase (assertEqual "תוכן בדיקה יחידתי" "ערך צפוי" "ערך אמיתי")

test2 :: Test
test2 = TestCase (assertEqual "תוכן בדיקה יחידתי" "ערך צפוי" "ערך משתנה")

-- סכימת הבדיקות והדפסת הפלט
main = runTestTT $ TestList [testFunction, test1, test2]
```

פלט:

```Haskell
Cases: 3  Tried: 3  Errors: 0  Failures: 0
```

## מעמד עמוק

כתיבת בדיקות תוכנה היא כלי חשוב לכל מתכנת. היא מאפשרת לנו לוודא שהתוכנית שכתבנו עובדת בדיוק כפי שציפינו ולגלות בעיות כבר בשלב מוקדם של הפיתוח. בנוסף, כתיבת בדיקות מעלה את רמת הביטחון בתוכנית ובדרך זו משפרת את איכות הקוד.
