## למה

כתיבת בדיקות היא חלק חשוב מתהליך הפיתוח של כל פרויקט ב-Haskell. כתיבת בדיקות עוזרת לנו לוודא שהקוד שכתבנו עובד בדיוק כפי שציפינו ומונעת תקלות ובאגים בעתיד. חשוב לציין שב-Haskell, כתיבת בדיקות היא חלק מהתרגול שלנו ויש לנו מנחים ספציפיים עבור כיצד לכתוב בדיקות באופן יעיל.

## איך לעשות זאת

בחלק זה נראה דוגמאות של כיצד לכתוב בדיקות ב-Haskell עם דוגמאות פשוטות ולא כל כך מורכבות.

קוד לתוכנית שלום עולם:

```Haskell
main = putStrLn "שלום עולם!"
```

פלט:

```
שלום עולם!
```

אפשרות לבדיקה של פונקציה:

```Haskell
-- פונקציה לחיבור של שני מספרים שלמים
add :: Int -> Int -> Int
add x y = x + y

-- בדיקה של פונקציה עם פרמטרים נתונים
main = print (add 2 3)

-- תוצאה צפויה: 5
```

פלט:

```
5
```

כדי להראות שהפונקציה שלנו עובדת כפי שציפינו, נכתוב עוד מספרי ציפיות לפונקציה שלנו ונראה שהכל עובד כצפוי:

```Haskell
-- בדיקה של פונקציה עם עוד ציפיות
main = do
  print (add 5 5) -- ציפייה: 10
  print (add 10 (-3)) -- ציפייה: 7
  print (add 0 0) -- ציפייה: 0
```

פלט:

```
10
7
0
```

## מעמקים

כאשר אנחנו מנסים לכתוב בדיקות לפונקציה מסוימת, חשוב לציין שהבדיקה עצמה אינה פונקציה חלקה מהקוד המקורי. בדיקות אינן עושות שימוש בקוד המקורי, אלא מכילות נתוני כניסה ותוצאות צפויות
