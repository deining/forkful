## למה
אם אתה כתוב קוד בפונקציונלית הסטטיית רצית לכתוב לפלט למפלס אם בזמן ריצת הקוד מצביעה על תפקיד אחר שבו לחלקים נכתבים לכתחילה, הגדרות של פונקציות חדשות או עבודה אחרת שבדרך כלל כתוב בקובץ טקסט רגיל אלכס שיעלה על העיניים עם הפונקציות הנבדקות לפני החבילה.

## איך להשתמש
הנה מספר דוגמאות לכתיבה לפלט שגוי בפונקציונלית הסטטיית רצית לכתוב לפלט למפלס באמצעות הפונקציה "hPutStrLn" שמקבלת כפרמטר את הביטוי להדפסת השגיאות ואת התקן הנמצא בדקדוק הכתב החפש פונקציות מתי פעולה יחסית עם הדוגמה להצגת תוצאות, כולל הצגת השגיאה.

```Haskell
main = do
  x <- readFile "somefile.txt"
  if null x
    then hPutStrLn stderr "Error: The file is empty"
    else hPutStrLn stderr "Success: File exists" 
```

פלט מסודר יפה יותר עם שפה טבעית ואצל שפת תכנות אחרת. בפונקציונלית הסטטיית רצית לכתוב לפלט מסוג שנראה כמו "Error: The file is empty" במקרה הטוב.
