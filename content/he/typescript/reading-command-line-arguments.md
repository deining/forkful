# למה

למה מתבקש מישהו לקרוא ארגומנטים בשורת הפקודה? ישנם מספר סיבות נוספות אבל החשובה ביותר היא כי זה מאפשר לנו ליישם קוד גמיש יותר ולאפשר למשתמשים להתאים אישית את התוכנה לכפייהם וצרכיהם. כמו כן, זה מאפשר לנו לתפעל תוכניות מרובות עם אותו קוד בקלות.

# איך לעשות זאת

וכתבנו בקוד שלנו, אנחנו יכולים לקרוא את הארגומנטים שהועברו לתוכנית בשורת הפקודה באמצעות הפונקציה process.argv. לדוגמה, אם אנחנו רוצים להדפיס את כל הארגומנטים שהועברו לתוכנית, נוכל לעשות זאת באמצעות הקוד הבא:

```TypeScript
console.log(process.argv);
```

והתוכנית תדפיס את הרשימה של ארגומנטים שהועברו לה. אם רצינו להדפיס רק את הארגומנט הראשון, נוכל לעשות זאת כך:

```TypeScript
console.log(process.argv[2]);
```

אנחנו יכולים להשתמש בארגומנטים המתקבלים ע"י התוכנית כדי להשתמש בתנאים וליישם פעולות שונות על פי הצורך.

# כיוון עמוק

ישנם כמה דברים שחשוב לזכור בעת קריאת ארגומנטים בשורת הפקודה. למעשה, המחרוזת הראשונה של התוכנית תהיה תמיד הנתיב לתוכנית עצמה. כמו כן, אם אנחנו רוצים להעביר מחרוזת מסוימת שמכילה רווחים כארגומנט, עלינו להשתמש בגריל המפרידה לציין שהוא לא חלק ממחרוזת אחת. לדוגמה:

```Shell
node myProgram.js "Hello World"
```
