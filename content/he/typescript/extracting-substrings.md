# למה:
התפקיד החשוב של מתכנת הוא להתמקד בפירוטים קטנים כדי לחשוף את הקוד שלו בדיוק לאותו רמת פירוט שלו. חלק מהפירוט הזה הוא דחיסת מחרוזות בכדי להבין את מבנה הנתון. לכן בדיוק למה לאסוף תתי מחרוזות נותן לנו ידע נוסף על מתכנת התכנית ומשפר את הקוד שלנו

# איך לעשות זאת:
ובמילים אחרים, תתי מחרוזות הם מתכון מחרוזת אלגנטי וחזק יותר. ב TypeScript, תתי מחרוזות ניתנים לחילוץ בדיוק כמו מחרוזות אחרות. השתמשו בפונקציות כמו slice, substring או substr כדי להחליץ את התת מחרוזת הרצויה. כאשר שימוש נכון, תתי המחרוזות יכולים לשפר ביצועים ולהפוך את הקוד שלנו ליותר מאורגן וקריא.

**דוגמאות:**

```TypeScript
// חילוץ תת מחרוזת מהמיקום המתאים
let str: string = "זו חרוזה ארוכה";
let substring: string = str.slice(4, 10);
console.log(substring); // חרוזה

// חילוץ תת מחרוזת מן הסוף של המחרוזת
let str: string = "זוהי כפית מחרוזת";
let substring: string = str.substring(6);
console.log(substring); // כפית מחרוזת

// חילוץ תת מחרוזת עד כמה תווים אחרי המיקום הנתון
let str: string = "זו עוד חרוזה ארוכה";
let substring: string = str.substr(8, 6);
console.log(substring); // חרוזה
```

**תוצאה:**

```
חרוזה
כפית מחרוזת
חרוזה
```
