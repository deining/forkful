## למה?

כתיבת בדיקות היא חלק חשוב וקריטי בתהליך התכנות. בדיקות מסייעות לנו לוודא שהקוד שלנו עובד כפי שרצינו בכל סינריו אפשרי וכן לזהות באופן מוקדם בעיות ותקלות בקוד. בנוסף, הן מאפשרות לנו לשמור על איכות ויציבות של האפליקציה שלנו על מנת לספק חוויה טובה יותר למשתמשים.

## איך לעשות זאת?

מתי כתבתם לאחרונה קוד בTypeScript ורציתם לוודא שהוא עובד במהירות ואמינות? אם עדיין לא התחלתם לכתוב בדיקות, כדאי להתחיל כעת! כאן נמצאת כתבה בסגנון קל ונוח עם דוגמאות קוד ותוצאות.

### בדיקות יחידה (Unit Tests)

בדיקות יחידה הן הבדיקות הכי יסודיות בתהליך התכנות. הן בודקות את חלקים קטנים ומעטים של הקוד ומוודאות שהם עובדים כפי שצפוי. לדוגמה, בדיקות יכולות לבדוק את הפונקציות שלנו באופן יחיד ולוודא שהן מחזירות את הערכים הנכונים וכו'.

 הצגת דוגמא של בדיקת יחידה ב-TypeScript:
```TypeScript
// בדיקת הספירה המכנית של התווים במחרוזת
function countCharacters(str: string): number {
  return str.split("").length;
}

// בדיקה שהפונקציה מחזירה את הערך הנכון
if (countCharacters("hello") === 5) {
  console.log("כל הכבוד! הבדיקה עברה בהצלחה");
} else {
  console.log("אוי לא! ישנה בעיה בפונקציה");
}
```
תוצאה:
```sh
כל הכבוד! הבדיקה עברה בהצלחה
```
