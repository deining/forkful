# כיצד לכתוב בדיקות ב-TypeScript
בדיקות הן חלק חשוב מאוד מתהליך הפיתוח ב-TypeScript. הן מאפשרות לנו לוודא שהקוד שאנחנו כותבים עובד כפי שצריך ולזהות ולתקן בעיות בזמן הכתיבה. בכתבה הזו, אנחנו נלמד כיצד לכתוב בדיקות ב-TypeScript.

## למה
לא ניתן להשקיע מספיק זמן בכתיבת בדיקות. הן מסייעות לנו לשאוב מפתחים ביכולות הכתיבה שלנו, לנטרל מקרי גילוי שגיאות ואפשרויות פתרון בצורה מהירה יותר, ולחזות על סגנונות רעים או שגיאות דגים טנפוראריות בצורה עקבית.

## כיצד לעשות זאת
אם אתם מקדימים בכתיבה שלכם, כל מה שיש לכם לעשות זה להוסיף דאגות כיצד לבדוק את קודכם החדש. תחילה, נגדיר קבצי בדיקה חדשים. כדי להגדיר קבצי בדיקה ב-TypeScript, נשתמש ב-directive, ואז נכתוב את הבדיקות הנדרשות. בסופו של דרישות הכתבה הזו, נתמקד בכתיבה ועשייה של בדיקות עם Jasmine.

```TypeScript
// קבצי בדיקה
// greeting.spec.ts

describe('Greeting', () => {
  it('כדי שדירוג מכיל שלום', () => {
    const greeting = new Greeting();
    greeting.sayHello();
    expect(greeting.message).toContain('שלום');
  });
});
```

העובדה שנוכל להשתמש ב-Jasmine תוכננה כך שנוכל להריץ בדיקות ולעבור לפסקולס ספציפיים או לוודא שהם נוכחים בצורה נכונה מנמלם ביותר. כדי לעשות זאת, נוכל להשתמש בדוגמאות השונות ב-Jasmine.
