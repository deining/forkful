# למה

ישנם מגוון רחב של סיבות לכתוב תוכניות ב TypeScript, ואחת מהן היא כי הוא מספק מנגנון משופר לשימוש בפונקציות קריאות יותר. במאמר זה, אנחנו נבין את הסיבה העמוקה מאחורי כתיבת קוד הגדרתית ב TypeScript ומדווחים על המאמר הראשון המפרט קולט סטרינג.

# כיצד

נתחיל על ידי יצירת פונקציה פשוטה המקבלת סטרינג ומחזירה אותו עם האות הראשונה מופעלת. אזהרה: זו לא פונקציה אידיאלית למצבים כניסה פשוטים כגון ראשי-אות, כי Pi צריך לכתוב באופן מיוחד בגיזרה הפרים (מפק הפרים) שלו. אותה פונקציה יכולה לראות את הקישור.

"במאמר זה אנחנו רוצים ללמוד כיצד לכתוב פונקציה המקבלת סטרינג ומחזירה את אותו סטרינג עם האות הראשונה מופעלת. נוכיח את זה על ידי ריצת קוד המשתמש בהרחבות המספר של TypeScript ```typescript
function capitalizeString(str) {
  return str.charAt(0).toUpperCase() + str.slice(1);
}

console.log(capitalizeString("typescript")); // Output ציון: "TypeScript"
```


# חפירה עמוקה

אחד היתרונות המשמעותיים ב TypeScript הוא שהוא מספק תמיכה מובנית עבור טיפוסים. על כן, בזמן ההירכה, לכתוב כקולט TypeScript הוא מעולה יותר מאשר לכתוב כקולט המהמרות. הפעם, נכתוב פונקציה יותר קלה המקבלת את הסטרינג וטיפוס לפני נכתב כל דבר מייד יצירת של כמה במשתמש לכתוב סטרינג עם האות הראשונה מופעלת בグמספ של נכתב על ידי. הנה כתיב
