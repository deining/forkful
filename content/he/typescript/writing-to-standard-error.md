# מדוע

למה לכתוב לצאן שגיאה בתכנות בטפסקריפט? ישנם מספר סיבות שיכולות להכניס אתכם לכתוב לתקן השגיאות. כאשר אתה מכתיב לתקן שגיאות, ניתן להיגדר כל הפיצ'רים מראש ולכן ביטול הטעויות האתמול אינו נדרש. וכמובן, זה ישפר את חוויית המשתמש ויעזור לאתר טעויות אחרי כך.

# איך לעשות

 לפני שנעבור לתרגול בפעולה, נצטארך לייבוא את המנגנון השגיאות הפורץ של טפסקריפט. נבצע זאת באמצעות הפקודה הבאה
"```TypeScript

import { error } from 'console';
```

כעת ניתן להשתמש בטקסט לאחזרה מטפסקריפט לתת לאחרים לכתוב לתקן השגיאות. במקרה הזה, ניתן להשתמש בפונקציות נוספות עם אותו דגש כאשר חוטם הוא בפיתוח בפונקציות להכתוב לתקן השגיאות שאנו מוצאים.

# טביעת רגל עמוקה

לכתוב לצאן שגיאה בטפסקריפט היא עבודה רבת כוח. לא רק שניתן להשתמש בתוכניות צד-לקוח ובספריית הפיתוח כדי לסיים אותם, אלא ניתן גם להשתמש במאמץ להחדיר בוודאות בתוכניות אחרות. לכן, כאשר מתארים תרגול שגיאות יש לעשות תרגול אמיתי בחומר אמיתי.
