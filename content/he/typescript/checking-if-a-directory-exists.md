# מה ולמה?

בתכנות, לבדיקה האם תיקייה קיימת, היא פקודה שמשמשת כדי לבדוק אם יש את התיקייה במחשב המתבצע הפקודה. תיקייה היא ארגון של קבצים ותתי תיקיות במחשב. מתכנתים מבצעים בדיקת תיקייה כדי לוודא שהתיקייה קיימת ולבצע פעולות נוספות בהתאם.

# איך לעשות?

הנה דוגמא מוצגת של פקודה לבדיקת תיקייה ב TypeScript:

```typescript
if (fs.existsSync(path)) {
    console.log('תיקייה נמצאת במיקום זה.');
} else {
    console.log('התיקייה לא נמצאת במיקום זה.');
}
```

יש לשים לב שכאשר משתמשים בפקודה זו, יש ליבדוק אם היא פועלת על המערכת הפעילה שלכם. בנוסף, יש גם לאפשר גישה לספרייה של "fs" כדי להשתמש בפקודה המתאימה.

דוגמא נוספת עם פקודת "readdirSync":

```typescript
fs.readdirSync(path).forEach(file => {
    console.log(file);
});
```

כאן, הפקודה מבצעת את הפעולה של לקרוא את כל הקבצים בתיקייה ולהדפיס אותם בשורה חדשה.

באם אתם מעונינים לבדוק אם תיקייה מסוימת קיימת בתוך תיקייה אחרת, אתם יכולים להשתמש בפקודה בשם "join":

```typescript
const pathToCheck = path.join(__dirname, 'folder', 'subfolder');
if (fs.existsSync(pathToCheck)) {
    console.log('תיקייה בתוך תיקייה זו נמצאת במיקום זה.');
}
```

כאן, הפקודה משמשת כדי לשכבל את המיקום המדויק של התיקייה שמתקיימת תת התיקייה בתוך תיקייה זו.

# חפירה עמוקה

- היסטוריה: בעבר, בעיקר בתכנות תת-תוכנית כמו C ופורטרן, היה חייב לבדוק אם קובץ קיים לפני השימוש בו בפעם ראשונה. זה נעשה כדי לאשר שהמערכת תוכל לפתור את הקובץ המבוקש.

- אלטרנטיבות: ישנן פקודות נוספות לבדיקת תיקייה בתוך מהלך תכנות כמו "stat" ו-"lstat" אבל "existSync" היא הפיתרון הפשוט יותר והנפוץ יותר.

- פירוט ביצוע: הפקודה המוצגת בדוגמאות משתמשת בספרייה של "fs" כדי לבדוק אם התיקייה קיימת.
