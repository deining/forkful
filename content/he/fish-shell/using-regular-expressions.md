# למה:

השתמשות בביטויים רגולריים היא דרך מאוד חזקה לעבוד עם טקסט בתוך תוכניות פיתוח. היא מאפשרת לנו לחפש, להחליף, ולהשתמש במידע בשיטה ממוסדת יותר מאשר פשוט לערוך אותו באופן ידני. אז למה לא להתחיל ללמוד איך להשתמש בביטויים רגולריים ולתת לנו כלי חזק יותר לתכנות?

# איך לעשות את זה:

כדי להשתמש בביטויים רגולריים בפיש של, קל למדי. ניתן להשתמש בפקודת `grep` עם האפשרות `-o` כדי להשמיט את התוספת בכל המתאימים. לדוגמה:

```
# נמצא את כל המספרים בארכיון הבינארי:

fish -c "grep -o '[0-9]\+' archive.tar"
```

יציאה:
```
123
456
789
```

אם אנחנו רוצים לסנוור במחרוזת כלשהי, ניתן להשתמש בפקודת `sed` עם האפשרות `-e` כדי להשתמש בתוספת. לדוגמה:

```
# סנוור את כל הסיסמאות בקובץ shadow:

fish -c "sed -e 's/:\([^:]\+\):\([^:]\+\)/:\1:\2:********/g' /etc/shadow"
```

יציאה:
```
root:********:********:********:********:********:********:********
bin:********:********:********:********:********:********:********
daemon:********:********:********:********:********:********:********
```

לחילופין, אם אנחנו רוצים להחליף ערכים בטקסט, אנחנו יכולים להשתמש בפקודת `sed` עם האפשרות `-i` כדי לשנות את הקובץ במקום להדפיס פלט. לדוגמה:

```
# החלפת כל המילים "מילה" ב "מילת מלחמה" בתיקיית הנוכחית:

fish -c "sed -i 's/מילה/מילת מלחמה/g' ./*"
```
