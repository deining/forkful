## למה

קוד יכול להיות מבלבל ומתיש במובנים שונים. לכן, הקפצות פרינט יכולות לתת לנו מידע חיוני על הקוד שלנו ולעזור לנו לפתור בעיות באופן יעיל יותר.

## איך לעשות זאת

תכירו את פיתרונות ה debug output שמופיעים בפיש שלך. הם תוכנית משתמשת בפקודות השורת פקודה 'set' כדי לעזור לך לפתח את הקוד שלך יותר טוב. כאן תוכלו למצוא מספר דוגמאות של הפקודות 'set' והפלט שלהן.

```fish
# פקודת set -x מציגה את הפקודה ה/ה אחרונה שהוצלה
function foo
  echo "This is a sample function"
end

set -x foo

# פקודת set -n מראה רק את הפקודה שהוגדרה, אך לא מבצעת אותה
function bar
  echo "This is a sample function"
end

set -n bar
set -n
```

### פלט:

```
+ foo
+ echo "This is a sample function"
This is a sample function
bar
```

הפלט שנראה לעיל מראה את הכנסה של שתי פקודות 'set' עם הארגומנטים '-x' ו'-n' בהתאמה. אם נפעיל פקודות משתנות נוספות, הפלט יהיה עדין יותר טוב.

## יריעה עמוקה

פקודת 'set' מספקת מספר אפשרויות לשימוש בזמני Debugging -x. למשל, נוכל להיות מוצלבים כך שלא נראה את כל הפלט, אלא רק את השורות שידועות כבעייתיות. באופן דומה, אנו גם יכולים להשתמש בפקודות ה set -n כדי להימנע מביצוע של הפקודה עצמה עם מטרה של מבצע תקנות ללא הקידול החכם. לא רק כך, אבל עם כמה אופציות יותר, אנו גם יכולים לכוון את הפלט הנכון שאותו אנו עובדים מול.
