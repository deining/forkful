## למה

בכתיבת תכניות מורכבות, חשוב לבדוק את התכנים כדי לוודא שהם עובדים כפי שצפוי מבחינת המפתחים. ניתן לכתוב בדיקות לשפת פיש שלך כדי לוודא שהתכניות שלך מתנהלות כראוי.

## כיצד לעשות זאת

כתיבת בדיקות בפיש היא פשוטה וקלה. השתמשו בדוגמאות הבאות כדי ללמוד כיצד לכתוב בדיקות בפיש שלכם:

```Fish Shell
# ייבוא הפונקציה לבדיקות
.דרוס בדיקות.fish

# פונקציית לבדיקה
function test_example
    # הגדרת משתנה לבדיקה
    set result "זה דוגמה"

    # בדיקת התנאי
    if test "$result" = "זה דוגמה"
        # הדפסת הפלט המתאים
        echo "מצויין, הבדיקה עברה!"

        # יציאה עם קוד ייצוא 0 - הצלחה
        set -e 0
    else
        # הדפסת הפלט המתאים במקרה שהבדיקה לא עברה
        echo "אוי לא, הבדיקה נכשלה!"

        # יציאה עם קוד ייצוא 1 - כישלון
        set -e 1
    end
end
```

על מנת להריץ תכנית זו כבדקת בפיש, יש להריץ את הפונקציה באמצעות הפקודה ".דרוס בדיקות.fish" ולקרוא "test_example". כאן להלן ייצא הפלט המתאים:

```Fish Shell
# הפלט שציפיתם לקבל
מצויין, הבדיקה עברה!

# יציאת קוד 0 מציינת הצלחה
```
