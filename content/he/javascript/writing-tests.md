## למה

כתיבת בדיקות היא תהליך חשוב וקריטי בתכנות ג'אווהסקריפט. הבדיקות מאפשרות לנו לוודא את תקינות הקוד שלנו ולזהות שגיאות ובאגים בקלות ובמהירות. הן נותנות לנו ניסיון וביטחון כשאנחנו מספקים קוד למשתמשים וכשאנחנו מבצעים שינויים בקוד קיים. בנוסף, כתיבת בדיקות מפחיתה את זמן הבדיקה והתיקונים בעת העבודה על קוד גדול ומורכב.

## איך לעשות זאת

כתיבת בדיקות בג'אווהסקריפט נעשית באמצעות כלי נבחר, כגון Jest או Mocha. ניתן להתחיל על ידי כתיבת פונקציות קטנות המבצעות פעולות פשוטות ולבדוק את תוצאותיהן. לדוגמה:

```Javascript
function add(x, y) {
  return x + y;
}

console.log(add(2, 3));
// output: 5
```

כשנדרוס את הפונקציה כדי להוסיף ייחודים ובדיקות שונות, ניתן להוסיף טסט לבדיקת הפונקציה:

```Javascript
test("adds 2 and 3 to equal 5", () => {
  expect(add(2, 3)).toBe(5);
});
```

כעת, כשנריץ את הטסט, הוא יציג את התוצאה הצפויה ואת תוצאת הפונקציה האמיתית. ניתן להוסיף גם טסטים למצבים מיוחדים ופניהם על ידי שימוש בפונקציות כמו beforeEach ו afterEach.

## Deep Dive

כשאנחנו כותבים טסטים, חשוב להדגיש את הדברים הבאים:

- כתיבת טסטים כבר בשלב התכנון ולא במהלך הפיתוח עצמו. כך נוכל להפחית קוד באגים ובעיות תקינות מוקדמות.
- הימנעות מטסטים קשוחים ומכוערים שיכולים
