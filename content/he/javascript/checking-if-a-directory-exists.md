## למה 

בברוקרמאתיש הנפוץ לכתוב אפליקציות ווב עבור מכשירים ניידים, מסווגת את התיקייה שבה מכיל הקובץ הבינארי לאפליקציה בתור תיקיית משאבים. כך, תיאור המכשיר הנייד הולך למקבל מסימון לתיקייה זו לפני שולנשתיוםשלתוכן טקסט כותרת האר. בדיקה של התיקייה הנוכחית כדי לדעת אם האפליקציה יכול לגשת אליה היא כלל נחוץ כדי לוודא פעמים רבות לפני שורה.

## איך לעשות את זה

דוגמאות כיצד לבדוק את התיקייה הנוכחית באמצעות קוד ג'אווהסקריפט:

```Javascript
if (fs.existsSync(path)) { 
    console.log('התיקייה קיימת'); 
} else { 
    console.log('התיקייה לא קיימת'); 
}
```

טקסט קוד המציג את הפלט של התנאי הנכון, אם התיקייה הנוכחית קיימת, והפלט השגוי אם היא לא קיימת.

ישנם גם אפשרויות לשלוט במצב פעיל רבים המתאימים לבדיקת אם גישה לתיקייה חלק מחל האפליקציה. מעדכן גם לוודא שהתיקייה המבוקשת קיימת ואינה מדגמה כדי למנוע ממיקה טעיות ובעיות באתר.

## מעלים מידי

בדיקת אם תיקייה קיימת היא חשובה לא רק במסגרת כתיבת קוד עדכני וטוב. כאשר מפתחים חשוב לראות שאין תיקיות חסרות כדי לא לפגוע בחלק ממכשירים ו, לאחר לבדוק שהתיקיות חסרות הן, לבדוק כל תיקייה אחת אם זהו שינוי של עמוד חלק או אם הוא עדיי לתיקיה חוקית.
