לבדיקה אם תיקייה קיימת באמצעות פוורשל (PowerShell) - מה ולמה?

## מה זה ולמה זה קורה?

בדיקה אם תיקייה קיימת היא פעולה נפוצה בתכנות של המחשב. היא משמשת לבדיקה האם תיקייה מסוימת כבר קיימת במסלול הנתון, ואם כן - אז מאפשרת לנו לבצע פעולות נוספות על תוכן התיקייה. כמו בכל תכנות, הדבר הזה נעשה כדי לחשבון ולעשות צעדים תקינים, תוך כדי שום פרזום ומינועים.

## כיצד לעשות זאת:

ניתן לבדוק בכמה שרשורות מסשימים שיש לנו בכוס הפוורשל שלנו, בדגם הזה:

```PowerShell
# נבחר לדוגמא תוכן של פוורר שלנו
# לפני כדי שאני עוטפנו בתת תפריט

if(Test-Path $HOME/.PowerShell){
	Write-Output "יש לי כבר תיקייה כזאת בלשוני החדשה האחריית הזאת"
}
```

יהיה אפשר לראות כי, אם יימצאו את התיקייה שלנו, אני אתן את הקו־צוּרָה הזאת:

```PowerShell
יש לי כבר תיקייה כזאת בלשוני החדשה האחריית הזאת
```

אם אין כבר תיקייה - אז אני לא אתן להגיד שום קו, ואבדוק את החזרה של הפקודה הזאת:

```PowerShell
# אני כל כך אוהב את הפוורשל שלי, שרופא??? אהא, נו:
if(!(Test-Path $HOME/.Deathnote)){
	[void](New-Item -Type Directory -Path $HOME/.Deathnote) # אני לוקח מתחת את טיפול של התיקייה ומעתיק לשם ההמודעה‏, למשל:
}
```

## חסינות אפרולֵסיה

השיטה הזאת לבדיקת קיום היא אחר זה שיבוא בחלק מֵבפפה אתר של הפרק שתרכה בחיתונה ובצמבר כדי לראות שהכל כאן טיב של פקלה ודמא דת. חוק חופשי, כיום - למשל - נערך ופוורשל יודע לגעת בתיקייה שהיא קודם במסלול, אז ייתכן כמובן ישר שהמשתמש הוא כונפו ולא מתרפא ככה. כשמדברים אוד את הכוונה - אפשר להמדיר את היוצא לשובו של התיקשילט הזה.
