# למה

זהו סעיף ראשון וחשוב בכתיבת בלוג עבור כל מתכנת ג'אווה. למה לנו לעסוק בכתיבת בדיקות? ולמה לשקול לכלול אותן בתהליך הכיבוד שלנו? ניתן להגיד ללא ספק שכתיבת בדיקות היא חלק חשוב וקריטי בפיתוח תוכניות. המטרה המרכזית של בדיקות היא לוודא שהתוכנה שאנו מפתחים עומדת בדרישות הפונקציונליות והתקינות שקובענו לה.

# איך לכתוב בדיקות

כתיבת בדיקות אינה קשה ויכולה להיות כיף גם לכל מתכנת ג'אווה. כדי לבצע כתיבת בדיקות אנו נשתמש בכמה מרכיבים בסיסיים של השפה. נהליך הכתיבה מתחיל עם יצירת מחלקות מתנהגות, שמושג חשוב ביכולת הטסטים לקרות אם התוכניות שלנו עובדות כראוי. הקובץ הבא מכיל דוגמה למחלקה כזאת, שהשם שלה הוא `Calculator`.

```Java
public class Calculator {
   private double result;
   
   public void add(double num1, double num2) {
      result = num1 + num2;
   }
   
   public double getResult() {
      return result;
   }
}

```

כעת, כדי לבדוק את תפקידה של `Calculator`, אנו נכתוב בדיקה פשוטה שתוודא אם התוצאה שהקבלנו בעזרת הפונקציה `getResult()` היא באמת תוצאת החיבור של המספרים שלנו.

```Java
public class CalculatorTest {
   private Calculator calculator;

   @Before
   public void createCalculator() {
      calculator = new Calculator();
   }

   @Test
   public void testAdd() {
      calculator.add(2, 3);
      assertEquals(calculator.getResult(), 5, 0);
   }
}
```

בדיקות יכולות להשתמש גם בפונקציות נוספות לא רק יצירת תוצאות, אלא גם בבדיקת תנאים ותכנים. בכך אנו יוצרים פתרונות בדיקה עבור כל חלק בתוכנית שלנו שאנחנו רוצים לוודא.
