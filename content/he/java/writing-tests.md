## Why - מדוע

ככל שתכתבו את הקוד שלכם, היכולת לבדוק את תקינותו ולזהות באגים עלולה להיות קשה יותר. בעזרת כתיבת בדיקות, תוכלו לוודא שהקוד עובד כמצופה ולמנוע באגים פוטנציאליים בתהליך הפיתוח.

## How To - איך

כאשר מתחילים לכתוב בדיקות בג'אווה (Java), ישנם שני סוגים שונים של בדיקות: בדיקות התיעוד ובדיקות היחידה. הבדיקות התיעוד מתמקדות בתיקים וכתוביות, במקום טקסט ממומש, ומשמשות כשכבת אימות נוספת כדי לוודא שהיצירה עובדת כצופה. לדוגמה:

```Java
// בדיקת הודעות

String message = "שלום!";
if (message.equals("שלום!")) {
  System.out.println("הבדיקה עברה בהצלחה");
}
else {
  System.out.println("הבדיקה נכשלה");
}
```

הבדיקות היחידה, מצד שני, מתבצעות על מתודות ופונקציות ספציפיות ומשמשות לוודא שהן מחזירות את הערך הרצוי. קודם כל, יש צורך ביצירת מופע של המחלקה או הממשק הנבדקים ולכן נשתמש בשפת התכנות Mockito ליצירת מופע מזויף של הממשק הנבדק. לדוגמה:

```Java
// בדיקת מחרוזות

String firstString = "כתיבת";
String secondString = "בדיקות";
String thirdString = "היא כיף!";

assertEquals("כתיבת בדיקות היא כיף!", firstString + secondString + thirdString);
```
