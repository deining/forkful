# למה

כשמתכנתים כותבים קוד, משהו בודק את התכנית ומציין אם יש בעיות. במקרים רבים, המצביע מדווח על השגיאות בעזרת כותרת השגיאה ומספר המשתמש. אך לעתים קרובות כדי להבין בדיוק מה גרם להודעת השגיאה, נדרש מידע נוסף. במאמר זה נדבר על השימוש בפקודה הפשוטה "תקן_תחתון" כדי להדפיס אינפורמציה נוספת על התכנית שאתם מפתחים. זה יכול לעזור לך לאתר את נקודת השגיאה ולתקן אותה בקלות.

# כיצד לעשות זאת

בכדי להדפיס לכיתוב התקן, יש להשתמש בפקודה "תקן_תחתון". ניתן להשתמש בה בכל מקום בקוד שברצונך להדפיס את ההודעה. כאשר התכנית מגיעה לפקודה זו, תוכן התכנית יודפס לכיתוב התקן ותוכל לראות איזה שורה ובאיזו קובץ נמצאת הבעיה. זה יכול להקל על המציאת השגיאה ותיקון התכנית.

לדוגמה, נניח שיש לך קובץ קוד בשם "myProgram.java" שמחשב ממוצע עבור מספרים שנכתבו בתוך גורם "args". אתה משתעשע בשפת ג'אווה ורוצה להוסיף קטע קוד דוגמה בסוף הקובץ. כשהתכנית מכילה שגיאה, היא עשויה להיראות כך:

```java
public class myProgram {
   public static void main(String[] args) {
      int sum = 0;
      for (int i = 0; i < args.length; i++) {
         sum += args[i];
      }
      int average = sum / args.length;
      System.out.println("The average is: " + average);
   }
}
```
