## למה
מודעתנות נכונה היא החצי השני של תכונות תכנות מוצלחות. דבג אאוטפוט הוא אחד הכלים הכי שימושיים לכתיבה ובדיקה של תכניות תכנות. במאמר זה נלמד כיצד להשתמש בדבג אאוטפוט כדי לזהות בעיות ולקבל מידע נוסף מצטרפת בעת פיתוח תכניות תכנות.

## כיצד להשתמש
ברוב המקרים, דבג אאוטפוט מייצא מידע בפורמט של מחרוזת. שימוש ביסודות יספק לנו את היכולת להדפיס את המחרוזת הזאת למסך. לדוגמה, כאשר אנו פותחים תכנית ראשונית ורצים אותה:

```Java
public class Main {
  public static void main(String[] args) {
    System.out.println("זהו דבג אאוטפוט ראשון שלי!");
  }
}
```

#### פלט:
זהו דבג אאוטפוט ראשון שלי!

ניתן גם לבצע הדפסת משתנים ופרמטרים באמצעות הצבת המחרוזות בין תווי משתנה.

```Java
public class Main {
  public static void main(String[] args) {
    int age = 25;
    System.out.println("הגיל שלי הוא: " + age);
  }
}
```

#### פלט:
הגיל שלי הוא: 25

כמו כן, ניתן להשתמש בדבג אאוטפוט גם בפונקציות וללולאות כדי לקבל מידע נוסף על הזרימה של הקוד. אנו יכולים להדפיס את ערכי המשתנים והפרמטרים ברחבי הקוד כדי ללמוד איפה בעיות את המשתמש הופעלו.
