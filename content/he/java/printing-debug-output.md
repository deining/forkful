תעלולי קידוד באמצעות יצוא פלט פיתוח בקונסולה (## Why)

מדוע שווה למתכנתים להכניס לבדקה פולטת בעת פיתוח? תשובה פשוטה - כי על ידי הדפסת שגיאות ותקלות בקונסולה, תוכלו לזהות ולתקן בעיות בקוד במהירות וביעילות רבה יותר. כשאתם מפעילים את הקוד ורואים תקלה, אתם יכולים לדרוס אותה כדי שתקבלו פלט מידי יותר.

איך להשתמש בדרכי פלט פיתוח (## How To)

אם אתם מתחילים לעבוד עם יצוא פלט, הנה דוגמה פשוטה לכיצד תוכלו להשתמש בזה ב-Java:

```Java
public class PrintDebugOutput {

    public static void main(String[] args) {
        System.out.println("מתחילים לבדוק פיתוח בקונסולה!");
        // כאן מופיעות שורות קוד נוספות על מנת לבדוק פיתוח
        System.out.println("צריך לתקן תקלות!");
    }
}
```

וזה יהיה הפלט המתאר את הבעיה:

תוצאות:

```Java
מתחילים לבדוק פיתוח בקונסולה!
צריך לתקן תקלות!
```

בקוד הזה, אנחנו משתמשים בפקודת `System.out.println()` כדי להדפיס את הטקסט שאנחנו רוצים. זו דרך פשוטה ויעילה להדפיס פלט פיתוח במהלך תהליך הפיתוח שלכם.
