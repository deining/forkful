(Hebrew Translation)

## מדוע
כדי להבין בדיוק למה כתיבת בדיקות היא חשובה, חשוב להבין את הערך שבהכנסה של כתיבת קוד וכתיבת בודקים שמתלווים לו. בדיקות מאפשרות לנו לוודא שהקוד שכתבנו עושה את מה שצפינו שהוא יעשה ושהוא אינו מפריע לקוד הקיים.

## כיצד לעשות זאת
```Elixir
defmodule MyModule do
  def add(x, y) do
    x + y
  end
end

defmodule MyModuleTest do
  use ExUnit.Case
    
  test "adding numbers" do
    assert MyModule.add(1, 2) == 3
  end
end
```

בדיוק כמו שאנו כותבים קוד לפונקציות שלנו, אנו יכולים לכתוב קטעי בדיקה כדי לוודא שהקוד עובד כמצופה. בקוד המופיע למעלה, אנו יוצרים מודול חדש של MyModule ומשתמשים בו בתוך MyModuleTest כדי לבדוק את הפונקציה add ולוודא שהיא מחשבת נכון. בדיקות מאפשרות לנו לתת משוב מהיר וקל כשאנו משנים את הפונקציות הקיימות או מוסיפים פונקציות חדשות.

## חפירה עמוקה
כתיבת בדיקות היא לא רק על זה לבדוק האם התוכנית שלנו עובדת כמצופה. בנוסף לכך, ישנם כמה טכניקות וכלים שניתן להשתמש בהם כדי לשפר את הבדיקות שלנו, כולל מודולים שחוזרים באופן אוטומטי, מודלי Fakes ועוד. חשוב ללמוד את היתרונות והאפשרויות המגוונות של כתיבת בדיקות סביבות אליקסיר.
