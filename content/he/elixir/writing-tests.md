## למה

בניגוד למה שהרבה אנשים חושבים, כתיבת בדיקות היא לא רק כדי להבטיח שהקוד עובד כמו שצריך. כתיבת בדיקות מביאה לקוד נקי יותר, יותר קל לניתוח ותחזוקה, ומסייעת למתכנתים להבין את הקוד שלהם בצורה טובה יותר.

## איך לעשות זאת

כדי לספק בדיקות מקיפות, עלינו לעבוד עם כמה כלים שמאפשרים לנו לבדוק את הקוד שלנו בצורה מכוונת יותר. נדגים כמה דרכים לכתיבת בדיקות ב-Elixir עם דוגמאות קוד ותוצאות.

במקרה הבא, ננסה לבדוק את פונקצית החיבור הפשוטה הבאה:

```Elixir
defmodule Calculator do
  def add(x, y) do
    x + y
  end
end
```

כדי לכתוב טסט לפונקציה זו, נוסיף פונקציה נוספת בשם "add_test" למודול שלנו:

```Elixir
defmodule CalculatorTest do
  use ExUnit.Case

  test "adding two numbers" do
    assert Calculator.add(2, 4) == 6
  end
end
```

הרץ את הטסט עם הפקודה "mix test" ותאמת שהוא עבר בהצלחה.

## חקר עמוק

בכתיבת בדיקות, כדאי לנו להיות תמיד מדווחים ולוודא שכל מקרה אפשרי מכוסה בטסט. למרבה המזל, ב-Elixir ישנם כמה כלים המסייעים לנו בכתיבת בדיקות בצורה נוחה יותר. נספר על כמה מהם:

- ExUnit: פריימוורק רישמי שמגיע עם Elixir ומאפשר לנו לכתוב טסטים בצורה מתוקשבת יותר.
- doctest: כלי שנועד לבדוק תיעוד של קוד בצורה מהירה ונוחה.
- meck: כלי שמאפשר לנו לחייב מוקים בטסטים כדי לבדוק פונקציות בלי תלות בקוד אחר.
