## なぜ標準エラーに書き込むのか

プログラムを実行している時にエラーが発生すると、プログラムは異常終了してしまいます。しかし、そのエラーはどこで起きたのかを知ることができない場合もあります。ここで標準エラーに書き込むことで、プログラムが正常に実行された場合には標準出力に、エラーが起きた場合には標準エラーにそれぞれ結果を出力することができます。これにより、エラーが起きた時にどこで起きたかを特定することができます。

## 標準エラーへの書き込み方

では、実際にどのように標準エラーに書き込むのかを見ていきましょう。以下のコードは、エラーが起きた場合に標準エラーにメッセージを出力する例です。

```C
#include <stdio.h>

int main(void)
{
    fprintf(stderr, "エラーが発生しました！\n");
    return 0;
}
```

上記のコードを実行すると、標準エラーに「エラーが発生しました！」というメッセージが出力されます。また、プログラムが正常に実行された場合には、標準出力には何も出力されません。

## 標準エラーへの深いダイブ

標準エラーに書き込むことで、プログラムのデバッグをより効率的に行うことができます。標準エラーにはエラーメッセージ以外にも、変数の値や実行時の情報を出力することもできます。また、標準エラーに書き込むことで、エラー情報をログファイルに保存することも可能です。

## では、次はどこへ

これで、標準エラーについての基礎知識を学びましたね。もしもプログラムでエラーが起きたときには、標準エラーにメッセージを出力してみてください。きっとデバッグがよりスムーズになることでしょう。
