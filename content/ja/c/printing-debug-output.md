## 何をして、なぜ？(What & Why?)
デバッグ出力（debug output）とは何か、プログラマーがそれを行う理由はなにかを説明します。

デバッグ出力とは、プログラムの実行時にコンソールやログファイルに表示することです。プログラムの実行中に何が起きているかを確認するため、デバッガーを使わずにプログラムの状態を把握するために行われます。

プログラマーはデバッグ出力を使用することで、プログラムの実行中にデータの値や特定のコードが実行されたかどうかを確認することができます。また、バグの原因を特定するのにも役立ちます。そのため、プログラマーがプログラムのバグを見つける際に欠かせないツールとなっています。

## 方法 (How to:)
下記のC言語のコードブロック内に、デバッグ出力を行う方法を示します。実行して得られる出力も一緒に記載しています。

```C
#include <stdio.h>

int main(void) {
  int num = 10;
  printf("デバッグ出力の例：%d", num);
  return 0;
}

// 出力：
// デバッグ出力の例：10
```

この例では、`printf()`を使用してデバッグ出力を行っています。`%d`という書式指定子を使用することで、`num`という変数の値を出力することができます。

デバッグ出力は、実行中の任意の箇所で行うことができますので、必要に応じて適宜コードを追加することでデバッグ出力を行うことができます。

## もっと深く (Deep Dive)
デバッグ出力はプログラミングにおいて必須のツールとなっていますが、もともとは実装の際のデバッグを助けるために使用されていました。デバッグ出力の代わりにデバッガーを使用する方法もありますが、デバッグ出力はプログラムの動作をリアルタイムに確認することができるため、依然多くのプログラマーに愛用されています。

また、デバッグ出力を使用する際には注意が必要です。あまりにも多くのデバッグ出力を行うと、プログラムの実行速度が低下する場合があります。そのため、デバッグ出力を使用する際には必要最小限の出力を行うよう心がけましょう。
