# なぜテストを書くのか

テストを書くことは、ソフトウェア開発者にとって重要なスキルです。テストは、コードの品質を保証し、バグを早期に発見することができるため、プロジェクトの成功に不可欠です。また、テストを書くことで開発プロセスがスムーズに進み、バグを修正するために必要な時間を減らすことができます。ここでは、TypeScriptにおけるテストの書き方について説明します。

## テストの書き方

テストは、開発の初期段階から最終段階まで、ソフトウェアの振る舞いを検証するために行われます。テストを書く際には、以下のような手順に従ってください。

まず、テストするファイルと同じディレクトリに「tests」というフォルダーを作成し、その中に「test.ts」というファイルを作成します。ここでは、テストするファイルを「Math.ts」とする例を示します。

```TypeScript
import { sum } from '../Math';

test('1 + 2 should equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});
```

上記のコードブロックでは、テストするファイルの関数をインポートし、テストの定義を行っています。ここでは、sum関数に対して引数として1と2を渡し、その結果が3になることを期待しています。テストを実行するには、コマンドラインで以下のコマンドを実行します。

```
npx jest tests/test.ts
```

このコマンドを実行すると、テストが実行され、以下のような結果が表示されます。

```
 PASS  tests/test.ts
  √ 1 + 2 should equal 3 (4ms)
```

テストがパスしたことを示す「PASS」のメッセージが表示されます。

これでテストの書き方の基本的な部分が把握できたと思います。次に、もう少し深くテストの書き方について説明します。

## ディープダイブ

テストの書き方にはさまざまな方法がありますが、ここでは一般的な方法として、Jestフレームワークを使用してテストを行う方法を紹介します。Jestは、TypeScriptをサポートしており、簡単にテストを書くことができます。また、デフォルトでカバレッジレポートを生成する機能も備えているため、テストを行うだけでコードのカバレッジも確認することができます。

テストを書く際には、ユニットテストや統合テストといった様々な種類のテストを行うことができます。ユニットテストでは、個々の関数やモジュールを単体でテストし、統合テストではそれらが組み合わさった際の動作をテストします。また、エンドツーエンドのテストでは、実際にアプリケーションを起動し、ユーザーの操作をシミュレートしてテストを行います。
