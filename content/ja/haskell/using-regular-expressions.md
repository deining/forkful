## なぜ正規表現を使うのか

正規表現は、文字列のパターンを特定するための強力なツールです。Haskellでは、正規表現を使うことで、文字列を効率的に処理することができます。

## 方法

まず、Haskellプロジェクトで正規表現を使うためには、`regex`モジュールをインポートする必要があります。

```Haskell
import Text.Regex
```

次に、正規表現を使って文字列パターンをマッチングする方法を見ていきましょう。

### マッチング

`=~`演算子を使って、文字列を正規表現にマッチングできます。

```Haskell
"hello world" =~ "l+" :: [[Char]]
-- ["ll"]
```

上記の例では、`l+`という正規表現にマッチする`ll`が返されます。また、`:: [[Char]]`として型注釈を行うことで、返される値の型をリスト型に指定することができます。

### サブマッチング

正規表現のグループ化を使って、マッチした部分のみを取得することができます。

```Haskell
"hello world" =~ "(l+)" :: [[Char]]
-- [["ll"], ["l"]]
```

上記の例では、`"hello world"`という文字列の中から、`"l"`という部分文字列を取得しています。`[[Char]]`の型では、2重のリストになっていることに注意してください。

### 置換

正規表現を使って、文字列中の特定の文字列を置換することができます。`subRegex`関数を使います。

```Haskell
subRegex (mkRegex "l+") "hello world" "e"
-- "heo world"
```

上記の例では、`"l+"`という正規表現にマッチする`l`を`e`に置換しています。
