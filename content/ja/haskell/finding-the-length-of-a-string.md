"## なぜ" 

文字列の長さを求めることに関心を持つべき理由をご紹介します。

文字列の長さを求めることは、プログラミングにおいて非常に一般的なタスクです。例えば、ユーザーからの入力を受け取った際に、その入力が指定した制限の長さを超えないかどうかを判断するために活用されます。また、文字列の長さを計算することにより、データベースやファイルなど、様々なソースから取得したデータを処理する際にも役立ちます。

"## 使い方" 

文字列の長さを計算するために、我々はHaskellの内蔵関数である`length`関数を使うことができます。この関数は、文字列を引数として受け取り、その長さを返します。例を示します。

```Haskell
length "こんにちは、世界" 
```

このコードを実行すると、文字列"こんにちは、世界"の長さである11が出力されます。

Haskellでは文字列は文字のリストとして扱われるため、`length`関数は文字列以外にもリストを引数として受け取ることができます。例えば、以下のように数値のリストを引数として渡すこともできます。

```Haskell
length [1, 2, 3, 4, 5] 
```

この場合、出力は5となります。リスト内の要素数が返されるため、文字列以外でも`length`関数は活用できることがわかりますね。

"## 深堀り" 

Haskellでは文字列の長さを求めるために`length`関数を使用することができますが、実際にこの関数がどのように動作しているか知ることも重要です。

`length`関数の定義は以下の通りです。

```Haskell
length :: Foldable t => t a -> Int 
```

この定義からわかるように、`length`関数は`Foldable`クラスのインスタンスである任意の型tを受け取り、整数型の値を返すということがわかります。また、関数の引数である`t a`は`Foldable`クラスのインスタンスである任意の型aを含むリストであることもわかります。

つまり、`length`関数はリスト内の要素数を数えるために、リストを走査するという処理を実行していることがわかります。
