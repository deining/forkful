## なぜ？

標準エラー出力に書き込むことは、デバッグやエラートラップに非常に便利です。エラーメッセージを書き出さないと、バグを修正するのが非常に難しくなります。そのため、Fish Shellを使用している場合は、標準エラー出力に書き込むことをお勧めします。

## 方法

まず、Fish Shellのターミナルで```echo```コマンドを使用して、標準出力と標準エラー出力を比較できるようにします。

```Fish Shell
echo "標準出力"
```
```
標準出力
```

次に、標準エラー出力に書き込むために、```&```を使用します。

```Fish Shell
echo "標準出力" >&2
```
```
(標準エラーコマンドが書き込まれたと仮定して)標準出力
```

これにより、```&```を使用して、標準エラー出力に文字列が書き込まれました。

## 深堀り

標準エラー出力に書き込む場合、エラーメッセージがスクリーンに表示されるだけでなく、ファイルに書き込むこともできます。これにより、後でエラーファイルを参照して問題箇所を確認することができます。

また、標準エラー出力には、標準出力と同じリダイレクト機能も利用することができます。例えば、```ls```コマンドを使用してファイルが存在するかどうかを確認する場合、存在しない場合は標準エラー出力にメッセージを書き込むように設定できます。

```Fish Shell
ls file.txt || echo "ファイルが存在しません" >&2
```

このように、標準エラー出力を活用することで、より効率的なデバッグやエラートラップが可能になります。

## 他の例

標準エラー出力に文字列を書き込む方法だけでなく、エラーの発生をシステムログに書き込むこともできます。また、Fish Shellでは、エラーメッセージをカラフルに表示することも可能です。詳細は公式ドキュメントをご確認ください。
