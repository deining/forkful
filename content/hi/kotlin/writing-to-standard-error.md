## क्यों

अभी प्रोग्रामिंग में सब कुछ अपरिचित हो सकता है, इसलिए किसी-न-किसी समय आपको आपके कोड में गलतिया आ गई होगी । हम अपने कोड को दुगुना जांचते हैं और फिर भी समस्या निकलती है तो हम परेशान हो जाते हैं। लेकिन इसलिए हार नहीं माननी चाहिए। हम इसमें आसानी से सगावट कर सकते हैं जिसे हम *standard error* कहते हैं।

## कैसे करें

कोट्लिन कम्पाइलर के साथ, हम ```System.err.println()``` का प्रयोग कर सकते हैं जो Standard Error पर बात करता है। इसमें हमें अपने कोड को दुगुना जांचने के लिए एक एकल निर्देशिका देने का फायदा मिलेगा। यह निम्नलिखित कोड को इस्तेमाल कर सकते हैं:

```Kotlin
fun main() {
    val number = 10
    if (number > 5) {
        System.err.println("यहां कुछ गलत हो रहा है!")
    }
}
```

अब जब हम इस कोड को कंपाइल और चलाएंगे, तो हमें कुछ इस तरह का आउटपुट मिलेगा:

```
यहां कुछ गलत हो रहा है!
```

## गहराई में देखें

स्ट्यांडर्ड एरर को समझने के लिए, हमें पहले समझना होगा कि कंपाइलर कैसे काम करता है। कंपाइलर कोड को नए फ़ाइल में विभिन्न लिंटर्स और और पेशेवर कोड चेक के साथ निष्पादित करता है। एक बार यह सफलतापूर्वक होता है तो, सम्मानित उत्पाद को सर्वर पर अलग रखा जाता है जो उत्पाद के बाद योग्य रूप से प्रसारित होता है। कई लोग अपने कोड में *standard error* के बारे में सोचते हैं कि उनको
