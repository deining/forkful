## क्यों

क्या आपने अपने कोटलिन प्रोग्राम में सीमांकन टाइप के आदेशों का उपयोग किया है? अगर हाँ, तो शायद आपने इन आदेशों को पाठ्यक्रम चर तर्कों के माध्यम से पारस्परिक श्रेणीबद्ध तरीकों से पढ़ा होगा। हालांकि, यदि आप यह नहीं कर पाए हैं तो आपके लिए ध्यान में रखने योग्य एक चीज है कि कैसे कमांड लाइन आपके कोड में आते हैं। यह आमतौर पर एक अन्यमक चर तरीके से असेट बतायाजाता है जो अपनी अपनी स्थति की रखा है जैसे (य़ह टारगेट कर जानकर देखें। प्राय: जैसा दिया गया है), इस आदेश को जार किएगा।

## कैसे

क्या आपको पता है कि कैसे कमांड लाइन आपके कोड को ग्रहण करता है? अगर नहीं, तो चिंता मत करें! हमने आपके लिए एक सरल कोड कोडिंग उदाहरण दिया है जो आपको इस प्रश्न का उत्तर देगा:

```Kotlin
fun main(args: Array<String>) {
    println("प्रारंभिक कमांड लाइन आदेश: " + args[0])
    for (i in 1 until args.size) {
        print("अतिरिक्त कमांड लाइन आदेश: ${args[i]}")
    }
}

```

जब आप इस कोड को चलाते हैं, तो आप निम्नलिखित आउटपुट देखेंगे:

```
प्रारंभिक कमांड लाइन आदेश: टॉपिक
अतिरिक्त कमांड लाइन आदेश: आधयूमी
अतिरिक्त कमांड लाइन आदेश: कॉन्फ़िगरेशन
अतिरिक्त कमांड लाइन आदेश: JavaConfig.kt
```
