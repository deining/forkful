## क्यों

गुरुत्वाकर्षण से सुलभता और प्रतिगम्भवता के साथ कोट्लिन आज कई डेवलपर्स के पसंदीदा भाषा है। इसमें छोटा संकलन, उन्नत और मज़ेदार सिंटैक्स और जंगली कार्यालय के साथ आसान भाषा फीचर्स शामिल हैं। यह जानना आवश्यक होता है कि टेस्टों को लिखना क्यों महत्वपूर्ण है और इसके क्या फायदे हैं।

## कैसे करें

टेस्टों को लिखने के लिए क्यों, हम स्ट्रिंग समकलन, लिस्ट्स, मैप्स और अन्य मूल्य को प्रस्तुत कर सकते हैं। क्योंकि वे आपके कोड को परीक्षण करते हैं और सुनिश्चित करते हैं कि आपका कोड अपेक्षित रूप से काम करता है। वहाँ हमारे पास विभिन्न तररा के टेस्ट हैं, जिन्हें हम इस लेख के साथ अन्य उदाहरणों के साथ प्रदर्शित करेंगे।

```Kotlin
// सैंपल फ़ंक्शन
fun addNumbers(a: Int, b: Int): Int {
    return a + b
}

// सैंपल टेस्ट केस
assert(addNumbers(5, 2) == 7)

// सैंपल थ्रो एक्सेप्शन टेस्ट केस
assertThrows<IllegalArgumentException> {
    addNumbers(5, "two")
}
```

ऊपर के कोड ब्लॉक में, हमने एक सैंपल फ़ंक्शन और उसके साथ कुछ टेस्ट केस दिखाए हैं। हमने `assert` फ़ंक्शन का उपयोग किया है जो हमारे आउटपुट को चेक करता है और गलत होने पर असत्यता उत्पन्न करता है। अंत में, हमने एक `assertThrows` फ़ंक्शन भी दिखाया है जो एक अप्रत्याशित एक्सेप्शन को हाथ में पकड़ता है।
