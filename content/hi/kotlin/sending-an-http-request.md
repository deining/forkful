What & Why?:

HTTP अनुरोध भेजना क्या है? HTTP (Hypertext Transfer Protocol) का उपयोग वेब और अन्य इंटरनेट संसाधनों को एक सर्वर से दूसरे सर्वर के बीच संवाद स्थापित करने के लिए दूसरी सर्वर से अनुरोध करने के लिए किया जाता है। जब हम एक वेबसाइट पर जाते हैं, तो हम अस्थायी रूप से एक HTTP अनुरोध को भेजते हैं और स्वरूप के अनुसार जवाब प्राप्त करते हैं।

प्रोग्रामर्स क्यों HTTP अनुरोध भेजते हैं? प्रोग्रामर्स HTTP अनुरोध को अपने एप्लिकेशन या वेबसाइट के साथ संवाद स्थापित करने के लिए भेजते हैं। यह सरल होता है क्योंकि HTTP अनुरोध का संरचना स्वरूप से निर्देशित होता है और यह दूसरे सर्वर से डेटा का आदान-प्रदान करने के लिए उपयोग किया जाता है।

How to:

```Kotlin
// HTTP अनुरोध भेजने के लिए Hurl का उपयोग करें।
val url = "https://example.com/api"
val client = OkHttpClient()

val request = Request.Builder()
    .url(url)
    .get()
    .build()

// अनुरोध प्रस्तुत करें और उत्तर लें।
val response = client.newCall(request).execute()
val responseBody = response.body()?.string()
println(responseBody)
```

Output:

```Kotlin
"Hello World!"
```

Deep Dive:

HTTP को जेम्स गोश्ट द्वारा 1991 में निर्मित किया गया था। यह थ्योरीटिकल प्रोटोकॉल वैश्विक विज्ञान संगठन द्वारा स्वीकृत और औद्योगिक तौर पर प्रयोग की अनुमति देने के लिए इंटरनेट इंजिनियरिंग टास्क फोर्स द्वारा स्पष्टीकृत किया गया था। वर्किंग ग्रुप मेटिंग के आधार पर XML-RPC प्रोटोकॉल के लिए प्रतिपादित प्रस्ताव के परिणाम के रूप में जेम्स गोश्ट द्वारा लिखे गए थे।

अन्य विकल्प शामिल हैं: 

1. वेब सॉकेट ब्राउज़र (WebSocket) - एक वेब प्रोटोकॉल जो दोनों पक्षों के बीच स्थायी समानांतर जोड़ बनाए रखता है जो दोनों पक्षों को एक आपसी रूप से संवाद स्थापित करने की अनुमति देता है। 
2. अपने स्वयं के HTTP बिल्डर को बनाएं - यदि आपके पास होमग्राउंड विवेक या कोटिन का मुख्यता निर्यात करने के लिए सक्षम होते हैं, तो आपको अपने एपीआई को धूर्त (druth) छोड़ देना चाहिए।
