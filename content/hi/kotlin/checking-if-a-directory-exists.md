## क्यों
कोई भी लोग प्रोग्रामिंग करते हैं तो वे अपनी फाइलों और फोल्डरों को अच्छी तरह से संगठित रखना चाहते हैं। इसलिए, जब हमें प्रोग्रामिंग में एक नया फाइल बनाना होता है तो हमें पहले से मौजूद फोल्डर या डायरेक्टरी को चेक करना पड़ता है कि वह मौजूद है या नहीं। इससे हमें गलतियों से बचने में मदद मिलती है और हम सही संरचना में कोडिंग कर सकते हैं।

## कैसे करें
```Kotlin
val directory = File("फोल्डर का पथ")
if (directory.exists()) {
    // फोल्डर मौजूद है
} else {
    // फोल्डर नहीं मौजूद है
}
```
यहां, हमने सरल कोड में एक नया फाइल बनाने के लिए फोल्डर के मौजूद होने का जाँच किया है। आप इस संरचना को अपनी फाइलें और फोल्डरों के लिए भी इस्तेमाल कर सकते हैं। यदि फोल्डर मौजूद नहीं होता है, तो हम उसे बना सकते हैं और उसमें फाइल लिख सकते हैं।

## गहराई में जाएं
फोल्डर मौजूद होने का कोड बनाना आसान हो सकता है, लेकिन इसमें कुछ और भी गहराई हो सकती है। आमतौर पर, हम अपनी फाइल सिस्टम की सीमाओं के कारण फाइलें या फोल्डर नहीं बना सकते हैं। इसलिए, हमें पूर्व में ही फोल्डर का मौजूद होने का जाँच करना आवश्यक हो सकता है। इसके अलावा, कुछ ऐसे विशेष प्रकार के फोल्डर हो सकते हैं जिनमें हमारे सामान्य जाँच
