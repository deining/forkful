## क्या और क्यों?
बेसिक प्रमाणीकरण के साथ HTTP अनुरोध भेजना क्या है और प्रोग्रामर इसे क्यों करते हैं।

HTTP अनुरोध एक प्रोटोकॉल है जो वेब सर्वर से डेटा को प्राप्त करने के लिए उपयोग किया जाता है। बेसिक प्रमाणीकरण HTTP अनुरोध में सुरक्षा सुनिश्चित करने के लिए उपयोग किया जाता है।

## कैसे करें:
```Fish Shell``` कोड ब्लॉक के भीतर कोडिंग उदाहरण और नमूना उत्पादन।

उदाहरण:
```fish
set -x username "john_doe"
set -x password "password123"

curl -u $username:$password https://www.example.com/api/data
```

आउटपुट:
```
{
  "data": [
    {
      "name": "John Doe",
      "age": 30,
      "city": "New York"
    },
    {
      "name": "Jane Doe",
      "age": 28,
      "city": "Los Angeles"
    }
  ]
}
```

## गहराई में जा
बेसिक प्रमाणीकरण के साथ HTTP अनुरोध भेजने का इतिहास, वैकल्पिक विकल्प और इसके विवरणों को गहराई में जान।

### इतिहास
HTTP अनुरोध में बेसिक प्रमाणीकरण का उपयोग 1999 ईसापूर्व में आरंभिक रूप से किया गया था। यह वर्तमान में प्रसिद्ध है और अन्य अनुरोध प्रोटोकॉलों में एक मानक के रूप में उपयोग किया जाता है।

### वैकल्पिक विकल्प
अन्य विकल्पों के बारे में अधिक जानकारी के लिए कृपया निम्नलिखित रिलीवेंट स्रोतों को देखें:
- HTTPS के साथ बेसिक प्रमाणीकरण का उपयोग करना
- दोगुना भागीरथी आयोजन (Digest Access Authentication)
- OAuth और मेनोआर तत्व (Nonce)
- संगतता मानक वैधनअंग (RFC) 7617 और 2617

### अंतर्निहित तत्व
HTTP अनुरोध में बेसिक प्रमाणीकरण सेट करने के लिए आपको अपने प्रोग्राम में दो पहलू सेट करने की आवश्यकता होगी:
- प्रयोक्तानाम (Username)
- पासवर्ड (Password)

ये विशेषताएं इस प्रक्रिया को सुरक्षित बनाती हैं और कोड क्रैकिंग से बचाती हैं। अतः, सुनिश्चित करें कि आप अपने दस्तावेजों में सार्वजनिक लेखों में प्रयोक्तानाम और पासवर्ड को सुरक्षित द्रष्टिकोण से रखते हैं।
