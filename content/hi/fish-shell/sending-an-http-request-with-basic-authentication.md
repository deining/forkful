## क्यों
क्षमतारूप साइटों से डेटा प्राप्त करने के लिए, आपको HTTP अनुरोध को भेजने के लिए बेसिक प्रमाणीकरण साथ चाहिए है। जो का अर्थ यह है कि आपको जोड़ने की आवश्यकता है कि आप सेवा को पहुंचने के माध्यम से सत्यापित कर सकें।

## कैसे करें
कोड उदाहरण और भी उदाहरण के साथ ```Fish Shell``` कोड ब्लॉक्स के भीतर बेसिक प्रमाणन साथ HTTP अनुरोध भेजने की बात करते हुए दिखाया जाएगा।

``` fish
set -x username "आपका यूजरनेम"
set -x password "आपका पासवर्ड"
set -x url "आपका हटप्प यूआरएल"

curl -u $username:$password $url
```

उपरोक्त कोड का अर्थ है कि आप अपने उपभोक्ता नाम और पासवर्ड को प्रमाणीकरण करने के लिए ```curl``` के साथ अपनी अनुरोध को भेज सकते हैं।

अब हम यह कुछ समझने के लिए देखेंगे कि कैसे हमारा रिक्वेस्ट प्रमाणीकरण के साथ आगे बढ़ता है।

## गहराई में

अपने हटप्प उपभोक्ताओं को प्रमाणीकरण के साथ एक HTTP अनुरोध भेजते समय, आप उनका सत्यापन कर रहे हैं कि वे आपके द्वारा सत्यापन उपभोक्ता हैं। यह दूसरे सेवा द्वारा डेटा अभिनीत होने या खराब होने के साथ कीमती सिद्धांत प्रबंधित करने में मदद कर सकता है। मानों, आप एक भुजा ट्रेडिंग एपीआई प्रदाता को अपने स्टॉक मार्ग खोलने की अनुमति देना चाहते हैं। आपको सुनिश्चित करना होगा कि आपका उपभोक्ता चाहता है क
