## क्या और क्यों?
अगर आप एक प्रोग्रामर हैं तो आपने शायद ``` फिश शेल``` के नाम से जाना होगा। यह एक कमांड लाइन इंटरफेस है जो आपको अपने प्रोग्रामों को तेजी से चलाने और स्क्रिप्ट करने में मदद करता है। अक्सर हमें अपने प्रोग्रामों में कुछ आवश्यक फोल्डर्स को चेक करना पड़ता है, इसलिए हम जांचते हैं कि क्या वे मौजूद हैं या नहीं।

## कैसे करें:
फिश शेल में, हम `test -d <directory>` का उपयोग करके फोल्डर की उपस्थिति को चेक कर सकते हैं। अगर फोल्डर मौजूद है, तो स्क्रिप्ट चलता रहेगा और यदि वह मौजूद नहीं है, तो एक त्रुटि संदेश दिखाई देगा। नीचे दिए गए उदाहरण में, हमें `test` का उपयोग करके `shellcheck` स्क्रिप्ट को कहते हैं कि क्या वह `src` फोल्डर में मौजूद है:

```fish
$ test -d src && echo "src directory exists"
```

यह हमें निम्न आउटपुट देगा:
```
src directory exists
```

## गहराई में जाएं:
फिश शेल `test` के साथ आता है, जो लिनक्स मोड्यूल की तरह है जो फोल्डर्स और फ़ाइलों को चेक करता है। अन्य विकल्पों में `[[ -d <directory> ]]` और `ls -d <directory>` भी मौजूद हैं। हालांकि, ये सभी अधिक पूर्णरूप से समान तरीके हैं और आप सुनिश्चित होने के लिए किसी भी एक का उपयोग कर सकते हैं। अन्य बहुत सारे लिनक्स मोड्यूल्स भी फोल्डर्स को चेक करने की सुविधा प्रदान करते हैं।
