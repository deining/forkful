# क्यों

फिश शेल प्रोग्रामिंग बहुत सारे लोगों के लिए एक आकर्षक विकल्प है। यह एक विकसित और उन्नत शेल है जो आपको कमांड लाइन में काम करने की अनेक सुविधाएं प्रदान करती है। यह शेल आपको अपनी जरूरतों के अनुसार अपनी खुद की सुविधाएं या क्षमताओं के साथ विकसित करने की अनुमति देती है। परिणामस्वरूप, फिश शेल प्रोग्रामिंग में अपनाने लेखन टेस्ट के लिए एक महत्वपूर्ण और आवश्यक कार्य है। इस लेख में हम फिश शेल के साथ कैसे टेस्ट लिख सकते हैं, उसके बारे में गहराई से जानेंगे।

# कैसे करें

टेस्ट लिखने के लिए आपको मानक संविधान में "test" फंक्शन का उपयोग करना होगा। आप स्टैंडर्ड संविधान के साथ एक नया फिश शेल सेशन शुरू कर सकते हैं और आपके मापदंडों को चालू कर सकते हैं।

```Fish Shell
set -ugi
test(value -eq 1); and echo "Test passed"
```

इस प्रकार, आप चालू किए जाने वाले चलन के साथ अपने टेस्ट को जनरेट कर सकते हैं। यदि आप अपने टेस्ट में किसी त्रुटि का पता लगाते हैं तो आपको त्रुटि को खारिज करना होगा।

```Fish Shell
test(value -ne 1); and echo "Test failed"
```

आप भी एक समलैंगिक बंदी द्वारा टेस्ट समय को स्पष्ट कर सकते हैं। उदाहरण के लिए, अपने टेस्ट को ट्रुटि पर खिसकने से पहले कुछ समय स्लीप करना सक्षम करें।

```Fish Shell
test_begin
sleep 5s
test(value -ne "test"); and echo "Test failed"
```
