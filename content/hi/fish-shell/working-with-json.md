## क्या है और क्यों?
JSON से काम करने की परिभाषा में, यह एक प्रोग्रामिंग भाषा है जो डेटा को संरचित और आसानी से पढ़ने के लिए डिजाइन किया गया है। प्रोग्रामर जो कोडिंग के साथ काम करते हैं वे अक्सर इस्तेमाल करते हैं क्योंकि यह स्ट्रक्चर्ड डाटा को असाधारण ढंग से अभिव्यक्त करने की अनुमति देता है।

## कैसे करें:
```Fish Shell``` का उपयोग करके, हम JSON फ़ाइलों से डेटा को कैसे पढ़ सकते हैं, क्या हम कैसे अलग प्रकार के डेटा प्रारूपों से काम कर सकते हैं और कैसे उन्हें एक समान रूप में प्रकट कर सकते हैं और कैसे हम JSON फ़ाइलों से डेटा को कैसे सेव कर सकते हैं, यह तथ्यों के साथ प्रदर्शित होता है। इससे हमें अलग-अलग प्रकार की डेटा प्रस्तुत करने का सुनहरा अवसर मिलता है, जीसका कोई प्रतिबंध नहीं है।

```fish
# अपनी JSON फाइल खोलें
$ cat data.json
{
  "नाम": "जॉन",
  "उम्र": 30,
  "रंग": "नीले"
}

# सामान्य कमांड का उपयोग करके फ़ाइल के डेटा को पढ़ें
$ cat data.json | jq

# एक JSON फ़ाइल से केवल एक डाटा पॉइंट प्राप्त करें
$ cat data.json | jq '.नाम'

# डेटा दोरान प्रदर्शित करें
$ echo '{ "नाम": "टीना", "उम्र": 25 }' | jq

# JSON फ़ाइल में नए डेटा जोड़ें
$ echo '{ "पहला नाम": "मार्क", "अंतिम नाम": "स्मिथ" }' > new_data.json | jq '. | . नाम + " " + . "अंतिम नाम"' > new_data.json

# डेटा को एक अलग प्रारूप में संशोधित करें
$ cat data.json | jq '. | . नाम + " " + . "उम्र" + "वर्ष"' > modified_data.json
```

## गहराई में जाएँ:
JSON प्रारम्भिक रूप में १९९९ में डिज़ाइन किया गया था और इसका उपयोग वर्तमान में अक्सर डेटाबेस्स, वेब सर्विसेस और अन्य सॉफ्टवेयरों में किया जाता है। कुछ अल्टर्नेटिव प्रारूप शामिल हैं XML और YAML, लेकिन JSON को आसान और सक्रिय बनाए रखने के लिए उनमें से कोई आमने-सामने नहीं होता है। कोडनाम "Postmodern प्रोग्रामिंग" के साथ।
