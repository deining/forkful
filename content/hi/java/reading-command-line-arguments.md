# क्या और क्यूँ?

कमांड लाइन एर्ग्युमेंट्स (command line arguments) को पढ़ना क्या है और इसे क्यों प्रोग्रामर्स करते हैं, यह जानने के लिए आपको यह लेख पढ़ना होगा। कमांड लाइन एर्ग्युमेंट्स हैं उन आपत्तिजनक शब्दों या मूल्यों के निर्देश जो आप अपने जावा प्रोग्राम को कमांड लाइन से बनाते हैं। ये एर्ग्युमेंट्स प्रोग्रामर्स को एक ऊपरी हाथ देते हैं ताकि वे अपने प्रोग्राम को बाहर से पैरामीटर्स और चुनाव द्वारा नियंत्रित कर सकें।

## कैसे:

कमांड लाइन एर्ग्युमेंट्स को जावा में पढ़ने के लिए हमें दो चीज़ों की आवश्यकता होती है: ```main()``` विधि और ```args[]``` इनपुट पैरामीटर। नीचे दिए गए उदाहरण को देखें:

```java
public static void main(String[] args) {
  // यहां हम कमांड लाइन एर्ग्युमेंट्स को पढ़ते हैं:
  for (int i = 0; i < args.length; i++) {
    System.out.println("आपको दिए गए एर्ग्युमेंट्स में से " + args[i] + " कोई भी पढ़ा गया।");
  }
}
```

उपरोक्त कोड में, हमने ```main()``` विधि के अंदर एक लूप बनाया है जो अंतिम एर्ग्युमेंट तक सभी एर्ग्युमेंट्स को पढ़ता है और एक-एक करके उन्हें प्रिंट करता है। इसका उपयोग हम उन एर्ग्युमेंट्स को देखने के लिए करते हैं जो हम प्रोग्राम के बाहर से देते हैं। अब यदि हम आप्षों (arguments) बिना चुनाव के प्रोग्राम को चलाते हैं तो वह तीन उपलब्ध एर्ग्युमेंट्स हमें प्रिंट करेगा:

```
आपको दिए गए एर्ग्युमेंट्स में से arg1 कोई भी पढ़ा गया।
आपको दिए गए एर्ग्युमेंट्स में से arg2 कोई भी पढ़ा गया।
आपको दिए गए एर्ग्युमेंट्स में से arg3 कोई भी पढ़ा गया।
```

अगर हम चुनाव सहित प्रोग्राम को चलाते हैं, तो हमें सिर्फ उन आप्षों को ही प्रिंट किया जाएगा जो हमने चुनाव द्वारा दिए हैं।
