## क्यों 

टेस्ट लिखने का प्रयास मुख्य रूप से डेवलपर्स को उनकी कोड या सॉफ्टवेयर के बग्स के बारे में सुनिश्चित करने के लिए किया जाता है। टेस्ट आपको सुनिश्चित करते हैं कि आपका कोड सुरक्षित और स्थिर है और उसमें कोई भी बग नहीं है। यह आपके प्रोजेक्ट की गुणवत्ता को बढ़ाता है और आपको किसी भी त्रुटि का पता चलता है जिसमें समस्याओं को पकड़ने और ठीक करने में मदद मिलती है।

## कैसे करें

```Java
public class Calculator {
  public int add(int num1, int num2) {
    return num1 + num2;
  }
}

public class CalculatorTest {
  @Test
  public void testAdd() {
    Calculator calculator = new Calculator();
    int result = calculator.add(5, 10);
    int expected = 15;
    assertEquals(expected, result);
  }
}
```

ऊपर दिए गए संगणक का एक उदाहरण है और साथ ही उसमें एक टेस्ट भी है। इसमें हमने कैलकुलेटर क्लास का एक ऑब्जेक्ट बनाया और `add()` मेथड को 5 और 10 से कॉल किया और आश्वासन किया है कि योग का नतीजा 15 है। इस तरह से हम अपने कोड को टेस्ट कर सकते हैं और सुनिश्चित कर सकते हैं कि कोड ठीक से काम कर रहा है।
