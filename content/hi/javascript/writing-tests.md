## क्यों
जब हम एक वेबसाइट या एप्लिकेशन डेवलप करते हैं, तो हम चाहते हैं कि आखिरी प्रोडक्ट ठीक से काम करे। इसके लिए हमें बहुत से विभिन्न मानकों का पालन करना पड़ता है, और इसमें से एक है टेस्टिंग। अपने कोड के साथ टेस्टिंग करना आपकी वेबसाइट या एप्लिकेशन की सुरक्षा, स्थिरता और गुणवत्ता बढ़ाता है।

## कैसे करें
टेस्ट केस का सबसे सरल उदाहरण एक गुणवत्ता चेक हो सकता है। जैसे की अगर हमारी एक फ़ंक्शन 2 नंबर को स्क्वेर करती है, तो उसका आउटपुट 4 होना चाहिए।

```Javascript
function square(x) {
  return x * x;
}
console.log(square(2)); // Output: 4
```

अगर यह आउटपुट नहीं होता है, तो आपको अपने कोड में कुछ गलती का पता चल जायेगा और आप उसे सुधार सकते हैं। आप एक बार अपने कोड को टेस्ट करने के बाद, स्वयं हुए मुश्किलात से बच सकते हैं।

## गहराई में जायें
टेस्टिंग का एक और महत्वपूर्ण अंश है की आप एक आरामदायक निर्माण प्रक्रिया बना पाते हैं। आप नए फीचर्स और बग फिक्स करने के साथ साथ अपने कोड में कोई बदलाव कर सकते हैं, और सुनिश्चित कर सकते हैं की आपका कोड अभी भी ठीक से काम कर रहा है। इससे आपको नए फीचर्स या बग फिक्स के साथ सुधारों को भी आसानी से डीप्लॉय कर सकते हैं।
