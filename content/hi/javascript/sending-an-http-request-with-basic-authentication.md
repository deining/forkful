## क्यों

HTTP अनुरोध भेजने में बेसिक प्रमाणिकरण के साथ लगाना क्यों जरूरी है, इसका स्पष्टीकरण करने का मुख्य प्रयास है।

## कैसे करें

जैसा कि नाम से पता चलता है, बेसिक प्रमाणिकरण एक आम प्रकार का प्रमाणित तरीका है जो वेब अनुरोधों को सुरक्षित बनाने के लिए उपयोग किया जाता है। यह मूलतः उपयोगकर्ता के लॉगिन और पासवर्ड के साथ एक अनुरोध की पुष्टि करने के लिए उपयोग किया जाता है।

जब हम एक HTTP अनुरोध को भेजते हैं, तब यह उपयोगकर्ता द्वारा प्रदत्त लॉगिन और पासवर्ड को प्रमाणित करता है। यदि प्रमाणिकरण सफल होता है, तो सर्वर अनुरोध को स्वीकार करता है और उपयोगकर्ता को अनुमति देता है। हालांकि अगर प्रमाणिकरण असफल होता है तो सर्वर अनुरोध को अस्वीकार कर देता है और अनुमति नहीं देता है।

यदि आप भी इस तरह का अनुरोध भेजना चाहते हैं, तो आपको निम्नलिखित चरणों का पालन करना होगा।

चरण 1: सर्वर के साथ समन्ध स्थापित करें

पहले आपको सर्वर के साथ समन्ध स्थापित करना होगा। यह आपके स्थानीय कंप्यूटर पर हो सकता है या आप किसी खास सर्वर के साथ प्रयोग कर सकते हैं।

```Javascript
const serverUrl = 'http://www.example.com';
```
