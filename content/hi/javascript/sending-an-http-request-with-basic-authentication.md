## क्या और क्यों?
बेसिक प्रमाणीकरण के साथ HTTP अनुरोध भेजना क्या है और क्यों प्रोग्रामर इसे करते हैं? बेसिक प्रमाणीकरण के साथ, हम अपने अनुरोधों को सुरक्षित तरीके से सर्वर को भेज सकते हैं और साथ ही डेटा को भेजकर सुरक्षित रूप से संग्रहित भी कर सकते हैं। 

## कैसे:
~~~~
अपने अनुरोध को बेसिक प्रमाणीकरण के साथ भेजने के लिए, हमें एक HTTP अनुरोध बनाना होगा और उसमें बेसिक प्रमाणीकरण चर पास करना होगा। नीचे दिए गए कोड स्निपेट में, हम एक साधारण GET अनुरोध भेजते हुए बेसिक प्रमाणीकरण का उपयोग करते हुए बताएंगे: 
~~~~
```Javascript
const url = 'https://www.example.com';
const username = 'myusername';
const password = 'mypassword';

fetch(url, {
  method: 'GET',
  headers: {
    'Authorization': 'Basic ' + btoa(username + ':' + password)
  }
}).then(response => response.json())
.then(data => {
  console.log(data);
})
.catch(error => {
  console.log(error);
});
```

यहां, हमने अपने अनुरोध में बासिक प्रमाणीकरण के लिए खुद को मैनुअली कन्वर्ट किया है और उसे अपने अनुरोध के हैडर में जोड़ा है। उसके बाद, हम fetch मेथड का उपयोग करके अनुरोध को सर्वर को भेजते हैं और उसका रिस्पॉन्स प्राप्त करते हैं। अंत में, हम प्राप्त डेटा को लॉग करते हैं और अगर कोई त्रुटि होती है तो हम उसे भी लॉग करते हैं। 

## गहराई में जाएं:
बेसिक प्रमाणीकरण के साथ HTTP अनुरोध भेजने का इतिहास बहुत पुराना है और शुरुआत में इसे एक सुरक्षित ढंग से सर्वरों को अपने अनुरोधों के साथ परिचय देने के लिए इस्तेमाल किया जाता था। आप अन्य प्रमाणीकरण मेथडों की तुलना में ज्यादा अस्पष्ट नहीं, लेकिन यह बेहतर है जब आपको अपने अनुरोधों को सुरक्षित तरीके से सर्वर को भेजने की आवश्यकता होती है। इसके आलावा, बहुत से अन्य प्रमाणीकरण और सुरक्षा मेथड भी हैं जैसे OAuth, JWT, आदि जो आप उपयोग कर सकते हैं अपने परियोजनाओं में।

बेसिक प्रमाणीकरण को अपने अनुरोध में शामिल करने के लिए, हमें सर्वर को उपयोगकर्ता का नाम और पासवर्ड भेजने की आवश्यकता होती है। यह आसान और तेजी से लागू होता है और सुरक्षित भी होता है जब आप अपने कॉड में उप
