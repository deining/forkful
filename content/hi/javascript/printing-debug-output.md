# क्या और क्यों?
Javascript में, कभी-कभी हमें अपनी प्रोग्राम के दौरान चीजों की स्टेटस या डेटा के बारे में जानकारी देखने की आवश्यकता होती है। इसलिए, हम डीबग आउटपुट को प्रिंट करके अपने कोड को जांचते हैं। इससे हम अपने कोड में त्रुटियों को संशोधित कर सकते हैं और सही तरीके से चलाने के लिए सुनिश्चित कर सकते हैं।

## कैसे करें:
```Javascript
// सामान्य तरीके से डीबग आउटपुट प्रिंट करने के लिए
console.log("यह मैसेज कंसोल में प्रिंट होगा");

// वेरिएबल्स या ऑब्जेक्ट की जानकारी प्रिंट करने के लिए
let name = "आदित्य";
console.log(`मेरा नाम ${name} है`);

// स्टेटमेंट की जांच करने के लिए
let num = 10;
if (num < 0) {
  console.log("यह स्टेटमेंट त्रुटिपूर्ण है");
} else {
  console.log("यह स्टेटमेंट सही है");
}
```

उपरोक्त भागों को अपने कोड में शामिल करके, आप डीबग आउटपुट प्रिंट कर सकते हैं और अपने कोड को चेक कर सकते हैं।

## गहराई में जाएँ:
डीबग आउटपुट प्रिंट करने का इतिहास काफी पुराना है। पहले, प्रोग्रामिंग में बड़े स्केल की विकसिति के साथ, लोगों को अपने कोड को जाँचने के लिए डीबग आउटपुट प्रिंट करने की आवश्यकता हुई। अल्टरनेटिव के रूप में, आप कंसोल में लॉग करने के साथ भी डीबग आउटपुट प्रिंट कर सकते हैं। कई विकासक डीबगिंग टूल्स का भी इस्तेमाल करते हैं जो आपको अपने कोड को चेक करने में मदद कर सकते हैं। डीबग आउटपुट प्रिंट करना एक महत्वपूर्ण स्किल है जो हर जाने वाले प्रोग्रामर को आनी चाहिए। 

## और भी देखें:
डीबग आउटपुट के बारे में अधिक जानने के लिए, आप निम्नलिखित स्रोतों पर जा सकते हैं:
- [MDN आर्टिकल](https://developer.mozilla.org/en-US/docs/Web/API/Console/log) - जानिए कैसे कंसोल में डीबग आउटपुट प्रिंट करें और अपने कोड को चेक करें
- [विकिपीडिया आर्टिकल](https://en.wikipedia.org/wiki/Debugging) - आपको डीबगिंग क्या है और इसके इतिहास के बारे में अधिक जानकारी देगा
