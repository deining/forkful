## क्यों

जब हम किसी प्रोग्राम बनाते हैं, तो अक्सर हमें अपने कोड के बीच से एक अंश का दिखावा करने के लिए अस्थायी फाइलें बनानी होती है। इन अस्थायी फाइलों के लिए हम एक अन्य स्थाई फाइल को कष्टप्रद ढंग से बचाना और उसे सुरक्षित रूप से हटाना चाहते हैं। इसलिए, हम अस्थायी फाइलें बनाते हैं।

## कैसे करें

```javascript
// अस्थायी फाइल नाम को डिफ़ॉल्ट अपने करनैम से बनाया जाएगा
const tmp = require('tmp'); // यह कस्टम फाइल नाम और पथ के लिए मॉड्यूल है
tmp.file(function _tempFileCreated(err, path, fd) {
  if (err) throw err;
 
  // अस्थायी फाइल पर कुछ लिखें
  fs.write(fd, 'Hello World!', function(err) {
    if (err) throw err;
 
    // अस्थायी फाइल बंद करें
    fs.close(fd, function(err) {
      if (err) throw err;
 
      // अस्थायी फाइल को हटाएं
      fs.unlink(path, function(err) {
        if (err) throw err;
      });
    });
  });
});
```

**नतीजा:**
जब आप यह कोड चलाएंगे, तो आपको करनैम/फाइल पथ के साथ एक अस्थायी फाइल बनाई जाएगी और 'Hello World!' के साथ भरी जाएगी। फिर वह फाइल बंद और हटा दी जाएगी।

## गहराई में जाएं

अस्थायी फाइलें बनाना एक अत्यंत महत्वपूर्ण हिस्सा है जब आप किसी प्रोग्राम में काम करते हैं। यह आपको अस्थायी स्थिति से बचाने और सुनिश्चित करने में मदद करता है कि आपका कोड सुरक्षित रूप से बनाए रहे। इसके अलावा, यह आपको अपने कोड को सुधारने के लिए अस्थायी फाइलों का प्रयोग करने की स्वतंत्रता भी
