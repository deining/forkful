## क्या और क्यों?
HTTP अनुरोध भेजना बुनियादी प्रमाणीकरण के साथ क्या होता है, यह हमें कम्प्लियांस को विश्वासपूर्ण बनाने के लिए हमारे द्वारा उपयोग किया जाता है।

## कैसे करें:
```Elixir
HTTPoison.get("http://www.example.com",
  headers: [
    {"Authorization", "Basic dXNlcm5hbWU6cGFzc3dvcmQ="}
  ]
)
```

उपरोक्त कोड ब्लॉक में, हम अपने HTTP अनुरोध में एक बुनियादी प्रमाणीकरण हैडर जोड़ रहे हैं। यह बेसिक प्रमाणीकरण के लिए आवश्यक उपयोगकर्ता का नाम और पासवर्ड शामिल करता है। यदि सही उपयोगकर्ता नाम और पासवर्ड प्रस्तुत किए गए हैं, तो यह अनुरोध सफलतापूर्वक पूरा होगा और हमें संबंधित डेटा का उत्तर प्राप्त होगा।

## गहराई में खोज:
जब भी हमारे द्वारा इंटरनेट कनेक्शन बनाए जाने के लिए एक HTTP अनुरोध भेजा जाता है, तो उसमे अधिकांश दृश्य हे कि हमें कुछ प्रकार का प्रमाणीकरण करना होता है। अक्सर इस उपयोगकर्ता नाम और पासवर्ड का उपयोग किया जाता है ताकि हमें कम्प्लियांस को सुनिश्चित कर सकें। इस मामले में, बुनियादी प्रमाणीकरण एक संभावित विकल्प है जो हमारे द्वारा उपयोग किया जा सकता है। इसके अलावा, हम अपने HTTP अनुरोध में अन्य प्रमाणीकरण तकनीकों का भी उपयोग कर सकते हैं, जैसे टोकन आधारित प्रमाणीकरण आदि।

## अधिक जानकारी के लिए:
अगर आप अधिक जानना चाहते हैं तो आप इस विषय पर Elixir डॉक्यूमेंटेशन और ब्लॉग पोस्ट के साथ-साथ बेहतर अध्ययन कर सकते हैं। शुरुआत करने के लिए, [Elixir की वेबसाइट](https://elixir-lang.org/) पर देखें। इसके अलावा, आप [HTTPoison लाइब्रेरी](https://hexdocs.pm/httpoison/HTTPoison.html) के बारे में भी अधिक जानकारी प्राप्त कर सकते हैं जो हमारे उदाहरणों में उपयोग किया गया है।
