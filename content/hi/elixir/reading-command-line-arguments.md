## क्या है और क्यों? 
कमांड लाइन आर्ग्यूमेंट पढ़ना क्‌या है और इसे क्यों प्रोग्रामर्स करते हैं? यह एलिक्सिर प्रोग्रामिंग स्किल में आपके लिए बेहद महत्वपूर्ण है। इसके माध्यम से आप अपने प्रोग्राम को डायनामिक बनाकर अपने कोड को बहुत सुविधाजनक बना सकते हैं।

## कैसे करें:
```Elixir
defmodule CommandLine do  # कमांड लाइन मॉड्यूल बनाएं
  def parse(args) do  # प्रथम पैरामीटर ग्राहण करें
    {:ok, parsed_args} = OptionParser.parse(args, switches: [first_name: :string, last_name: :string])  # ऑप्शन्स द्वारा पैरामीटर्स ग्राहण करें
    IO.puts "नमस्ते #{parsed_args.first_name} #{parsed_args.last_name}"  # प्रथम और द्वितीय नाम प्रिंट करें
  end
end

CommandLine.parse(["--first_name", "जॉन", "--last_name", "स्मिथ"])  # मॉड्यूल कॉल करें
```
आउटपुट:
```
नमस्ते जॉन स्मिथ
```

## गहराई से जानें:
### इतिहास:
कमांड लाइन आर्ग्युमेंट पढ़ने एक लंबे समय से आज कल स्वभाविक है। अपने कोड को डायनामिक बनाने और प्रोग्रामर्स को त्वरित काम करने के लिए यह बहुत ही उपयोगी है।

### विकल्प:
एक अल्टरनेटिव विकल्प यह है कि आप एक फ़ाइल से अपने पैरामीटर ग्राहण कर सकते हैं। यह थोड़ा ज़्यादा उपयोगी हो सकता है क्योंकि आप एक सिंगल फ़ाइल से पैटर्न्स को संभाल सकते हैं और इसे ऑप्शन्स के रूप में इस्तेमाल भी किया जा सकता है।

### इंप्लीमेंटेशन डिटेल्स:
प्रोग्रामर्स एलिक्सिर में फंक्शंस और मॉड्यूल्स का इस्तेमाल करके एक मॉड्यूल बना सकते हैं जो कीमतों को रियल ऑब्जेक्ट्स के रूप में कंपाइल कर सकता है। यह मॉड्यूल से आप आर्ग्यूमेंट्स को पार्स कर सकते हैं और वे प्रोग्राम में आपके लिए उपलब्ध हो जाएंगे।

## इसके अलावा देखें:
अगर आपको और जानना है कि कमांड लाइन आर्ग्युमेंट प्रोग्रामिंग में कितना महत्वपूर्ण है, तो आप [Elixir Command Line Args Library](https://hexdocs.pm/command_key/CommandLine.Args.html) की विस्तृत डॉक्यूमेंटेशन देख सकते हैं। यहां आपको अपने कोड में अन्य आर्ग्युमेंट ग्राहण विकल्पों के बारे में पता चलेगा जो आप आराम से इस्तेमाल कर सकते हैं।
