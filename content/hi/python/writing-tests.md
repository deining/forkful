## क्यों

टेस्ट क्रिएट करना वह अहम कदम है जो हर प्रोग्रामर को लेना चाहिए क्योंकि यह सुनिश्चित करता है कि उनका कोड सही तरीके से काम कर रहा है और कोई भी बग नहीं है। यह स्टेटमेंट प्रोफेशनल प्रोग्रामर्स और नए शुरुआती प्रोग्रामर्स के लिए एक साथ लागू होती है।

## कैसे करें

टेस्ट हमेशा संगित कोड से अलग रखें। इसके लिए, हम `unittest` मॉड्यूल का उपयोग करेंगे। 

```Python
import unittest

# हमारी एक साधारण फंक्शन बनाएंगे जो दो संख्याओं का बाहुल्य से जोड़ने के लिए है।
def add(x, y):
    return x + y

# अब हमारे पास एक unittest टेस्टकेस्ट क्लास है जो हमारी add फंक्शन को टेस्ट करेगा।
class TestAddFunction(unittest.TestCase):
    def test_add(self):
        # हमारे फंक्शन को कॉल करेंगे और सही जवाब मिलने की जांच करेंगे।
        result = add(2, 3)
        # हमारे अपेक्षित परिणाम को हम यहां दर्शाएंगे। यदि यह सही है, तो टेस्ट पास हो चुका है।
        self.assertEqual(result, 5)

# हम अपनी टेस्टकेस्ट को सेटअप करते हैं और टेस्ट सुरु करते हैं।
if __name__ == '__main__':
    unittest.main()
```

अब हमारा कोड टेस्ट करने के लिए तैयार है। ऊपर दिए गए कोड को एक `.py` फाइल में सहेजें और टर्मिनल में निम्नलिखित आदेश को चलाएं:

```
python test_add_function.py
```

यदि सब कुछ सही से कॉम्पाइल होता है और टेस्ट पास हो जाता है, तो आपको इस प्रकार का आउटपुट मिलना चाहिए:

```
.
----------------------------------------------------------------------
Ran 1 test in 0.000s

OK
```
