## क्यों
शायद आपने पायथन कोड के आगे # योग्यता को पढ़ा है लेकिन परीक्षण को पढ़ने के बाद आप आश्चर्यचकित हो सके हैं कि आपको टेस्ट क्यों करना चाहिए। यह अपने कोड को परीक्षण क्यों सुनिश्चित करने के लिए होता है और इससे आपके कोड में संशोधनों को लगातार चालू रखने में मदद मिलती है।

## कैसे करें
टेस्टिंग कोड में बढ़िन चहते हैं? यहां हम कुछ मामलों के साथ उदाहरण दे रहे हैं:

```Python
# गणना फ़ंक्शन
def add_numbers(x, y):
    return x + y

# टेस्ट कैसे?
assertEqual(add_numbers(2, 3), 5)
assertNotEqual(add_numbers(2, 3), 6)
```

आपको गणना फ़ंक्शन कोड को परीक्षण के लिए गणना फ़ंक्शन में गलतियां ढूंढना करे सकते हो। इससे आपको दोबारा कोड को जगोह से देखनें की आवश्यकता नहीं होमें गी।

## गहराई
टेस्ट लिखने के बाद लोग अक्सर कुछ एक्स्टरा प्रश्नों को भी पता करना चाहते हैं। एक गहरा अन्दशास्त्री के लिए यह महत्वपूर्ण है कि आपका कोड प्रभावित होता है भले ही आप अपने स्वयम से इंतरफेस को सेबातिय् करे। टेस्टिंग आपके स्वयम से इंटरफेस सेबातिय् निरंतर पहनने क्यों करतें हैं, क्योंकि आप अपना कोड को कैसे बस पसंद कसतें हों।
