# क्यों

क्या आपको प्रोग्रामिंग में डिबग आउटपुट के प्रिंट करने का मन नहीं होता है? जब हम किसी कोड में ग़लती को ढूँढ़ने के लिए दीगर प्रिंट करके और ग़लती को सुधारने के लिए प्रोग्राम को चलते रहते हैं, तो हम डिबग आउटपुट का उपयोग कर सकते हैं। इससे हमारा कोड थोड़ा सा सरल बन जाता है और हमे ग़लती को पकड़ने में घूमने की ज़रूरत नहीं पड़ती।

# कैसे करें

जीवम में डिबग आउटपुट को प्रिंट करना बहुत आसान है। हम तुरंत '```gleam debug ... ```' के साथ अपना रिकवरी (debug) प्रिंट कर सकते हैं। नीचे एक उदाहरण हो सकता हैं:

```gleam
import gleam/debug

pub fn main() {
  debug.log("आपका रेज़ल्ट यह हैं: %{fun(2, 3)}")
}

fn fun(x, y) {
  x + y
}
```

यह आपको निम्न आउटपुट देगा:

आपका रेज़ल्ट यह हैं: 5


आप "आपका रेज़ल्ट यह हैं:" स्ट्रिंग को अपनी पसंद की हिसाब से कोई भी नंबर या अन्य डेटा के साथ बदल सकते हैं। इसी तरह से, आप '```info```', '```warn```', या '```error```' को भी अपना डिबग आउटपुट मानचित्र बनाने के लिए प्रयोग कर सकते हैं।
