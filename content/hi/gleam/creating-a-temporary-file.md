## क्यों

अगर आप एक Gleam प्रोग्रामर हैं, आपने शायद इस तकनीक को अज्ञात नहीं रखा होगा - बाकी प्रोग्रामिंग भाषाओं की तरह, Gleam भी अस्थाई फ़ाइल बनाने के लिए आपको किसी अलग तरह की फ़ंक्शन का उपयोग करता है। यह तरीका उपयोगकर्ताओं को अस्थाई डाटा या किसी अस्थाई प्रक्रिया पर नियंत्रण पाने में मदद करता है, प्रोग्राम के साथ संवाद स्थगन, और अन्य सामान्य स्थितियों में एकाधिक क्लास्टरिंग या प्रोपर्टी फ़ाइलसिस्टम में स्वचालित ऊपरिमेंटल छोड़ सकता है।

## कैसे करें

```Gleam
import gleam/temporary

let { ok, fd } = temporary.file("~/.my_temp_file")
case ok {
  Ok -> "सफलतापूर्वक अस्थाई फ़ाइल बनाई गई है"
  Err -> "अस्थाई फ़ाइल बनाने में त्रुटि हुई है"
}
temporary.close(fd)
```

यहां हमने एक अस्थाई फ़ाइल बनाने का उदाहरण दिया है जो `~/.my_temp_file` में बनाई गई है। हमने फ़ंक्शन `temporary.file` को कॉल करते हुए उर्जा की, और `let` और `case` को छोड़कर हमने हममार स्थान उस डाक्यूमेंट को छोड़कर हमने अस्थाई फ़ाइल बनाई है। हमने एक स्थिर डाक्यूमेंट में संकेत किया जो शुरुआत से ही `temporary.file` को bag देगा। आप अपने स्वयं द्वारा संश्लिषित डाउनलोड पोइट में हम आपको यदि आपके कोड पर संशोधित तौर पूर्व नहीं है - [gleam/tmpfile](https://github.com/gleam-lang/gleam/tree/master/lib/gleam) को वावेर कर सकते हैं
