## इसलिए

गर आप अपनी ग्लीम प्रोग्रामिंग स्किल्स को सुधारना चाहते हैं और अपने कोड को अधिक scalable बनाना चाहते हैं, तो आपको अपने कोड को टेस्ट करना चाहिए। इससे आपने अपने कोड में किसी भी बग को पकड़ाकर उसे ठीक किया जा सकता है और आपका कोड अधिक reliable हो जाता है।

## कैसे करें

चलिए शुरू करते हैं। उपरोक्त कारणों की रौशनी में, यह बेहद महत्वपूर्ण है कि हम अपने कोड को टेस्ट कैसे करें। हम ग्लीम में ```fn test() { ... } ``` कोड ब्लॉक का उपयोग करके आसानी से टेस्ट लिख सकते हैं। नीचे दिखाए गए उदाहरण में, हमने दो संख्याओं के योग को परीक्षण करने के लिए `test_add` नाम का टेस्ट बनाया है। कृपया ध्यान से देखें कि कैसे हमने विभिन्न प्रकार के डेटा टेस्ट करने के लिए विभिन्न पट्टियों का उपयोग किया है। आप इस तरह के संतुलित टेस्ट के साथ अपने कोड की परीक्षण कर सकते हैं और अपने एप्लिकेशन का विश्वास कर सकते हैं।

```Gleam
test_add() {
  assert.eq(2, 1 + 1)
  ::assert.neq(-1, 1 + 1)
  
  ::assert.eq(2.0, 1.0 + 1.0)
  ::assert.eq(2.0, 1 + 1.0)
  
  ::assert.eq("Hello world", "Hello" ++ " world")
  
  foo = [1, 2, 3]
  bar = [4, 5, 6]
  ::assert.eq([1, 2, 3, 4, 5, 6], foo ++ bar)
}
```

जैसा कि आप देख सकते हैं, हमने `assert` फंक्शन का उपयोग अपने टेस्ट केस की पुष्टि करने के लिए किया है। इसमें हम अपनी निर्दिष्ट परामर्शों को प्रदर
