## क्या और क्यों?
कमांड लाइन आर्ग्यूमेंट पढ़ना क्या है और क्यों प्रोग्रामर्स इसे करते हैं, यह बताने के लिए हमें इस तरह से कोड करेंगे।

## कैसे:
अगर आपको प्रोग्रामिंग में उपयुक्त आर्ग्यूमेंट पास करने की आवश्यकता है, तो निम्न उदाहरण में ```Gleam io``` का उपयोग करके कैसे किया जाता है, देखें।

```
Gleam io = Keyword . parse_dst ( "-इनपुट फाइल.txt" )
छाप दें (देखाओ (io) )
```

उपरोक्त कोड का आउटपुट निम्नलिखित होगा:

```
{ checks : { write : {“डेस्कटॉप फाइल.txt "} }, encodings : [] }
```

## गहराई में जाएं:
कमांड लाइन आर्ग्यूमेंट पढ़ना प्रोग्रामिंग में बहुत उपयोगी है, क्योंकि यह आपको प्रोग्राम में दिए गए आवश्यक आर्ग्यूमेंट प्राप्त करने की क्षमता देता है। इसके अलावा, आप गलत आर्ग्यूमेंट देने के लिए भी प्रोग्राम को संशोधित कर सकते हैं। कमांड लाइन आर्ग्यूमेंट को पढ़ने के अलावा, आप इसे बदलने और नए आर्ग्यूमेंट जोड़ने के लिए भी उपयोग कर सकते हैं।

इसके अतिरिक्त आप खोज सकते हैं कि क्या अन्य भाषाओं में भी कमांड लाइन आर्ग्यूमेंट पढ़ने के लिए कुछ अन्य तरीके हैं, जैसे इंगित कमांडिंग या यूनिक्स स्क्रिप्टिंग।

आप इस लेख को पढ़कर आपको एक अच्छा ज्ञान हो गया होगा कि Gleam में कमांड लाइन आर्ग्यूमेंट पढ़ना कैसे करते हैं। अगर कोई दिक्कत आती है तो आप आराम से इस विषय में और भी अधिक चर्चा कर सकते हैं।

## और भी देखें:
अगर आप और अधिक जानना चाहते हैं तो कृपया निम्नलिखित संसाधनों को देखें:

- [ग्लीम दस्तावेज़ीकरण] (https://gleam.run/documentation/): ग्लीम भाषा की अधिक जानकारी के लिए दस्तावेज़ीकृत स्रोत।
- [कमांड लाइन आर्ग्यूमेंट पार्सिंग के तरीके] (https://en.wikipedia.org/wiki/Command-line_argument_parsing): अन्य प्रोग्रामिंग भाषाओं में कमांड लाइन आर्ग्यूमेंट पार्सिंग के बारे में सामान्य जानकारी।
