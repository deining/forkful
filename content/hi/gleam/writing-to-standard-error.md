## क्यों

आज के समय में, डेवलपर्स अपने कोड में बग्स को सही करने के लिए कई तरह के लॉगिंग विकल्पों का इस्तेमाल करते हैं। इसमें से एक है "standard error" जो कोड के बग्स को स्पष्ट करने में मदद करता है। इस ब्लॉग पोस्ट में, हम आपको बताएंगे कि आप इस लोग लेवलिंग ओपरेशन को कैसे कर सकते हैं और अपने कोड को और अधिक अनुकूलित और बग फ्री बना सकते हैं।

## कैसे

```Gleam
fn main() {
    let a = 5;
    let b = 0;
    let result = divide(a, b);
    io::stderr.print("Result: {{ result }}").unwrap()
}

fn divide(x, y) {
    match y {
        0 => fail!
        _ => x / y
    }
}
```

उपरोक्त कोड ब्लॉक में, हमने ```io::stderr``` का उपयोग किया है जो कि standard error पर लोग करता है। यहां हमने अपने कोड में एक बग पाया जो कि हमने गणना करने के लिए 0 से भाग करने की कोशिश की है। हमारी लाइन ```io::stderr.print("Result: {{ result }}").unwrap()``` standard error पर "Result: Infinity" प्रिंट करेगी। यह उस बग को स्पष्ट करेगी कि हमने कैसे 0 से भागने की कोशिश की है।

## गहराई में जाएं

standard error के साथ काम करना न केवल आपको अपने कोड में बग्स को स्पष्ट करने में मदद करता है, बल्कि जब आप बग्स को धुंध रहे होते हों उसके लिए यह एक अहम उपकरण भी हो सकता है। हमेशा अपने कोड में लोगिंग विकल्पों को शामिल करके, आप अपने कोड को थोड़ा अधिक सुरक्षित और प्रदर्शनशील बना सकते हैं।
