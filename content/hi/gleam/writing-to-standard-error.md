मछलीन्यासः

हम सभी कम से कम एक बार अपने कोड में त्रुटियों का सामना करते हैं। इसलिए, समय और प्रयास को बचाने के लिए हमें अपने त्रुटि सुचना को सामान्यतया *चरण मार्दयम* की श्रेणी में नहीं समायोजित करते हैं। इसलिए, हम अपने कोड में त्रुटियों को लघु हो जाने से रोकने के लिए *standard error* में लिख सकते हैं। तो चलिए, हम देखते हैं कि इसे कैसे किया जाता है।

## कैसे?

```Gleam
import gleam/stdio

let main = fn() {
  stdio.write_error("यह एक त्रुटि है।")
}
```

जैसा कि आप देख सकते हैं, हमने `gleam/stdio` को इम्पोर्ट करके `stdio.write_error` का उपयोग किया है। फिर हमने अपनी त्रुटि संदेश को स्ट्रिंग के रूप में लिखा है। आप इस संदेश को अपनी त्रुटि सुचना के अनुसार बदल सकते हैं।

जब आप इस कोड को चलाएंगे, आपको निम्नलिखित सामाप्ति के साथ कोई भी आउटपुट नहीं मिलेगा:


लेकिन, आपको फिर भी अपनी त्रुटि सुचना अपने कंसोल में देख सकते हैं।

इसके अलावा, आप इसे फाइल में भी लिख सकते हैं।

```Gleam
import gleam/stdio

let main = fn() {
  stdio.write_file_error("errors.log", "यह एक त्रुटि है।")
}
```

जैसा कि आप देख सकते हैं, हमने `stdio.write_file_error` का उपयोग केसिस्टमेंग `errors.log` में त्रुटि संदेश लिखने के लिए किया है। आप फाइल नाम और त्रुटि संदेश को अपनी आवश्यकतानुसार बदल सकते हैं।
