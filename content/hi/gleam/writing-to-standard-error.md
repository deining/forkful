## क्यों

स्टैण्डर्ड एरर में लिखने का काम करने का कोई स्पेशल कारण हो सकता है। किसी ऐसे कोड के समय जो हमारे लिए अनुमानित या उम्मीद से भिन्न तरीके से काम करता है।

## कैसे करें

कोड मामलों का आकार कम से कम करने के लिए, हमें अक्सर स्टैंडर्ड एरर को उचित स्थान पर लिखना होता है। इसके अलावा, हम अपने एप्लिकेशन को सुधारने के लिए भी स्टैंडर्ड एरर को उपयोग कर सकते हैं। नीचे दिए गए उदाहरण में, हमारे एक्साम्पल कोड में थ्रो मैक्रो उपयोग करते हुए हम स्टैंडर्ड एरर में संदेश प्रिंट करने के लिए दिखाते हैं।

```Gleam
pub fn main() {
  let result = divide(10, 0);
  match result {
    Ok(num) -> std.fmt.println("The result is {}", [num]),
    Error(err) -> std.fmt.println(err),
  }
}

fn divide(x, y) {
  if y == 0 {
    throw("Cannot divide by zero");
  } else {
    Ok(x / y)
  }
}
```

उपरोक्त कोड का आउटपुट हमेशा स्टैंडर्ड एरर में दिखेगा।

## गहराई में जाएं

स्टैंडर्ड एरर में लिखने का यह एक बहुत ही उपयोगी कार्य है। इससे हम बग रिपोर्टिंग, डिबगिंग, और अन्य प्रयोजनों के लिए स्टैंडर्ड एरर को आसानी से प्रबंधित कर सकते हैं। हम अपने कोड में एरर हैंडलिंग में स्टैंडर्ड एरर को उपयोग करने की सलाह देते हैं। इसके अलावा, हम अपने एप्लिकेशन को और उन्नत बनाने के लिए भी स्टैंडर्ड एरर को सही तरीके से प्रयोग कर सकते हैं।
