## क्यों

टेम्पररी फ़ाइल बनाने के पीछे कारण क्या है? ठीक है, हम आमतौर पर अपने प्रोजेक्ट में इस्तेमाल करने वाली फ़ाइलें बनाते हैं जो प्रोग्राम को स्थानांतरित किए बिना बिना हो सकता हैं। इससे हम सुनिश्चित करते हैं कि हमारा प्रोग्राम स्थाई रूप से तैयार होगा और अन्य मशीनों पर भी काम करेगा।

## कैसे करें

टेम्पररी फ़ाइल बनाना काफी आसान है। सबसे पहले, हम `Tempfile` क्लास का एक ऑब्जेक्ट बनाते हैं और `open` फ़ंक्शन को उसके साथ लगाते हैं। जब हम `write` फ़ंक्शन को ऑब्जेक्ट पर कॉल करते हैं, तो उस टेम्पररी फ़ाइल में डेटा लिखा जाता है। यहां एक उदाहरण है:

```Ruby
require 'tempfile'

# एक टेम्पररी फ़ाइल बनाएं 
temp_file = Tempfile.open("my_temp_file") 

# फ़ाइल में कुछ डेटा लिखें 
temp_file.write("यह एक संग्रह है")

# फ़ाइल से डेटा पढ़ें 
temp_file.rewind 
puts temp_file.read 

# फ़ाइल बंद करें 
temp_file.close 
```
इसका आउटपुट होगा:
```
यह एक संग्रह है
```

## गहराई में

टेम्पररी फ़ाइलें असली फ़ाइल नामों के स्थानांतरित नामों के रूप में जानी जाती हैं। यह करने के लिए, Ruby में `Tempfile` क्लास और स्थानांतरित फ़ाइलों के बीच एक प्रत्याय को इस्तेमाल करता है। इस तरह से, टेम्पररी फ़ाइलें अपने आप ही स्थानांतरित हो जाती हैं जब वे कोड में अधिकारिक रूप से बनाए जाते हैं।
