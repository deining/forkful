## क्यों

एक उपयोगकर्ता को एचटीटीपी अनुरोध भेजने के लिए बेसिक प्रमाणीकरण का उपयोग करने के पीछे कई कारण हो सकते हैं। एक ऐसा कारण हो सकता है कि एक वेब सर्वर प्रमाणीकरण के साथ एक्सेस को संयंत्रित करना चाहता हो। इससे सुरक्षित तरीके से साइट या अन्य सेवाओं को एक्सेस किया जा सकता है। 

## कैसे करें 

```Ruby 
require 'net/http'

uri = URI('https://www.example.com')
http = Net::HTTP.new(uri.host, uri.port)
http.use_ssl = true

request = Net::HTTP::Get.new(uri.request_uri)

# Add basic authentication headers
request.basic_auth('username', 'password')

response = http.request(request)

puts response.body
```

उपरोक्त कोड ब्लॉक में, हमने नेटः:एचटीटीपी को इंस्टॉल किया है और एक यूआरआई और नेटः: एचटीटीपी की स्थापना की है। उसके बाद हमने एचटीटीपी के इंस्टेंस को एक सुरक्षित कनेक्शन के साथ, डीफॉल्ट अनुरोध मोड के साथ इनलाइन कर हत्थ रिक्येस्ट बनाया है। आखर में, हमने एचटीटीपी को भेजते समय बेसिक प्रमाणीकरण हेडर्स को जोड़ा है और एचटीटीपी द्वारा दिए गए प्रतिक्रिया के बॉडी को प्रिंट किया है। 

## गहराई में जाएं

भेजे गए एचटीटीपी अनुरोध में बेसिक प्रमाणीकरण का उपयोग करने से पहले, हमें अपने एप्लिकेशन में हस्तांतरित करने की आवश्यकता हो सकती है। हम अपने खाते या यूजरनेम और पासवर्ड जैसे संज्ञाएं प्राप्त कर सकते हैं और उन्हें उपयोग करके उन्हें एचटीटीपी अनुरोध में भेज सकत
