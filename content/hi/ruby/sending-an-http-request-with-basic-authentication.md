## क्या और क्यों?
HTTP अनुरोध को बेसिक प्रमाणीकरण के साथ भेजना क्या है और प्रोग्रामर्स इसे क्यों करते हैं।

## कैसे करें:
बेसिक प्रमाणीकरण के साथ HTTP अनुरोध भेजने के लिए कुछ कोडिंग उदाहरण और सेम्पल आउटपुट।
```Ruby
require 'net/http'
uri = URI('https://example.com')
req = Net::HTTP::Get.new(uri)
req.basic_auth 'username', 'password'
res = Net::HTTP.start(uri.hostname, uri.port) {|http|
  http.request(req)
}
puts res.body
```

## गहराई में खोजें:
HTTP अनुरोध को बेसिक प्रमाणीकरण के साथ भेजने के साथ उसका इतिहास, विकल्प और कार्यान्वयन विवरण।

### इतिहास:
HTTP अनुरोध भेजने के मूल स्वरूप में एक चौभद्र ब्राज़ याद कीजिए, जो 1995 के दशक में इंटरनेट पर सबसे आम था। बेसिक प्रमाणीकरण वर्जन में प्रत्येक बिट (8 बिट) को किसी भी वास्तविक संख्या से अलग किया गया था।

### विकल्प:
HTTP अनुरोध को भेजने के अधिक प्रमाणीकरण विकल्प शामिल हैं जैसे कि इनटेग्रेटेड Windows प्रमाणीकरण, डाइजेस्ट प्रमाणीकरण और Oauth प्रमाणीकरण।

### कार्यान्वयन विवरण:
HTTP अनुरोध को भेजने के लिए बेसिक प्रमाणीकरण, वेब सर्वर द्वारा एक निश्चित उपयोगकर्ता नाम और पासवर्ड प्रदान करता है। उपयूक्त परीक्षण के बाद, एक 401 अधिसूचना साथ पुनरागामी त्रुटि नया अनुरोध भेजने के लिए प्रलंबित रहता है।

## और देखें:
सीखने के लिए अनुरोध शब्द को और उसके साथ प्रमाणीकरण कैसे लॉगिंग और प्रबंधित करने को।
