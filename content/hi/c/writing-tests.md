# क्यों
प्रोग्रामिंग में टेस्टिंग लिखने का क्या महत्व है? आईए जानते हैं। टेस्टिंग से हम अपने कोड की सटीकता और काम करने की क्षमता को जांच सकते हैं, जिससे हमें बेहतर कोडिंग चुनौतियों का सामना करने की क्षमता मिलती है। अपसे कहा जाए तो टेस्टिंग कोड का 'safety net' होता है, जो की हमें कोड के स्वास्थ्य को सुनिश्चित करने में मदद करता है।

# कैसे करें
चलिए समझते हैं की हम कैसे अपने तैयार किये हुए टेस्ट के माध्यम से कोड को जांच सकते हैं। अपने कोड में आप आमतौर पर `assert()` फंक्शन यूज़ कर सकते हैं। ये आपको दो तरीकों में सक्षम करता है: या तो आपके कोड स्विच ऑन और ऑफ हो जायेंगे या तो आपका टेस्ट केस फेल हो जायेगा और आपको इसके बारे में चेतावनी मिलेगी। नीचे एक उत्पाद जारी कर दिया है जहां हमने `assert()` का प्रयोग किया है, उसमें हमने `foo` नामक आईडेंटिफायर को अपने ब्लाक से फ्री होने से पहले 5 लहिरों के साथ अपडेट किया है। इस उत्पाद से हमारे कोड के तीन दुष्प्रभावों को परीक्षण कर सकते हैं जो की `foo` को संबद्ध करते हैं और लॉजिकल त्रुटियों को पकड़ सकते हैं।

```C
#include <assert.h>

int main()
{
    int foo = 10;
    assert(foo == 10); // पहला टेस्ट केस: सभी काम ठीक से हो रहा है
    foo++;
    assert(foo == 11); // दूसरा टेस्ट केस: वेख़ ठीक है पर लहिर नामक संबंध ठीक
