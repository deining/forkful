## क्या और क्यों?
डिबग आउटपुट को प्रिंट करना क्या है और क्यों प्रोग्रामर इसे करते हैं, इसे बताने के लिए कुछ सेंटेंसेस।

डिबग आउटपुट को प्रिंट करना एक तरीका है जिसके द्वारा हम अपने कोड के विभिन्न चरणों को ट्रैक कर सकते हैं। प्रोग्रामर इसे अक्सर अपने द्वारा बनाई गई गलतियों को सुधारने के लिए, कोड के जानने के लिए, और प्रोग्राम के प्रवाह को समझने के लिए करते हैं।

## कैसे करें?
यहां हम कुछ उदाहरण सहित कोडिंग करेंगे और ```C ... ``` कोड ब्लॉक्स के भीतर सैंपल आउटपुट दिखाएंगे।

```
#include <stdio.h>

int main() {
    int x = 5;
    int y = 10;
    int result = x + y;

    printf("Adding x and y gives result: %d", result);

    return 0;
}
```

आउटपुट:
```
Adding x and y gives result: 15
```

```
#include <stdio.h>

int main() {
    int num = 1;

    while (num <= 5) {
        printf("%d", num);
        num++;
    }

    return 0;
}
```

आउटपुट:
```
12345
```

## गहराई में जाएं
इस अनुभाग में हम (1) इतिहासी संदर्भ, (2) वैकल्पिकों, और (3) डिबग आउटपुट को प्रिंट करने के कुछ विवरण जैसे प्रदर्शित करेंगे।

### इतिहासी संदर्भ
डिबग आउटपुट को प्रिंट करने का आरम्भ बहुत पहले किया गया। इसे पहले प्रोग्रामर अपनी गलतियों को सुधारने के लिए अपने कोड में हार्ड कोडेड मैसेज या परिचर्या संकेतों का उपयोग करते थे। लेकिन जैसे तकनीकी प्रगति हुई, यह विधि प्रोग्रामिंग में आम बन गई है।

### वैकल्पिक
कुछ प्रोग्रामर अपने कोड में डिबग आउटपुट को प्रिंट करने के बजाय डिबगर का उपयोग करते हैं। डिबगर एक सहेजनीय ऑप्शन है जिसके द्वारा प्रोग्रामर चर, भांग, या वेरियबल्स को सामान्य रोगों वाले प्रोग्राम पर क्रॉस कर सकते हैं। इससे बहुत समय बचाए जाते हैं और कोड को अधिक प्रोफेशनल तरीके से सुधारा जा सकता है।

### डिबग आउटपुट को प्रिंट करने की विधि
जब डिबग आउटपुट प्रिंट किया जाता है, सामान्यतया त्रुटि संदेश मूल रूप से एक चेतावनी को दर्शाते हैं, जो प्रोग्राम के स्थिति को समझने में मदद करती है। कुछ प्रोग्रामर एक साथ कई त्रुटि
