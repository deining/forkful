# क्या और क्यों?

लेखकों को अपने कोड को टेस्ट करने के लिए संगठित सेट ऑफ़ टेस्ट केस बनाने को यह करना होता है कि उनका कोड स्टेबल, निश्चित और विश्वसनीय हो।
आमतौर पर, लेखकों इसको सुनिश्चित करने के लिए करते हैं कि उनका कोड आवश्यकताओं को समाप्त करता है, इनवायरमेंट और चेतावनियों के लिए तैयार होता है और साइबर सुरक्षा के संकेत और सुरक्षा द्वारा आक्रमणों से निपटने के लिए तैयार होता है। टेस्टिंग कोड के महत्व को समझने के लिए, ऑनलाइन बैंकिंग और अन्य सुरक्षित वेबसाइटों को सोचें जहां टेस्टिंग काएमल तत्र अत्यंत महत्वपूर्ण होता है।

# कैसे करें?

**C++** में टेस्टिंग को आसान बनाने के लिए, हम एक उदाहरण साथ में देते हैं। यहाँ हम पहले एक फंक्शन बनाते हैं जो दो मूल्यों के बीच फर्क परीक्षण करता है। फिर हम उसकी गुणवत्ता को टेस्ट करते हैं।

```C++
// यह फंक्शन दो मूल्यों के बीच अन्तर को टेस्ट करता है
bool checkDifference(int x, int y) {
    if (x > y) {
        return true;
    } else {
        return false;
    }
}

// यह पूर्वावलोकन फंक्शन को योग्य मूल्यों के साथ टेस्ट करता है
int main() {
    // यहां हमारे फंक्शन को योग्य मानों के साथ टेस्ट करते हैं
    assert(checkDifference(10, 5)); // सही होना चाहिए
    assert(!checkDifference(5, 10)); // गलत होना चाहिए
    return 0;
}
```

जैसा कि आप देख सकते हैं, हमने प्रोग्राम में ```assert()``` फंक्शन का इस्तेमाल किया है जो कि सामान्य टेस्टिंग में उपयोगी होता है। इससे हम सुनिश्चित कर सकते हैं कि हमारा कोड सही है या नहीं। हम अपने कोड में भी अन्य टेस्टिंग फंक्शन और लिब्रेरी का इस्तेमाल कर सकते हैं, जैसे कि **Google Test** और **CppUnit**। 

# गहराई में जाइए

टेस्टिंग के बारे में तो आप पहले से भी ही जानते हैं, लेकिन इसका इतिहास आपके लिए रोचक हो सकता है। पूर्व में, लेखकों को खुद खुद अपने कोड को टेस्ट करना होता था। परंतु ऑटोमेटेड टेस्टिंग के आगमन से, लेखकों को अपने कोड को पहचानने और गल्तियों को सुधारने के लिए अधिक समय और ऊर्जा काटन
