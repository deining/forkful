## क्यों

टेस्ट लिखने का प्रमुख कारण है कि एक डेवलपर अपनी कोड को सुनिश्चित करना चाहता है कि वह उन सभी प्रत्येक स्थिति को सुसंगत और सही ढंग से देख सकता है जो कि उन्हें तेज़ी से उनके कोड में परिवर्तन प्रभावित कर सकती है।

## कैसे करें

```C++
#include <iostream>

using namespace std;
 
 // फ़ंक्शन बनाएं जो दो संख्याओं को जोड़ता है
 int add(int x, int y) {
     return x + y;
 }
 
 int main() {
     // अपनी जोड़ी गई संख्याओं को प्रिंट करें
     cout << "2 + 3 = " << add(2, 3) << endl;
     return 0;
 }

```

इस कोड में, हमने दो संख्याओं को जोड़ने के लिए एक add() फ़ंक्शन बनाया है और इसे main() फ़ंक्शन में कॉल किया है। मैं main() फ़ंक्शन के भीतर जोड़ी गई संख्याओं को प्रिंट कर रहा हूँ। यदि कोड सही है तो कंपाइलर add() फ़ंक्शन को बुलाने के लिए कॉल करेगा और 2 और 3 को तोड़ देगा जो एक देंगे। फिर कंपाइलर main() फ़ंक्शन को बुलाएगा जिससे 5 होगा जो डेटायप (data type) दूसरे अर्गुमेंट देगा। आप 2 + 3 = 5 दो संख्याओं को जोड़ते हैं।
