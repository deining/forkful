## क्या और क्यों?

कोई भी प्रोग्रामर कभी-कभी ऐसी स्थिति से गुजरता है जब उन्हें अपने कोड को एक साथ चलाने के लिए कुछ स्थान की आवश्यकता होती है। इस समस्या का एक समाधान है अस्थायी फ़ाइल (temporary file) बनाना। अस्थायी फ़ाइल एक गुणवत्ता कोडिंग प्रैक्टिस है जो प्रोग्रामर्स को कोड को चलाने में आसानी प्रदान करता है।

## कैसे:

अस्थायी फाइल बनाने के लिए आप पहले ```fstream``` लाइब्रेरी को इनपुट ऑब्जेक्ट के साथ इनिश्लीज कर सकते हैं। नीचे दिए गए कोड ब्लॉक में दिखाए गए कंपाइल और एक्सिक्यूट करने के उत्पाद है।

```C++
// अस्थायी फ़ाइल बनाने के लिए फ़ाइल स्ट्रिम जोड़ें
#include <fstream>

int main() {
    // फ़ाइल का नाम और पथ निर्धारित करें
    char* fileName = "tempFile.txt";
    
    // अस्थायी फ़ाइल बनाएं
    std::ofstream tempFile;
    // फ़ाइल ओपन करें
    tempFile.open (fileName);
    
    // अस्थायी फ़ाइल में कुछ लिखें
    tempFile << "यह अस्थायी फ़ाइल से आया है" << std::endl;
    
    // अस्थायी फ़ाइल को बंद करें
    tempFile.close();
    
    // सफलतापूर्वक फ़ाइल बनाएं और उसे पढ़ें
    return 0;
}
```

अस्थायी फ़ाइल बनाने का प्रोसेस संबंधित हो सकता है और यह सभी लिनक्स, विंडोज और मैक ओएस प्लेटफ़ॉर्म्स पर उपलब्ध है।

## गहराई में जाएं:

### इतिहासिक पृष्ठभूमि:

अस्थायी फाइल का प्रयोग प्राथमिकतापूर्वक पेशकश से किया गया था। इसका उद्देश्य एक तरह से यूसर डेटा को स्टोर करना था जो प्रोग्रामर से इन्पुट लेकर उसे चलाते हुए पास कर सकता था।

### वैकल्पिक:

अस्थायी फ़ाइल बनाने के लिए कुछ अन्य विकल्प भी हैं जैसे कि सोकेट फ़ाइल और रीड ऑनली फ़ाइल जेस्ट (ROT)।

### अनुमानित विवरण:

अस्थायी फ़ाइल बनाने का प्रक्रिया अन्य फ़ाइल्स को बनाने के लिए साधारण रूप से उपयोग किए जाने वाले सिस्टम कॉल से मिलता है। अनुमानित रूप से, कई लिनक्स यूज़र अर्ज़ में उपयोग किया जाता है और वे किसी भी समय फ़ाइल को डिलीट कर सकते हैं।
