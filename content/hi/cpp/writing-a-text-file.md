## क्या और क्यों?
एक पाठ फ़ाइल लिखना क्या है और प्रोग्रामर इसे क्यों करते हैं, इसके बारे में दो तीन वाक्यों में व्याख्या।

जब हम किसी कंप्यूटर पर जानकारी को संग्रहीत करना चाहते हैं या अपने प्रोग्राम के दौरान कुछ डाटा रिकॉर्ड करना चाहते हैं, तो हम पाठ फ़ाइल लिखते हैं। पाठ फ़ाइल एक प्रकार का डॉक्युमेंट होता है जो टेक्स्ट को संग्रहीत करता है और हम अपने प्रोजेक्ट के लिए उस डाटा को बाद में उपयोग कर सकते हैं। 

## कैसे करें:
कोड उदाहरण और ```C++...``` कोड ब्लॉक के भीतर सैंपल आउटपुट।

```C++
#include <fstream> 
using namespace std;

int main() 
{ 
  ofstream file("sample.txt"); // file object to write to sample.txt

  if(file.is_open()) // checks if file is opened successfully
  {
    file << "Hello, World!"; // writing text to file
    file.close(); // closing the file
    cout << "Data written to sample.txt"; // output to console 
  }
  else
  {
    cout<<"Unable to open file"; // prints error message if file cannot be opened
  }
  
  return 0; 
}
```
और यदि हम sample.txt पर चलाते हैं, तो हमें "Hello, World!" लिखा हुआ दिखेगा।

## गहराई में जाएं:
पाठ फाइल लेखन से संबंधित (1) ऐतिहासिक पृष्ठभूमि, (2) वैकल्पिक विकल्प और (3) इम्प्लीमेंटेशन विवरण इस अनुभाग में दिया गया है। 

### ऐतिहासिक पृष्ठभूमि:
पहले, पाठ फाइलें सिर्फ टाइपराइटर या स्क्रीन डिस्प्ले के रूप में लिखी जाती थीं। इसके बाद समय के साथ, कंप्यूटर संग्रहीत किए गए डाटा को स्टॉर करने के लिए हार्ड ड्राइव का उपयोग करने लगे जो अब भी उपयोग किए जाते हैं।

### वैकल्पिक विकल्प:
अलग फार्मेटों के साथ पाठ फाइलें भी लिखी जा सकती हैं जैसे कि .doc, .pdf, .rtf, आदि। हालांकि, पाठ फ़ाइल अब भी देर से प्रभावी है अगर हमें सिर्फ पाठ को संग्रहीत करना हो तो।

### इम्प्लीमेंटेशन विवरण:
पाठ फ़ाइलें पाठ को ASCII, UTF-8 या अन्य कैरिएक्टर सेट में लिखती हैं। मानक ASCII स्वरूप के अलावा, UTF-8 भी उच्च शार्ट कोड के लिए उपयोग किया जाता है।
