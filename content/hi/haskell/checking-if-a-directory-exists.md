## क्या व क्यों?

डायरेक्टरी की मौजूदगी जांचना क्या है और क्यों प्रोग्रामर्स इसे जांचते हैं? डायरेक्टरी की मौजूदगी जांचना एक प्रोग्रामिंग टेक्निक है जो वहाँ है या नहीं है, उन फ़ाइलों को पहचानने में मदद करती है जो आपके कोड के भीतर आपको निर्दिष्ट निर्देशों अनुसार संगठित करती है।

## कैसे करें:

```Haskell
import System.Directory

doesDirectoryExist :: FilePath -> IO Bool
```

डायरेक्टरी की मौजूदगी की जांच करने के लिए, हमें फ़ाइल पथ को प्रदान करना है और कार्यक्रम को प्राप्त होने वाले बाहरी डायरेक्ट्री के सामने या नीचे सोचने के लिए विकल्प को संबोधित करना है। डायरेक्टरी मौजूद है तो यही चीज़ की जांच करेगा और True वापस देगा, यदि नहीं, तो False वापस देगा।

उदाहरण:
```Haskell
> doesDirectoryExist "C:\\Users\\username\\Downloads"  -- फ़ाइल सिस्टम पर Windows
True
> doesDirectoryExist "/home/username/Documents" -- फ़ाइल सिस्टम पर Unix
True
> doesDirectoryExist "C:\\Users\\username\\Music\\Playlist"  -- एक फाल्डर नहीं है या पुष्टि है
False
```

## गहरी परख़:

### ऐतिहासिक संदर्भ:

डायरेक्टरी की मौजूदगी जाँचना अन्य प्रोग्रामिंग भाषाओं में भी मौजूद है, जैसे कि C और Python में os.path.exists() फ़ंक्शन होता है। Haskell में, हम System.Directory मॉड्यूल का उपयोग करते हैं, जो डायरेक्टरी की मौजूदगी जाँचता है और हमें यहाँ उपलब्ध कराता है।

### विकल्प:

यदि हम डायरेक्टरी की मौजूदगी को पता करने के लिए आगे बढ़ते हैं, तो हम अन्य कई विकल्पों का उपयोग कर सकते हैं, जैसे कि System.Posix.Files और System.Posix.Directory मॉड्यूल। हालांकि, हम यहां सिस्टम डायरेक्टरी मॉड्यूल का उपयोग करने की सलाह देते हैं, जिसे कि आसानी से उपयोग किया जा सकता है और इसमें जितना काम हो सकता हो, हमारे पास वह है।

### अंतर्निहित जानकारी:

डायरेक्टरी की मौजूदगी की जाँच करने के लिए, मॉड्यूल System.Directory एपीआई प्रदान करता है। इस एपीआई का उपयोग करने के लिए, हमें System.Directory मॉड्यूल ने आप Indians में निर्दिष्ट किया है और हम यहाँ पहले द्वारा उपलब्ध कुछ My India कई फुंग्क्सन्स हमें प्राप्त ह
