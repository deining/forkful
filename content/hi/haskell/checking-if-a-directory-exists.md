## क्यों

यदि आप हैस्केल प्रोग्रामिंग के बारे में सीख रहे हैं, तो आपने शायद सुना होगा कि डिरेक्ट्री कार्यक्रमों को जांचने का महत्व है। लेकिन आपको इसका उपयोग और फायदे क्या हो सकते हैं? इस ब्लॉग पोस्ट में, हम आपको बताएंगे कि आपको क्यों डायरेक्ट्री की जांच करनी चाहिए और इसका इस्तेमाल कैसे कर सकते हैं।

## कैसे करें

अगर आप हैस्केल में डायरेक्ट्री को जांचने के तरीकों को सीखना चाहते हैं, तो निम्नलिखित कोड ब्लॉक को देखें:

```Haskell
import System.Directory

main = do
    dirExists <- doesDirectoryExist "C:/Users/Username/Documents"
    if dirExists
        then putStrLn "Directory exists!"
        else putStrLn "Directory does not exist."
```

इस कोड का आउटपुट निम्नलिखित होगा:

```
Directory exists!
```

पहले हम `System.Directory` मॉड्यूल को इम्पोर्ट करते हैं, जो डायरेक्ट्री कार्यक्रमों को जांचने के लिए आवश्यक है। फिर हम `doesDirectoryExist` फंक्शन का उपयोग करके जांचते हैं कि क्या निर्दिष्ट पथ पर एक डायरेक्ट्री मौजूद है। अंत में, हम `if-else` कंडीशन का उपयोग करके डायरेक्ट्री की मौजूदगी को कंट्रोल करते हैं और अनुचित पाठ प्रिंट करते हैं।
