## क्यों

प्रोग्रामिंग में टेस्ट कैसे लिखा जाता है, यह आपके कोड की गुणवत्ता को सुनिश्चित करने का एक महत्वपूर्ण हिस्सा है। यह आपको बग्स से बचाता है और आपके ऐप्स को बेहतर बनाता है।

## कैसे करें

```Go
func Add(x int, y int) int {
	return x + y
}

func TestAdd(t *testing.T) {
	result := Add(2, 3)
	if result != 5 {
		t.Errorf("Add function did not return expected result! Expected: %v, Got: %v", 5, result)
	}
}
```

पहले स्थान पर, हम टेस्ट करने वाले फंक्शन को बनाते हैं, जिसमें हम अपने कोड को टेस्ट करेंगे। `testing` पैकेज का उपयोग करके हम एक नया टेस्ट केस बनाते हैं जो हमारे कोड को चलाएगा और इसका एआउटपुट चेक करेगा। यदि एक्सपेक्टेड रिजल्ट और गोट रिजल्ट मिलते हैं, तो टेस्ट केस सफल होगा, और यदि नहीं, तो एक एरर मैसेज प्रिंट होगा। इस तरह, हम आसानी से अपने कोड को टेस्ट कर सकते हैं और उसमें से बग्स को धुंध सकते हैं।

## गहराई में जाएं

टेस्ट केस कोडिंग के अलावा, यह भी महत्वपूर्ण है कि हम अपने टेस्ट कोड को और भी अधिक समझें। टेस्ट केस लिखने के लिए `testing` पैकेज के भीतर `t` वेरिएबल का उपयोग करना अनिवार्य है, जो हमें एक `Fail` या `Error` फंक्शन को कॉल करना हो सकता है। इसके अलावा, हम अपने कोड में टेस्ट करने के लिए आवश्यक फंक्शन को बनाना चाहिए। यह सुनिश्चित करेगा कि हमारे कोड में हर चलने में कोई दोष नहीं ह
