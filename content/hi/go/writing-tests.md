# क्यों

टेस्ट लेखन क्यों महत्वपूर्ण है? टेस्टिंग एक क्रिटिकल अंग है जो कोड की सहीता और फंक्शनलिटी को सुनिश्चित करता है। टेस्ट सुनिश्चित करते हैं कि कोड ठीक से चल रहा है और कोई नए बग निकाले जाते हैं।

# कैसे करें

व भाषा में टेस्ट लेखन बहुत सरल है! आप सीधे अपने कोड में टेस्ट कर सकते हैं। नीचे गो भाषा में कुछ कोड ब्लॉक है जो आपको समझने में मदद करेंगे:

```Go
// यह एक साधारण कॉन्सोल आउटपुट है
package main

import "fmt"

func main() {
  fmt.Println("नमस्ते दुनिया!")
}

// यह एक बैरीयर चेक करता है
package main

import "testing"

func TestAdd(t *testing.T) {
  result := Add(2, 2)

  if result != 4 {
    t.Errorf("गलत परिणाम, उम्मीद से अलग: %d", result)
  }
}

func Add(x, y int) int {
  return x + y
}
```

# गहराई तक जाएं

टेस्टिंग कोड लिखना सरल तो हो सकता है, लेकिन अधिक गहराई से इसे समझना महत्वपूर्ण है। टेस्ट कस्टम कोड को सही करने में मदद कर सकते हैं और सुनिश्चित कर सकते हैं कि सभी यूनिट्स ठीक से काम कर रहे हैं। आप अपनी अपनी वेबसाइट में टेस्ट चला सकते हैं जो सुरक्षा को सुनिश्चित करेगा।
