## क्या और क्यों?

कोई समय-सीमित फ़ाइल तैयार करना प्रोग्रामरों के लिए काफी उपयोगी हो सकता है क्योंकि यह अनेक प्रोग्रामिंग कार्यों को सुगम और आसान बना देता है। टेम्प फाइल तैयार करने का मुख्य उद्देश्य है कि इससे अस्थायी डेटा को संचित और संपादित किया जा सके बिना मूल फ़ाइल को हानि पहुँचाने के।

## कैसे करें:

स्टैंडार्ड गो पैकेज इनपुट/आउटपुट सुविधाओं का उपयोग करके हम अस्थायी फ़ाइल तैयार कर सकते हैं। नीचे दिए गए कोड उदाहरण द्वारा इसका दिखाया गया है।

```Go
// टेम्प फाइल तैयार करने के लिए, यूनिक नाम तक पहुँच का उपयोग करें
f, err := ioutil.TempFile("", "example")

if err != nil {
    fmt.Println("फाइल तैयार नहीं हो सकी।", err)
}
defer os.Remove(f.Name()) // प्रोग्राम समाप्त होने पर फ़ाइल हटा दी जाएगी

// फाइल में डेटा टाइप करें
if _, err := f.Write([]byte("धन्यवाद।")); err != nil {
    fmt.Println("फाइल लिखने में त्रुटि।", err)
}
if err := f.Close(); err != nil {
    fmt.Println("फाइल बंद करने में त्रुटि।", err)
}
```

उपरोक्त कोड अस्थायी फ़ाइल बनाने और उसमे डेटा लिखने के लिए यूनिक नाम तक पहुँच का उपयोग करता है। फ़ाइल बंद होने पर यह स्वचालित रूप से हटा दी जाती है, इसलिए कोई भी मौजूदा फ़ाइल को नुकसान पहुँचाई नहीं जानी चाहिए।

## गहराई में जाएं:

अस्थायी फ़ाइलों का अवसर अत्यधिक स्मार्ट नहीं था। पहले यह सिस्टम से बंद फ़ोल्डर में फ़ाइलें बनाता था, लेकिन आजकल इसे आसानी से नियंत्रित करने के लिए बनाया गया है। अन्य विकल्प आती हैं, जैसे कि टेम्प फाइलों के साथ व्यापक विकल्प, जो जीवनकाल, सुरक्षा और संपादन क्षमताओं को समायोजित करती हैं।

टेम्प फाइल तैयार करने के अलावा, हम गो प्रोग्रामिंग में उपयोगी अन्य I/O सुविधाओं के बारे में भी सीख सकते हैं। इस विषय पर संबंधित लिंक नीचे दिए गए हैं।
