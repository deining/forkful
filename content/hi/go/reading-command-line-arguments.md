## क्या और क्यों?
कमांड लाइन आर्ग्यूमेंट्स को समझना एक प्रोग्रामिंग में जोर देता है। ये आर्ग्यूमेंट प्रोग्राम के साथ इनपुट के रूप में काम करते हैं और प्रोग्रामर्स को अपने कोड को प्रोग्राम की तरह चलाने की अनुमति देते हैं। 
प्रोग्रामर्स को कमांड लाइन आर्ग्यूमेंट्स की जरूरत होती है जब उनके पास अपनी प्रोग्राम को स्पष्ट करने और उनकी जरूरतों के अनुसार निर्देशों को पारिभाषित करने की आवश्यकता होती है।

## कैसे करें:
कमांड लाइन आर्ग्यूमेंट्स को समझने के लिए, आपको अपने गो प्रोग्राम में 'os' और 'fmt' लाइब्रेरी को इंपोर्ट करना होगा। आप इस लाइब्रेरी को उपयोग करके 'os.Args' में से अपने कमांड लाइन आर्ग्यूमेंट्स को प्राप्त कर सकते हैं। यहां आपको एक उदाहरण ऊपर की तरह दिया गया है:

```Go
package main

import (
  "fmt"
  "os"
)

func main() {
  fmt.Println(os.Args[0], os.Args[1:])
}
```

जब आप अपने प्रोग्राम को एक आर्ग्यूमेंट के साथ कॉल करेंगे, आपको उस आर्ग्यूमेंट को अपने प्रोग्राम द्वारा प्रिंट किया जाएगा। उपरोक्त उदाहरण में, प्रोग्राम को स्थानीय वर्ग में चलाने के लिए निम्नलिखित कमांड इस्तेमाल किया गया है:

```Go
go run main.go "Hello World"
```

इससे आपको "Hello World" का आउटपुट मिलेगा।

## विस्तृत अध्ययन:
कमांड लाइन आर्ग्यूमेंट्स को समझने के अलावा, प्रोग्रामर्स कई अन्य विकल्प भी अपनाते हैं जैसे फ़्लैग्स, वॉरियेबल्स या कॉन्फिगरेशन फाइल्स। कमांड लाइन आर्ग्यूमेंट्स से अधिक लंबा होने का कारण यह है कि वे प्रोग्राम की प्रवेश सरणी (entry array) के रूप में उपयोग किए जाते हैं,जो प्रोग्राम के साथ जुड़ा रहता है।
