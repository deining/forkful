## क्या और क्यों?
HTTP अनुरोध भेजना एक प्रायोगिक तकनीक है जो किसी अनुप्रयोग को एक सिद्धांत से संपर्क करने के लिए सरल और सुरक्षित ढंग से सेट करता है। यह अपने BASIC प्रमाणीकरण विधि से जाना जाता है जो एक उपयोगकर्ता नाम और पासवर्ड का उपयोग करते हुए डेटा को एन्क्रिप्ट करता है। प्रोग्रामर्स अपने अनुप्रयोगों में सुरक्षा को बढ़ाने के लिए इस तकनीक का उपयोग करते हैं।

## कैसे करें:
इस तकनीक का उपयोग करने के लिए, हम क्लोजर के [clj-http](https://github.com/dakrone/clj-http) पुस्तकालय का उपयोग करते हैं। सही URL और प्रमाणीकरण जानकारी के साथ `http://client/get` के साथ `:basic-auth` खंड को उपयोग करके HTTP GET अनुरोध भेजा जा सकता है, जैसे कि निम्न दिखाया गया है:

```clojure
(require 'clj-http.client)

(def url "https://example.com")
(def username "user")
(def password "pass")

(clj-http.client/get url 
                      {:basic-auth [username password]})
```

उपरोक्त कोड ने एक एचटीटीपी अनुरोध भेजने के लिए, URL का उपयोग करते हुए, सरल एचटीटीपी प्रमाणीकरण के साथ एक डेटा विनिर्देश वृत्ति का उपयोग कर एक उत्तर प्राप्त किया है। यह प्रतिक्रिया सफलतापूर्वक प्राप्त हुई है दर्शाती है। हम चूंकि इस URL को साधारणता से पथ्यय पहुंचके हुए हैं, हम अपनी प्रमाणीकरण जानकारी को जानते हैं, इसलिए हमारे अनुरोध में आसानी से अतिरिक्त सुरक्षा है।

## गहराई में जाएं:
HTTP की प्राचीन एवं विकास का परिचय कुछ तत्वों पर आधारित है जो वर्तमान मुख्य एचटीटीपी प्रोटोकॉल में शामिल है। अल्टर्नेटिव के रूप में, एचटीटीपी प्रोटोकॉल ने बहुत सारे सुरक्षा विसंगतियों का आतंक रखा है लेकिन यह पहली चरण में BASIC उपयोगकर्ता प्रमाण की अभ्यास तकनीक को पिछली संस्करण की तुलना में सुरक्षित बनाने के लिए जोड़ दिया गया था।

## उर्इदर्ए और देखिए:
इस लेख में हमने एक सरल और उपयोगी तकनीक हाल की उदाहरणों के साथ जाना है जो Clojure में HTTP अनुरोध भेजने के लिए BASIC प्रमाणीकरण की सुरक्षा को सुनिश्चित करता है। अधिक जानने के लिए, आप [clj-http documentation](https://github.com/dakrone/clj-http) के लिए जाएं।
