## क्यों

टेस्ट लिखने का हमें क्यों होना चाहिए? वह इसलिए कि यह हमारे कोड को बेहतर बनाता है, इसे अधिक सुरक्षित बनाता है और स्पष्टीकरण देता है कि हमारा कोड ठीक काम कर रहा है या नहीं।

## कैसे करें

```Swift
func addNumbers(num1: Int, num2: Int) -> Int {
    return num1 + num2
}

print(addNumbers(num1: 5, num2: 10))
// Output: 15
```

टेस्ट लिखने के लिए, हम स्विफ्ट कोड में टेस्ट केस बनाते हैं जो हमारे फंक्शन को कॉल करें और उसका आउटपुट सही है या नहीं वह जांचते हैं। अगर आउटपुट सही है, तो टेस्ट केस पास हो जाएगा और यदि आउटपुट गलत है, तो टेस्ट फेल होगा। स्विफ्ट में, हम टेस्टिंग के लिए इनबिल्ट XCTest फ्रेमवर्क का उपयोग कर सकते हैं।

## गहराई में जाएं

टेस्ट लिखना हमारी कोड की गुणवत्ता को बढ़ाएं, इसे एक आसान तरीके से परीक्षण करने में मदद करता है, और एप्लिकेशन को अनुरोधों का मुंहतोड़ निपटान करने में मदद करता है। स्विफ्ट में, हम पुनर्निर्माण (refactoring) को आसान बनाने के लिए टेस्ट कवरेज का भी उपयोग कर सकते हैं। टेस्ट लिखना ज़रूरी है याद रखें कि हो सकता है यह थोड़ा समय ले, लेकिन यह उस समय को बचाने में हमें भावनाएं (emotions) देंगे!
