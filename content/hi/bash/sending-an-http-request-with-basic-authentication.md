## क्यों

कोई भी प्रोग्रामर क्यूँ HTTP रिक्वेस्ट भेजने में साधारण प्रमाणीकरण का उपयोग करता है, उसके पीछे का कारण क्या है? यह ये सुनिश्चित करने के लिए करता होगा कि सभी उपयोगकर्ताओं की पहचान सत्यापित हो जाती है और गलत उपयोग से सुरक्षा रखी जा सकती है।

## कैसे करे

इस लेख में हम आपको कैसे Bash में HTTP रिक्वेस्ट भेजने में प्रमाणीकरण का उपयोग करते हैं यह बताएंगे।

हम चाहेंगे कि आपका क्लाइंट यूआरएल हो। इसका मतलब है कि कोई पंक्तियां घोषणा नहीं होने चाहिए हम आपको एक स्लौग्रा यूआरएल दर्ज करने के लिए बाध्य करते हैं। इस आदेश को सामान्य भाषा में लंबा बनाता है कि क्या हे।

```Bash
curl -u username:password URL
```
जहा *username* और *password* की जगह आपके प्रमाणीकरण क्रेडेंशियल होने चाहिए और *URL* होना चाहिए। इस आदेश में, हमने "केक्डेंशियल" को एक चर्चाओं के तरह क्रियाकलापों के लिए वह किया है जो आप आम यूज़र के लिये कर रहे होने स्हान पर किया है।
