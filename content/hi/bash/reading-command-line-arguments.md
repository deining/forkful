## क्यों
अगर आप अपने बैश स्क्रिप्ट में उपयोगकर्ताओं से आउटपुट को सामान्य रूप से ही लेना चाहते हैं तो आपको कमांड लाइन आर्ग्यूमेंट्स को पढ़ने के लिए एक अच्छा समझ होना चाहिए।

## कैसे करें
एक उदाहरण के साथ आप कैसे बैश में कमांड लाइन आर्ग्यूमेंट्स को पढ़ सकते हैं और एक सामान्य आर्ग्यूमेंट को ओर से एक मेसेज दे सकते हैं, यह निम्न उदाहरण में दिखाया गया है।

```Bash
#!/bin/bash

if [ $# -eq 1 ]; then
    echo "आपने कह दिया कि $1 कुछ है।"
else
    echo "आपने कुछ कहा ही नहीं।"
fi
```

जब आप यह स्क्रिप्ट रन करेंगे, CMD में आप अपने आर्ग्यूमेंट को देख सकेंगे।

```Bash
$ ./script.sh अनुपम
आपने कह दिया कि अनुपम कुछ है।
```

और अगर आप स्क्रिप्ट रन करके कुछ भी नहीं बोलते हो तो आपको निम्न मैसेज मिलेगा।

```Bash
$ ./script.sh
आपने कुछ कहा ही नहीं।
```

## गहराई में जाएं
कमांड लाइन आर्ग्यूमेंट्स को पढ़ने का एक प्रभावी तरीका ```getopts``` का उपयोग करना है। इस टूल को स्क्रिप्ट के शुरु के हिसाब से उपयोगकर्ता से आर्ग्यूमेंट पढ़ने का अनुमति देता है। इससे किसी भी आर्ग्यूमेंट की जगह नहीं है वैसे जो आप कहते हैं। ```getopts``` के साथ आप स्क्रिप्ट को अधिक अभिन्नता दे सकते हैं और उसे में सही गाइड बनाने के लिए सिद्ध कर सकते हैं
