## क्या और क्यों?

HTTP अनुरोध भेजना एक आम सी तकनीक है जो विभिन्न वेब साइटों और एप्लिकेशनों को कनेक्ट करता है। प्रोग्रामर्स इस काम को भी करते हैं अपने प्रोग्राम्स में डेटा को पाने और उपयोग करने के लिए।

## कैसे करें:

```Bash
curl http://www.example.com
```
यह उदाहरण एक प्रोग्रामर को कमांड लाइन पर एक HTTP अनुरोध भेजने के लिए दिखाता है। तब आप वेब साइट से प्राप्त डेटा की एक सादी सूची देखेंगे।

```Bash
HTTP/2 200
server: nginx
content-type: text/html; charset=UTF-8
```

यह देखने के लिए है कि अनुरोध सफलतापूर्वक प्राप्त हुआ और वेब साइट द्वारा भेजे गए जवाब में क्या है। यह उत्तर कोड, सर्वर इमेर, और कंटेंट का प्रकार दिखाता है।

## गहराई में जाएं:

HTTP अनुरोधों की शुरुआत 1991 में टिम बर्नस-ली द्वारा की गई थी और यह अब भी एक प्रमुख तकनीक है। इसके अलावा, कई अन्य तकनीक भी हैं जो इंटरनेट या दूसरे संचार सर्वरों के साथ संवाद करने के लिए प्रयुक्त हो सकते हैं, जैसे कि FTP या SMTP। पहले से HTTP अनुरोध भेजने के लिए कई उपकरण मौजूद हैं जैसे कि cURL या Wget। इनमें से कुछ कमांड लाइन उपयोगकर्ताओं के लिए ही हैं, जबकि दूसरे भी उपयोगकर्ताओं को समर्थित करते हैं, जैसे कि अनुरोधों को अपने प्रोग्राम में शामिल करना।
