## क्यों

बश को एक लोकप्रिय शेल प्रोग्रामिंग भाषा के रूप में जाना जाता है जो लिनक्स और अन्य ऑपरेटिंग सिस्टमों पर उपलब्ध है। लिखने के दौरान स्टैंडर्ड त्रुटि पर लिखना काफी उपयोगी हो सकता है क्योंकि इससे हम दिशानिर्देशों और संदेशों को स्टैंडर्ड त्रुटि में देख सकते हैं और अपनी स्क्रिप्ट में किसी भी समस्या को आसानी से ठीक कर सकते हैं।

## कैसे करें

यहां कुछ कोड उदाहरण है जो आपको बताएंगे कि कैसे आप एक स्टैंडर्ड त्रुटि में लिख सकते हैं।

```Bash
# एक समान संदेश लिखना
echo "यह एक समान संदेश है" >&2

# विभिन्न सूचनाओं को स्टैंडर्ड त्रुटि से भिन्न रंगों में प्रिंट करना
# पीला - warning, लाल - error, हरे - success
echo -e "\033[33mयह एक पीला सूचना है\033[0m" >&2
echo -e "\033[31mयह एक लाल सूचना है\033[0m" >&2
echo -e "\033[32mयह एक हरा सूचना है\033[0m" >&2
```

आपका त्रुटि संदेश इस तरह से दिखेगा:
```
यह एक समान संदेश है
यह एक पीला सूचना है
यह एक लाल सूचना है
यह एक हरा सूचना है
```

## गहराई में जाएं

आप अपने बश स्क्रिप्ट में `>&2` का उपयोग करके स्टैंडर्ड त्रुटि में लिख सकते हैं। यह कमांड स्टैंडर्ड एरर को स्टैंडर्ड आउटपुट के स्थान पर भेज देता है, जिससे हम उपयोगकर्ताओं को अपने स्क्रिप्ट में कोई भी त
