## क्या और क्यों?
टेस्ट लिखना क्या है और उसे कैसे करते हैं? यह वह तकनीक है जो डेवलपर्स को अपनी कोड की गुणवत्ता और स्थिरता को जांचने में मदद करती है। आमतौर पर, प्रोग्रामर अपने कोड में गलतियां खोजने और उन्हें ठीक करने के लिए टेस्ट लिखते हैं।

## कैसे करें:
 चलिए अब हम देखें कि टाइपस्क्रिप्ट में टेस्ट को कैसे लिखा जाता है।
 ```Typescript
// add function
function add(x: number, y: number): number {
  return x + y;
}

// example test case
console.log(add(2, 3)); // output: 5
```
इस कोड में, हमने एक `add` फंक्शन बनाया है जो दो नंबरों को जोड़ता है। फिर हमने एक उदाहरण टेस्ट केस दिया है जो प्रस्तुत संख्याओं को जोड़ता है और उसका आउटपुट 5 है। हम आशा करते हैं कि प्रोग्राम बिना किसी एरर के सफलतापूर्वक चलेगा और हमारा टेस्ट केस पास हो जाएगा। 

## डीप डाइव:
यूनिट टेस्टिंग को अलग-अलग तकनीकों का एक समूह माना जाता है जो एक साथ काम करके एक कोड का आँकलन करते हैं। वे सिस्टम टेस्ट, एक से अधिक फंक्शनालिटी को मिलाकर चैक करने या एक नए फीचर की विकास के दौरान सिस्टम विश्लेषण करने जैसे कई स्थितियों के लिए प्रयोग किए जा सकते हैं। कई अन्य उपयोगी तकनीकें हैं, जैसे की टेस्ट ड्राइवन डेवलपमेंट (TDD), जिसमें पहले टेस्ट लिखे जाते हैं और फिर उसके आधार पर कोड बनाया जाता है, इससे हम बेहतर और घोषणा वाले क्योंकि सभी टेस्ट केस अपने कोड में ही टेस्ट करने के लिए उपलब्ध होते हैं। लेकिन आज के घरेलू राजमार्ग में उन्नत अपरिहार्य होम गार्ड वा मिनों के साथ भी-भीड़ परीक्षाएं उपयोगी हो सकती हैं।

## देखें भी:
- [Jasmine](https://jasmine.github.io/): टेस्ट ड्राइवन डेवलपमेंट (TDD) और सिस्टम टेस्टिंग के लिए एक प्रसिद्ध ओपन स्रोत फ्रेमवर्क।
- [Mocha](https://mochajs.org/): एक उन्नत टेस्टिंग फ्रेमवर्क जो अनुक्रमणिका, असिन्क्रॉनस टेस्टिंग और अधिक को समर्थन करता है।
