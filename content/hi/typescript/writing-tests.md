## क्यों

क्या आप अपने कोड में सुरक्षा सुनिश्चित करना चाहते हैं? या अपने कोड को आसानी से सम्भालना चाहते हैं? उत्तर हाँ है! आपको अपने कोड में टेस्ट कोड लिखने की आवश्यकता है। टेस्ट लिखने के साथ, आप अपने कोड को सुरक्षित और त्रुटि मुक्त बना सकते हैं और उसे आसानी से मैन्टेन कर सकते हैं।

## कैसे करें

```typescript
// एक सार्वजनिक फ़ंक्शन जो दो संख्याओं को जोड़ता है
function add(x: number, y: number): number {
  return x + y;
}

// add() फ़ंक्शन टेस्ट
console.log(add(2, 3)); // Output: 5
console.log(add(11, 6)); // Output: 17
```

ऊपर दिए गए उदाहरण में, हमने `add()` फ़ंक्शन को टेस्ट किया है और हम जान सकते हैं कि फ़ंक्शन सही रूप से काम कर रहा है। हमें अपने कोड में इस तरह के टेस्ट कोड लिखने के लिए `jasmine` जैसी लाइब्रेरी का उपयोग करना होगा। यह लाइब्रेरी एक बाहरी टेस्टिंग फ्रेमवर्क है जो आपको टेस्ट सूट, एसर्ट कन्डिशन्स और मॉक ऑब्जेक्ट्स के साथ आपके कोड को टेस्ट करने की सुविधा प्रदान करता है।
