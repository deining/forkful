## क्यों
किसी अन्य कोड को पुनर्यापन करने के लिए या किसी प्रोग्राम का चलाने के लिए हमेशा अस्थायी फ़ाइलों की आवश्यकता होती है। इस ब्लॉग पोस्ट में हम अस्थायी फ़ाइल के बारे में बात करेंगे और यह क्यों ज़रूरी है।

## कैसे करें
अस्थायी फ़ाइल बनाने के लिए हम `fs` लाइब्रेरी का उपयोग कर सकते हैं। यह पाठ्यक्रम एक अस्थायी फ़ाइल बनाने का उदाहरण देता है और कैसे हम उसे हटा सकते हैं।

```TypeScript
import fs from 'fs';

const temporaryFile = fs.mkdtempSync('/tmp/'); // '/tmp/' के अंत में स्लैश जोड़ें
console.log(`स्थानिक अस्थायी फ़ाइल: ${temporaryFile}`);

fs.rmdirSync(temporaryFile); // अस्थायी फ़ाइल हटाएं
```

आप देख सकते हैं कि हम `mkdtempSync()` फ़ंक्शन को `'/tmp/'` के साथ कॉल करते हैं, जो मानदंड के रूप में स्थानिक अस्थायी फ़ाइल के नाम का एक अनुचित मान हो सकता है। इसलिए, हम `rmdirSync()` फ़न्क्शन का उपयोग करके उसे हटा सकते हैं।

## गहरी जांच
अस्थायी फ़ाइल बनाने के कई तरीके हैं, जो `fs.mkdtempSync()` फ़ंक्शन के साथ पाठ से जुड़ी हैं। यह फ़ंक्शन यह श्रृंखला को बनाने के लिए `fs.mkdtemp('prefix')` फ़ंक्शन का उपयोग करता है, जहां `prefix` आपके द्वारा दिए गए मान के पहले शब्दों को आपत्तिजनक रूप से सकारात्मक बना सकता है।
