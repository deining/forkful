## क्यों

डायरेक्टरी का अस्तित्व जाँचने में क्यों लगाया जाता है कुछ लोगों को? 

डायरेक्टरी का अस्तित्व जाँचना अपने प्रोग्राम में निर्देशित मानचित्रन फ़ाइलों को स्थायी रूप से संबंधित करने के लिए महत्वपूर्ण हो सकता है। यह केवल फ़ाइल संग्रहलय को प्रभावित नहीं करता है बल्कि आपके प्रोग्राम को स्थायी और दक्ष स्तर पर प्रभावित कर सकता है। 

## कैसे करें

आप कैसे TypeScript में डायरेक्टरी का अस्तित्व जाँच सकते हैं? 
```TypeScript
import fs from 'fs';

// चेक करें कि डायरेक्टरी मौजूद है 
if (fs.existsSync('./my-directory')) {
    console.log('डायरेक्टरी मौजूद है।');
} else {
    console.log('डायरेक्टरी मौजूद नहीं है।');
}
```

```TypeScript
import path from 'path';

// चेक करें कि स्थानांतरण से ऑपरेशन डायरेक्टरी को समर्थन करता है 
if (path.posix.supportsDirectives()) {
    console.log('ऑपरेशन डायरेक्टरी समर्थित है।');
} else {
    console.log('ऑपरेशन डायरेक्टरी असमर्थित है।');
}
```

## गहराई में जाएँ

जब आप `fs.existsSync()` को उपयोग करते हैं, दो चीजें समझनी चाहिए। पहले, यह फ़ंक्शन स्ट्रिंग रूप में पथ को स्वीकार करता है। अतः, पथ स्ट्रिंग लेटिलिटर्स बार से अलग होने चाहिए। दूसरे, यह फ़ंक्शन केवल सब पैरमीटर की अवैधता की जाँच करेगा। यह मान स्वीकारता है कि अगर पथ एक मौजूदा फ़ाइल है, तो यह वैधता लालिक का संकेत देता है।
