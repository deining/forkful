## क्या और क्यों?

HTTP अनुरोध भेजना क्या है, और क्यों प्रोग्रामर इसे करते हैं।

HTTP अनुरोध एक प्रोग्राम से दूसरे प्रोग्राम तक डेटा भेजने का एक तरीका है। प्रोग्रामर इसका उपयोग उन अन्य प्रोग्रामों के साथ डेटा साझा करने या एप्लिकेशन के साथ अनुरोध प्रसंस्करण करने के लिए करते हैं। 

## कैसे करें:

```TypeScript
import http from 'http';

// एक HTTP अनुरोध निर्मित करें
const options = {
  hostname: 'www.example.com',
  path: '/data',
  method: 'GET'
}

// ग्राहक निर्माण
const request = http.request(options, (response) => {
  console.log(`अनुरोध का प्रतिक्रिया कोड: ${response.statusCode}`);

  // डेटा पार्स करें
  response.on('data', (data) => {
    console.log(data.toString());
  })
})

// अनुरोध को भेजें
request.end();
```

उपरोक्त कोड में, हमने `http` मॉड्यूल को इम्पोर्ट किया है जो HTTP अनुरोध भेजने के लिए उपयोगी मेथड और क्लास प्रदान करता है। हमने एक अनुरोध बनाया है और उसे निर्मित करके सर्वर को भेजा है। सबसे अंत में हमने अनुरोध को `end()` के माध्यम से भेज दिया है। उत्तर को प्राप्त करने के लिए, हमने सुविधाओं को प्रसारित किया है और उसे पार्स किया है।

## गहराई तक जाएं:

### इतिहास:

HTTP का अनुरोध भेजना साल 1989 में डॉ. टिम बर्नर्स-ली द्वारा बनाया गया था। यह इन्टरनेट कॉनेक्शन के माध्यम से सुरक्षित डेटा साझा करने का एक तरीका है। आज, यह भी एप्लिकेशन विकास में एक महत्वपूर्ण टूल है।

### वैकल्पिक:

HTTP अनुरोध भेजने के लिए अन्य माध्यमों को भी उपयोग किया जा सकता है, जैसे AJAX और Fetch API। AJAX ब्राउज़र जावास्क्रिप्ट को सर्वर के साथ डेटा को असिंक्रोनस रूप से संवर्धित करता है, जबकि Fetch API एक नए डेटा एक्सचेंज वर्तमान कंसोल ऐप्लिकेशन प्रोग्राम इंटरफेस है। 

### कार्यान्वयन विवरण:

HTTP अनुरोध भेजने के लिए आपको सर्वर के होस्टनाम और पथ के साथ विशेषताओं की आवश्यकता होती है। आप भी अनुरोध का विधि निर्दिष्ट कर सकते हैं, जैसे GET, POST और DELETE। आप अनुरोध पूरा करने के लिए कॉलबैक आर्ग्यूमेंट भी पास कर सकते हैं। इसके अलावा, आप सुरक्षा को बढ़ाने के लिए सुरक्षा स्तर भी सेट कर सकते हैं
