## क्यों

डीबग आउटपुट प्रिंट करना, एल्म प्रोग्रामिंग में एक आम तकनीक है जो हमें हमारे कोड को जांचने और समस्याओं को निराकरण करने में मदद करती है। इससे हमें अपने कोड के भागों को पहचानने और सुधार करने में आसानी होती है।

## कैसे

इस तकनीक का प्रयोग करने के लिए, हम `Debug` मॉड्यूल को इम्पोर्ट कर सकते हैं जो `toString` और `log` जैसे फ़ंक्शंस प्रदान करता है। तो चलिए कुछ कोड के माध्यम से देखें कि यह कैसे काम करता है:

```Elm
import Debug exposing (toString, log)

-- एक डेटा स्ट्रक्चर को परिभाषित करें
type alias Person =
    { name : String
    , age : Int
    }

-- डेटा स्ट्रक्चर के कुछ डेटा संरचित करें
person : Person
person =
    { name = "आदित्य"
    , age = 28
    }

-- डबग आउटपुट प्रिंट करें
main : Html msg
main =
    div []
        [ text <| toString person
        , log person
        ]
```

आप उपरोक्त कोड को कॉपी-पेस्ट करके चला सकते हैं और नीचे दिए गए आउटपुट को प्राप्त कर सकते हैं:

```Elm
{ age = 28, name = "आदित्य" }
{ age = 28, name = "आदित्य" }
```

इस तरह, हम अपने कोड में उपयोगी जानकारी को प्रिंट करके उसे समस्याओं से छुटकारा पा सकते हैं।

## गहराई में

अब हम इस तकनीक को और गहराई से देखें। डेबग आउटपुट प्रिंट करने के अलावा, हम `log` फ़ंक्शन के माध्यम से अपने कोड में अन्य फ़ंक्शंस भी प्रिंट कर सकते हैं। इससे हमें अपने कोड के स्टेटस, वापसी मूल्यों और दूसरे विशेषताओं को प्रद
