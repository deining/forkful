## क्यों

टेस्ट स्क्रिप्टिंग में समय लगता है, लेकिन यह आपके कोड को अधिक सुरक्षित और स्थिर बनाने में मदद करता है। इससे आपको भविष्य में अधिक समय बचाने में मदद मिल सकती है।

## तरीके

```Rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn main() {
    assert_eq!(add(2, 3), 5);
    assert_eq!(add(5, 7), 12);
}
```

जैसा कि आप देख सकते हैं, हमने `assert` फंक्शन का उपयोग करके अपने कोड की योगिता को जांचा है। इस तरह के फंक्शन सही जानकारी के साथ परिणाम वापस देते हैं, कि कोड कितना स्थिर और सुरक्षित है।

## गहरी तलाश

टेस्ट लिखने का सबसे बड़ा लाभ यह है कि आप अपने कोड को सुरक्षित बनाने में मदद मिलती हैं। इसलिए, आपको अपने टेस्ट को कम से कम समय-अवधि में लिखने के लिए समय निकालना चाहिए। यह सुनिश्चित करने से सुनिश्चित करें कि आपका कोड बेहतरीन ढंग से काम करता है और आपके उपभोक्ताओं को कम समस्याओं का सामना करना पड़ता है।
