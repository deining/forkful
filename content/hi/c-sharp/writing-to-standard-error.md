# Standard error पर लिखना क्या है और इसे क्यों किया जाता है?

 एक समझदार डेवलपर बनने के लिए हमेशा ट्रबलशूटिंग का जोखिम उठाना पड़ता है। इसका मतलब है कि हमारे कोड में गलतियां हो सकती हैं और हमें उन्हें ठीक करने की जरूरत हो सकती है। इसलिए, हमें हमेशा परिस्थितियों को नियंत्रित करने और इससे सम्बंधित जानकारी प्राप्त करने के लिए कुशल होना चाहिए। जब हम कोड लिखते हैं तो हमारे पास डिबग करने के लिए कुछ विकल्प होते हैं, जिसमें से एक है "Standard error पर लिखना"। हम जानेंगे कि यह क्या है और यह क्यों जरूरी है।

## कैसे करें:

जैसा कि हमने पहले बताया, "Standard error पर लिखना" एक विकल्प है जो हमें गलतियों का पता लगाने में मदद कर सकता है। यह C# में काफी आसान है। नीचे दिए गए कोड ब्लॉक में हम इसका उदाहरण देख सकते हैं।

```C#
try
{
    // कोड लिखें
}
catch(Exception e)
{
    Console.Error.WriteLine("गलती उठ गई: " + e.Message);
}
```

जब हमारे कोड में कोई गलती होगी तो हम C# console में इस लाइन को देख सकते हैं: "गलती उठ गई: [गलती का संदेश]"। यह उदाहरण आपको समझने में मदद करेगा।

## गहराई में जाएँ:

"Standard error पर लिखना" कोडिंग में बहुत उपयोगी है, लेकिन क्या आप जानते हैं कि इसका इतिहास क्या है? पहले, कई डेवलपर्स ने अपने code में साधारण print statements या log files का उपयोग किया था। लेकिन यह दोनों तरीके जैसे ही code base बढ़ता हैं, उन्हें manage करना मुश्किल हो जाता है। "Standard error पर लिखना" एक बेहतर और सुगम विकल्प है, इसमें हम कोई भी error message सीधे console में देख सकते हैं। इसके अलावा, ये काफी सरल तरीके से डिबग करने में मदद करता है। दूसरे विकल्प के तुलना में, यह कोड लिखने और उसे manage करने में आसान है।
