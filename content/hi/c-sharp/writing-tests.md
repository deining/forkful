## क्यों

क्या आप कभी सोचा है कि आपको अपने लिखित कोड को कई बार चलाना चाहिए? यदि हां, तो आप अच्छी तरह से कोड टेस्टिंग के बारे में जानते होंगे। कोड इस तरह के टेस्टों की मदद से चलाने योग्य होता है जिससे हम गर्भवती स्थितियों को मिलकर ढंग से जांच सकते हैं। आपका कोड जितना ताजगी और तरक्की भरा होगा, आपको उतने ही मुश्किल स्थितियों से जूझना पड़ सकता है। इसलिए, टेस्ट केस लिखना अति आवश्यक है जो आपको सुरक्षित बनाए रख सकता है।

## लिखने का तरीका

कोड टेस्ट केस लिखने के लिए, हमारे पास दो प्रमुख विकल्प हैं। पहले, हम मौजूदा कोड में टेस्ट केस जोड़ सकते हैं जो स्नाइपलचित होते हैं ताकि आपको बाद में अपने कोड को और बढ़िया बनाने में आसानी हो। दूसरा तरीका, और सबसे ज्यादा उचित तरीका, यह है कि आप लिखित कोड के साथ साथ टेस्ट केस भी लिखें जो कोड को सचेत रखने में मदद करते हैं।

यहां हमारे पास एक सरल उदाहरण है:

```C#
public int AddNumbers(int num1, int num2)
{
   return num1 + num2;
}

[Test]
public void Test_AddNumbers()
{
   int result = AddNumbers(5, 10);
   Assert.AreEqual(15, result);
}
```

इसके अलावा, हमारे पास कई टेस्टिंग फ्रेमवर्क भी हैं जिनका उपयोग करके आप अपने टेस्ट केस को और बढ़िया बना सकते हैं, जैसे NUnit, xUnit और MSTest।
