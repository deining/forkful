## อะไร & ทำไม?

การเริ่มโปรเจคใหม่เหมือนกับการปลูกเมล็ดในสวนดิจิตอลของคุณ—คุณกำลังเริ่มต้นกลุ่มความคิดใหม่ๆ และแปลงพวกมันเป็นโค้ดที่ทำงานบางอย่างให้มีประโยชน์ โปรแกรมเมอร์เริ่มโปรเจคใหม่เพื่อแก้ปัญหา, สำรวจแนวคิด, หรือเพียงแค่เพื่อความสนุกสนานในการสร้างสรรค์สิ่งใหม่ๆ

## การทำ:

ดังนั้น, คุณพร้อมที่จะเปลี่ยนความคิดเหล่านั้นเป็นโปรเจค Ruby หรือยัง? ลุยเลย ลองเริ่มจากพื้นฐานก่อน.

```Ruby
# ติดตั้ง Ruby, หากคุณยังไม่ได้ติดตั้ง
# ตรวจสอบเวอร์ชั่น Ruby เพื่อให้แน่ใจว่าคุณใช้เวอร์ชั่นล่าสุด:
$ ruby -v

# ผลลัพธ์ควรเป็นเวอร์ชั่นปัจจุบันของ Ruby:
# ruby 3.x.x

# ต่อไป, ลองสร้าง directory สำหรับโปรเจคของคุณ:
$ mkdir my_new_project
$ cd my_new_project

# ตอนนี้, เริ่มต้นกับ repository Git ใหม่หากคุณต้องการการควบคุมเวอร์ชั่น (แนะนำอย่างยิ่ง):
$ git init

# จากนั้นสร้างไฟล์เริ่มต้น, เราเรียกมันว่า 'app.rb':
$ touch app.rb

# เริ่มเขียนโค้ด! เขียนการแสดงผลง่ายๆเพื่อให้แน่ใจว่ามันทำงาน:
puts "Hello New Project!"

# รันไฟล์ของคุณ:
$ ruby app.rb

# ผลลัพธ์ควรเป็น:
# Hello New Project!
```

## ดำดิ่งลงไป

ในอดีต, การเริ่มต้นโปรเจค Ruby ใหม่อาจจะดูเรียบง่าย—เพียงแค่คุณ, โปรแกรมแก้ไขข้อความ, และไฟล์ `.rb` เป็นกอง ตามที่ภาษานี้เติบโตขึ้น, เครื่องมือต่างๆ ก็ปรากฏขึ้นเพื่อทำให้กระบวนการนี้ง่ายขึ้น

ตัวอย่างเช่น, Bundler จัดการ gems—ไลบรารีของ Ruby—ของคุณ เพื่อให้คุณติดตามและติดตั้งการพึ่งพาได้อย่างง่ายดาย เพียงรัน `bundle init` หลังจากที่คุณตั้งค่า directory โปรเจคเพื่อสร้าง `Gemfile`, ที่คุณรายการ gems ของคุณ

จากนั้นเรามี Ruby Version Manager (RVM) และ Ruby Environment (rbenv) ซึ่งช่วยให้สามารถเปลี่ยนเวอร์ชั่น Ruby ตามโปรเจค ค่อนข้างสะดวกถ้าคุณกำลังจัดการกับโค้ดเก่า

และเกี่ยวกับเฟรมเวิร์กล่ะ? Ruby on Rails เป็นชื่อดังสำหรับเว็บแอป แต่ถ้าคุณต้องการอะไรที่เบากว่านั้น (เช่น สำหรับบริการหรือ API), ลองดู Sinatra หรือ Roda
