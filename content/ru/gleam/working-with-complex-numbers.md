## Что и почему?
Комплексные числа состоят из действительной и мнимой части (`a + bi`). Они пригождаются в различных областях, таких как электротехника и квантовые вычисления. Программисты используют их для моделирования уравнений, которые нельзя решить, используя только действительные числа.

## Как это сделать:
В Gleam отсутствует встроенная поддержка комплексных чисел. Обычно вы создаёте свою реализацию или ищете библиотеку. Вот краткий пример того, как можно реализовать базовые операции:

```gleam
type Complex {
  Complex(Float, Float)
}

fn add(c1: Complex, c2: Complex) -> Complex {
  let Complex(a, b) = c1
  let Complex(x, y) = c2
  Complex(a + x, b + y)
}

fn multiply(c1: Complex, c2: Complex) -> Complex {
  let Complex(a, b) = c1
  let Complex(x, y) = c2
  Complex(a*x - b*y, a*y + b*x)
}

fn main() {
  let num1 = Complex(1.0, 2.0)
  let num2 = Complex(3.0, 4.0)
  let sum = add(num1, num2)
  let product = multiply(num1, num2)

  sum // Complex(4.0, 6.0)
  product // Complex(-5.0, 10.0)
}
```

## Глубокое погружение

Комплексные числа были впервые более формально задокументированы Жероламо Кардано в XVI веке. Они являются естественным расширением действительных чисел. Однако, в молодом языке, как Gleam, который придает приоритет производительности и типовой безопасности, такие возможности представлены в минимальном виде (или вы делаете их самостоятельно).

В некоторых других языках, таких как Python, комплексные числа встроены (`3+4j`), что облегчает жизнь. В Rust или Haskell есть библиотеки, которые из коробки предлагают расширенные функциональности.

Подход Gleam означает, что вам нужно обрабатывать все аспекты: арифметику, полярные координаты, экспоненциальные формы и т. д. Реализация эффективных, точных операций требует внимательного программирования с учетом того, как поведение с плавающей точкой может влиять на ваши результаты.

Не забывайте тщательно тестировать, особенно крайние случаи! Обработка бесконечности и значений NaN (не число) в комплексных числах может подстерегать вас, если вы не будете осторожны.
