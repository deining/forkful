## Что и почему?
Отправка HTTP-запроса - это способ, которым ваша программа запрашивает данные из другого места в интернете. Программисты делают это для взаимодействия с веб-сервисами, получения актуальных данных или для интеграции внешних API в свои приложения.

## Как:
В Gleam у нас еще нет встроенного HTTP-клиента, поэтому давайте использовать библиотеку `gleam_http`. Вот простой GET-запрос:

```gleam
import gleam/http.{Response, Error}
import gleam/httpc

pub fn main() -> Result(Response, Error) {
  // Отправить GET-запрос на example.com
  httpc.send(httpc.Request(
    method: httpc.Get,
    url: "http://example.com",
    headers: [],
    body: httpc.Empty,
  ))
}
```

Запуск этого кода отправит запрос на example.com и вернет ответ. Помните, что вам нужно будет обработать результат, чтобы действительно его использовать.

## Подробнее
Исторически отправка HTTP-запросов была задачей для специализированных инструментов, таких как `curl`, или библиотек в других языках. Это довольно ново, что сами языки включают удобные функции для отправки HTTP-запросов. Альтернативы для отправки HTTP-запросов в Gleam включают сторонние библиотеки, такие как `gleam_http`, и привязки, специфичные для платформы.

Что касается реализации, есть две части: создание запроса и получение ответа. Запросы имеют методы (GET, POST и т. д.), URL, заголовки и тело, в то время как ответы несут статус-коды, заголовки и тело.

Система типов и сопоставление с образцом в Gleam здесь блестяще себя показывают, обеспечивая исчерпывающую обработку ошибок и четкий разбор ответов. Это не просто отправка данных в пустоту и надежда на лучшее; это контролируемый, структурированный диалог.
