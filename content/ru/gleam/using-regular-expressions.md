## Что и зачем?

Регулярные выражения (regex) — это шаблоны, используемые для поиска сочетаний символов в строках. Программисты используют их для поиска, валидации и манипулирования текстом, поскольку они быстрые и эффективные.

## Как использовать:

В Gleam нет встроенной поддержки регулярных выражений, поэтому мы используем модуль `gleam_otp/re`. Вот что нужно сделать:

Сначала добавьте `gleam_otp` в зависимости вашего `rebar.config`.

```erlang
{deps, [
    {gleam_otp, "0.1.0"}
]}
```

Теперь напишите немного кода на Gleam для поиска по шаблонам:

```rust
import gleam/otp/re

fn main() {
  let pattern = "^Hello, (\\w+)!"
  let text = "Hello, World!"
  
  let result = re.run(pattern, text)
  jsonecho(result)
}

fn jsonecho(result) {
  case result {
    Ok(matches) -> 
      case matches {
        [] -> 
          "Нет совпадений"
        [_, name] -> 
          "Совпало с именем: " <> name
        _ -> 
          "Неожиданное количество совпадений"
      }
    Error(_) -> 
      "Шаблон не скомпилировался"
  }
}
```

Запустите его, и вы увидите `Совпало с именем: World`.

## Подробнее

Регулярные выражения существуют с 1950-х годов; их использование в инструментах Unix в 1970-х годах утвердило их положение. К альтернативам относятся строковые функции, но они могут быть многословными. С точки зрения реализации, регулярные выражения обычно компилируются во внутреннюю машину состояний, что обеспечивает скорость, но потенциальную сложность при написании сложных шаблонов.
