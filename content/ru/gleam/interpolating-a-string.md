## Что и Почему?

Интерполяция строк - это процесс встраивания выражений в строковые литералы для создания новых строковых значений. Программисты используют интерполяцию строк для динамического построения сообщений, часто для вывода пользователю или логирования.

## Как:

В Gleam интерполяция строк проста. Используйте синтаксис `#{}` для вставки выражений в строки. Вот быстрый пример:

```gleam
fn main() {
  let name = "world"
  let greeting = "Привет, #{name}!"
  greeting
}

// Это выведет: "Привет, world!"
```

## Подробнее

Исторически сложилось так, что нормой была конкатенация строк, когда вы вручную соединяли строки и значения. Это быстро становится запутанным. Интерполяция проще и понятнее.

Языки различаются в синтаксисе интерполяции; `#{}` в Gleam отражает синтаксис Ruby и Elixir. Это согласованность полезна для людей, переключающихся между языками.

Под капотом компилятор Gleam преобразует интерполированные строки в серию операций конкатенации строк до их компиляции в Erlang, язык, в который компилируется Gleam. Так:

```gleam
"Привет, #{name}!"
```

становится чем-то вроде (в псевдокоде Erlang):

```erlang
"Привет, " ++ name ++ "!"
```

Выбор между интерполяцией и конкатенацией обычно базируется на удобочитаемости и удобстве, хотя разницы в производительности почти нет из-за оптимизации компилятора.
